# 2.3.1 工具使用与动作选择

## 概念讲解：什么是Agent？

### 从"聊天机器人"到"智能代理"

想象两种AI应用：

```
❌ 聊天机器人：
   用户："北京天气怎么样？"
   AI："我的知识截止于训练数据，无法获取实时天气信息。"
   
✅ 智能代理（Agent）：
   用户："北京天气怎么样？"
   AI："让我查一下..." [调用天气API]
   AI："北京现在是晴天，25°C，空气质量优。"
```

> **核心要点**：Agent的本质是**让LLM具备使用工具的能力**，从而突破模型的知识边界。

### Agent的工作循环

```
┌─────────────────────────────────────────────────────┐
│                 Agent执行循环                        │
│                                                     │
│  用户输入 → 思考：需要做什么？                        │
│              ↓                                        │
│         选择：用哪个工具？                             │
│              ↓                                        │
│         执行：调用工具获取结果                         │
│              ↓                                        │
│         观察：工具返回了什么？                         │
│              ↓                                        │
│      ┌─── 够了吗？ ──否──┐                            │
│      ↓                   ↓                           │
│     是                继续思考-选择-执行               │
│      ↓                                                │
│    生成最终回复                                         │
└─────────────────────────────────────────────────────┘
```

这个循环被称为**ReAct模式**（Reason + Act）：推理下一步该做什么，然后执行。

## 核心要点：工具的三要素

### 要素1：工具定义

在LangChain中，用`@tool`装饰器将函数转换为工具：

```python
from langchain_core.tools import tool

@tool
def search_weather(city: str) -> str:
    """查询指定城市的天气信息。
    
    Args:
        city: 城市名称
    """
    # 实际项目中调用天气API
    return f"{city}：晴，25°C"
```

**关键点**：函数名、参数类型、docstring会被自动解析，LLM通过这些信息理解工具的用途。

### 要素2：工具绑定

将工具列表传递给模型，模型会自动学习如何调用：

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")
tools = [search_weather]

# 绑定工具到模型
llm_with_tools = llm.bind_tools(tools)
```

### 要素3：Agent循环

LangChain提供了多种Agent实现方式，最常用的是`create_tool_calling_agent`：

```python
from langchain.agents import create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有帮助的助手"),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad"),
])

agent = create_tool_calling_agent(llm, tools, prompt)
```

## 简单示例：完整的Agent

### 场景：天气查询+数学计算

```python
from langchain_core.tools import tool
from langchain_openai import ChatOpenAI
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

# 1. 定义工具
@tool
def search_weather(city: str) -> str:
    """查询指定城市的天气信息"""
    weather_data = {
        "北京": "晴，25°C，湿度45%",
        "上海": "多云，23°C，湿度60%",
        "广州": "阵雨，28°C，湿度75%",
    }
    return weather_data.get(city, f"未找到{city}的天气信息")

@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    try:
        result = eval(expression, {"__builtins__": {}}, {})
        return f"{expression} = {result}"
    except Exception as e:
        return f"计算错误：{e}"

# 2. 创建Agent
llm = ChatOpenAI(model="gpt-4o-mini")
tools = [search_weather, calculate]

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个智能助手，可以使用工具帮助用户"),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad"),
])

agent = create_tool_calling_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 3. 使用Agent
result = agent_executor.invoke({
    "input": "北京天气怎么样？另外帮我计算(15 + 27) * 3"
})
print(result["output"])
```

**执行过程**（verbose=True时可见）：

```
> Entering new AgentExecutor chain...

[{'name': 'search_weather', 'args': {'city': '北京'}, 'id': 'call_1'}]
[{'name': 'calculate', 'args': {'expression': '(15 + 27) * 3'}, 'id': 'call_2'}]

Observation: 北京：晴，25°C，湿度45%
Observation: (15 + 27) * 3 = 126

AI回复：北京现在是晴天，25°C，湿度45%。另外，(15 + 27) * 3 = 126。
```

## 进阶理解：Agent如何决策？

### 模型是如何选择工具的？

LLM选择工具的过程不是硬编码的规则，而是**基于语义理解的自然决策**：

```
用户："今天气温多少度？"
    ↓
LLM分析关键词：气温、温度、天气
    ↓
查找工具描述中包含相关关键词的工具
    ↓
发现search_weather工具，docstring提到"天气信息"
    ↓
决定调用search_weather
```

**这就是为什么工具的docstring如此重要**：它是LLM理解工具的唯一途径。

### 工具参数如何传递给模型？

```python
from pydantic import BaseModel, Field

class WeatherInput(BaseModel):
    city: str = Field(description="城市名称，如北京、上海")

@tool(args_schema=WeatherInput)
def search_weather(city: str) -> str:
    """查询指定城市的天气信息"""
    return f"{city}的天气信息..."
```

通过`args_schema`，模型能更精确地理解每个参数的含义。

## 常见问题

### Q1: 我应该选择哪种Agent类型？

**A:** 根据场景选择：

| Agent类型 | 适用场景 | 创建方法 |
|-----------|---------|---------|
| 工具调用Agent | 大多数场景 | `create_tool_calling_agent` |
| ReAct Agent | 需要逐步推理 | `create_react_agent` |
| 自定义Prompt | 需要特殊提示 | 自定义prompt + `create_tool_calling_agent` |

> **建议**：大多数情况下，`create_tool_calling_agent`是最佳选择，它原生支持OpenAI的function calling格式，效率高且稳定。

### Q2: 工具执行失败怎么办？

**A:** 工具应该内部处理异常，返回有意义的错误信息：

```python
@tool
def search_weather(city: str) -> str:
    """查询天气"""
    try:
        response = weather_api.get(city)
        return response.text
    except Exception as e:
        return f"查询失败：{str(e)}"  # 返回错误给LLM，让它决定下一步
```

> **关键**：不要抛出异常，而是返回错误信息给LLM，让Agent自己决定如何处理。

### Q3: Agent会无限循环调用工具吗？

**A:** LangChain内置了循环保护机制：

```python
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=15,      # 最大循环次数
    handle_parsing_errors=True,  # 处理解析错误
)
```

超过最大次数后，Agent会强制停止并返回当前结果。

## 本节总结

### 核心认知
1. **Agent = LLM + 工具 + 循环**：LLM负责推理，工具负责执行，循环负责迭代
2. **工具docstring至关重要**：这是LLM理解工具的唯一信息来源
3. **ReAct模式**：思考→选择→执行→观察，循环直到问题解决

### 关键启示
- 工具是Agent的能力扩展
- 好的工具定义比复杂的Agent逻辑更重要
- 让错误信息返回给LLM，让它自己决策
