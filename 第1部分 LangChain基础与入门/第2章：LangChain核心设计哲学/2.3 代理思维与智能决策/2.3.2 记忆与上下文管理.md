# 2.3.2 记忆与上下文管理

## 概念讲解：为什么Agent需要记忆？

### 从"失忆症"到"连续对话"

想象两种对话体验：

```
❌ 没有记忆：
   用户："我叫张三"
   AI："你好张三"
   用户："你记得我叫什么吗？"
   AI："抱歉，我不知道您的名字"
   
✅ 有记忆：
   用户："我叫张三"
   AI："你好张三"
   用户："你记得我叫什么吗？"
   AI："当然，你叫张三"
```

> **核心要点**：LLM本身是无状态的，每次调用都是独立的。记忆系统让AI应用**记住历史，理解上下文**。

## 核心要点：记忆的两种实现方式

### 方式一：消息列表（简单场景）

最直接的方式是把历史消息传给模型：

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage

llm = ChatOpenAI(model="gpt-4o-mini")

# 维护一个消息列表
messages = [
    HumanMessage(content="我叫张三"),
    AIMessage(content="你好张三！"),
    HumanMessage(content="你记得我叫什么吗？"),
]

# 把历史一起发送给模型
response = llm.invoke(messages)
print(response.content)  # → "你叫张三"
```

**问题**：消息越来越多，会超出模型的token限制。

### 方式二：trim_messages智能修剪

LangChain提供`trim_messages`来管理上下文窗口：

```python
from langchain_core.messages.utils import trim_messages

# 消息太多时，智能修剪
trimmed = trim_messages(
    messages=messages,
    strategy="last",      # 保留最近的
    max_tokens=1000,      # 最大token数
    start_on="human",     # 从用户消息开始
)
```

**修剪策略**：

```
原始消息：[H1, A1, H2, A2, H3, A3, H4, A4, H5, A5]
                                    ↓
trim_messages(max_tokens=1000)
                                    ↓
保留消息：[H3, A3, H4, A4, H5]
```

> **关键**：总是保留最近的消息，因为它们最相关。

## 简单示例：带记忆的对话链

### 场景：多轮问答

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage

llm = ChatOpenAI(model="gpt-4o-mini")

# 用MessagesPlaceholder动态插入历史消息
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个友好的助手"),
    MessagesPlaceholder("history", optional=True),  # 历史消息占位
    ("human", "{input}"),
])

chain = prompt | llm

# 维护历史消息
history = []

# 第一轮
result1 = chain.invoke({
    "history": history,
    "input": "我叫李四",
})
history.extend([
    HumanMessage(content="我叫李四"),
    result1,
])

# 第二轮（模型能记住历史）
result2 = chain.invoke({
    "history": history,
    "input": "你还记得我叫什么吗？",
})
print(result2.content)  # → "你叫李四"
```

## 进阶应用：LangGraph的持久化记忆

### Checkpointer：状态持久化

LangGraph提供Checkpointer机制，可以保存和恢复对话状态：

```python
from langgraph.graph import StateGraph, MessagesState, START
from langgraph.checkpoint.memory import InMemorySaver

# 定义节点
def chatbot(state: MessagesState):
    return {"messages": [llm.invoke(state["messages"])]}

# 创建图
builder = StateGraph(MessagesState)
builder.add_node("chatbot", chatbot)
builder.add_edge(START, "chatbot")

# 编译时添加checkpointer
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 使用thread_id隔离不同会话
config = {"configurable": {"thread_id": "user_001"}}

# 第一轮
graph.invoke(
    {"messages": [HumanMessage(content="我叫王五")]},
    config=config
)

# 第二轮（状态自动恢复）
result = graph.invoke(
    {"messages": [HumanMessage(content="我叫什么？")]},
    config=config
)
```

**执行流程**：

```
第一次invoke → 保存状态到checkpointer
第二次invoke → 从checkpointer恢复状态 → 继续对话
```

### 多用户隔离

不同的thread_id，状态完全隔离：

```python
config_user1 = {"configurable": {"thread_id": "user_001"}}
config_user2 = {"configurable": {"thread_id": "user_002"}}

# 两个用户的对话互不影响
graph.invoke({"messages": [HumanMessage(content="我叫张三")]}, config=config_user1)
graph.invoke({"messages": [HumanMessage(content="我叫李四")]}, config=config_user2)
```

## 常见问题

### Q1: 应该选择哪种记忆方式？

**A:** 根据场景选择：

| 场景 | 推荐方案 |
|------|---------|
| 简单对话（<10轮） | 消息列表 + trim_messages |
| 长对话 | 摘要记忆（定期总结） |
| 生产环境 | LangGraph + Checkpointer |
| 跨会话记忆 | 数据库持久化 |

### Q2: 如何控制token使用量？

**A:** 三种策略：

```python
# 1. trim_messages：保留最近N条
trimmed = trim_messages(messages, max_tokens=2000)

# 2. 只保留最后K条
recent_messages = messages[-10:]

# 3. 摘要替换
# 定期让LLM总结历史对话
summary_prompt = ChatPromptTemplate.from_template("总结以下对话：{history}")
summary = summary_prompt | llm
```

### Q3: 记忆数据如何持久化？

**A:** 生产环境应该使用数据库存储：

```python
from langgraph.checkpoint.sqlite import SqliteSaver

# 使用SQLite持久化状态
with SqliteSaver.from_conn_string("memory.db") as checkpointer:
    graph = builder.compile(checkpointer=checkpointer)
```

## 本节总结

### 核心认知
1. **LLM本身无状态**：需要外部系统来维护对话历史
2. **trim_messages管理窗口**：智能修剪，控制token使用
3. **Checkpointer持久化**：LangGraph提供开箱即用的状态保存

### 关键启示
- 简单场景用消息列表就够了
- 生产环境用Checkpointer实现持久化
- 用thread_id隔离不同用户的状态
