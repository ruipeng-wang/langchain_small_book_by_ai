# 2.3.3 多轮对话与状态保持

## 概念讲解：多轮对话的本质

### 从"一问一答"到"任务型对话"

```
❌ 简单问答：
   用户："今天天气？"
   AI："晴天"
   
✅ 任务型对话（多轮）：
   用户："帮我订个餐厅"
   AI："好的，请问人数？"
   用户："4个人"
   AI："有偏好的菜系吗？"
   用户："川菜"
   AI："为你推荐XX川菜馆，今晚7点4位，确认吗？"
   用户："确认"
   AI："预订成功！"
```

> **核心要点**：多轮对话的本质是**收集信息 → 逐步推进 → 完成目标**。

### StateGraph：对话状态机

LangGraph的StateGraph将对话建模为**状态转移图**：

```
┌─────────────────────────────────────────────────────┐
│              餐厅预订对话状态机                       │
│                                                     │
│  START → 问候 → 收集人数 → 收集菜系 → 确认 → 完成    │
│              ↓          ↓          ↓                │
│           人数不明确   菜系不明确  用户取消           │
│              ↓          ↓          ↓                │
│           重新询问   重新询问    结束对话            │
└─────────────────────────────────────────────────────┘
```

## 核心要点：状态图的三要素

### 要素1：状态定义

用TypedDict定义对话状态的结构：

```python
from typing import TypedDict, Annotated
import operator

class DialogState(TypedDict):
    messages: Annotated[list, operator.add]  # 消息列表（自动累积）
    current_step: str                         # 当前步骤
    collected_info: dict                      # 收集的信息
```

> **关键**：`Annotated[list, operator.add]`表示新消息会追加到列表中，而不是替换。

### 要素2：节点函数

每个节点处理一个对话步骤：

```python
def greet(state: DialogState) -> DialogState:
    """问候用户"""
    return {
        "messages": [{"role": "assistant", "content": "你好！请问有什么可以帮你？"}],
        "current_step": "greeted"
    }

def collect_info(state: DialogState) -> DialogState:
    """收集用户信息"""
    last_msg = state["messages"][-1]
    # 处理用户输入...
    return {"current_step": "collected"}
```

### 要素3：边和路由

定义状态转移路径：

```python
from langgraph.graph import StateGraph, START, END

# 创建图
builder = StateGraph(DialogState)

# 添加节点
builder.add_node("greet", greet)
builder.add_node("collect", collect_info)

# 添加边
builder.add_edge(START, "greet")
builder.add_edge("greet", "collect")
builder.add_edge("collect", END)

# 编译
graph = builder.compile()
```

## 简单示例：餐厅预订对话

### 完整实现

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage

llm = ChatOpenAI(model="gpt-4o-mini")

# 定义状态（使用内置的MessagesState）
class BookingState(MessagesState):
    current_step: str
    booking_info: dict

# 定义节点
def ask_people(state: BookingState) -> dict:
    """询问人数"""
    return {
        "messages": [AIMessage(content="请问用餐人数？")],
        "current_step": "asking_people"
    }

def ask_cuisine(state: BookingState) -> dict:
    """询问菜系"""
    # 从用户消息中提取人数
    last_msg = state["messages"][-1]
    people = last_msg.content if last_msg.type == "human" else ""
    
    return {
        "messages": [
            AIMessage(content=f"好的{people}位，请问有偏好的菜系吗？")
        ],
        "current_step": "asking_cuisine",
        "booking_info": {"people": people}
    }

def confirm_booking(state: BookingState) -> dict:
    """确认预订"""
    last_msg = state["messages"][-1]
    cuisine = last_msg.content if last_msg.type == "human" else ""
    
    booking = state.get("booking_info", {})
    return {
        "messages": [
            AIMessage(
                content=f"已为您预订{booking.get('people', '')}位的{cuisine}，请确认。"
            )
        ],
        "current_step": "confirmed"
    }

# 构建图
builder = StateGraph(BookingState)
builder.add_node("ask_people", ask_people)
builder.add_node("ask_cuisine", ask_cuisine)
builder.add_node("confirm", confirm_booking)

builder.add_edge(START, "ask_people")
builder.add_edge("ask_people", "ask_cuisine")
builder.add_edge("ask_cuisine", "confirm")
builder.add_edge("confirm", END)

graph = builder.compile()

# 模拟对话
messages = [HumanMessage(content="我想订餐厅")]
result = graph.invoke({"messages": messages, "current_step": "start", "booking_info": {}})

for msg in result["messages"]:
    print(f"{msg.type}: {msg.content}")
```

## 进阶应用：条件路由

### 场景：根据用户输入动态路由

```python
from typing import Literal

def classify_input(state: BookingState) -> Literal["book_restaurant", "query_info", "end"]:
    """根据用户输入分类"""
    content = state["messages"][-1].content.lower()
    if "订" in content or "预订" in content:
        return "book_restaurant"
    elif "查询" in content or "什么" in content:
        return "query_info"
    return "end"

# 添加条件边
builder.add_conditional_edges(
    START,
    classify_input,
    {
        "book_restaurant": "ask_people",
        "query_info": "query_node",
        "end": END
    }
)
```

**执行流程**：

```
用户输入："我想订餐厅"
    ↓
classify_input → "book_restaurant"
    ↓
路由到 ask_people 节点
```

### 场景：带Checkpointer的持久化对话

```python
from langgraph.checkpoint.memory import InMemorySaver

# 编译时添加checkpointer
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 使用thread_id隔离会话
config = {"configurable": {"thread_id": "user_001"}}

# 第一轮
graph.invoke(
    {"messages": [HumanMessage(content="订餐厅")]},
    config=config
)

# 第二轮（状态自动恢复）
graph.invoke(
    {"messages": [HumanMessage(content="4个人")]},
    config=config
)
```

## 常见问题

### Q1: StateGraph和普通Chain有什么区别？

**A:** 

| 特性 | Chain | StateGraph |
|------|-------|------------|
| 执行方式 | 线性顺序 | 图结构，支持分支和循环 |
| 状态管理 | 手动维护 | 内置状态对象 |
| 适用场景 | 简单管道 | 复杂多轮对话 |

### Q2: 如何处理用户中途改变意图？

**A:** 在每个节点检查用户输入，如果检测到新意图，跳转到对应节点：

```python
def check_intent_change(state: BookingState) -> Literal["ask_people", "query_info", "continue"]:
    content = state["messages"][-1].content.lower()
    if "查询" in content:
        return "query_info"
    return "continue"  # 继续当前流程
```

### Q3: 如何限制对话轮数？

**A:** 在状态中记录轮数，条件边判断是否超限：

```python
def check_max_turns(state: BookingState) -> Literal["continue", "end"]:
    if state.get("turn_count", 0) > 10:
        return "end"  # 超过10轮，结束对话
    return "continue"
```

## 本节总结

### 核心认知
1. **StateGraph是状态机**：节点处理逻辑，边定义转移
2. **MessagesState自动累积消息**：`Annotated[list, operator.add]`是关键
3. **条件边实现智能路由**：根据用户输入动态选择路径

### 关键启示
- 简单对话用Chain就够了
- 多轮、分支、循环用StateGraph
- 用Checkpointer实现持久化

### 第2章总结

本章学习了LangChain的三大设计哲学：

| 节 | 核心概念 | 关键组件 |
|----|---------|---------|
| 2.1 模块化设计 | 标准化+可组合 | Runnable、管道符 |
| 2.2 链式思维 | 声明式工作流 | RunnableSequence、RunnableParallel |
| 2.3 代理思维 | 自主决策 | Agent、StateGraph、Checkpointer |

这三层设计从简单到复杂，覆盖了从线性管道到智能代理的完整 spectrum。
