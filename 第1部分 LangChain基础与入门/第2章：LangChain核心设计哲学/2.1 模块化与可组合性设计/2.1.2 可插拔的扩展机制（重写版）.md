# 2.1.2 可插拔的扩展机制（重写版）

## 概念讲解：什么是可插拔架构？

### 从"焊接电路"到"USB接口"

```
❌ 紧耦合（焊接电路）：
   每个组件直接连接，无法单独更换
   
✅ 可插拔（USB接口）：
   只要遵循USB协议，任何设备都能即插即用
```

> **核心要点**：可插拔架构的核心是**定义接口协议，实现即插即用**。

### AI应用的可插拔需求

```
同一个应用，可能需要切换：
┌────────────────────────────────────────┐
│  模型：OpenAI → Anthropic → 本地模型     │
│  向量库：Chroma → FAISS → Milvus        │
│  嵌入模型：OpenAI → Cohere → 本地模型    │
│  存储：内存 → Redis → PostgreSQL        │
└────────────────────────────────────────┘

如果每个组件都要改代码，那就不是可插拔
```

## 核心要点：LangChain的扩展协议

### Provider接口：统一模型接入

所有模型提供商都实现相同的BaseChatModel接口：

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic
from langchain_google_genai import ChatGoogleGenerativeAI

# 三种模型，使用方式完全一致
models = {
    "openai": ChatOpenAI(model="gpt-4o-mini"),
    "anthropic": ChatAnthropic(model="claude-3-haiku"),
    "google": ChatGoogleGenerativeAI(model="gemini-pro"),
}

# 切换模型只需改一行
chain = prompt | models["openai"] | parser
```

### 嵌入模型的可插拔

```python
from langchain_openai import OpenAIEmbeddings
from langchain_cohere import CohereEmbeddings

# 两种嵌入模型，接口相同
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
# 或
embeddings = CohereEmbeddings(model="embed-english-v3.0")
```

## 简单示例：配置驱动的可插拔系统

### 场景：通过配置切换所有组件

```python
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_anthropic import ChatAnthropic

# 配置字典
CONFIGS = {
    "production": {
        "llm": ChatOpenAI(model="gpt-4o"),
        "embedding": OpenAIEmbeddings(),
    },
    "cost_saving": {
        "llm": ChatAnthropic(model="claude-3-haiku"),
        "embedding": OpenAIEmbeddings(model="text-embedding-3-small"),
    },
    "local": {
        "llm": ChatOpenAI(model="llama-3-8b", base_url="http://localhost:11434/v1"),
        "embedding": OpenAIEmbeddings(base_url="http://localhost:11434/v1"),
    },
}

def build_chain(config_name: str):
    """根据配置名构建链"""
    config = CONFIGS[config_name]
    return prompt | config["llm"] | parser

# 生产环境
prod_chain = build_chain("production")
# 测试环境
test_chain = build_chain("cost_saving")
```

## 常见问题

### Q1: 如何扩展自定义的模型？

**A:** 继承BaseChatModel并实现核心方法：

```python
from langchain_core.language_models import BaseChatModel
from langchain_core.messages import BaseMessage

class CustomLLM(BaseChatModel):
    def _generate(self, messages, stop=None, run_manager=None):
        # 实现你的模型调用逻辑
        pass
    
    @property
    def _llm_type(self):
        return "custom_llm"
```

### Q2: 如何知道某个组件有哪些实现？

**A:** 查看langchain生态的集成包：

```
pip install langchain-openai      # OpenAI
pip install langchain-anthropic   # Anthropic
pip install langchain-cohere      # Cohere
pip install langchain-chroma      # Chroma向量库
pip install langchain-qdrant      # Qdrant向量库
```

## 本节总结

### 核心认知
1. **接口统一**：所有同类组件实现相同接口
2. **配置驱动**：通过配置而非代码切换组件
3. **生态丰富**：各厂商维护自己的langchain集成包

### 关键启示
- 选择组件时考虑未来可能的切换需求
- 优先使用官方维护的集成包，质量更有保障
