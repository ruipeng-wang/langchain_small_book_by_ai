# 2.1.3 统一接口与抽象层（重写版）

## 概念讲解：为什么需要统一接口？

### 从"多种插头"到"万能插座"

不同AI提供商的API完全不同：

```python
# ❌ OpenAI的调用方式
response = openai_client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}]
)
result = response.choices[0].message.content

# ❌ Anthropic的调用方式（完全不同）
response = anthropic_client.messages.create(
    model="claude-3",
    max_tokens=1000,
    messages=[{"role": "user", "content": prompt}]
)
result = response.content[0].text
```

> **核心要点**：统一接口屏蔽底层差异，让使用者无需关心具体实现。

## 核心要点：Runnable协议

### 什么是Runnable？

Runnable是LangChain的核心抽象，所有组件实现统一接口：

```python
# 所有LangChain组件都支持这四个方法
component.invoke(input)     # 同步调用
component.stream(input)     # 流式调用
component.batch(inputs)     # 批量调用
await component.ainvoke(input)  # 异步调用
```

**统一接口的实际效果**：

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# 三种不同类型的组件，使用相同的方法
prompt = ChatPromptTemplate.from_template("解释{topic}")
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

prompt.invoke({"topic": "AI"})   # ✅
llm.invoke("你好")                # ✅
parser.invoke(llm_response)      # ✅
```

> **关键洞察**：学会一个接口，就学会所有组件的使用方式。

## 简单示例：管道组合

用`|`操作符串联组件：

```python
# 构建问答链
chain = prompt | llm | parser

# 调用时，数据依次流经每个组件
result = chain.invoke({"topic": "机器学习"})
```

**执行流程**：

```
chain.invoke({"topic": "机器学习"})
    ↓
prompt → "请解释机器学习"
    ↓
llm → AIMessage(content="机器学习是...")
    ↓
parser → "机器学习是..."（纯字符串）
    ↓
返回最终结果
```

## 进阶理解：Runnable的组合能力

### 并行执行（RunnableParallel）

```python
from langchain_core.runnables import RunnableParallel

# 同时执行多个任务
parallel = RunnableParallel({
    "translation": translate_chain,
    "summary": summary_chain,
})

results = parallel.invoke({"text": "Hello"})
# → {"translation": "你好", "summary": "一句问候"}
```

### 自定义处理（RunnableLambda）

```python
from langchain_core.runnables import RunnableLambda

def format_output(x):
    return x.content.upper()

chain = prompt | llm | RunnableLambda(format_output)
```

### 增强能力：包装器

```python
# 配置参数
configured_llm = llm.with_config({"temperature": 0.7})

# 自动重试
retry_llm = llm.with_retry(stop_after_attempt=3)

# 失败回退
fallback_llm = llm.with_fallbacks([backup_llm])
```

## 常见问题

### Q1: Runnable和普通函数有什么区别？

| 特性 | 普通函数 | Runnable |
|------|---------|----------|
| 调用方式 | 各不相同 | 统一invoke/stream/batch |
| 组合能力 | 手动编排 | 管道符直接组合 |
| 流式支持 | 额外实现 | 内置stream方法 |
| 批量处理 | 循环调用 | 内置并行优化 |

### Q2: 如何自定义Runnable组件？

```python
from langchain_core.runnables import Runnable

class MyProcessor(Runnable):
    def invoke(self, input, config=None):
        return f"处理结果: {input}"

# 可直接放入链中使用
chain = prompt | llm | MyProcessor() | parser
```

### Q3: Runnable和LangGraph有什么关系？

- **Runnable**：提供统一接口和简单组合（线性、并行）
- **LangGraph**：在Runnable基础上添加状态管理、循环、条件分支

## 本节总结

### 核心认知
1. **Runnable是统一接口**：所有组件使用相同方法调用
2. **管道符是组合方式**：`|`让组件像管道一样串联
3. **包装器是增强机制**：with_config/with_retry/with_fallbacks

### 关键启示
- 统一接口降低学习成本
- 组合优于继承，管道优于嵌套
