# 2.1.3 统一接口与抽象层（重写版）

## 概念讲解：为什么需要统一接口？

### 从"多种插头"到"万能插座"

想象你出国旅行，会遇到什么问题？

```
❌ 欧洲旅行：需要Type C插头
❌ 英国旅行：需要Type G插头
❌ 美国旅行：需要Type B插头
❌ 澳洲旅行：需要Type I插头

每次都要带不同的转换头，繁琐且容易遗忘

✅ 万能转换器：一个适配器，全球通用
   输入 → [万能适配器] → 输出
   不管去哪，带上它就够了
```

> **核心要点**：统一接口的本质是**屏蔽底层差异**，让使用者无需关心具体实现细节。

### AI生态的"插头混乱"问题

不同AI提供商的API设计完全不同：

```python
# ❌ 不同模型的不同调用方式
import openai
import anthropic
import cohere

# OpenAI的调用方式
response = openai_client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}]
)
result = response.choices[0].message.content

# Anthropic的调用方式（完全不同）
response = anthropic_client.messages.create(
    model="claude-3",
    max_tokens=1000,
    messages=[{"role": "user", "content": prompt}]
)
result = response.content[0].text

# Cohere的调用方式（又是一种风格）
response = cohere_client.generate(
    model="command",
    prompt=prompt
)
result = response.generations[0].text
```

**问题**：每种模型都要学一套不同的调用方式，代码无法复用。

## LangChain的解决方案：Runnable协议

### 什么是Runnable？

Runnable是LangChain的**核心抽象协议**，所有组件都实现统一接口：

```
┌──────────────────────────────────────────────────────┐
│              Runnable协议：统一的执行接口               │
│                                                      │
│  所有LangChain组件都支持以下方法：                      │
│                                                      │
│  invoke(input)        同步调用，等待完整结果返回        │
│  stream(input)        流式调用，逐chunk返回结果         │
│  batch(inputs)        批量调用，并行处理多个输入        │
│  ainvoke(input)       异步调用，非阻塞执行              │
│  astream(input)       异步流式调用                     │
│  abatch(inputs)       异步批量调用                     │
│                                                      │
│  无论你是LLM、Tool、Prompt还是Parser，接口完全一致！    │
└──────────────────────────────────────────────────────┘
```

### 统一接口的实际效果

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser
from langchain_core.tools import tool

# 四种不同类型的组件
prompt = ChatPromptTemplate.from_template("解释{topic}")
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

@tool
def get_time() -> str:
    """获取当前时间"""
    return "2024-01-01 12:00:00"

# 所有组件都使用相同的方法调用
prompt.invoke({"topic": "AI"})     # ✅
llm.invoke("你好")                  # ✅
parser.invoke(llm_response)        # ✅
get_time.invoke({})                # ✅

# 所有组件都支持流式
for chunk in llm.stream("讲个故事"):
    print(chunk, end="")

# 所有组件都支持批量
results = llm.batch(["问题1", "问题2", "问题3"])
```

> **关键洞察**：学会一个接口，就学会了所有组件的使用方式。这就是统一接口的力量。

## 核心要点：Runnable的三大组合能力

### 组合一：管道序列（RunnableSequence）

用`|`操作符串联组件，数据从左到右流动：

```python
# 构建问答链
chain = prompt | llm | parser

# 调用时，数据依次流经每个组件
# input → prompt → llm → parser → output
result = chain.invoke({"topic": "机器学习"})
```

**底层执行流程**：

```
┌──────────────────────────────────────────────────┐
│              RunnableSequence执行流程              │
│                                                  │
│  chain.invoke({"topic": "机器学习"})               │
│       ↓                                          │
│  1. prompt.invoke({"topic": "机器学习"})           │
│     → 输出: "请解释机器学习"                        │
│       ↓                                          │
│  2. llm.invoke("请解释机器学习")                    │
│     → 输出: AIMessage(content="机器学习是...")     │
│       ↓                                          │
│  3. parser.invoke(AIMessage)                      │
│     → 输出: "机器学习是..."（纯字符串）             │
│       ↓                                          │
│  返回最终结果                                      │
└──────────────────────────────────────────────────┘
```

### 组合二：并行执行（RunnableParallel）

同时执行多个分支，结果合并为字典：

```python
from langchain_core.runnables import RunnableParallel

# 并行执行多个链
parallel_chain = RunnableParallel({
    "translation": translate_prompt | llm,
    "summary": summary_prompt | llm,
    "sentiment": sentiment_prompt | llm,
})

# 一次调用，三个结果同时返回
results = parallel_chain.invoke({"text": "Hello World"})
# 输出: {
#     "translation": "你好世界",
#     "summary": "一句简单的问候",
#     "sentiment": "积极"
# }
```

### 组合三：灵活路由（RunnableLambda）

用自定义函数处理复杂逻辑：

```python
from langchain_core.runnables import RunnableLambda

def extract_key_info(x):
    """从LLM输出中提取关键信息"""
    return {"context": x.content, "question": x.content[:20]}

# 在链中间插入自定义逻辑
chain = (
    prompt
    | llm
    | RunnableLambda(extract_key_info)  # 自定义处理
    | next_step_prompt
    | llm
)
```

## 简单示例：构建一个完整的RAG链

### 场景：基于文档的智能问答

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# 1. 定义组件
retriever = vector_store.as_retriever(search_kwargs={"k": 3})

prompt = ChatPromptTemplate.from_template("""
根据以下上下文回答问题：
{context}

问题：{question}
""")

llm = ChatOpenAI(model="gpt-4o-mini")

# 2. 组合为RAG链
def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | RunnableLambda(format_docs), "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# 3. 使用（只需一行代码）
answer = rag_chain.invoke("LangChain的主要优势是什么？")
```

## 进阶理解：Runnable的设计哲学

### 开闭原则的实践

Runnable协议体现了**开闭原则**（对扩展开放，对修改封闭）：

```
┌──────────────────────────────────────────────────────┐
│              开闭原则在Runnable中的体现                 │
│                                                      │
│  如果你想添加新组件：                                  │
│  ┌──────────────────────────────────────┐             │
│  │  1. 实现invoke/stream/batch方法       │             │
│  │  2. 自动获得|操作符支持                │             │
│  │  3. 自动兼容所有现有链                 │             │
│  └──────────────────────────────────────┘             │
│                                                      │
│  你不需要：                                           │
│  • 修改现有链的代码                                   │
│  • 重写调用逻辑                                       │
│  • 处理兼容性                                         │
└──────────────────────────────────────────────────────┘
```

**自定义Runnable示例**：

```python
from langchain_core.runnables import Runnable

class MyCustomProcessor(Runnable):
    """自定义Runnable组件"""
    
    def invoke(self, input, config=None):
        # 处理输入
        return f"处理结果: {input}"
    
    def stream(self, input, config=None):
        # 支持流式输出
        result = self.invoke(input)
        for char in result:
            yield char

# 创建实例
custom = MyCustomProcessor()

# 可以直接放入链中使用
chain = prompt | llm | custom | parser
```

### Runnable内置的高级能力

除了基本的invoke/stream/batch，Runnable还提供丰富的链式方法：

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")

# 1. 配置包装：动态修改行为
configured_llm = llm.with_config({"temperature": 0.9})

# 2. 重试包装：自动重试失败请求
retry_llm = llm.with_retry(stop_after_attempt=3)

# 3. 回退包装：失败时切换到备用模型
fallback_llm = llm.with_fallbacks([backup_llm])

# 4. 监听器：监控执行过程
def on_start(input):
    print(f"开始处理: {input}")

def on_end(output):
    print(f"处理完成: {output}")

observed_llm = llm.with_listeners(on_start=on_start, on_end=on_end)

# 这些包装器可以组合使用，且不影响原有接口
final_llm = llm.with_config({"temperature": 0.7}).with_retry(stop_after_attempt=3)
```

Runnable支持定义输入输出的类型约束：

## 常见问题

### Q1: Runnable和普通函数有什么区别？

**A:** Runnable是一个协议（接口），它定义了标准化的执行方式：

| 特性 | 普通函数 | Runnable |
|------|---------|----------|
| 调用方式 | 各自不同 | 统一invoke/stream/batch |
| 组合能力 | 需要手动编排 | 支持管道符直接组合 |
| 流式支持 | 需要额外实现 | 内置stream方法 |
| 批量处理 | 需要循环调用 | 内置batch并行优化 |
| 配置传递 | 需要手动处理 | 支持with_config |

### Q2: 什么时候需要用RunnableLambda？

**A:** 当你需要在链中间插入自定义逻辑时：

```python
# ✅ 适合用RunnableLambda的场景
chain = (
    prompt
    | llm
    | RunnableLambda(lambda x: x.content.upper())  # 简单转换
    | save_to_db
)

# ✅ 复杂逻辑建议定义为独立函数
def process_llm_output(ai_message):
    content = ai_message.content
    # 复杂处理逻辑...
    return processed_data

chain = prompt | llm | RunnableLambda(process_llm_output)
```

### Q3: Runnable和LangGraph有什么关系？

**A:** Runnable是基础组件，LangGraph构建在Runnable之上：

- **Runnable**：提供单个组件的统一接口和简单的线性/并行组合
- **LangGraph**：在Runnable基础上添加了**状态管理、循环、条件分支**等高级编排能力

可以把Runnable理解为"砖块"，LangGraph是"建筑图纸"。

## 本节总结

### 核心认知
1. **Runnable是统一接口**：所有组件使用相同的invoke/stream/batch方法
2. **管道符是组合方式**：`|`操作符让组件像管道一样串联
3. **包装器是增强机制**：with_config/with_retry/with_fallbacks提供非侵入式增强

### 关键启示
- 统一接口降低了学习成本和使用复杂度
- 开闭原则让系统具备强大的扩展能力
- 组合优于继承，管道优于嵌套调用
