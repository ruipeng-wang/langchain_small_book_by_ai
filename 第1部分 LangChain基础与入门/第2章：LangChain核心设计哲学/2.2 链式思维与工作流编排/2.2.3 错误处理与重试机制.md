# 2.2.3 错误处理与重试机制

## 概念讲解：企业级应用中的容错设计哲学

在大规模AI应用中，错误处理不仅仅是技术实现，更是一种**系统设计哲学**。与传统应用程序不同，LLM应用面临独特的挑战：API限流、网络抖动、模型超时、上下文溢出、格式解析失败等不确定性因素。LangChain v1.2.22通过系统化的错误处理机制，将复杂的容错逻辑抽象为可配置的中间件和策略，体现了"声明式配置优于命令式代码"的设计理念。

### 错误处理的演进：从手动到声明式

**传统错误处理的局限性：**
1. **代码重复**：每个API调用都需要独立的错误处理逻辑
2. **策略分散**：重试、熔断、降级等策略难以统一管理
3. **监控缺失**：缺乏统一的错误监控和指标收集
4. **维护困难**：错误处理逻辑与业务逻辑深度耦合

**LangChain的错误处理哲学：**
1. **统一抽象**：所有组件（模型、工具、链）共享相同的错误处理接口
2. **声明式配置**：通过配置文件而非代码定义错误处理策略
3. **可组合性**：错误处理策略可以像乐高积木一样组合和嵌套
4. **框架透明**：业务逻辑与错误处理逻辑分离，代码更清晰

### 核心设计原则

1. **优雅降级（Graceful Degradation）**：系统在部分失败时仍能提供有限但可用的服务
2. **快速失败（Fail Fast）**：尽早发现并报告错误，避免错误传播
3. **自动恢复（Self-healing）**：系统能够从错误状态自动恢复
4. **可观察性（Observability）**：所有错误和重试行为都可监控和追踪

## 核心要点：LangChain v1.2.22错误处理体系

### 1. 三层错误处理架构

根据Context7最新文档，LangChain提供了完整的三层错误处理体系：

| 层级 | 组件 | 功能 | 适用场景 |
|------|------|------|----------|
| **模型层** | ModelRetryMiddleware | 处理模型API调用的错误 | API限流、网络超时、模型不可用 |
| **工具层** | ToolRetryMiddleware | 处理外部工具调用的错误 | 数据库连接失败、第三方API错误 |
| **系统层** | Circuit Breaker + Fallback | 整体系统容错 | 级联故障、资源耗尽 |

### 2. 重试策略类型

1. **指数退避重试**：每次重试的延迟时间按指数增长，避免加剧服务压力
2. **抖动重试**：在重试延迟中添加随机抖动，避免多个客户端同时重试
3. **条件重试**：根据错误类型决定是否重试（如只重试瞬态错误）
4. **降级重试**：重试失败后使用备用方案或简化服务

### 3. 关键配置参数

根据Context7文档，关键配置参数包括：
- `max_retries`：最大重试次数（默认2次，共3次尝试）
- `backoff_factor`：退避因子（默认2.0，指数退避）
- `initial_delay`：初始延迟（默认1秒）
- `max_delay`：最大延迟（默认60秒，限制指数增长）
- `jitter`：是否添加抖动（默认True，±25%随机变化）
- `retry_on`：重试条件（错误类型或自定义函数）
- `on_failure`：失败处理策略（"continue"、"error"或自定义函数）

```python
    max_delay=30.0,         # 最大延迟30秒

    """自定义重试条件判断"""

    if isinstance(error, TimeoutError):
        return True
    # 或检查特定HTTP状态码
    if hasattr(error, "status_code"):
        return error.status_code in (429, 503)  # 429:限流, 503:服务不可用
    return False

# 使用自定义条件

    max_retries=3,
    retry_on=should_retry,  # 使用自定义条件函数
```

### 应用1：智能重试策略管理器

```python
# 智能重试策略管理器
from typing import Dict, Any, Optional
from dataclasses import dataclass
from datetime import datetime, timedelta
import random

@dataclass
class RetryPolicy:
    """重试策略配置"""
    name: str
    max_retries: int
    backoff_factor: float
    initial_delay: float
    max_delay: float
    jitter: bool
    retry_conditions: list
    fallback_strategy: str

class IntelligentRetryManager:
    """智能重试管理器"""
    
    def __init__(self):
        self.policies = self._load_policies()
        self.metrics = {
            "total_requests": 0,
            "successful_requests": 0,
            "failed_requests": 0,
            "retry_attempts": 0
        }
        self.circuit_breakers = {}  # 熔断器状态
    
    def _load_policies(self) -> Dict[str, RetryPolicy]:
        """加载重试策略"""
        return {
            "aggressive": RetryPolicy(
                name="aggressive",
                max_retries=5,
                backoff_factor=1.5,
                initial_delay=0.5,
                max_delay=30.0,
                jitter=True,
                retry_conditions=["timeout", "rate_limit"],
                fallback_strategy="use_cache"
            ),
            "conservative": RetryPolicy(
                name="conservative",
                max_retries=2,
                backoff_factor=2.0,
                initial_delay=2.0,
                max_delay=60.0,
                jitter=True,
                retry_conditions=["timeout"],
                fallback_strategy="return_error"
            ),
            "critical": RetryPolicy(
                name="critical",
                max_retries=1,
                backoff_factor=0.0,  # 无退避
                initial_delay=0.0,
                max_delay=0.0,
                jitter=False,
                retry_conditions=[],  # 不重试
                fallback_strategy="fail_fast"
            )
        }
    
    def select_policy(self, operation_type: str, error: Optional[Exception] = None) -> RetryPolicy:
        """根据操作类型和错误选择策略"""
        self.metrics["total_requests"] += 1
        
        if operation_type == "model_inference":
            # 模型推理：使用保守策略
            return self.policies["conservative"]
        elif operation_type == "data_processing":
            # 数据处理：使用激进策略
            return self.policies["aggressive"]
        elif operation_type == "user_authentication":
            # 用户认证：关键操作，不重试
            return self.policies["critical"]
        else:
            # 默认策略
            return self.policies["conservative"]
    
    def should_retry(self, policy: RetryPolicy, error: Exception, attempt: int) -> bool:
        """判断是否应该重试"""
        if attempt >= policy.max_retries:
            return False
        
        # 检查错误类型是否在重试条件中
        error_str = str(error).lower()
        for condition in policy.retry_conditions:
            if condition in error_str:
                self.metrics["retry_attempts"] += 1
                return True
        
        return False
    
    def calculate_delay(self, policy: RetryPolicy, attempt: int) -> float:
        """计算重试延迟"""
        # 基础延迟计算
        delay = policy.initial_delay * (policy.backoff_factor ** attempt)
        
        # 限制最大延迟
        delay = min(delay, policy.max_delay)
        
        # 添加抖动
        if policy.jitter:
            jitter_factor = random.uniform(0.75, 1.25)  # ±25%抖动
            delay *= jitter_factor
        
        return delay
    
    def get_metrics_summary(self) -> Dict[str, Any]:
        """获取指标摘要"""
        success_rate = (
            self.metrics["successful_requests"] / self.metrics["total_requests"] * 100
            if self.metrics["total_requests"] > 0 else 0
        )
        
        return {
            "total_requests": self.metrics["total_requests"],
            "success_rate": f"{success_rate:.1f}%",
            "retry_rate": f"{(self.metrics['retry_attempts'] / self.metrics['total_requests'] * 100):.1f}%",
            "failure_rate": f"{(self.metrics['failed_requests'] / self.metrics['total_requests'] * 100):.1f}%"
        }

# 使用智能重试管理器
retry_manager = IntelligentRetryManager()

# 模拟错误处理
def process_with_retry(operation_type: str, data: Any):
    """使用智能重试处理操作"""
    attempt = 0
    last_error = None
    
    # 选择策略
    policy = retry_manager.select_policy(operation_type)
    
    while attempt <= policy.max_retries:
        try:
            # 执行操作（这里简化为模拟）
            if random.random() < 0.3:  # 30%失败率
                raise TimeoutError("Simulated timeout")
            
            # 成功
            retry_manager.metrics["successful_requests"] += 1
            return "Success"
            
        except Exception as e:
            last_error = e
            attempt += 1
            
            if retry_manager.should_retry(policy, e, attempt):
                delay = retry_manager.calculate_delay(policy, attempt)
                print(f"尝试 {attempt} 失败，{delay:.2f}秒后重试...")
                import time
                time.sleep(delay)
            else:
                break
    
    # 所有尝试都失败
    retry_manager.metrics["failed_requests"] += 1
    
    # 执行降级策略
    if policy.fallback_strategy == "use_cache":
        return "Using cached response"
    elif policy.fallback_strategy == "return_error":
        return f"Operation failed: {last_error}"
    elif policy.fallback_strategy == "fail_fast":
        raise last_error
    
    return "Fallback response"

# 测试
results = []
for i in range(10):
    result = process_with_retry("model_inference", f"data_{i}")
    results.append(result)

print("\n处理结果统计:")
print(f"成功: {results.count('Success')}")
print(f"使用缓存: {results.count('Using cached response')}")
print(f"失败: {len([r for r in results if 'failed' in str(r)])}")

print("\n性能指标:")
print(retry_manager.get_metrics_summary())
```

### 应用2：多层熔断器系统

```python
# 多层熔断器系统
from enum import Enum
from typing import Set

class CircuitState(Enum):
    """熔断器状态"""
    CLOSED = "CLOSED"    # 正常状态，允许请求
    OPEN = "OPEN"        # 熔断状态，拒绝请求
    HALF_OPEN = "HALF_OPEN"  # 半开状态，允许少量测试请求

class MultiLayerCircuitBreaker:
    """多层熔断器系统"""
    
    def __init__(self, layers: Dict[str, Dict]):
        """
        初始化多层熔断器
        
        Args:
            layers: 各层配置，例如：
                {
                    "api": {"failure_threshold": 5, "reset_timeout": 60},
                    "database": {"failure_threshold": 3, "reset_timeout": 120},
                    "cache": {"failure_threshold": 10, "reset_timeout": 30}
                }
        """
        self.layers = {}
        for layer_name, config in layers.items():
            self.layers[layer_name] = {
                "config": config,
                "state": CircuitState.CLOSED,
                "failure_count": 0,
                "last_failure_time": None,
                "half_open_successes": 0
            }
        
        # 依赖关系图
        self.dependencies = {
            "api": ["database", "cache"],
            "database": [],
            "cache": []
        }
    
    def can_execute(self, layer: str) -> bool:
        """检查指定层是否允许执行"""
        layer_info = self.layers.get(layer)
        if not layer_info:
            return True  # 未知层默认允许
        
        # 检查依赖层状态
        for dep in self.dependencies.get(layer, []):
            if not self.can_execute(dep):
                print(f"依赖层 {dep} 熔断，阻止 {layer} 执行")
                return False
        
        state = layer_info["state"]
        
        if state == CircuitState.OPEN:
            # 检查是否超过重置超时
            reset_timeout = layer_info["config"]["reset_timeout"]
            last_failure = layer_info["last_failure_time"]
            
            if last_failure:
                from datetime import datetime
                elapsed = (datetime.now() - last_failure).total_seconds()
                if elapsed > reset_timeout:
                    # 切换到半开状态
                    layer_info["state"] = CircuitState.HALF_OPEN
                    layer_info["half_open_successes"] = 0
                    print(f"层 {layer} 从OPEN切换到HALF_OPEN")
                    return True
            return False
        
        elif state == CircuitState.HALF_OPEN:
            # 半开状态：允许少量测试请求
            max_test_requests = 3  # 最多允许3个测试请求
            if layer_info["half_open_successes"] >= max_test_requests:
                # 测试成功，切换到CLOSED状态
                layer_info["state"] = CircuitState.CLOSED
                layer_info["failure_count"] = 0
                print(f"层 {layer} 从HALF_OPEN切换到CLOSED")
                return True
            return True
        
        return True  # CLOSED状态
    
    def record_success(self, layer: str):
        """记录成功"""
        layer_info = self.layers.get(layer)
        if not layer_info:
            return
        
        if layer_info["state"] == CircuitState.HALF_OPEN:
            layer_info["half_open_successes"] += 1
        
        # 重置失败计数
        layer_info["failure_count"] = 0
    
    def record_failure(self, layer: str, error: Exception):
        """记录失败"""
        layer_info = self.layers.get(layer)
        if not layer_info:
            return
        
        from datetime import datetime
        
        layer_info["failure_count"] += 1
        layer_info["last_failure_time"] = datetime.now()
        
        failure_threshold = layer_info["config"]["failure_threshold"]
        
        if layer_info["failure_count"] >= failure_threshold:
            layer_info["state"] = CircuitState.OPEN
            print(f"层 {layer} 熔断（失败次数: {layer_info['failure_count']}）")
        
        # 传播失败到依赖层
        self._propagate_failure(layer, error)
    
    def _propagate_failure(self, source_layer: str, error: Exception):
        """传播失败到上游依赖层"""
        # 查找哪些层依赖于此层
        for layer_name, deps in self.dependencies.items():
            if source_layer in deps:
                # 上游层也记录部分失败
                self.layers[layer_name]["failure_count"] += 0.5  # 部分失败
                
                # 如果部分失败累积到阈值，也触发熔断
                if self.layers[layer_name]["failure_count"] >= self.layers[layer_name]["config"]["failure_threshold"]:
                    self.layers[layer_name]["state"] = CircuitState.OPEN
                    self.layers[layer_name]["last_failure_time"] = datetime.now()
                    print(f"层 {layer_name} 因依赖层 {source_layer} 失败而熔断")
    
    def get_status_report(self) -> Dict[str, Any]:
        """获取状态报告"""
        report = {}
        for layer_name, info in self.layers.items():
            report[layer_name] = {
                "state": info["state"].value,
                "failure_count": info["failure_count"],
                "last_failure": info["last_failure_time"].isoformat() if info["last_failure_time"] else None,
                "half_open_successes": info.get("half_open_successes", 0)
            }
        return report

# 使用多层熔断器
layers_config = {
    "api": {"failure_threshold": 5, "reset_timeout": 60},
    "database": {"failure_threshold": 3, "reset_timeout": 120},
    "cache": {"failure_threshold": 10, "reset_timeout": 30}
}

circuit_breaker = MultiLayerCircuitBreaker(layers_config)

# 模拟服务调用
def call_service_with_circuit_breaker(layer: str, operation: callable):
    """使用熔断器保护的服务调用"""
    if not circuit_breaker.can_execute(layer):
        raise Exception(f"Circuit breaker for {layer} is OPEN")
    
    try:
        result = operation()
        circuit_breaker.record_success(layer)
        return result
    except Exception as e:
        circuit_breaker.record_failure(layer, e)
        raise e

# 测试
def simulate_api_call():
    """模拟API调用"""
    import random
    if random.random() < 0.7:  # 70%成功率
        return "API Success"
    else:
        raise Exception("API Failure")

def simulate_database_call():
    """模拟数据库调用"""
    import random
    if random.random() < 0.8:  # 80%成功率
        return "DB Success"
    else:
        raise Exception("DB Failure")

# 执行测试
print("多层熔断器测试:")
for i in range(10):
    try:
        result = call_service_with_circuit_breaker("api", simulate_api_call)
        print(f"调用 {i}: API成功")
    except Exception as e:
        print(f"调用 {i}: API失败 - {e}")
    
    try:
        result = call_service_with_circuit_breaker("database", simulate_database_call)
        print(f"调用 {i}: 数据库成功")
    except Exception as e:
        print(f"调用 {i}: 数据库失败 - {e}")

print("\n熔断器状态报告:")
print(circuit_breaker.get_status_report())
```

## 常见问题

### Q1: 什么时候应该使用重试机制？什么时候不应该？
**A:** 应该使用重试的情况：
1. **瞬态错误**：网络超时、API限流、临时服务不可用
2. **可恢复错误**：连接中断后可以重新建立
3. **外部依赖错误**：第三方服务暂时不可用

不应该使用重试的情况：
1. **业务逻辑错误**：参数错误、权限不足（重试不会改变结果）
2. **永久性错误**：配置错误、代码bug
3. **用户输入错误**：格式错误、必填项缺失
4. **安全相关错误**：认证失败、访问被拒绝

### Q2: ModelRetryMiddleware和ToolRetryMiddleware有什么区别？
**A:** 主要区别：
1. **作用对象不同**：
   - `ModelRetryMiddleware`：专门处理模型API调用的错误
   - `ToolRetryMiddleware`：处理外部工具（如数据库、API）调用的错误
2. **错误类型不同**：
   - 模型错误：API限流、网络超时、模型不可用
   - 工具错误：连接失败、超时、格式错误
3. **配置重点不同**：
   - 模型重试更关注API限制和配额管理
   - 工具重试更关注连接稳定性和数据一致性

### Q3: 如何配置合适的重试参数？
**A:** 推荐配置原则：
1. **重试次数**：根据错误恢复时间确定（通常2-5次）
2. **初始延迟**：根据服务响应时间确定（通常1-2秒）
3. **退避因子**：根据服务压力确定（通常1.5-2.0）
4. **最大延迟**：根据用户体验容忍度确定（通常30-60秒）
5. **抖动**：在分布式系统中应该启用（避免同步重试）

### Q4: 熔断器和重试机制如何配合使用？
**A:** 推荐配合策略：
1. **分层保护**：重试处理单个请求的瞬态错误，熔断器保护整个服务
2. **熔断触发**：当重试失败率达到阈值时触发熔断
3. **半开状态**：熔断器半开时使用保守重试策略（较少重试次数）
4. **监控联动**：重试指标作为熔断器决策的重要输入

### Q5: 如何监控和调试错误处理机制？
**A:** 建议监控指标：
1. **成功率**：成功请求占总请求的比例
2. **重试率**：需要重试的请求比例
3. **平均重试次数**：每次失败请求的平均重试次数
4. **错误类型分布**：各类错误的发生频率
5. **熔断器状态**：各层熔断器的开闭状态和时间
6. **降级比例**：使用降级策略的请求比例
