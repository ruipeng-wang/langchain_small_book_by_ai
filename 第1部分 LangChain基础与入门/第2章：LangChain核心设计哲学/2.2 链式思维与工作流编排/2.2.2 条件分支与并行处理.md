# 2.2.2 条件分支与并行处理

## 从线性思维到分支并行的演进

### 传统线性思维的局限性

在复杂的AI应用中，纯线性执行流程往往无法满足实际需求。传统开发方式在处理条件分支和并行任务时面临诸多挑战：

```python
# 传统线性思维示例：客户支持系统
class LinearCustomerSupport:
    def __init__(self):
        self.classifier = IntentClassifier()
        self.faq_processor = FAQProcessor()
        self.ticket_creator = TicketCreator()
        self.escalation_manager = EscalationManager()
        self.sentiment_analyzer = SentimentAnalyzer()
    
    def handle_customer_query(self, query: str, customer_info: Dict[str, Any]) -> Dict[str, Any]:
        """线性处理客户查询（问题：无法处理复杂分支）"""
        response = {"status": "pending", "steps": []}
        
        try:
            # 步骤1: 分析意图（硬编码顺序）
            response["steps"].append("intent_classification")
            intent = self.classifier.classify(query)
            
            # 步骤2: 根据意图选择处理路径
            if intent == "faq":
                # FAQ处理路径
                response["steps"].append("faq_processing")
                answer = self.faq_processor.find_answer(query)
                response["answer"] = answer
                response["resolution"] = "faq_resolved"
                
            elif intent == "complaint":
                # 投诉处理路径
                response["steps"].append("sentiment_analysis")
                sentiment = self.sentiment_analyzer.analyze(query)
                
                if sentiment == "angry":
                    # 愤怒客户需要升级
                    response["steps"].append("escalation")
                    escalation_result = self.escalation_manager.escalate(
                        query, customer_info, priority="high"
                    )
                    response["ticket_id"] = escalation_result["ticket_id"]
                    response["resolution"] = "escalated"
                else:
                    # 普通投诉创建工单
                    response["steps"].append("ticket_creation")
                    ticket = self.ticket_creator.create_ticket(
                        query, customer_info, category="complaint"
                    )
                    response["ticket_id"] = ticket["id"]
                    response["resolution"] = "ticket_created"
                    
            elif intent == "technical":
                # 技术问题路径
                response["steps"].append("technical_analysis")
                # 这里可能有复杂的分析逻辑...
                # 但都是线性执行，无法并行处理子任务
                
                # 需要同时检查多个方面（但只能顺序执行）
                issue_type = self._analyze_issue_type(query)
                severity = self._assess_severity(query)
                affected_systems = self._identify_affected_systems(query)
                
                # 问题：这些分析可以并行，但硬编码为顺序
                response["technical_details"] = {
                    "issue_type": issue_type,
                    "severity": severity,
                    "affected_systems": affected_systems
                }
                
                # 根据分析结果决定下一步
                if severity == "critical":
                    response["steps"].append("critical_escalation")
                    escalation = self.escalation_manager.escalate(
                        query, customer_info, priority="critical"
                    )
                    response["ticket_id"] = escalation["ticket_id"]
                else:
                    response["steps"].append("standard_ticket")
                    ticket = self.ticket_creator.create_ticket(
                        query, customer_info, category="technical"
                    )
                    response["ticket_id"] = ticket["id"]
                    
            else:
                # 未知意图
                response["steps"].append("fallback")
                response["answer"] = "I'm not sure how to help with that."
                response["resolution"] = "unknown_intent"
            
            response["status"] = "completed"
            
        except Exception as e:
            response["status"] = "error"
            response["error"] = str(e)
        
        return response
    
    def _analyze_issue_type(self, query: str) -> str:
        # 模拟耗时分析
        import time
        time.sleep(0.5)
        return "performance" if "slow" in query else "functionality"
    
    def _assess_severity(self, query: str) -> str:
        time.sleep(0.3)
        return "critical" if "urgent" in query else "normal"
    
    def _identify_affected_systems(self, query: str) -> List[str]:
        time.sleep(0.4)
        systems = []
        if "database" in query:
            systems.append("database")
        if "api" in query:
            systems.append("api")
        if "ui" in query:
            systems.append("ui")
        return systems
```

**传统线性思维的局限：**

1. **条件分支复杂**：嵌套的if-else语句难以理解和维护
2. **无法并行执行**：可以并行的任务被迫顺序执行，性能低下
3. **代码重复**：相似的分支逻辑在不同地方重复
4. **可扩展性差**：添加新的处理路径需要修改核心逻辑
5. **错误处理困难**：每个分支需要单独的错误处理
6. **测试复杂度高**：需要测试所有可能的分支组合
7. **业务逻辑与执行逻辑混杂**：难以分离关注点

### LangChain的分支并行解决方案

LangChain v1.2.22通过**RunnableBranch**和**RunnableParallel**提供了声明式的条件分支和并行处理方案：

```python
# LangChain分支并行示例：客户支持系统
from langchain_core.runnables import RunnableBranch, RunnableParallel, RunnableLambda
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from typing import Dict, Any, List
import asyncio

class BranchingCustomerSupport:
    def __init__(self):
        # 初始化LLM
        self.llm = ChatOpenAI(model="gpt-4", temperature=0.7)
        
        # 1. 意图分类组件
        self.intent_classifier = RunnableLambda(
            lambda x: self._classify_intent(x["query"])
        )
        
        # 2. FAQ处理组件
        self.faq_prompt = ChatPromptTemplate.from_messages([
            ("system", "You are a helpful customer support assistant."),
            ("human", "Answer this FAQ question: {query}")
        ])
        self.faq_processor = self.faq_prompt | self.llm | StrOutputParser()
        
        # 3. 情感分析组件
        self.sentiment_analyzer = RunnableLambda(
            lambda x: self._analyze_sentiment(x["query"])
        )
        
        # 4. 工单创建组件
        self.ticket_creator = RunnableLambda(
            lambda x: self._create_ticket(x["query"], x["customer_info"], x.get("category", "general"))
        )
        
        # 5. 升级管理组件
        self.escalation_manager = RunnableLambda(
            lambda x: self._escalate_issue(x["query"], x["customer_info"], x.get("priority", "normal"))
        )
        
        # 6. 技术分析并行组件
        self.technical_analyzer = RunnableParallel({
            "issue_type": RunnableLambda(lambda x: self._analyze_issue_type(x["query"])),
            "severity": RunnableLambda(lambda x: self._assess_severity(x["query"])),
            "affected_systems": RunnableLambda(lambda x: self._identify_affected_systems(x["query"]))
        })
        
        # 构建主工作流
        self.support_workflow = (
            # 初始输入处理
            RunnableLambda(lambda x: {
                "query": x["query"],
                "customer_info": x.get("customer_info", {}),
                "original_input": x
            })
            # 意图分类
            | RunnableParallel({
                "intent": self.intent_classifier,
                "query": RunnableLambda(lambda x: x["query"]),
                "customer_info": RunnableLambda(lambda x: x["customer_info"])
            })
            # 基于意图的分支处理
            | RunnableBranch(
                # 分支1: FAQ意图
                (
                    lambda x: x["intent"] == "faq",
                    RunnableSequence(
                        RunnableLambda(lambda x: {"query": x["query"]}),
                        self.faq_processor,
                        RunnableLambda(lambda answer: {
                            "resolution": "faq_resolved",
                            "answer": answer,
                            "status": "completed"
                        })
                    )
                ),
                # 分支2: 投诉意图
                (
                    lambda x: x["intent"] == "complaint",
                    RunnableSequence(
                        # 并行执行情感分析和基本信息提取
                        RunnableParallel({
                            "sentiment": RunnableLambda(lambda x: x["query"]) | self.sentiment_analyzer,
                            "query": RunnableLambda(lambda x: x["query"]),
                            "customer_info": RunnableLambda(lambda x: x["customer_info"])
                        })
                        # 基于情感的分支
                        | RunnableBranch(
                            # 子分支2.1: 愤怒客户（需要升级）
                            (
                                lambda x: x["sentiment"] == "angry",
                                RunnableSequence(
                                    RunnableLambda(lambda x: {
                                        "query": x["query"],
                                        "customer_info": x["customer_info"],
                                        "priority": "high"
                                    }),
                                    self.escalation_manager,
                                    RunnableLambda(lambda result: {
                                        "resolution": "escalated",
                                        "ticket_id": result["ticket_id"],
                                        "status": "completed"
                                    })
                                )
                            ),
                            # 子分支2.2: 普通投诉（创建工单）
                            RunnableSequence(
                                RunnableLambda(lambda x: {
                                    "query": x["query"],
                                    "customer_info": x["customer_info"],
                                    "category": "complaint"
                                }),
                                self.ticket_creator,
                                RunnableLambda(lambda result: {
                                    "resolution": "ticket_created",
                                    "ticket_id": result["ticket_id"],
                                    "status": "completed"
                                })
                            )
                        )
                    )
                ),
                # 分支3: 技术问题意图
                (
                    lambda x: x["intent"] == "technical",
                    RunnableSequence(
                        # 并行技术分析
                        RunnableLambda(lambda x: {"query": x["query"]})
                        | self.technical_analyzer
                        # 基于严重程度的分支
                        | RunnableBranch(
                            # 子分支3.1: 严重问题（紧急升级）
                            (
                                lambda x: x["severity"] == "critical",
                                RunnableSequence(
                                    RunnableLambda(lambda x: {
                                        "query": x["original_input"]["query"],
                                        "customer_info": x["original_input"]["customer_info"],
                                        "priority": "critical",
                                        "technical_details": {
                                            "issue_type": x["issue_type"],
                                            "severity": x["severity"],
                                            "affected_systems": x["affected_systems"]
                                        }
                                    }),
                                    self.escalation_manager,
                                    RunnableLambda(lambda result: {
                                        "resolution": "critical_escalation",
                                        "ticket_id": result["ticket_id"],
                                        "technical_details": result.get("technical_details", {}),
                                        "status": "completed"
                                    })
                                )
                            ),
                            # 子分支3.2: 普通技术问题（创建工单）
                            RunnableSequence(
                                RunnableLambda(lambda x: {
                                    "query": x["original_input"]["query"],
                                    "customer_info": x["original_input"]["customer_info"],
                                    "category": "technical",
                                    "technical_details": {
                                        "issue_type": x["issue_type"],
                                        "severity": x["severity"],
                                        "affected_systems": x["affected_systems"]
                                    }
                                }),
                                self.ticket_creator,
                                RunnableLambda(lambda result: {
                                    "resolution": "standard_ticket",
                                    "ticket_id": result["ticket_id"],
                                    "technical_details": result.get("technical_details", {}),
                                    "status": "completed"
                                })
                            )
                        )
                    )
                ),
                # 默认分支: 未知意图
                RunnableSequence(
                    RunnableLambda(lambda x: {"query": x["query"]}),
                    RunnableLambda(lambda x: {
                        "resolution": "unknown_intent",
                        "answer": "I'm not sure how to help with that.",
                        "status": "completed"
                    })
                )
            )
            # 添加元数据和步骤跟踪
            | RunnableLambda(lambda x: {
                **x,
                "timestamp": time.time(),
                "workflow_version": "1.0"
            })
        )
    
    # 辅助方法（模拟实现）
    def _classify_intent(self, query: str) -> str:
        query_lower = query.lower()
        if any(word in query_lower for word in ["how", "what", "why", "faq"]):
            return "faq"
        elif any(word in query_lower for word in ["problem", "issue", "error", "not working"]):
            return "technical"
        elif any(word in query_lower for word in ["complaint", "angry", "unhappy", "dissatisfied"]):
            return "complaint"
        return "unknown"
    
    def _analyze_sentiment(self, query: str) -> str:
        query_lower = query.lower()
        if any(word in query_lower for word in ["angry", "furious", "outraged", "terrible"]):
            return "angry"
        return "neutral"
    
    def _create_ticket(self, query: str, customer_info: Dict, category: str) -> Dict:
        # 模拟创建工单
        import uuid
        return {
            "ticket_id": f"TICKET-{uuid.uuid4().hex[:8]}",
            "query": query,
            "customer": customer_info.get("id", "unknown"),
            "category": category,
            "created_at": time.time()
        }
    
    def _escalate_issue(self, query: str, customer_info: Dict, priority: str) -> Dict:
        # 模拟升级处理
        import uuid
        return {
            "ticket_id": f"ESCALATED-{uuid.uuid4().hex[:8]}",
            "query": query,
            "customer": customer_info.get("id", "unknown"),
            "priority": priority,
            "escalated_at": time.time(),
            "assigned_to": "senior_support"
        }
    
    def _analyze_issue_type(self, query: str) -> str:
        import time
        time.sleep(0.5)  # 模拟耗时操作
        return "performance" if "slow" in query.lower() else "functionality"
    
    def _assess_severity(self, query: str) -> str:
        import time
        time.sleep(0.3)
        return "critical" if "urgent" in query.lower() else "normal"
    
    def _identify_affected_systems(self, query: str) -> List[str]:
        import time
        time.sleep(0.4)
        query_lower = query.lower()
        systems = []
        if "database" in query_lower:
            systems.append("database")
        if "api" in query_lower:
            systems.append("api")
        if "ui" in query_lower or "interface" in query_lower:
            systems.append("ui")
        return systems
    
    def handle_customer_query(self, query: str, customer_info: Dict[str, Any] = None) -> Dict[str, Any]:
        """使用分支并行工作流处理客户查询"""
        return self.support_workflow.invoke({
            "query": query,
            "customer_info": customer_info or {}
        })
    
    async def handle_customer_query_async(self, query: str, customer_info: Dict[str, Any] = None) -> Dict[str, Any]:
        """异步处理客户查询"""
        return await self.support_workflow.ainvoke({
            "query": query,
            "customer_info": customer_info or {}
        })

# 使用示例
support = BranchingCustomerSupport()

# 测试不同场景
test_cases = [
    ("How do I reset my password?", {"id": "user123", "name": "John Doe"}),  # FAQ
    ("The system is very slow when loading reports", {"id": "user456", "plan": "premium"}),  # 技术问题
    ("I'm angry about the service outage yesterday!", {"id": "user789", "status": "vip"}),  # 愤怒投诉
    ("I want to cancel my subscription", {"id": "user101", "name": "Alice"}),  # 普通投诉
]

for query, customer_info in test_cases:
    result = support.handle_customer_query(query, customer_info)
    print(f"\n查询: {query[:50]}...")
    print(f"处理结果: {result['resolution']}")
    print(f"状态: {result['status']}")
    if 'ticket_id' in result:
        print(f"工单ID: {result['ticket_id']}")
    if 'answer' in result:
        print(f"回答: {result['answer'][:100]}...")
```

**LangChain分支并行的核心优势：**

1. **声明式分支**：使用RunnableBranch清晰定义条件逻辑
2. **天然并行**：RunnableParallel自动并行执行独立任务
3. **模块化设计**：每个分支独立，易于测试和维护
4. **性能优化**：并行执行大幅提升处理速度
5. **可组合性**：分支可以嵌套和组合
6. **错误隔离**：分支失败不影响其他分支
7. **动态路由**：支持基于运行时数据的条件判断

# 传统方式：硬编码内容路由
class HardcodedContentRouter:
    def route_content(self, content: str) -> str:
        """硬编码的内容路由"""
        content_lower = content.lower()
        
        # 复杂的条件逻辑
        if "weather" in content_lower:
            if "forecast" in content_lower:
                return "weather_forecast"
            elif "current" in content_lower:
                return "current_weather"
            else:
                return "general_weather"
        
        elif "news" in content_lower:
            if "sports" in content_lower:
                return "sports_news"
            elif "business" in content_lower:
                return "business_news"
            elif "technology" in content_lower:
                return "tech_news"
            else:
                return "general_news"
        
        elif "calculation" in content_lower or "math" in content_lower:
            return "calculator"
        
        elif "translate" in content_lower:
            # 提取目标语言
            if "to french" in content_lower:
                return "translate_french"
            elif "to spanish" in content_lower:
                return "translate_spanish"
            else:
                return "translate_general"
        
        else:
            return "general_chat"
    
    def process_content(self, content: str) -> str:
        """基于路由处理内容"""
        route = self.route_content(content)
        
        if route == "weather_forecast":
            return self._process_weather_forecast(content)
        elif route == "current_weather":
            return self._process_current_weather(content)
        elif route == "sports_news":
            return self._process_sports_news(content)
        # ... 更多elif分支
        else:
            return self._process_general_chat(content)
    
    def _process_weather_forecast(self, content: str) -> str:
        return f"Weather forecast for: {content}"
    
    def _process_current_weather(self, content: str) -> str:
        return f"Current weather: {content}"
    
    def _process_sports_news(self, content: str) -> str:
        return f"Sports news: {content}"
    
    def _process_general_chat(self, content: str) -> str:

from langchain_core.runnables import RunnableBranch

class DynamicContentRouter:
    def __init__(self):
        # 定义路由条件
        self.content_router = RunnableBranch(
            # 天气相关
            (
                lambda x: "weather" in x.lower(),

                    # 一般天气
                    RunnableLambda(lambda x: {"route": "general_weather", "content": x})
                )
            ),
            # 新闻相关
            (
                lambda x: "news" in x.lower(),
                RunnableBranch(
                    # 体育新闻
                    (lambda x: "sports" in x.lower(),
                     RunnableLambda(lambda x: {"route": "sports_news", "content": x})),
                    # 商业新闻
                    (lambda x: "business" in x.lower(),
                     RunnableLambda(lambda x: {"route": "business_news", "content": x})),
                    # 科技新闻
                    (lambda x: "technology" in x.lower() or "tech" in x.lower(),

                    # 一般新闻
                    RunnableLambda(lambda x: {"route": "general_news", "content": x})
                )
            ),
            # 计算相关
            (
                lambda x: "calculation" in x.lower() or "math" in x.lower(),
                RunnableLambda(lambda x: {"route": "calculator", "content": x})
            ),
            # 翻译相关
            (
                lambda x: "translate" in x.lower(),
                RunnableBranch(
                    # 法语翻译
                    (lambda x: "to french" in x.lower(),
                     RunnableLambda(lambda x: {"route": "translate_french", "content": x})),
                    # 西班牙语翻译
                    (lambda x: "to spanish" in x.lower(),
                     RunnableLambda(lambda x: {"route": "translate_spanish", "content": x})),
                    # 一般翻译
                    RunnableLambda(lambda x: {"route": "translate_general", "content": x})
                )
            ),
            # 默认路由
            RunnableLambda(lambda x: {"route": "general_chat", "content": x})
        )
        
        # 定义处理组件
        self.processors = {
            "weather_forecast": RunnableLambda(lambda x: f"Weather forecast for: {x['content']}"),
            "current_weather": RunnableLambda(lambda x: f"Current weather: {x['content']}"),
            "sports_news": RunnableLambda(lambda x: f"Sports news: {x['content']}"),
            "business_news": RunnableLambda(lambda x: f"Business news: {x['content']}"),
            "tech_news": RunnableLambda(lambda x: f"Tech news: {x['content']}"),
            "calculator": RunnableLambda(lambda x: f"Calculating: {x['content']}"),
            "translate_french": RunnableLambda(lambda x: f"Translating to French: {x['content']}"),
            "translate_spanish": RunnableLambda(lambda x: f"Translating to Spanish: {x['content']}"),
            "translate_general": RunnableLambda(lambda x: f"Translating: {x['content']}"),
            "general_chat": RunnableLambda(lambda x: f"General response to: {x['content']}"),
            "general_weather": RunnableLambda(lambda x: f"General weather: {x['content']}"),
            "general_news": RunnableLambda(lambda x: f"General news: {x['content']}"),
        }
        
        # 构建完整工作流
        self.workflow = (
            self.content_router
            | RunnableLambda(lambda x: {
                "route": x["route"],
                "content": x["content"],
                "processor": self.processors[x["route"]]
            })
            | RunnableLambda(lambda x: x["processor"].invoke(x))
        )
    
    def process_content(self, content: str) -> str:
        """声明式的内容路由和处理"""
        return self.workflow.invoke(content)

# 传统方式：复杂业务规则
class ComplexBusinessRules:
    def evaluate_loan_application(self, application: Dict[str, Any]) -> Dict[str, Any]:
        """复杂的贷款申请评估规则"""
        result = {
            "status": "pending",
            "decision": "undecided",
            "reasons": [],
            "conditions": []
        }
        
        # 规则1: 收入要求
        monthly_income = application.get("monthly_income", 0)
        loan_amount = application.get("loan_amount", 0)
        
        if monthly_income <= 0:
            result["decision"] = "rejected"
            result["reasons"].append("Invalid income")
            return result
        
        # 规则2: 债务收入比
        monthly_debt = application.get("monthly_debt", 0)
        debt_to_income = monthly_debt / monthly_income if monthly_income > 0 else float('inf')
        
        if debt_to_income > 0.5:
            result["decision"] = "rejected"
            result["reasons"].append("Debt-to-income ratio too high")
            return result
        
        # 规则3: 信用评分
        credit_score = application.get("credit_score", 0)
        
        if credit_score < 600:
            result["decision"] = "rejected"
            result["reasons"].append("Credit score too low")
            return result
        
        # 规则4: 就业稳定性
        employment_years = application.get("employment_years", 0)
        
        if employment_years < 2:
            result["conditions"].append("Require co-signer")
        
        # 规则5: 贷款金额与收入比
        loan_to_income = loan_amount / (monthly_income * 12)  # 年收入
        
        if loan_to_income > 5:
            result["decision"] = "rejected"
            result["reasons"].append("Loan amount too high relative to income")
            return result
        
        # 规则6: 抵押品检查
        has_collateral = application.get("has_collateral", False)
        collateral_value = application.get("collateral_value", 0)
        
        if loan_amount > 100000 and not has_collateral:
            result["decision"] = "rejected"
            result["reasons"].append("Large loan requires collateral")
            return result
        
        # 规则7: 最终决策逻辑
        if credit_score >= 750 and debt_to_income < 0.3:
            result["decision"] = "approved"
            result["interest_rate"] = 3.5
        elif credit_score >= 700:
            result["decision"] = "approved"
            result["interest_rate"] = 4.5
        elif credit_score >= 650:
            result["decision"] = "approved"
            result["interest_rate"] = 6.0
            result["conditions"].append("Higher interest rate due to credit score")
        else:
            result["decision"] = "approved"
            result["interest_rate"] = 8.0
            result["conditions"].append("Highest interest rate tier")
        
        # 规则8: 特殊条件
        if application.get("is_first_time_buyer", False):
            result["conditions"].append("First-time buyer program")
            if result.get("interest_rate"):
                result["interest_rate"] -= 0.5
        
        result["status"] = "completed"
        return result

from pydantic import BaseModel, Field, validator
from typing import List, Optional

class LoanApplication(BaseModel):
    """贷款申请数据模型"""
    monthly_income: float = Field(..., gt=0, description="月收入")
    loan_amount: float = Field(..., gt=0, description="贷款金额")
    monthly_debt: float = Field(default=0, ge=0, description="月债务")
    credit_score: int = Field(..., ge=300, le=850, description="信用评分")
    employment_years: int = Field(default=0, ge=0, description="工作年限")
    has_collateral: bool = Field(default=False, description="是否有抵押品")
    collateral_value: float = Field(default=0, ge=0, description="抵押品价值")
    is_first_time_buyer: bool = Field(default=False, description="是否首次购买")

class LoanDecision(BaseModel):
    """贷款决策结果"""
    status: str
    decision: str
    reasons: List[str] = Field(default_factory=list)
    conditions: List[str] = Field(default_factory=list)
    interest_rate: Optional[float] = None
    message: Optional[str] = None

class DeclarativeBusinessRules:
    def __init__(self):
        # 构建规则评估管道
        self.rule_evaluator = (
            # 输入验证
            RunnableLambda(lambda x: LoanApplication(**x))
            # 并行评估所有规则
            | RunnableParallel({
                # 规则1: 收入有效性（已由Pydantic验证）
                "income_valid": RunnableLambda(lambda x: x.monthly_income > 0),
                # 规则2: 债务收入比
                "debt_to_income": RunnableLambda(
                    lambda x: x.monthly_debt / x.monthly_income if x.monthly_income > 0 else float('inf')
                ),
                # 规则3: 信用评分
                "credit_score": RunnableLambda(lambda x: x.credit_score),
                # 规则4: 就业稳定性
                "employment_years": RunnableLambda(lambda x: x.employment_years),
                # 规则5: 贷款收入比
                "loan_to_income": RunnableLambda(
                    lambda x: x.loan_amount / (x.monthly_income * 12)
                ),
                # 规则6: 抵押品检查
                "has_collateral": RunnableLambda(lambda x: x.has_collateral),
                "collateral_value": RunnableLambda(lambda x: x.collateral_value),
                # 原始申请数据
                "application": RunnableLambda(lambda x: x),
            })
            # 基于规则结果的分支决策
            | RunnableBranch(
                # 分支1: 收入无效
                (
                    lambda x: not x["income_valid"],
                    RunnableLambda(lambda x: LoanDecision(
                        status="completed",
                        decision="rejected",
                        reasons=["Invalid income"],
                        message="Monthly income must be positive"
                    ))
                ),
                # 分支2: 债务收入比过高
                (
                    lambda x: x["debt_to_income"] > 0.5,
                    RunnableLambda(lambda x: LoanDecision(
                        status="completed",
                        decision="rejected",
                        reasons=["Debt-to-income ratio too high"],
                        message=f"Debt-to-income ratio {x['debt_to_income']:.2f} exceeds 0.5 limit"
                    ))
                ),
                # 分支3: 信用评分过低
                (
                    lambda x: x["credit_score"] < 600,
                    RunnableLambda(lambda x: LoanDecision(
                        status="completed",
                        decision="rejected",
                        reasons=["Credit score too low"],
                        message=f"Credit score {x['credit_score']} below minimum 600"
                    ))
                ),
                # 分支4: 贷款金额过高
                (
                    lambda x: x["loan_to_income"] > 5,
                    RunnableLambda(lambda x: LoanDecision(
                        status="completed",
                        decision="rejected",
                        reasons=["Loan amount too high relative to income"],
                        message=f"Loan-to-income ratio {x['loan_to_income']:.2f} exceeds 5.0 limit"
                    ))
                ),
                # 分支5: 大额贷款无抵押品
                (
                    lambda x: x["application"].loan_amount > 100000 and not x["has_collateral"],
                    RunnableLambda(lambda x: LoanDecision(
                        status="completed",
                        decision="rejected",
                        reasons=["Large loan requires collateral"],
                        message="Loans over 100,000 require collateral"
                    ))
                ),
                # 默认分支: 批准贷款（根据信用评分确定利率）
                RunnableSequence(
                    # 确定利率等级
                    RunnableBranch(
                        # 优秀信用
                        (
                            lambda x: x["credit_score"] >= 750 and x["debt_to_income"] < 0.3,
                            RunnableLambda(lambda x: {"rate_tier": "excellent", "base_rate": 3.5})
                        ),
                        # 良好信用
                        (
                            lambda x: x["credit_score"] >= 700,
                            RunnableLambda(lambda x: {"rate_tier": "good", "base_rate": 4.5})
                        ),
                        # 一般信用
                        (
                            lambda x: x["credit_score"] >= 650,
                            RunnableLambda(lambda x: {"rate_tier": "fair", "base_rate": 6.0})
                        ),
                        # 基本批准
                        RunnableLambda(lambda x: {"rate_tier": "basic", "base_rate": 8.0})
                    )
                    # 应用条件和调整
                    | RunnableLambda(lambda x: {
                        **x,
                        "conditions": self._determine_conditions(x),
                        "final_rate": self._calculate_final_rate(x)
                    })
                    # 生成最终决策
                    | RunnableLambda(lambda x: LoanDecision(
                        status="completed",
                        decision="approved",
                        conditions=x["conditions"],
                        interest_rate=x["final_rate"],
                        message=f"Loan approved at {x['final_rate']}% interest rate"
                    ))
                )
            )
        )
    
    def _determine_conditions(self, data: Dict) -> List[str]:
        """确定贷款条件"""
        conditions = []
        app = data.get("application")
        
        if app and app.employment_years < 2:
            conditions.append("Require co-signer due to short employment history")
        
        if data.get("rate_tier") in ["fair", "basic"]:
            conditions.append(f"Higher interest rate due to {data['rate_tier']} credit tier")
        
        if app and app.is_first_time_buyer:
            conditions.append("First-time buyer program")
        
        return conditions
    
    def _calculate_final_rate(self, data: Dict) -> float:
        """计算最终利率"""
        base_rate = data.get("base_rate", 8.0)
        app = data.get("application")
        
        # 首次购买者折扣
        if app and app.is_first_time_buyer:
            base_rate -= 0.5
        
        # 就业稳定奖励
        if app and app.employment_years >= 5:
            base_rate -= 0.25
        
        # 抵押品奖励
        if app and app.has_collateral and app.collateral_value >= app.loan_amount * 0.5:
            base_rate -= 0.5
        
        return max(base_rate, 2.5)  # 最低利率2.5%
    
    def evaluate_loan_application(self, application: Dict[str, Any]) -> Dict[str, Any]:
        """声明式的贷款申请评估"""
        result = self.rule_evaluator.invoke(application)
        return result.dict()

    def __init__(self):

        self.step1_processor = RunnableLambda(self._step1_processing)
        self.step2_processor = RunnableLambda(self._step2_processing)

        )

        )

        import time
        time.sleep(0.05)
        return {**item, "step1": "processed"}
    
    def _step2_processing(self, item: Dict) -> Dict:
        import time
        time.sleep(0.08)
        return {**item, "step2": "processed"}

        return {**item, "step3": "processed", "final": True}
    
    def _process_stream_real_time(self, data_stream):
        """真正的流式并行处理"""
        from concurrent.futures import ThreadPoolExecutor, as_completed
        import queue
        
        # 创建处理队列
        processing_queue = queue.Queue()
        result_queue = queue.Queue()
        
        def worker():
            """工作线程：从队列获取项目并处理"""
            while True:
                try:
                    item = processing_queue.get(timeout=1)
                    if item is None:  # 终止信号
                        break
                    
                    try:
                        result = self.item_pipeline.invoke(item)
                        result_queue.put({"success": True, "result": result})
                    except Exception as e:
                        result_queue.put({"success": False, "error": str(e), "item": item})
                    
                    processing_queue.task_done()
                except queue.Empty:
                    continue
        
        # 启动工作线程池
        num_workers = 5
        with ThreadPoolExecutor(max_workers=num_workers) as executor:
            # 启动工作线程
            worker_futures = [executor.submit(worker) for _ in range(num_workers)]
            
            # 将数据流推送到处理队列
            items_processed = 0
            for item in data_stream:
                processing_queue.put(item)
                items_processed += 1
                
                # 立即获取并返回可用结果（真正的流式）
                while not result_queue.empty():
                    yield result_queue.get()
            
            # 等待所有项目处理完成
            processing_queue.join()
            
            # 发送终止信号
            for _ in range(num_workers):
                processing_queue.put(None)
            
            # 获取剩余结果
            while not result_queue.empty():
                yield result_queue.get()
            
            # 等待工作线程完成
            for future in worker_futures:
                future.result()
    
    def process_stream(self, data_stream):
        """流式处理数据"""
        return self.stream_pipeline.invoke(data_stream)

class ParallelismOptimization:
    def __init__(self):
        # 定义不同复杂度的处理任务
        self.simple_task = RunnableLambda(lambda x: self._simple_processing(x))
        self.medium_task = RunnableLambda(lambda x: self._medium_processing(x))

        # CPU密集型任务
        self.cpu_intensive_task = RunnableLambda(lambda x: self._cpu_intensive_processing(x))
    
    def _simple_processing(self, data: str) -> str:
        """简单处理（<10ms）"""
        import time
        time.sleep(0.005)  # 5ms
        return f"simple_{data}"
    
    def _medium_processing(self, data: str) -> str:
        """中等处理（50-100ms）"""
        import time
        time.sleep(0.075)  # 75ms
        return f"medium_{data}"
    
    def _complex_processing(self, data: str) -> str:
        """复杂处理（200-500ms）"""
        import time
        time.sleep(0.3)  # 300ms
        return f"complex_{data}"
    
    def _io_intensive_processing(self, data: str) -> str:
        """I/O密集型处理（模拟网络/磁盘I/O）"""
        import time
        time.sleep(0.5)  # 500ms I/O等待
        return f"io_{data}"
    
    def _cpu_intensive_processing(self, data: str) -> str:
        """CPU密集型处理（模拟计算）"""
        import time
        # 模拟CPU计算
        start = time.time()
        result = 0
        for i in range(100000):
            result += i * len(data)
        processing_time = time.time() - start
        return f"cpu_{data}_{result}_{processing_time:.3f}s"
    
    def optimize_parallel_execution(self, task_type: str, data_list: List[str], max_workers: int = None) -> List[str]:
        """优化并行执行"""
        from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
        import multiprocessing
        
        # 选择任务
        if task_type == "simple":
            task = self.simple_task
        elif task_type == "medium":
            task = self.medium_task
        elif task_type == "complex":
            task = self.complex_task
        elif task_type == "io":
            task = self.io_intensive_task
        elif task_type == "cpu":
            task = self.cpu_intensive_task
        else:
            raise ValueError(f"Unknown task type: {task_type}")
        
        # 根据任务类型选择执行器和优化参数
        if task_type == "cpu":
            # CPU密集型任务使用进程池
            executor_class = ProcessPoolExecutor
            # 推荐工作进程数 = CPU核心数
            if max_workers is None:
                max_workers = multiprocessing.cpu_count()
        else:
            # I/O密集型或混合任务使用线程池
            executor_class = ThreadPoolExecutor
            # 对于I/O密集型，可以设置更高的并行度
            if max_workers is None:
                if task_type == "io":
                    max_workers = 50  # 高并行度应对I/O等待
                elif task_type == "simple":
                    max_workers = 20  # 简单任务可以高并行
                else:
                    max_workers = 10  # 中等复杂度任务
        
        print(f"任务类型: {task_type}, 执行器: {executor_class.__name__}, 最大工作数: {max_workers}")
        
        # 执行并行处理
        results = []
        with executor_class(max_workers=max_workers) as executor:
            # 提交任务
            future_to_data = {
                executor.submit(task.invoke, data): data
                for data in data_list
            }
            
            # 收集结果
            for future in as_completed(future_to_data):
                try:
                    result = future.result()
                    results.append(result)
                except Exception as e:
                    results.append(f"error_{future_to_data[future]}: {e}")
        
        return results

# 容错并行处理实践
class FaultTolerantParallelProcessing:
    def __init__(self):
        # 定义可能失败的任务
        self.unreliable_task = RunnableLambda(self._unreliable_processing)
        self.slow_task = RunnableLambda(self._slow_processing)
        self.resource_intensive_task = RunnableLambda(self._resource_intensive_processing)
    
    def _unreliable_processing(self, data: str) -> str:
        """不可靠的处理（可能失败）"""
        import random
        import time
        
        time.sleep(0.1)
        
        # 模拟随机失败
        if random.random() < 0.3:  # 30%失败率
            raise ValueError(f"Processing failed for {data}")
        
        return f"processed_{data}"
    
    def _slow_processing(self, data: str) -> str:
        """慢速处理（可能超时）"""
        import random
        import time
        
        # 模拟随机延迟
        delay = random.uniform(0.05, 2.0)  # 50ms到2秒
        time.sleep(delay)
        
        return f"slow_processed_{data}_{delay:.3f}s"
    
    def _resource_intensive_processing(self, data: str) -> str:
        """资源密集型处理（可能耗尽资源）"""
        import random
        
        # 模拟内存使用
        large_list = [f"item_{i}" for i in range(10000)]
        
        # 模拟随机失败（内存不足）
        if random.random() < 0.2:  # 20%失败率
            # 模拟内存错误
            raise MemoryError(f"Out of memory processing {data}")
        
        return f"resource_intensive_{data}_{len(large_list)}"
    
    def fault_tolerant_parallel_process(self, data_list: List[str], task_type: str, 
                                       max_workers: int = 5, timeout: float = 1.0,
                                       max_retries: int = 3) -> List[Dict]:
        """容错并行处理"""
        from concurrent.futures import ThreadPoolExecutor, TimeoutError as FutureTimeoutError
        import asyncio
        
        # 选择任务
        if task_type == "unreliable":
            task = self.unreliable_task
        elif task_type == "slow":
            task = self.slow_task
        elif task_type == "resource":
            task = self.resource_intensive_task
        else:
            raise ValueError(f"Unknown task type: {task_type}")
        
        results = []
        
        with ThreadPoolExecutor(max_workers=max_workers) as executor:
            # 提交所有任务
            future_to_data = {}
            for data in data_list:
                future = executor.submit(task.invoke, data)
                future_to_data[future] = {
                    "data": data,
                    "retries": 0,
                    "succeeded": False
                }
            
            # 处理完成的任务
            completed = set()
            
            while len(completed) < len(future_to_data):
                # 检查已完成的任务
                for future in list(future_to_data.keys()):
                    if future in completed:
                        continue
                    
                    if future.done():
                        data_info = future_to_data[future]
                        
                        try:
                            result = future.result(timeout=0)  # 立即获取结果
                            results.append({
                                "status": "success",
                                "data": data_info["data"],
                                "result": result,
                                "retries": data_info["retries"]
                            })
                            data_info["succeeded"] = True
                            
                        except Exception as e:
                            # 任务失败
                            if data_info["retries"] < max_retries:
                                # 重试
                                data_info["retries"] += 1
                                print(f"重试 {data_info['data']} (尝试 {data_info['retries']}/{max_retries}): {e}")
                                
                                # 重新提交任务
                                new_future = executor.submit(task.invoke, data_info["data"])
                                future_to_data[new_future] = data_info
                                
                                # 移除旧future
                                del future_to_data[future]
                            else:
                                # 达到最大重试次数
                                results.append({
                                    "status": "failed",
                                    "data": data_info["data"],
                                    "error": str(e),
                                    "retries": data_info["retries"]
                                })
                        
                        completed.add(future)
                
                # 短暂休眠避免忙等待
                import time
                time.sleep(0.01)
        
        return results
    
    def process_with_circuit_breaker(self, data_list: List[str], task_type: str) -> List[Dict]:
        """使用熔断器模式的容错处理"""
        # 简单熔断器实现
        class CircuitBreaker:
            def __init__(self, failure_threshold: int = 5, reset_timeout: float = 10.0):
                self.failure_threshold = failure_threshold
                self.reset_timeout = reset_timeout
                self.failure_count = 0
                self.last_failure_time = 0
                self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
            
            def can_execute(self) -> bool:
                import time
                current_time = time.time()
                
                if self.state == "OPEN":
                    # 检查是否应该进入HALF_OPEN状态
                    if current_time - self.last_failure_time > self.reset_timeout:
                        self.state = "HALF_OPEN"
                        print("熔断器: OPEN -> HALF_OPEN")
                        return True
                    return False
                
                return True
            
            def on_success(self):
                if self.state == "HALF_OPEN":
                    self.state = "CLOSED"
                    self.failure_count = 0
                    print("熔断器: HALF_OPEN -> CLOSED (成功)")
                else:
                    self.failure_count = 0
            
            def on_failure(self):
                import time
                self.failure_count += 1
                self.last_failure_time = time.time()
                
                if self.failure_count >= self.failure_threshold:
                    self.state = "OPEN"
                    print(f"熔断器: CLOSED -> OPEN (失败次数: {self.failure_count})")
        
        # 选择任务
        if task_type == "unreliable":
            task = self.unreliable_task
        elif task_type == "slow":
            task = self.slow_task
        else:
            task = self.unreliable_task
        
        circuit_breaker = CircuitBreaker(failure_threshold=3, reset_timeout=5.0)
        results = []
        
        for data in data_list:
            if not circuit_breaker.can_execute():
                results.append({
                    "status": "circuit_breaker_open",
                    "data": data,
                    "error": "Circuit breaker is OPEN, request rejected"
                })
                continue
            
            try:
                result = task.invoke(data)
                circuit_breaker.on_success()
                
                results.append({
                    "status": "success",
                    "data": data,
                    "result": result
                })
                
            except Exception as e:
                circuit_breaker.on_failure()
                
                results.append({
                    "status": "failed",
                    "data": data,
                    "error": str(e)
                })
        
        return results

## 总结：条件分支与并行处理的核心价值

### 技术价值对比

| 维度 | 传统方式 | LangChain方式 |
|------|---------|--------------|
| **条件分支复杂度** | 高（嵌套if-else） | 低（扁平化RunnableBranch） |
| **并行处理能力** | 有限（手动线程管理） | 强大（声明式RunnableParallel） |
| **代码可读性** | 差（业务逻辑混杂） | 好（声明式组合） |
| **错误处理** | 复杂（每个分支单独处理） | 统一（链式错误传播） |
| **性能优化** | 困难（手动优化） | 简单（自动并行化） |
| **可测试性** | 困难（需要模拟复杂状态） | 简单（组件可独立测试） |
| **可维护性** | 低（修改影响范围广） | 高（模块化设计） |
