# 2.2.2 条件分支与并行处理

## 概念讲解：为什么需要分支和并行？

### 线性管道的局限性

想象一个客服系统：

```
❌ 纯线性处理：
   所有问题走同一流程
   接收 → 分析 → 回答 → 结束
   
   问题：
   - 简单FAQ和复杂投诉走同一流程，效率低
   - 无法根据客户情绪调整处理方式
   
✅ 带分支的处理：
   接收 → 意图识别 → 分流处理
                ↓
        ┌───────┼───────┐
        ↓       ↓       ↓
      FAQ     投诉    技术问题
        ↓       ↓       ↓
      直接    情感    并行分析
      回答    处理    问题+严重度
```

> **核心要点**：分支让系统能够**根据运行时数据选择不同的处理路径**，并行让系统能够**同时执行独立的任务**。

## 核心要点：RunnableBranch条件分支

### 什么是RunnableBranch？

RunnableBranch是LangChain提供的**声明式条件路由组件**，根据条件选择执行不同的分支：

```python
from langchain_core.runnables import RunnableBranch

branch = RunnableBranch(
    (条件1, 分支1),  # 如果条件1满足，执行分支1
    (条件2, 分支2),  # 否则如果条件2满足，执行分支2
    默认分支         # 否则执行默认分支
)
```

### 简单示例：意图路由

```python
from langchain_core.runnables import RunnableBranch, RunnableLambda

# 定义路由条件
router = RunnableBranch(
    # 天气查询
    (lambda x: "天气" in x or "气温" in x, 
     RunnableLambda(lambda x: f"查询天气: {x}")),
    
    # 计算请求
    (lambda x: "计算" in x or "等于" in x,
     RunnableLambda(lambda x: f"执行计算: {x}")),
    
    # 翻译请求
    (lambda x: "翻译" in x,
     RunnableLambda(lambda x: f"翻译内容: {x}")),
    
    # 默认：闲聊
    RunnableLambda(lambda x: f"闲聊回复: {x}")
)

# 测试不同输入
print(router.invoke("今天北京天气怎么样？"))  # → 查询天气
print(router.invoke("1+1等于多少？"))        # → 执行计算
print(router.invoke("翻译Hello"))            # → 翻译内容
print(router.invoke("你好啊"))               # → 闲聊回复
```

## 核心要点：RunnableParallel并行处理

### 什么是RunnableParallel？

RunnableParallel可以**同时执行多个独立的任务**，结果合并为字典：

```python
from langchain_core.runnables import RunnableParallel

parallel = RunnableParallel({
    "结果1": 任务1,
    "结果2": 任务2,
    "结果3": 任务3,
})
```

### 简单示例：并行分析

```python
from langchain_core.runnables import RunnableParallel, RunnableLambda
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")

# 定义多个分析任务
summarize_prompt = ChatPromptTemplate.from_template("总结以下内容：{text}")
sentiment_prompt = ChatPromptTemplate.from_template("分析以下文本的情感：{text}")
keyword_prompt = ChatPromptTemplate.from_template("提取以下文本的关键词：{text}")

# 并行执行三个分析任务
analysis_chain = RunnableParallel({
    "summary": summarize_prompt | llm,
    "sentiment": sentiment_prompt | llm,
    "keywords": keyword_prompt | llm,
})

# 一次调用，三个结果同时返回
text = "人工智能技术正在改变各行各业的发展模式，许多企业开始采用AI来提高效率和降低成本。"
results = analysis_chain.invoke({"text": text})

print("摘要:", results["summary"].content)
print("情感:", results["sentiment"].content)
print("关键词:", results["keywords"].content)
```

**执行流程**：

```
输入文本
    ↓
┌─────────────────────────────────────┐
│          RunnableParallel           │
│                                     │
│  ┌─────────┐ ┌─────────┐ ┌───────┐ │
│  │  summary│ │sentiment│ │keyword│ │
│  │   ↓     │ │   ↓     │ │  ↓    │ │
│  │ prompt  │ │ prompt  │ │prompt │ │
│  │   ↓     │ │   ↓     │ │  ↓    │ │
│  │   llm   │ │   llm   │ │ llm   │ │
│  └─────────┘ └─────────┘ └───────┘ │
└────────────────┬────────────────────┘
                 ↓
    {"summary": ..., "sentiment": ..., "keywords": ...}
```

> **关键洞察**：RunnableParallel不是真正的多线程并行，而是**在LLM调用层面可以并发执行**。对于同步调用，它按顺序执行；对于异步调用（`ainvoke`），它会真正并行执行。

## 简单示例：完整的条件+并行工作流

### 场景：智能客服系统

```python
from langchain_core.runnables import RunnableBranch, RunnableParallel, RunnableLambda
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4o-mini")

# 意图分类器
classifier_prompt = ChatPromptTemplate.from_template(
    "判断以下问题的意图类型（faq/complaint/technical/other）：{question}"
)
classifier = classifier_prompt | llm | StrOutputParser()

# 各类型处理器
faq_prompt = ChatPromptTemplate.from_template("作为客服，回答这个FAQ：{question}")
complaint_prompt = ChatPromptTemplate.from_template("作为客服，安抚客户并处理投诉：{question}")
technical_prompt = ChatPromptTemplate.from_template("作为技术支持，分析这个问题：{question}")

# 并行分析器（技术问题需要多维分析）
technical_analyzer = RunnableParallel({
    "issue_type": RunnableLambda(lambda x: "网络" if "网络" in x or "连接" in x else "其他"),
    "severity": RunnableLambda(lambda x: "紧急" if "无法" in x or "崩溃" in x else "一般"),
    "suggestion": technical_prompt | llm | StrOutputParser(),
})

# 构建完整工作流
support_workflow = (
    # 第一步：意图分类
    RunnableParallel({
        "intent": classifier,
        "question": RunnableLambda(lambda x: x)
    })
    # 第二步：根据意图分流
    | RunnableBranch(
        # FAQ → 直接回答
        (lambda x: "faq" in x["intent"].lower(),
         faq_prompt | llm | StrOutputParser()),
        
        # 投诉 → 安抚处理
        (lambda x: "complaint" in x["intent"].lower(),
         complaint_prompt | llm | StrOutputParser()),
        
        # 技术问题 → 并行分析
        (lambda x: "technical" in x["intent"].lower(),
         RunnableLambda(lambda x: x["question"]) | technical_analyzer),
        
        # 其他 → 默认回复
        RunnableLambda(lambda x: "抱歉，我没有理解您的问题，请重新描述。")
    )
)

# 测试
test_questions = [
    "如何重置密码？",                    # FAQ
    "你们的系统太差了，我要退款！",         # 投诉
    "我的网络连接一直不稳定，无法正常使用",  # 技术问题
]

for q in test_questions:
    result = support_workflow.invoke(q)
    print(f"问题: {q}")
    print(f"回复: {result}")
    print("-" * 50)
```

## 进阶理解：分支与并行的组合模式

### 模式1：嵌套分支（多级路由）

```python
# 多级条件分支
complex_router = RunnableBranch(
    # 第一级：天气相关
    (lambda x: "天气" in x,
     RunnableBranch(
         (lambda x: "预报" in x, RunnableLambda(lambda x: "天气预报处理")),
         (lambda x: "现在" in x, RunnableLambda(lambda x: "当前天气处理")),
         RunnableLambda(lambda x: "一般天气处理")
     )),
    
    # 第二级：新闻相关
    (lambda x: "新闻" in x,
     RunnableBranch(
         (lambda x: "体育" in x, RunnableLambda(lambda x: "体育新闻")),
         (lambda x: "科技" in x, RunnableLambda(lambda x: "科技新闻")),
         RunnableLambda(lambda x: "一般新闻")
     )),
    
    # 默认
    RunnableLambda(lambda x: "闲聊处理")
)
```

### 模式2：并行后分支

```python
# 先并行获取多个信息，再根据信息选择路径
parallel_then_branch = (
```

## 常见问题

### Q1: RunnableBranch和if-else有什么区别？

**A:** 功能相同，但RunnableBranch更适合链式环境：

```python
# ❌ if-else在链中不好用
def route(x):
    if "天气" in x:
        return weather_chain.invoke(x)
    else:
        return chat_chain.invoke(x)

# ✅ RunnableBranch可以直接嵌入链
chain = RunnableBranch(
    (lambda x: "天气" in x, weather_chain),
    chat_chain
)
result = chain.invoke(input)  # 一行搞定
```

### Q2: RunnableParallel真的是并行执行吗？

**A:** 取决于调用方式：
- `invoke()`：按顺序执行（同步）
- `ainvoke()`：真正并行执行（异步）

```python
# 同步调用：顺序执行
results = parallel_chain.invoke(input)

# 异步调用：并行执行（更快）
results = await parallel_chain.ainvoke(input)
```

### Q3: 分支条件可以很复杂吗？

**A:** 可以，但建议保持简单。复杂的条件逻辑应该拆分为独立的函数：

```python
# ✅ 好的做法：条件函数职责单一
def is_weather_query(x):
    return any(kw in x for kw in ["天气", "气温", "下雨"])

def is_urgent(x):
    return any(kw in x for kw in ["紧急", "马上", "立刻"])

branch = RunnableBranch(
    (is_weather_query, weather_chain),
    (is_urgent, urgent_chain),
    default_chain
)
```

## 本节总结

### 核心认知
1. **RunnableBranch**：声明式条件路由，替代复杂的if-else嵌套
2. **RunnableParallel**：并行执行独立任务，结果合并为字典
3. **组合使用**：分支和并行可以嵌套，构建复杂工作流

### 关键启示
- 分支让系统具备智能路由能力
- 并行提升执行效率
- 声明式优于命令式
