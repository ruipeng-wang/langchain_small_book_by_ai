# 2.2.1 任务分解与顺序执行

## 概念讲解：什么是链式思维？

### 从"面条代码"到"声明式管道"

想象一个工厂流水线：

```
❌ 传统方式（面条代码）：
   一个工人包揽所有工序
   设计 → 生产 → 质检 → 包装 → 发货
   问题：修改任何一个环节都要改整体逻辑

✅ 链式方式（声明式管道）：
   每个工序独立，通过传送带连接
   [设计站] → [生产站] → [质检站] → [包装站]
   好处：可以单独替换任何一个站点
```

> **核心要点**：链式思维的本质是**把复杂任务分解为独立的步骤，用声明式的方式串联**。

### AI应用中的任务分解

大多数AI应用都可以分解为以下基本步骤：

```
典型的AI应用流程：
┌────────────────────────────────────────────────┐
│                                                │
│  1. 输入处理 → 2. 上下文检索 → 3. 提示构建      │
│       ↓                                            │
│  4. 模型调用 → 5. 输出解析 → 6. 结果返回          │
│                                                │
│  每个步骤都可以独立开发、测试、替换                │
└────────────────────────────────────────────────┘
```

## 核心要点：LCEL链式执行

### 什么是LCEL？

LCEL（LangChain Expression Language）是LangChain的**声明式链式语言**，用简单的`|`操作符连接组件：

```python
# LCEL的核心语法
chain = component_a | component_b | component_c
```

这行代码的含义是：**数据先经过component_a处理，输出作为component_b的输入，以此类推**。

### LCEL的优势

| 特性 | 传统方式 | LCEL方式 |
|------|---------|----------|
| 代码结构 | 嵌套调用 | 线性管道 |
| 组件替换 | 修改代码 | 替换参数 |
| 流式支持 | 需要额外实现 | 自动支持 |
| 批量处理 | 需要循环 | 内置优化 |
| 异步支持 | 需要重写 | 自动支持 |

## 简单示例：从传统代码到LCEL

### 场景：构建一个简单的问答链

**传统方式（硬编码）**：

```python
# ❌ 传统方式：步骤硬编码在函数中
def answer_question(question: str) -> str:
    # 步骤1：构建提示
    prompt_text = f"请回答以下问题：{question}"
    
    # 步骤2：调用模型
    response = llm.invoke(prompt_text)
    
    # 步骤3：解析结果
    return response.content

# 问题：
# 1. 提示模板写死在代码中
# 2. 组件之间紧耦合
# 3. 难以复用和测试
```

**LCEL方式（声明式）**：

```python
# ✅ LCEL方式：组件独立，声明式组合
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# 1. 定义独立组件
prompt = ChatPromptTemplate.from_template("请回答以下问题：{question}")
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

# 2. 用管道符连接
chain = prompt | llm | parser

# 3. 使用
result = chain.invoke({"question": "什么是机器学习？"})
print(result)
```

**执行流程可视化**：

```
chain.invoke({"question": "什么是机器学习？"})
        ↓
┌───────────────────────────────────────┐
│  prompt                               │
│  输入: {"question": "什么是机器学习？"}  │
│  输出: "请回答以下问题：什么是机器学习？" │
└──────────────────┬────────────────────┘
                   ↓
┌───────────────────────────────────────┐
│  llm                                  │
│  输入: "请回答以下问题：..."            │
│  输出: AIMessage(content="机器学习是...")│
└──────────────────┬────────────────────┘
                   ↓
┌───────────────────────────────────────┐
│  parser                               │
│  输入: AIMessage                      │
│  输出: "机器学习是..."（纯字符串）      │
└──────────────────┬────────────────────┘
                   ↓
              最终结果返回
```

## 进阶应用：更复杂的任务分解

### 场景：带检索的问答系统（简化RAG）

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough, RunnableParallel

# 假设已有向量存储
retriever = vector_store.as_retriever(search_kwargs={"k": 3})

# 定义提示模板
prompt = ChatPromptTemplate.from_template("""
根据以下上下文回答问题：

{context}

问题：{question}
""")

llm = ChatOpenAI(model="gpt-4o-mini")

# 辅助函数：格式化检索结果
def format_context(docs):
    return "\n\n".join(doc.page_content for doc in docs)

# 构建RAG链
rag_chain = (
    # 第一步：并行获取上下文和问题
    RunnableParallel({
        "context": retriever | (lambda docs: format_context(docs)),
        "question": RunnablePassthrough()
    })
    # 第二步：提示构建
    | prompt
    # 第三步：模型调用
    | llm
    # 第四步：输出解析
    | StrOutputParser()
)

# 使用
answer = rag_chain.invoke("LangChain的主要优势是什么？")
```

**数据流解析**：

```
用户输入："LangChain的主要优势是什么？"
              ↓
┌─────────────────────────────────────────┐
│           RunnableParallel              │
│  ┌─────────────┐    ┌──────────────┐    │
│  │  retriever  │    │ Passthrough  │    │
│  │  ↓          │    │      ↓       │    │
│  │  文档列表    │    │   原始问题    │    │
│  │  ↓          │    │              │    │
│  │  格式化的   │    │              │    │
│  │  上下文      │    │              │    │
│  └─────────────┘    └──────────────┘    │
└────────────────┬────────────────────────┘
                 ↓
      {"context": "...", "question": "..."}
                 ↓
            prompt → llm → parser
                 ↓
            最终回答
```

### 场景：多步骤内容处理管道

```python
from langchain_core.runnables import RunnableLambda

# 定义各步骤组件
steps = {
    "clean": RunnableLambda(lambda x: x.strip().lower()),
    "summarize": prompt_summary | llm | StrOutputParser(),
    "translate": prompt_translate | llm | StrOutputParser(),
}

# 构建处理管道
processing_chain = (
    steps["clean"]           # 步骤1：清洗
    | steps["summarize"]     # 步骤2：摘要
    | steps["translate"]     # 步骤3：翻译
)

# 使用
result = processing_chain.invoke("  这是一段需要处理的文本...  ")
```

## 常见问题

### Q1: 什么时候用`|`操作符，什么时候用函数调用？

**A:** 简单场景用函数，复杂场景用管道：

```python
# 简单场景：一两步，函数更直观
result = parser.parse(llm.invoke(prompt.format()))

# 复杂场景：多步骤，管道更清晰
chain = step1 | step2 | step3 | step4
result = chain.invoke(input)
```

### Q2: 如何在链中传递多个参数？

**A:** 使用字典传递参数：

```python
# 正确方式：用字典包装多个参数
chain = prompt | llm
result = chain.invoke({
    "question": "什么是AI？",
    "context": "AI是人工智能..."
})
```

### Q3: LCEL的性能开销大吗？

**A:** 几乎可以忽略。LCEL的管道组合只是在内存中建立执行图，执行时的额外开销在微秒级别。真正的性能瓶颈在于：
- LLM API调用（秒级）
- 网络传输（百毫秒级）
- 向量检索（十毫秒级）

## 本节总结

### 核心认知
1. **链式思维**：把复杂任务分解为独立步骤，声明式地串联
2. **LCEL语法**：`|`操作符连接组件，数据从左到右流动
3. **组件独立**：每个组件职责单一，易于测试和复用

### 关键启示
- 声明式优于命令式
- 组合优于继承
- 管道优于嵌套
