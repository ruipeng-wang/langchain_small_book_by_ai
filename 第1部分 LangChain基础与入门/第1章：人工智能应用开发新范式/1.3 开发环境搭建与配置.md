# 1.3 开发环境搭建与配置

## 概念讲解：为什么AI项目需要专门的环境管理？

AI应用开发与传统的Python开发有着**截然不同的环境要求**：

### AI项目的特殊环境需求

1. **Python版本要求严格**：LangChain v1.2+要求Python 3.10+
2. **依赖包数量庞大**：一个完整的AI项目可能涉及50+依赖包
3. **版本兼容性敏感**：不同版本的模型SDK可能导致API调用失败
4. **GPU环境配置复杂**：深度学习框架需要特定版本的CUDA支持

### 环境管理的核心原则

```
┌─────────────────────────────────────────────────────┐
│                环境管理三大原则                        │
├─────────────────────────────────────────────────────┤
│ 1. 隔离性：每个项目使用独立的虚拟环境                  │
│ 2. 可复现性：通过锁文件确保任何机器上一致的环境         │
│ 3. 最小化：只安装必要的依赖，减少冲突风险              │
└─────────────────────────────────────────────────────┘
```

> **核心要点**：良好的环境管理是AI项目开发的基础，可以避免90%以上的"在我机器上能运行"问题。

## 核心要点：环境搭建三步曲

### 步骤一：Python版本准备

**LangChain v1.2.22版本要求**：

| Python版本 | 支持状态 | 推荐程度 |
|-----------|---------|---------|
| 3.12      | 完全支持 | ⭐⭐⭐ |
| 3.11      | 完全支持 | ⭐⭐⭐⭐⭐（推荐）|
| 3.10      | 完全支持 | ⭐⭐⭐ |
| 3.9及以下  | 不支持  | ❌ |

**检查Python版本**：
```bash
python --version  # 或 python3 --version
```

如果版本不符合要求，推荐使用以下工具安装：

| 工具 | 适用平台 | 特点 |
|------|---------|------|
| **uv** | 全平台 | 极速（Rust实现），推荐 |
| **conda** | 全平台 | 科学计算优化，企业常用 |
| **pyenv** | macOS/Linux | 轻量级，纯shell |

### 步骤二：创建虚拟环境

**推荐方案：使用uv（最快最现代）**

```bash
# 1. 安装uv（如果尚未安装）
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows PowerShell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 2. 创建项目目录并初始化
mkdir my-langchain-project
cd my-langchain-project
uv init

# 3. 指定Python版本
uv python pin 3.11

# 4. 添加LangChain核心依赖
uv add "langchain>=1.2.22"
uv add "langchain-openai"

# 5. 同步环境
uv sync

# 6. 验证安装
uv run python -c "import langchain; print(f'LangChain版本: {langchain.__version__}')"
```

**备选方案：使用conda（企业常用）**

```bash
# 1. 创建环境
conda create -n langchain-env python=3.11

# 2. 激活环境
conda activate langchain-env

# 3. 安装LangChain
pip install langchain>=1.2.22 langchain-openai

# 4. 验证安装
python -c "import langchain; print(f'LangChain版本: {langchain.__version__}')"
```

### 步骤三：配置API密钥

大多数AI模型需要API密钥。推荐通过环境变量管理：

**创建`.env`文件**（不要提交到Git）：
```bash
# .env 文件
OPENAI_API_KEY=sk-your-api-key-here
LANGSMITH_API_KEY=lsv2-your-api-key-here
LANGSMITH_TRACING=true
LANGSMITH_PROJECT=my-project
```

**在Python中加载环境变量**：
```python
# 安装python-dotenv
# uv add python-dotenv

from dotenv import load_dotenv
import os

load_dotenv()  # 从.env文件加载环境变量

# 现在API密钥已经被自动设置了
from langchain_openai import ChatOpenAI
llm = ChatOpenAI(model="gpt-4")  # 自动使用环境变量中的API_KEY
```

## 简单示例：验证环境是否就绪

创建一个简单的测试脚本来验证环境：

```python
# test_env.py - 环境验证脚本
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

def test_environment():
    """验证LangChain环境是否正确"""
    
    try:
        # 1. 测试模型初始化
        llm = ChatOpenAI(model="gpt-4")
        print("✅ 模型初始化成功")
        
        # 2. 测试消息构建
        message = HumanMessage(content="你好，请用一句话介绍自己")
        print("✅ 消息构建成功")
        
        # 3. 测试调用（需要有效的API密钥）
        # response = llm.invoke([message])
        # print(f"✅ 模型调用成功: {response.content[:50]}...")
        
        print("\n🎉 环境验证完成！可以开始LangChain开发了。")
        return True
        
    except ImportError as e:
        print(f"❌ 导入失败: {e}")
        print("请运行: uv add langchain langchain-openai")
        return False
    except Exception as e:
        print(f"⚠️  {type(e).__name__}: {e}")
        print("请检查API密钥是否正确配置")
        return False

if __name__ == "__main__":
    test_environment()
```

运行测试：
```bash
uv run python test_env.py
```

## 进阶配置：项目结构模板

一个标准的LangChain项目应该包含以下结构：

```
my-langchain-project/
├── .env                    # 环境变量（不提交到Git）
├── .gitignore              # Git忽略配置
├── pyproject.toml          # 项目配置和依赖
├── uv.lock                 # 依赖锁定文件（确保可复现）
├── src/
│   ├── __init__.py
│   ├── chains/             # 链式组件
│   ├── agents/             # 代理系统
│   ├── tools/              # 自定义工具
│   └── utils/              # 工具函数
├── tests/                  # 测试代码
└── notebooks/              # Jupyter探索笔记
```

**pyproject.toml模板**：
```toml
[project]
name = "my-langchain-project"
version = "0.1.0"
description = "基于LangChain的AI应用项目"
requires-python = ">=3.11"

dependencies = [
    "langchain>=1.2.22",
    "langchain-openai",
    "python-dotenv>=1.0.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.4.0",
    "jupyter>=1.0.0",
]
```

## 常见问题

### Q1: 安装时遇到依赖冲突怎么办？
**A:** 最常见的原因是Python版本过低或包版本不兼容。解决方案：
1. 确认Python版本 >= 3.10
2. 使用新的虚拟环境（不要在全局Python中安装）
3. 使用`uv`或`poetry`等现代包管理器自动解决冲突

### Q2: 没有OpenAI API密钥可以用其他模型吗？
**A:** 完全可以。LangChain支持多种免费或本地模型：
```python
# 使用Ollama运行本地模型（免费，无需API密钥）
from langchain_ollama import ChatOllama
llm = ChatOllama(model="llama3.1")

# 使用Google Gemini（有免费额度）
from langchain_google_genai import ChatGoogleGenerativeAI
llm = ChatGoogleGenerativeAI(model="gemini-pro")
```

### Q3: 如何在团队间共享环境配置？
**A:** 提交`pyproject.toml`和锁文件（`uv.lock`或`poetry.lock`）到Git，团队成员运行`uv sync`即可获得完全相同的环境。

## 本节总结

### 核心步骤
1. **Python版本**：确保使用3.10+（推荐3.11）
2. **虚拟环境**：使用uv或conda创建隔离环境
3. **依赖安装**：安装langchain和所需集成包
4. **API配置**：通过.env文件管理API密钥
5. **环境验证**：运行测试脚本确认安装成功
