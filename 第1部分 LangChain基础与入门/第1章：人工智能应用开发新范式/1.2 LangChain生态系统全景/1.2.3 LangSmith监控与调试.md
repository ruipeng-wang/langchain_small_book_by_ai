# 1.2.3 LangSmith监控与调试

## 概念讲解：为什么AI应用需要专门的监控工具？

### AI应用的调试困境

传统软件调试有成熟的工具链，但AI应用面临着全新的挑战：

```
┌────────────────────────────────────────────────────────┐
│              AI应用调试的独特挑战                        │
│                                                        │
│  1. 非确定性输出                                       │
│     - 相同输入可能产生不同输出                           │
│     - 传统断点调试无法复现问题                           │
│                                                        │
│  2. 黑盒模型调用                                       │
│     - 无法直接查看模型内部推理过程                       │
│     - 只能观察输入和输出，中间过程不可见                 │
│                                                        │
│  3. 多组件链路                                         │
│     - 一个请求可能经过多个模型、工具、检索器              │
│     - 问题可能出在链路的任何环节                         │
│                                                        │
│  4. 提示词迭代                                         │
│     - 提示词的微小变化可能导致输出剧变                   │
│     - 需要对比不同版本的提示词效果                       │
│                                                        │
│  5. 质量评估困难                                       │
│     - 没有标准的"正确答案"来断言                         │
│     - 需要人工评审或专门的评估模型                       │
└────────────────────────────────────────────────────────┘
```

> **核心要点**：AI应用的调试不是"找Bug"，而是"理解行为"和"优化质量"。LangSmith正是为了解决这些独特挑战而设计的平台。

## 核心要点：LangSmith的三大核心能力

### 能力一：链路追踪（Tracing）

记录AI应用中每次请求的完整执行过程：

```python
# 启用LangSmith追踪
import os
os.environ["LANGSMITH_TRACING"] = "true"
os.environ["LANGSMITH_API_KEY"] = "your-api-key"
os.environ["LANGSMITH_PROJECT"] = "my-ai-app"

# 正常编写LangChain代码，追踪自动开启
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

llm = ChatOpenAI(model="gpt-4")
prompt = ChatPromptTemplate.from_template("请回答：{question}")
chain = prompt | llm

result = chain.invoke({"question": "什么是机器学习？"})
```

### 能力二：评估与测试（Evaluation）

系统化地评估AI应用的输出质量：

```python
from langsmith import Client
from langsmith.evaluation import evaluate, EvaluationResult

# 定义评估器
def correctness_evaluator(run, example) -> EvaluationResult:
    """评估回答是否正确"""
    # 可以使用LLM作为评判器
    llm_judge = ChatOpenAI(model="gpt-4")
    
    judgment_prompt = f"""
    请评估以下回答是否正确：
    
    问题：{example.inputs['question']}
    预期答案：{example.outputs['answer']}
    实际回答：{run.outputs['output']}
    
    请回答"是"或"否"：
    """
    
    judgment = llm_judge.invoke(judgment_prompt)
    score = 1 if "是" in judgment.content else 0
    
    return EvaluationResult(key="correctness", score=score)

# 运行评估
client = Client()
results = evaluate(
    lambda inputs: chain.invoke(inputs),  # 被测函数
    data="test_dataset",                   # 测试数据集
    evaluators=[correctness_evaluator],    # 评估器列表
    experiment_prefix="v1.0"               # 实验标识
)
```


## 进阶理解：LangSmith在生产环境的应用

### 实时监控与告警

```python
from langsmith import Client
from datetime import datetime, timedelta

client = Client()

# 查询最近1小时的请求统计
runs = client.list_runs(
    project_name="production-app",
    start_time=datetime.now() - timedelta(hours=1),
    filter="and(eq(status, 'error'))"  # 只查看失败的请求
)

# 分析错误模式
error_types = {}
for run in runs:
    error = run.error
    if error:
        error_type = error.get("type", "unknown")
        error_types[error_type] = error_types.get(error_type, 0) + 1

print("错误类型统计:", error_types)
```

### 反馈循环优化

```python
from langsmith import Client

client = Client()

# 收集用户反馈
def collect_feedback(run_id: str, score: int, comment: str = ""):
    """
    收集用户对回答的反馈
    score: 1-5分
    """
    client.create_feedback(
        run_id=run_id,
        key="user_satisfaction",
        score=score,
        comment=comment
    )

# 分析低分回答，找出优化方向
low_score_runs = client.list_runs(
    project_name="production-app",
    filter="and(eq(feedback.user_satisfaction, 1))"
)

for run in low_score_runs:
    print(f"问题: {run.inputs}")
    print(f"回答: {run.outputs}")
    print("---")
```

## 常见问题

### Q1: LangSmith是必须使用的吗？
**A:** 不是必须的，但强烈推荐。开发阶段可以不启用，但进入生产环境后，LangSmith提供的可观测性是保障AI应用质量的必要工具。没有监控的AI应用就像"盲飞"——无法发现问题，也无法持续优化。

### Q2: LangSmith是否会增加延迟？
**A:** 追踪是异步进行的，对正常请求延迟的影响通常<50ms。对于极度延迟敏感的场景，可以：
- 关闭详细追踪，只记录元数据
- 采用抽样追踪（只记录部分请求）
- 批量异步发送追踪数据

### Q3: LangSmith的数据安全性如何？
**A:** LangSmith由LangChain官方运营，提供：
- 数据加密传输和存储
- 访问权限控制
- 企业版支持私有部署
- 符合SOC 2等安全认证

对于高度敏感的数据，建议：
- 脱敏后再发送到LangSmith
- 使用私有部署版本
- 关闭详细输入/输出记录

## 本节总结

### 核心认知
1. **AI调试需要新范式**：传统调试工具无法应对AI应用的非确定性和黑盒特性
2. **可观测性是生产基石**：没有监控的AI应用无法保证质量和持续优化
3. **数据驱动迭代优化**：通过评估和反馈循环，系统化地提升AI应用效果

### 关键启示
- 开发阶段启用追踪可以加速调试和问题定位
- 生产环境必须建立完善的监控和告警机制
- 评估数据集是持续优化AI应用的核心资产
