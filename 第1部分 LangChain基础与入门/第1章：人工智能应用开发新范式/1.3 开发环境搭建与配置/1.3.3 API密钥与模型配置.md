# 1.3.3 API密钥与模型配置

## 概念讲解：安全地管理API密钥

### 为什么不能硬编码API密钥？

新手开发者常犯的错误是将API密钥直接写在代码中：

```python
# ❌ 危险做法：API密钥硬编码在代码中
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="gpt-4",
    api_key="sk-1234567890abcdef..."  # 绝对不要这样做！
)
```

**硬编码密钥的风险**：

```
┌──────────────────────────────────────────────────────────┐
│              硬编码API密钥的风险                           │
│                                                          │
│  1. 代码泄露 = 密钥泄露                                   │
│     - 上传到GitHub后被爬虫自动收集                          │
│     - 同事/合作伙伴访问代码后可能滥用                       │
│                                                          │
│  2. 经济损失                                             │
│     - 他人使用你的密钥产生高额API费用                       │
│     - OpenAI等服务商不会为密钥泄露退款                     │
│                                                          │
│  3. 安全风险                                             │
│     - 恶意使用可能导致账户被封禁                           │
│     - 可能被用于生成违规内容                               │
│                                                          │
│  4. 管理困难                                             │
│     - 多环境（开发/测试/生产）需要不同密钥                   │
│     - 密钥轮换需要修改代码并重新部署                        │
└──────────────────────────────────────────────────────────┘
```

> **核心要点**：API密钥是敏感信息，应该通过环境变量或专门的密钥管理服务来管理，绝不写入代码。

### 环境变量：管理密钥的标准方式

环境变量是操作系统级别的键值对，程序运行时可以读取：

```
┌──────────────────────────────────────────────────────┐
│              环境变量的工作原理                         │
│                                                      │
│  .env文件（本地开发）                                  │
│       ↓                                              │
│  python-dotenv加载到进程环境                            │
│       ↓                                              │
│  os.environ["OPENAI_API_KEY"] 可读取                   │
│       ↓                                              │
│  LangChain自动读取环境变量初始化模型                    │
│       ↓                                              │
│  无需在代码中暴露密钥                                   │
└──────────────────────────────────────────────────────┘
```

## 核心要点：配置模型连接

### 步骤一：创建.env文件

在项目根目录创建`.env`文件：

```bash
# .env 文件（绝不要提交到Git！）

# OpenAI API配置
OPENAI_API_KEY=sk-your-api-key-here

# Anthropic API配置（如使用Claude）
ANTHROPIC_API_KEY=sk-ant-your-api-key-here

# Google AI配置（如使用Gemini）
GOOGLE_API_KEY=your-google-api-key

# LangSmith配置（监控与调试）
LANGSMITH_API_KEY=lsv2-your-api-key
LANGSMITH_TRACING=true
LANGSMITH_PROJECT=my-project

# 可选：设置默认模型
DEFAULT_LLM_MODEL=gpt-4o
```

### 步骤二：配置.gitignore

确保`.env`文件不会被提交：

```bash
# .gitignore
.env
.env.*
*.env
```

### 步骤三：在代码中加载环境变量

```python
# 使用python-dotenv加载环境变量
from dotenv import load_dotenv
import os

# 从.env文件加载环境变量
load_dotenv()

# 验证密钥是否加载成功
api_key = os.getenv("OPENAI_API_KEY")
if not api_key:
    raise ValueError("未找到OPENAI_API_KEY环境变量，请检查.env文件")
if api_key == "sk-your-api-key-here":
    raise ValueError("请替换为真实的API密钥")

print("✅ API密钥加载成功")
```

### 步骤四：初始化模型

```python
from langchain_openai import ChatOpenAI

# LangChain会自动从环境变量读取OPENAI_API_KEY
llm = ChatOpenAI(model="gpt-4o")

# 也可以显式传入（优先级高于环境变量）
# llm = ChatOpenAI(
#     model="gpt-4o",
#     api_key=os.getenv("OPENAI_API_KEY")
# )

# 测试模型连接
response = llm.invoke("你好，请用一句话介绍自己")
print(response.content)
```

## 简单示例：完整的配置流程

```python
# config.py - 项目配置模块
"""
项目配置模块
负责加载环境变量、初始化模型、验证配置有效性
"""

from dotenv import load_dotenv
import os
import logging

# 配置日志
logger = logging.getLogger(__name__)

def setup_environment():
    """加载环境变量并验证"""
    # 加载.env文件
    load_dotenv()
    
    # 验证必要的环境变量
    required_vars = {
        "OPENAI_API_KEY": "OpenAI API密钥",
    }
    
    missing_vars = []
    for var_name, var_desc in required_vars.items():
        if not os.getenv(var_name):
            missing_vars.append(var_desc)
    
    if missing_vars:
        logger.warning(f"缺少必要的环境变量: {', '.join(missing_vars)}")
        logger.warning("请检查.env文件是否正确配置")
        return False
    
    logger.info("✅ 所有必要的环境变量已加载")
    return True

def create_llm(model: str = None, **kwargs):
    """创建LLM实例"""
    from langchain_openai import ChatOpenAI
    
    model = model or os.getenv("DEFAULT_LLM_MODEL", "gpt-4o")
    
    llm = ChatOpenAI(
        model=model,
        temperature=kwargs.get("temperature", 0.7),
        max_tokens=kwargs.get("max_tokens"),
        streaming=kwargs.get("streaming", True),
    )
    
    logger.info(f"LLM已初始化: {model}")
    return llm

# 模块加载时自动执行环境配置
setup_environment()
```

使用配置模块：

```python
# main.py
from config import create_llm

# 使用默认配置
llm = create_llm()
response = llm.invoke("你好")

# 使用自定义配置
llm = create_llm(model="gpt-4", temperature=0.1)
response = llm.invoke("请精确回答以下问题")
```

## 进阶理解：多模型与高级配置

### 配置多个模型

企业级应用通常需要配置多个模型：

```python
# 不同场景使用不同模型
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

# 快速/便宜的模型：处理简单任务
fast_llm = ChatOpenAI(model="gpt-4o-mini")

# 强大/贵的模型：处理复杂推理
smart_llm = ChatOpenAI(model="gpt-4o")

# Claude：擅长长文本和代码
claude_llm = ChatAnthropic(model="claude-3-5-sonnet")

# 根据任务类型选择模型
def get_llm_for_task(task_type: str):
    """根据任务类型选择最合适的模型"""
    models = {
        "simple_chat": fast_llm,
        "complex_reasoning": smart_llm,
        "code_generation": claude_llm,
        "translation": fast_llm,
    }
    return models.get(task_type, fast_llm)
```

### 模型参数详解

```python
llm = ChatOpenAI(
    # 基础配置
    model="gpt-4o",            # 模型名称
    temperature=0.7,           # 创造性 (0-1, 越低越确定)
    max_tokens=1000,           # 最大输出token数
    top_p=1.0,                 # 核采样参数 (0-1)
    
    # 高级配置
    streaming=True,            # 启用流式输出
    timeout=30,                # 请求超时（秒）
    max_retries=3,             # 最大重试次数
    
    # 成本优化
    presence_penalty=0,        # 存在惩罚 (-2到2)
    frequency_penalty=0,       # 频率惩罚 (-2到2)
    
    # 安全配置
    stop=["END"],              # 停止词列表
)
```

**关键参数说明**：

| 参数 | 范围 | 低值效果 | 高值效果 | 推荐场景 |
|------|------|---------|---------|---------|
| **temperature** | 0-1 | 确定性高，回答一致 | 创造性高，变化多样 | 问答:0.1-0.3, 创作:0.7-0.9 |
| **max_tokens** | 1-模型上限 | 回答简短 | 回答详细 | 根据需求设定 |
| **top_p** | 0-1 | 只选最高概率词 | 允许更多选择 | 通常保持1.0 |
| **frequency_penalty** | -2到2 | 可能重复用词 | 避免重复用词 | 长文本:0.3-0.5 |

### 流式输出

对于用户体验要求高的场景，流式输出可以让用户更快看到响应：

```python
llm = ChatOpenAI(model="gpt-4o", streaming=True)

# 流式调用
for chunk in llm.stream("请写一段关于春天的文字"):
    print(chunk.content, end="", flush=True)
# 输出效果：文字逐字出现，类似ChatGPT的打字效果
```

## 常见问题

### Q1: 没有API密钥可以学习LangChain吗？
**A:** 完全可以。多种免费/低成本方案：

```python
# 方案1: Ollama本地运行（完全免费）
from langchain_ollama import ChatOllama
llm = ChatOllama(model="llama3.1")  # 本地运行，无需API密钥

# 方案2: Google Gemini（免费额度）
from langchain_google_genai import ChatGoogleGenerativeAI
llm = ChatGoogleGenerativeAI(model="gemini-1.5-flash")  # 每月有免费额度

# 方案3: Groq（快速推理，免费额度充足）
from langchain_groq import ChatGroq
llm = ChatGroq(model="llama-3.1-70b")  # 速度快，免费额度多
```

### Q2: 如何获取OpenAI API密钥？
**A:** 获取步骤：
1. 访问 https://platform.openai.com/
2. 注册/登录账户
3. 进入 API Keys 页面
4. 点击 "Create new secret key"
5. 复制密钥到.env文件

**费用参考**：
- GPT-4o: $2.50/百万输入token, $10/百万输出token
- GPT-4o-mini: $0.15/百万输入token, $0.60/百万输出token
- 新用户通常有$5-$18的免费额度

### Q3: 生产环境如何管理API密钥？
**A:** 生产环境推荐使用：

| 方案 | 适用场景 | 特点 |
|------|---------|------|
| **云服务商密钥管理** | AWS/GCP/Azure用户 | AWS Secrets Manager, GCP Secret Manager |
| **专门密钥管理服务** | 多云/混合云 | HashiCorp Vault, Doppler |
| **CI/CD变量** | 部署时使用 | GitHub Secrets, GitLab CI Variables |
| **Kubernetes Secrets** | K8s环境 | 原生支持，需加密存储 |

**基本原则**：
- 永远不要将密钥提交到代码仓库
- 开发、测试、生产使用不同的密钥
- 定期轮换密钥
- 监控API使用情况，设置告警

### Q4: 如何验证API密钥是否有效？
**A:** 简单测试方法：

```python
from langchain_openai import ChatOpenAI

def test_api_key():
    """测试API密钥是否有效"""
    try:
        llm = ChatOpenAI(model="gpt-4o-mini")
        response = llm.invoke("你好")
        print(f"✅ API密钥有效，响应: {response.content[:30]}...")
        return True
    except Exception as e:
        print(f"❌ API密钥无效: {e}")
        return False

test_api_key()
```
