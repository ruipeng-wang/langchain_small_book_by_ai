# 1.3.1 Python环境与依赖管理

## 概念讲解：为什么AI项目需要专门的环境管理？

### AI项目的特殊环境需求

AI应用开发与传统的Python开发有着**截然不同的环境要求**：

```
┌──────────────────────────────────────────────────────────┐
│                AI项目环境管理的挑战                        │
│                                                          │
│  1. Python版本要求严格                                    │
│     - LangChain要求Python 3.10+                          │
│     - 某些模型SDK要求特定Python版本                       │
│                                                          │
│  2. 依赖包数量庞大                                       │
│     - 一个完整的AI项目可能涉及50+依赖包                    │
│     - numpy、pydantic等基础库版本敏感                      │
│                                                          │
│  3. 版本兼容性敏感                                       │
│     - LangChain更新频繁，API可能变化                      │
│     - 不同模型SDK版本之间可能存在冲突                      │
│                                                          │
│  4. GPU环境配置复杂（如需本地推理）                        │
│     - CUDA版本需要与深度学习框架匹配                      │
│     - GPU驱动版本影响可用性和性能                          │
└──────────────────────────────────────────────────────────┘
```

> **核心要点**：良好的环境管理是AI项目开发的基础，可以避免90%以上的"在我机器上能运行"问题。

### 环境管理的三大原则

```
┌─────────────────────────────────────────────────────┐
│              环境管理三大原则                           │
├─────────────────────────────────────────────────────┤
│ 1. 隔离性：每个项目使用独立的虚拟环境                  │
│    → 避免不同项目的依赖冲突                           │
│                                                     │
│ 2. 可复现性：通过锁文件确保任何机器上一致的环境         │
│    → 团队成员、CI/CD、生产部署都使用相同版本           │
│                                                     │
│ 3. 最小化：只安装必要的依赖，减少冲突风险              │
│    → 按需安装集成包，不全局安装所有包                  │
└─────────────────────────────────────────────────────┘
```

## 核心要点：Python环境管理工具对比

### 主流工具对比

| 工具 | 特点 | 适用场景 | 推荐度 |
|------|------|---------|--------|
| **uv** | Rust实现，极速，现代依赖管理 | 所有场景，特别推荐 | ⭐⭐⭐⭐⭐ |
| **conda** | 科学计算优化，支持非Python依赖 | 数据科学、GPU环境 | ⭐⭐⭐⭐ |
| **venv + pip** | Python内置，生态最广 | 简单项目，学习阶段 | ⭐⭐⭐ |
| **poetry** | 依赖解析优秀，项目结构规范 | 库开发，规范项目 | ⭐⭐⭐⭐ |

### 推荐方案：使用uv（现代且快速）

`uv`是由Astral开发（Ruff的同一团队）的Python包管理器，用Rust编写，速度极快：

```bash
# 安装uv
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 验证安装
uv --version
```

**uv的核心优势**：
- **速度**：比pip快10-100倍
- **统一**：一个工具管理Python版本、虚拟环境、依赖
- **锁文件**：自动生成交付锁文件，确保可复现性
- **兼容**：完全兼容pip的包格式和pyproject.toml

## 简单示例：使用uv创建项目环境

### 步骤一：初始化项目

```bash
# 创建项目目录
mkdir my-langchain-project
cd my-langchain-project

# 初始化项目（创建pyproject.toml）
uv init

# 指定Python版本
uv python pin 3.11
```

### 步骤二：添加依赖

```bash
# 添加LangChain核心包
uv add "langchain>=1.2.22"

# 添加OpenAI集成
uv add langchain-openai

# 添加环境变量管理
uv add python-dotenv
```

### 步骤三：验证环境

```bash
# 在虚拟环境中运行Python
uv run python -c "
import langchain
from langchain_openai import ChatOpenAI
print(f'LangChain版本: {langchain.__version__}')
print('环境验证成功！')
"
```

## 进阶理解：依赖管理的最佳实践

### pyproject.toml 配置详解

```toml
[project]
name = "my-langchain-project"
version = "0.1.0"
description = "基于LangChain的AI应用项目"
requires-python = ">=3.11"
readme = "README.md"

# 核心依赖：只列项目直接依赖
dependencies = [
    "langchain>=1.2.22",
    "langchain-openai",
    "python-dotenv>=1.0.0",
]

# 可选依赖：按场景分组
[project.optional-dependencies]
# 开发工具
dev = [
    "pytest>=8.0",
    "pytest-asyncio>=0.23",
    "jupyter>=1.0.0",
    "ruff>=0.3.0",
]

# 向量数据库（按需选择）
vector-db = [
    "langchain-chroma",
    "chromadb>=0.4.0",
]

# 本地模型
local-llm = [
    "langchain-ollama",
]

# 构建系统配置
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

# uv配置
[tool.uv]
# 使用阿里云镜像加速（国内用户）
index-url = "https://mirrors.aliyun.com/pypi/simple/"
```

### 依赖锁文件的重要性

```bash
# uv add 会自动更新锁文件（uv.lock）
# 锁文件记录了所有依赖的精确版本和哈希值

# 团队成员同步环境
git add pyproject.toml uv.lock
git commit -m "添加LangChain依赖"

# 其他成员克隆后只需一行命令
uv sync  # 根据锁文件安装完全相同的环境
```

**锁文件解决的问题**：
- **可复现性**：确保任何机器上安装的环境完全一致
- **CI/CD稳定性**：避免依赖更新导致的构建失败
- **问题排查**：锁定版本可以排除"版本差异"导致的问题

### 虚拟环境管理

```bash
# 查看当前Python版本
uv python list

# 安装特定Python版本
uv python install 3.11.8

# 创建虚拟环境（uv自动管理）
uv venv

# 激活虚拟环境（可选，uv run会自动使用）
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate

# 查看已安装的包
uv pip list

# 检查过时包
uv pip list --outdated
```

## 备选方案：使用conda

对于数据科学团队或需要GPU支持的项目，conda是常用选择：

```bash
# 1. 安装Miniconda（推荐，比Anaconda轻量）
# 从 https://docs.conda.io/en/latest/miniconda.html 下载

# 2. 创建环境
conda create -n langchain-env python=3.11

# 3. 激活环境
conda activate langchain-env

# 4. 安装LangChain
pip install "langchain>=1.2.22" langchain-openai python-dotenv

# 5. （可选）安装GPU支持
# conda install pytorch pytorch-cuda=12.1 -c pytorch -c nvidia

# 6. 导出环境配置（供团队共享）
conda env export > environment.yml

# 团队成员导入环境
conda env create -f environment.yml
```

## 常见问题

### Q1: 为什么要用虚拟环境？直接全局安装不行吗？
**A:** 强烈不建议全局安装。原因：
- **依赖冲突**：不同项目可能需要同一包的不同版本
- **权限问题**：全局安装可能需要管理员权限
- **环境污染**：难以追踪安装了哪些包
- **复现困难**：无法确保其他机器上有相同环境

虚拟环境让每个项目拥有独立的Python和包，互不干扰。

### Q2: uv和conda可以同时使用吗？
**A:** 可以，但不推荐在同一项目中混用。建议：
- **纯Python项目**：使用uv
- **需要非Python依赖（如CUDA）**：使用conda
- **团队统一**：选择一种工具，全员使用

### Q3: 如何加速国内pip下载？
**A:** 使用国内镜像源：

```bash
# uv使用镜像
uv pip install langchain --index-url https://mirrors.aliyun.com/pypi/simple/

# 或者配置到pyproject.toml中
[tool.uv]
index-url = "https://mirrors.aliyun.com/pypi/simple/"

# pip使用镜像
pip install langchain -i https://mirrors.aliyun.com/pypi/simple/
```

常用国内镜像：
- 阿里云：https://mirrors.aliyun.com/pypi/simple/
- 清华：https://pypi.tuna.tsinghua.edu.cn/simple/
- 腾讯云：https://mirrors.cloud.tencent.com/pypi/simple/

## 本节总结

### 核心步骤
1. **选择工具**：推荐uv（快速）或conda（数据科学）
2. **创建环境**：为每个项目创建独立的虚拟环境
3. **管理依赖**：使用pyproject.toml和锁文件确保可复现性
4. **按需安装**：只安装项目需要的包，减少冲突

### 关键启示
- 环境管理是项目成功的基石
- 锁文件是团队协作的重要保障
- 现代工具（如uv）可以大幅提升开发效率
