# 3.2.1 Pipeline操作符的实现原理

## 概念讲解：从命令式到声明式

### 传统函数调用的问题

```python
# ❌ 命令式：一步步告诉计算机怎么做
def traditional_workflow(text):
    processed = preprocess(text)      # 步骤1
    response = call_model(processed)  # 步骤2
    parsed = parse_response(response) # 步骤3
    return format_output(parsed)      # 步骤4
```

### LCEL Pipeline的声明式风格

```python
# ✅ 声明式：只关心做什么，不关心怎么做
chain = preprocess | call_model | parse_response | format_output
result = chain.invoke(text)
```

> **核心要点**：Pipeline操作符将工作流从命令式变为声明式，代码更简洁。

## 核心要点：`|`操作符的实现

### 操作符重载原理

```python
from langchain_core.runnables import Runnable

class MyRunnable(Runnable):
    def __or__(self, other):
        """实现 | 操作符"""
        return RunnableSequence(self, other)
```

### RunnableSequence执行流程

```python
chain = prompt | llm | parser
```

等价于：

```python
chain = RunnableSequence(
    first=prompt,
    last=parser,
    middle=[llm],
)
```

**invoke时**：

```
input → prompt.invoke() → llm.invoke() → parser.invoke() → output
```

## 简单示例：构建链式管道

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# 三步管道
chain = (
    ChatPromptTemplate.from_template("翻译{word}成英文")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

result = chain.invoke({"word": "人工智能"})
```

## 进阶理解：管道组合的灵活性

### 部分组合再组合

```python
# 先组合部分组件
prompt_chain = prompt | llm
final_chain = prompt_chain | parser

# 等价于
final_chain = prompt | llm | parser
```

### RunnableLambda插入自定义逻辑

```python
from langchain_core.runnables import RunnableLambda

def add_timestamp(x):
    return f"[{time.now()}] {x}"

chain = prompt | llm | RunnableLambda(add_timestamp) | parser
```

## 常见问题

### Q1: `|`操作符支持哪些类型组合？

| 左边 | 右边 | 结果 |
|------|------|------|
| Prompt | LLM | RunnableSequence |
| LLM | Parser | RunnableSequence |
| Any Runnable | Any Runnable | RunnableSequence |

### Q2: 管道中如何传递字典？

```python
# 使用字典解构
from langchain_core.runnables import RunnablePassthrough

chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
)
```

### Q3: 管道执行是同步还是异步？

```python
# 同步
result = chain.invoke(input)

# 异步
result = await chain.ainvoke(input)
```

## 本节总结

### 核心认知
1. **`|`是操作符重载**：将多个Runnable组合为RunnableSequence
2. **数据从左到右流动**：每个步骤的输出作为下一步的输入
3. **声明式优于命令式**：关注做什么而非怎么做

### 关键启示
- Pipeline是可读的，无需手动管理中间状态
- 任何Runnable都可以用`|`组合
- 配合RunnableLambda插入自定义逻辑
