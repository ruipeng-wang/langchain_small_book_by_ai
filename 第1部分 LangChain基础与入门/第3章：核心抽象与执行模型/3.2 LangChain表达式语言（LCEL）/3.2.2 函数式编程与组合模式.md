# 3.2.2 函数式编程与组合模式

## 概念讲解

### 什么是函数式编程？

函数式编程将计算视为数学函数求值，核心是**纯函数、不可变数据和声明式代码**。

```python
# ❌ 命令式：关注步骤
result = ""
for word in text.split():
    result += word.upper() + " "

# ✅ 函数式：关注数据转换
result = " ".join(map(str.upper, text.split()))
```

> **核心要点**：关注"做什么"而非"如何做"。

### LangChain中的函数式编程价值

1. **确定性**：相同输入产生相同输出
2. **可组合性**：像乐高积木自由组合
3. **并发友好**：不可变数据天然支持并行
4. **错误隔离**：函数边界清晰

## 核心要点

### 1. 纯函数（Pure Functions）

纯函数无副作用，引用透明：

```python
# ❌ 非纯函数：修改外部状态
counter = 0
def impure_increment():
    global counter
    counter += 1
    return counter

# ✅ 纯函数：无副作用
def pure_increment(x):
    return x + 1
```

### 2. 函数组合（Function Composition）

```python
from langchain_core.runnables import RunnableLambda

# 定义转换函数
trim = RunnableLambda(lambda x: x.strip())
upper = RunnableLambda(lambda x: x.upper())
exclaim = RunnableLambda(lambda x: x + "!")

# 组合成处理链
chain = trim | upper | exclaim
result = chain.invoke("  hello  ")  # → "HELLO!"
```

### 3. 条件分支：RunnableBranch

```python
from langchain_core.runnables import RunnableBranch

smart_processor = RunnableBranch(
    (lambda x: len(x) > 50, RunnableLambda(lambda x: f"长文本: {x[:50]}...")),
    (lambda x: len(x) > 20, RunnableLambda(lambda x: f"中等文本: {x}")),
    RunnableLambda(lambda x: f"短文本: {x}")
)
```

## 简单示例：文本分析管道

```python
from langchain_core.runnables import RunnableParallel

text_analyzer = RunnableParallel({
    "length": RunnableLambda(lambda x: len(x)),
    "word_count": RunnableLambda(lambda x: len(x.split())),
    "has_numbers": RunnableLambda(lambda x: any(c.isdigit() for c in x)),
})

result = text_analyzer.invoke("Hello 123 World")
# → {'length': 14, 'word_count': 3, 'has_numbers': True}
```

## 进阶理解：函数记忆化

```python
from functools import lru_cache
from langchain_core.runnables import RunnableLambda

# 缓存计算结果，避免重复计算
@lru_cache(maxsize=128)
def expensive_computation(text):
    return hash(text) % 1000

cached_fn = RunnableLambda(expensive_computation)

# 相同输入直接返回缓存结果
cached_fn.invoke("test")
cached_fn.invoke("test")  # 不重新计算
```

## 常见问题

### Q1: 函数式编程与面向对象编程有什么区别？

两者互补：
- **面向对象**：关注数据和行为封装，适合建模实体
- **函数式**：关注数据转换和组合，适合数据处理
- **LangChain**：Runnable是对象，但支持函数式组合

### Q2: 如何处理函数式编程中的错误？

```python
# 使用RunnableLambda处理可能失败的操作
safe_parse = RunnableLambda(lambda x: json.loads(x) if x else {})

# 配合with_fallbacks
safe_chain = llm.with_fallbacks([fallback_llm])
```

### Q3: 如何调试函数式代码？

1. **分步执行**：单独测试每个Runnable
2. **RunnableLambda添加日志**：在lambda中打印中间值
3. **单元测试**：纯函数易于测试

## 本节总结

### 核心认知
1. **纯函数**：无副作用，引用透明
2. **函数组合**：用`|`操作符串联多个函数
3. **不可变数据**：避免状态突变，提升可维护性

### 关键启示
- 声明式代码更易读易维护
- RunnableLambda将任何函数融入管道
- 纯函数天然支持缓存和并行
