# 3.2.3 链式调用的语法糖设计

## 概念讲解：什么是语法糖？

语法糖（Syntactic Sugar）是不增加语言功能但提升代码可读性的语法特性。

```python
# ❌ 没有语法糖：嵌套调用，难以阅读
result = parser(llm(prompt.invoke({"topic": "AI"})))

# ✅ 有语法糖：管道表达式，一目了然
chain = prompt | llm | parser
result = chain.invoke({"topic": "AI"})
```

> **核心要点**：代码不仅是给机器执行的指令，更是给人阅读的文档。

## 核心要点：LCEL的核心语法糖

### 1. 管道操作符 `|`

```python
chain = (
    ChatPromptTemplate.from_template("翻译{word}")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)
```

### 2. 字典展开：并行处理

```python
from langchain_core.runnables import RunnableParallel

# 字典字面量表示并行分支
parallel = RunnableParallel({
    "summary": summary_chain,
    "translation": translation_chain,
})
```

### 3. RunnablePassthrough：透明传递

```python
from langchain_core.runnables import RunnablePassthrough

# 保持原始输入的同时添加新数据
chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
)
```

### 4. RunnableBranch：条件分支

```python
from langchain_core.runnables import RunnableBranch

router = RunnableBranch(
    (lambda x: "?" in x, qa_chain),           # 问号 → 问答
    (lambda x: len(x) > 100, summary_chain),  # 长文本 → 摘要
    general_chain,                            # 默认 → 通用
)
```

## 简单示例：文本处理流水线

```python
from langchain_core.runnables import RunnableLambda, RunnableParallel, RunnablePassthrough

text_pipeline = (
    RunnableLambda(lambda x: x.strip().lower())
    | RunnableParallel({
        "clean": RunnablePassthrough(),
        "stats": RunnableLambda(lambda x: {
            "words": len(x.split()),
            "chars": len(x),
        })
    })
    | RunnableLambda(lambda x: f"文本: {x['clean']}, 统计: {x['stats']}")
)

result = text_pipeline.invoke("  Hello World  ")
# → "文本: hello world, 统计: {'words': 2, 'chars': 11}"
```

## 进阶理解：混合使用语法糖

### 场景：智能路由系统

```python
from langchain_core.runnables import RunnableLambda, RunnableBranch

# 根据输入类型路由到不同处理器
smart_router = RunnableBranch(
    (lambda x: isinstance(x, dict) and x.get("type") == "qa", qa_pipeline),
    (lambda x: isinstance(x, dict) and x.get("type") == "summarize", summarize_pipeline),
    default_pipeline,
)

result = smart_router.invoke({"type": "qa", "question": "什么是AI？"})
```

## 常见问题

### Q1: 什么时候用语法糖，什么时候用传统写法？

| 场景 | 推荐 |
|------|------|
| 数据处理流水线 | 管道 `|` |
| 并行多个计算 | 字典展开 |
| 复杂条件逻辑 | 传统if/else |
| 细粒度状态管理 | 传统写法 |

### Q2: 语法糖会影响性能吗？

不会。LangChain会对管道进行静态分析和优化：
- 纯函数的计算结果可被缓存
- RunnableParallel内部并行执行
- 惰性求值避免不必要的计算

### Q3: 如何调试管道表达式？

```python
# 方法1：分步执行
step1 = prompt.invoke({"topic": "AI"})
step2 = llm.invoke(step1)
result = parser.invoke(step2)

# 方法2：给Runnable命名，便于追踪
named_chain = (
    RunnableLambda(lambda x: x, name="input")
    | prompt
    | llm
    | RunnableLambda(lambda x: x, name="output")
)
```

## 本节总结

### 核心认知
1. **管道 `|`**：函数组合的直观表达
2. **字典展开**：并行处理的声明式语法
3. **RunnablePassthrough**：保持原始数据传递
4. **RunnableBranch**：条件逻辑的声明式表达

### 关键启示
- 语法糖让代码意图更清晰
- 声明式优于命令式
- 合理控制复杂度，避免过度嵌套
