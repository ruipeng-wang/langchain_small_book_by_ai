# 3.3.1 结构化消息类型体系

## 概念讲解：为什么需要结构化消息？

### 传统字典消息的问题

```python
# ❌ 传统方式：字典表示
messages = [
    {"role": "system", "content": "你是助手"},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！"},
]

# 问题：类型不安全、无法附加元数据、难以验证
invalid = {"role": "unknown", "content": "测试"}  # 运行时才报错
```

### 结构化消息的优势

```python
# ✅ LangChain方式：类型化消息
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

messages = [
    SystemMessage(content="你是助手"),
    HumanMessage(content="你好"),
    AIMessage(content="你好！"),
]
```

> **核心要点**：结构化消息提供类型安全、语义明确、元数据丰富。

## 核心要点：消息类型层次

```
BaseMessage (基类)
├── SystemMessage     # 系统指令
├── HumanMessage      # 用户输入
├── AIMessage         # AI响应
├── ToolMessage       # 工具执行结果
└── ChatMessage       # 通用角色消息
```

### 各类型用途

| 类型 | 用途 | 典型场景 |
|------|------|---------|
| SystemMessage | 设定AI行为 | 角色定义、系统指令 |
| HumanMessage | 用户输入 | 提问、上传内容 |
| AIMessage | AI响应 | 回复、工具调用 |
| ToolMessage | 工具结果 | 函数返回值 |

## 简单示例：基础使用

```python
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

# 构建对话
conversation = [
    SystemMessage(content="你是天气助手"),
    HumanMessage(content="北京今天天气？"),
    AIMessage(content="北京今天晴天，22°C"),
]

# 每条消息都有类型和属性
for msg in conversation:
    print(f"{type(msg).__name__}: {msg.content}")
```

### 消息属性

```python
msg = HumanMessage(
    content="你好",
    additional_kwargs={"user_id": "123"},  # 自定义元数据
)

print(msg.content)          # 主要内容
print(msg.additional_kwargs)  # 附加元数据
print(msg.type)             # 类型标识
```

## 进阶理解：多模态消息

HumanMessage支持多模态内容（文本+图片）：

```python
multimodal_msg = HumanMessage(
    content=[
        {"type": "text", "text": "请分析这张图片"},
        {"type": "image_url", "image_url": {"url": "https://example.com/img.jpg"}},
    ]
)
```

### 序列化和反序列化

```python
from langchain_core.messages import messages_to_dict, messages_from_dict

# 序列化
msg_dict = messages_to_dict(conversation)

# 反序列化
restored = messages_from_dict(msg_dict)
```

## 常见问题

### Q1: 什么时候用SystemMessage？

设定AI行为时使用，通常是对话的第一条消息：
- 定义角色（"你是翻译助手"）
- 设定语言（"请用中文回答"）

### Q2: AIMessage中的tool_calls是什么？

当AI决定调用工具时，AIMessage会包含tool_calls：

```python
ai_msg = AIMessage(
    content="",
    tool_calls=[{"name": "get_weather", "args": {"location": "北京"}, "id": "call_001"}]
)
```

### Q3: 如何添加自定义元数据？

```python
msg = HumanMessage(
    content="内容",
    additional_kwargs={"user_id": "123", "priority": "high"}
)
```

## 本节总结

### 核心认知
1. **结构化消息**：类型安全，替代原始字典
2. **消息类型**：SystemMessage、HumanMessage、AIMessage、ToolMessage各有职责
3. **元数据支持**：additional_kwargs存储任意信息

### 关键启示
- 始终使用类型化消息，避免原始字典
- 消息类型让代码更易读、更安全
- 多模态消息支持图片等非文本内容
