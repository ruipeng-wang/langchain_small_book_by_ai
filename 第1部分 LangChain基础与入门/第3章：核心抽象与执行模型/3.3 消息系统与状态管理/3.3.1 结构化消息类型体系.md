# 3.3.1 结构化消息类型体系

## 概念讲解

### 什么是结构化消息？

在传统的AI对话系统中，消息通常使用简单的字符串或字典表示，如 `{"role": "user", "content": "你好"}`。这种方式虽然简单，但存在诸多问题：**缺乏类型安全、难以扩展、元数据支持有限、验证困难**。

LangChain v1.2.22引入了**结构化消息类型体系**，将消息从简单的数据容器提升为具有明确语义、丰富元数据和类型安全的通信单元。这一体系的核心思想是：**不同类型的消息应该有不同的类型定义，每种类型都有明确的角色和用途**。

### 为什么需要结构化消息？

传统消息处理的局限性：
- **类型不安全**：错误的role值只能在运行时发现
- **缺乏元数据**：无法附加额外的结构化信息
- **序列化复杂**：需要手动处理格式转换
- **难以验证**：缺少结构化验证机制
- **扩展困难**：添加新字段需要修改所有处理逻辑

LangChain的结构化消息体系通过以下方式解决这些问题：
1. **类型安全**：每种消息类型都有对应的Python类
2. **语义明确**：SystemMessage、HumanMessage等名称直观反映其角色
3. **元数据丰富**：支持附加任意结构化数据
4. **序列化友好**：内置标准化序列化机制
5. **易于扩展**：通过继承可创建自定义消息类型

### LangChain消息体系的设计哲学

LangChain的消息设计遵循"约定优于配置"的原则，为常见的对话场景提供了预定义的消息类型，同时保留了足够的灵活性供开发者扩展。这种设计屏蔽了底层模型API的差异，为开发者提供了统一的消息处理接口。

## 核心要点

### 消息类型的层次结构

```
BaseMessage (基类)
├── SystemMessage     # 系统指令消息
├── HumanMessage      # 用户输入消息
├── AIMessage         # AI响应消息
├── ChatMessage       # 通用聊天消息
├── FunctionMessage   # 函数调用结果消息
└── ToolMessage       # 工具执行结果消息
```

### 每种消息类型的关键特性

#### SystemMessage
- **定位**：定义AI行为的系统级指令
- **特点**：通常是对话的第一条消息，持久影响整个对话
- **使用场景**：设定AI角色、配置行为参数、传递系统级上下文

#### HumanMessage
- **定位**：表示用户的输入
- **特点**：支持多模态内容（文本、图像、音频、文件）
- **使用场景**：用户提问、上传文件、发送图像等交互

#### AIMessage
- **定位**：表示AI模型的响应
- **特点**：包含工具调用、结构化响应、流式输出支持
- **使用场景**：AI回复、工具调用决策、中间推理过程

#### ChatMessage
- **定位**：灵活的角色化消息
- **特点**：支持任意角色定义，与传统格式兼容
- **使用场景**：多智能体对话、自定义角色系统

#### FunctionMessage / ToolMessage
- **定位**：工具交互结果
- **特点**：关联具体的函数/工具调用，包含执行结果和元数据
- **使用场景**：函数执行结果返回、工具调用反馈

### 消息的核心属性

所有消息类型都继承自`BaseMessage`，具有以下核心属性：

1. **content**：消息的主要内容（字符串或结构化内容）
2. **additional_kwargs**：附加的关键字参数，用于存储任意元数据
3. **response_metadata**：响应元数据，通常由模型填充
4. **type**：消息类型标识

## 简单示例

### 传统方法与LangChain方法对比

```python
# ============================================
# 传统方法：使用字典表示消息（问题多多）
# ============================================
messages_traditional = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮助你的？"},
]

# 问题1：类型不安全，错误的role不会在编译时发现
invalid_message = {"role": "unknown", "content": "测试"}  # 运行时才会报错

# 问题2：缺乏元数据支持，难以附加额外信息
# 问题3：序列化复杂，需要手动转换格式

# ============================================
# LangChain方法：使用结构化消息（类型安全）
# ============================================
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

messages_langchain = [
    SystemMessage(content="你是一个助手"),
    HumanMessage(content="你好"),
    AIMessage(content="你好！有什么可以帮助你的？"),
]

print("LangChain结构化消息示例:")
for i, msg in enumerate(messages_langchain):
    print(f"[{i}] {type(msg).__name__}: {msg.content}")
```

### 基础消息类型使用

```python
# 导入消息类型
from langchain_core.messages import (
    SystemMessage, HumanMessage, AIMessage,
    ChatMessage, FunctionMessage, ToolMessage
)
from datetime import datetime

# 1. SystemMessage - 系统指令
system_msg = SystemMessage(
    content="你是一个专业的天气助手，请提供准确的天气信息。",
    additional_kwargs={
        "temperature": 0.7,
        "max_tokens": 1000,
        "timestamp": datetime.now().isoformat()
    }
)

# 2. HumanMessage - 用户输入
human_msg = HumanMessage(
    content="今天北京的天气怎么样？",
    name="user_123",  # 用户标识
    additional_kwargs={
        "user_context": {
            "user_id": "user_123",
            "language": "zh-CN"
        }
    }
)

# 3. AIMessage - AI响应（带工具调用）
ai_msg = AIMessage(
    content="让我查询一下北京的天气...",
    tool_calls=[
        {
            "name": "get_weather",
            "args": {"location": "北京"},
            "id": "call_001"
        }
    ],
    response_metadata={
        "model": "gpt-4",
        "tokens_used": 150
    }
)

# 4. FunctionMessage - 函数调用结果
function_msg = FunctionMessage(
    name="get_weather",
    content='{"temperature": 22, "condition": "sunny"}',
    additional_kwargs={
        "execution_time_ms": 150,
        "success": True
    }
)

# 5. ToolMessage - 工具执行结果
tool_msg = ToolMessage(
    content="天气查询完成：北京今天晴朗，22°C",
    tool_call_id="call_001"
)

# 构建完整对话
conversation = [
    system_msg,
    human_msg,
    ai_msg,
    function_msg,
    tool_msg
]

print("完整对话流程:")
for i, msg in enumerate(conversation):
    print(f"[{i}] {type(msg).__name__}: {msg.content[:30]}...")
```

### 消息序列化与反序列化

```python
import json
from langchain_core.messages import messages_to_dict, messages_from_dict

# 创建简单对话
simple_conversation = [
    SystemMessage(content="你是一个助手"),
    HumanMessage(content="你好"),
    AIMessage(content="你好！有什么可以帮助你的？"),
]

# 序列化为字典
conversation_dict = messages_to_dict(simple_conversation)
print("序列化结果（JSON格式）:")
print(json.dumps(conversation_dict, ensure_ascii=False, indent=2))

# 反序列化还原消息
restored_messages = messages_from_dict(conversation_dict)
print("\n反序列化验证:")
for orig, restored in zip(simple_conversation, restored_messages):
    print(f"✓ {type(orig).__name__}: 内容一致 - {orig.content == restored.content}")
```

### 消息工厂模式

对于需要批量创建消息或统一消息格式的场景，可以使用工厂模式。

```python
from typing import Dict, Any, Optional
from dataclasses import dataclass
from enum import Enum

class MessagePriority(Enum):
    LOW = 1
    NORMAL = 2
    HIGH = 3
    URGENT = 4

@dataclass
class MessageConfig:
    content: str
    priority: MessagePriority = MessagePriority.NORMAL
    metadata: Optional[Dict[str, Any]] = None

class MessageFactory:
    """消息工厂类"""
    
    @staticmethod
    def create_system_message(config: MessageConfig) -> SystemMessage:
        """创建系统消息"""
        additional_kwargs = config.metadata or {}
        additional_kwargs["priority"] = config.priority.value
        
        return SystemMessage(
            content=config.content,
            additional_kwargs=additional_kwargs
        )
    
    @staticmethod
    def create_user_message(
        content: str,
        user_id: Optional[str] = None
    ) -> HumanMessage:
        """创建用户消息"""
        additional_kwargs = {}
        if user_id:
            additional_kwargs["user_id"] = user_id
        
        return HumanMessage(
            content=content,
            name=user_id,
            additional_kwargs=additional_kwargs
        )
    
    @staticmethod
    def create_ai_message(
        content: str,
        tool_calls: Optional[list] = None
    ) -> AIMessage:
        """创建AI消息"""
        return AIMessage(
            content=content,
            tool_calls=tool_calls or []
        )

# 使用消息工厂
factory = MessageFactory()

system_config = MessageConfig(
    content="你是一个客服助手",
    priority=MessagePriority.HIGH,
    metadata={"department": "customer_service"}
)

system_msg = factory.create_system_message(system_config)
user_msg = factory.create_user_message("我的订单有问题", user_id="customer_123")
ai_msg = factory.create_ai_message("已收到您的问题，正在处理...")

print("工厂创建的消息:")
print(f"系统消息: {system_msg.content} (优先级: {system_msg.additional_kwargs.get('priority')})")
print(f"用户消息: {user_msg.content} (用户: {user_msg.name})")
print(f"AI消息: {ai_msg.content}")
```

### 企业级消息验证

在企业应用中，消息验证是确保系统安全稳定的关键环节。

```python
import re
from typing import List

class MessageSecurityValidator:
    """消息安全验证器"""
    
    def __init__(self):
        # 定义安全策略
        self.blocked_patterns = [
            r"<script.*?>.*?</script>",  # 脚本标签
            r"javascript:",  # JavaScript协议
            r"on\w+=\".*?\"",  # 事件处理器
            r"eval\s*\(",  # eval函数
        ]
        self.compiled_patterns = [
            re.compile(pattern, re.IGNORECASE | re.DOTALL)
            for pattern in self.blocked_patterns
        ]
    
    def validate_message(self, message) -> List[str]:
        """验证消息安全性"""
        issues = []
        
        # 检查内容长度
        if isinstance(message.content, str) and len(message.content) > 10000:
            issues.append("消息内容过长")
        
        # 检查危险内容
        if isinstance(message.content, str):
            for pattern in self.compiled_patterns:
                if pattern.search(message.content):
                    issues.append("检测到危险内容")
                    break
        
        # 检查附加参数大小
        if message.additional_kwargs:
            import sys
            kwargs_size = sys.getsizeof(str(message.additional_kwargs))
            if kwargs_size > 1024 * 1024:  # 1MB
                issues.append("附加参数过大")
        
        return issues
    
    def sanitize_message(self, message) -> SystemMessage:
        """净化消息内容"""
        if not isinstance(message.content, str):
            return message
        
        sanitized_content = message.content
        
        # 移除危险模式
        for pattern in self.compiled_patterns:
            sanitized_content = pattern.sub("[REMOVED]", sanitized_content)
        
        # 创建净化后的消息
        return SystemMessage(
            content=sanitized_content,
            additional_kwargs={
                **message.additional_kwargs,
                "security_sanitized": True
            }
        )

# 使用验证器
validator = MessageSecurityValidator()

# 测试消息
test_message = HumanMessage(
    content="正常内容<script>alert('xss')</script>更多内容",
    additional_kwargs={"test": True}
)

# 验证
issues = validator.validate_message(test_message)
if issues:
    print(f"安全验证发现问题: {issues}")
    sanitized = validator.sanitize_message(test_message)
    print(f"净化后内容: {sanitized.content}")
else:
    print("✓ 消息安全验证通过")
```

## 常见问题

### Q1: 什么时候使用SystemMessage？
**A:** SystemMessage用于设定AI的上下文和行为，通常是对话的第一条消息。例如：
- 设定AI的角色（"你是一个翻译助手"）
- 配置AI的行为参数（temperature, max_tokens等）
- 传递系统级指令（"请用中文回答"）

### Q2: HumanMessage和ChatMessage有什么区别？
**A:** HumanMessage专门表示用户输入，支持多模态内容和用户标识。ChatMessage更通用，可以表示任意角色的消息，适合多智能体对话或需要自定义角色的场景。

### Q3: AIMessage中的tool_calls如何使用？
**A:** 当AI决定调用工具时，会在AIMessage的tool_calls属性中添加工具调用信息。然后需要：
1. 执行对应的工具
2. 创建ToolMessage或FunctionMessage返回结果
3. 将结果消息添加到对话中，供AI继续处理

### Q4: 如何添加自定义元数据？
**A:** 通过additional_kwargs参数添加任意自定义元数据：
```python
message = HumanMessage(
    content="内容",
    additional_kwargs={
        "custom_field": "value",
        "timestamp": "2024-01-15T10:30:00",
        "priority": "high"
    }
)
```

### Q5: 消息序列化有什么注意事项？
**A:** 
1. 使用`messages_to_dict()`和`messages_from_dict()`进行标准序列化
2. 自定义序列化时需确保能正确还原消息类型
3. 注意JSON不支持Python特定对象，需先转换为基本类型

### Q6: 如何处理消息过大问题？
**A:** 
1. 使用分块处理，将大消息拆分为多个小消息
2. 对内容进行摘要或压缩
3. 设置消息大小限制并进行验证

1. **结构化消息的优势**：类型安全、语义明确、元数据丰富、易于扩展
2. **消息类型的选择**：

   - 始终使用类型化的消息，避免使用原始字典
