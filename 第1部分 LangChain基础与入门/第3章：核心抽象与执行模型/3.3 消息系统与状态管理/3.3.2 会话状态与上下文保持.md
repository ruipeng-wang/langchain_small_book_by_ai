# 3.3.2 会话状态与上下文保持

## 概念讲解：为什么需要记忆？

### 没有记忆的问题

```
用户: 我叫小明
AI: 你好，小明！

用户: 我叫什么名字？
AI: 我不知道...  ❌ 忘记了
```

### 有记忆的对话

```
用户: 我叫小明
AI: 你好，小明！

用户: 我叫什么名字？
AI: 你叫小明！ ✅ 记住了
```

> **核心要点**：记忆系统让AI记住之前的对话，维持上下文连贯性。

## 核心要点：记忆组件类型

LangChain提供多种记忆组件：

| 组件 | 特点 | 适用场景 |
|------|------|---------|
| ConversationBufferMemory | 保存所有消息 | 短对话、调试 |
| ConversationBufferWindowMemory | 滑动窗口，保留最近k条 | 长对话 |
| ConversationSummaryMemory | 生成摘要 | 超长对话 |
| ConversationSummaryBufferMemory | 混合模式 | 平衡详细和摘要 |

## 简单示例：基础记忆

```python
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_openai import ChatOpenAI

# 存储会话
store = {}

def get_session_history(session_id: str):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

llm = ChatOpenAI(model="gpt-4o-mini")

# 创建带记忆的链
chain = RunnableWithMessageHistory(
    llm,
    get_session_history,
    input_messages_key="input",
)

# 多轮对话
chain.invoke({"input": "我叫小明"}, config={"configurable": {"session_id": "1"}})
chain.invoke({"input": "我叫什么名字？"}, config={"configurable": {"session_id": "1"}})
# AI会回答"你叫小明"
```

## 进阶理解：消息历史管理

### InMemoryChatMessageHistory

```python
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.messages import HumanMessage, AIMessage

# 创建消息历史
history = InMemoryChatMessageHistory()

# 添加消息
history.add_messages([
    HumanMessage(content="你好"),
    AIMessage(content="你好！"),
])

# 获取所有消息
print(history.messages)
```

### 自定义持久化

```python
import json
from langchain_core.chat_history import BaseChatMessageHistory

class FileChatHistory(BaseChatMessageHistory):
    """文件存储的消息历史"""
    
    def __init__(self, file_path: str):
        self.file_path = file_path
    
    def add_message(self, message):
        messages = self.messages
        messages.append(message)
        with open(self.file_path, 'w') as f:
            json.dump([msg.model_dump() for msg in messages], f)
    
    @property
    def messages(self):
        try:
            with open(self.file_path, 'r') as f:
                return [HumanMessage.model_validate(m) for m in json.load(f)]
        except:
            return []
```

## 常见问题

### Q1: 如何处理长对话的Token限制？

使用滑动窗口或摘要：

```python

# 方案1：滑动窗口 - 只保留最近的消息
def get_recent_history(history, k=10):
    return history.messages[-k:]
```

### Q2: 多会话如何隔离？

使用session_id区分：

```python
def get_session_history(session_id: str):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

# 不同session_id隔离
chain.invoke({"input": "你好"}, config={"configurable": {"session_id": "user1"}})
chain.invoke({"input": "你好"}, config={"configurable": {"session_id": "user2"}})
```

## 本节总结

### 核心认知
1. **记忆组件**：RunnableWithMessageHistory简化会话管理
2. **消息历史**：InMemoryChatMessageHistory是基础存储
3. **会话隔离**：通过session_id区分不同用户

### 关键启示
- 短对话用InMemoryChatMessageHistory即可
- 长对话需要考虑Token限制和摘要
- 生产环境使用Redis/数据库持久化
