# 3.3.3 流式输出与增量更新

## 概念讲解：为什么需要流式输出？

### 等待完整响应 vs 实时输出

```
❌ 完整等待：用户看到进度条3秒，然后突然显示完整回答
✅ 流式输出：用户立即看到第一个字，逐步显示完整回答
```

> **核心要点**：流式输出提升用户体验，降低感知延迟。

## 核心要点：三种流式模式

### 1. stream() - 同步流式

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")

# 同步流式
for chunk in llm.stream("讲个故事"):
    print(chunk.content, end="", flush=True)
```

### 2. astream() - 异步流式

```python
async for chunk in llm.astream("讲个故事"):
    print(chunk.content, end="", flush=True)
```

### 3. astream_events() - 事件流式

提供细粒度事件追踪，适合监控和调试：

```python
async for event in llm.astream_events("你好", version="v2"):
    print(f"事件: {event['event']}")
    if event["event"] == "on_chat_model_stream":
        print(event["data"]["chunk"].content, end="")
```

## 简单示例：Web SSE流式输出

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from langchain_openai import ChatOpenAI

app = FastAPI()
llm = ChatOpenAI(model="gpt-4o-mini")

@app.get("/chat")
async def chat_stream(message: str):
    async def generate():
        async for chunk in llm.astream(message):
            yield f"data: {chunk.content}\n\n"
    
    return StreamingResponse(generate(), media_type="text/event-stream")
```

## 进阶理解：流式回调处理器

```python
from langchain_core.callbacks import StreamingStdOutCallbackHandler

# 方法1：创建时指定
llm = ChatOpenAI(
    model="gpt-4o-mini",
    streaming=True,
    callbacks=[StreamingStdOutCallbackHandler()]
)
llm.invoke("你好")  # 自动流式输出

# 方法2：调用时指定
result = llm.invoke(
    "你好",
    config={"callbacks": [StreamingStdOutCallbackHandler()]}
)
```

## 常见问题

### Q1: stream和astream有什么区别？

| 特性 | stream | astream |
|------|--------|---------|
| 类型 | 同步 | 异步 |
| 阻塞 | 是 | 否 |
| Web场景 | 不推荐 | 推荐 |

### Q2: 如何在链中使用流式？

```python
chain = prompt | llm | parser

# 整个链都支持流式
for chunk in chain.stream({"topic": "AI"}):
    print(chunk, end="")
```

### Q3: stream_mode参数有什么用？

```python
# values - 只输出最终值（默认）
# messages - 输出中间消息
# updates - 输出状态更新
# debug - 输出调试信息

async for event in chain.astream_events(input, version="v2"):
    print(event)
```

## 本节总结

### 核心认知
1. **stream/astream**：同步/异步流式迭代
2. **astream_events**：细粒度事件流
3. **StreamingStdOutCallbackHandler**：自动流式输出

### 关键启示
- Web应用优先使用astream
- 简单脚本用stream即可
- 监控调试用astream_events获取详细信息
