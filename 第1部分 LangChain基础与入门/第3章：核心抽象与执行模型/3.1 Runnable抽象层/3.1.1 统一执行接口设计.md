# 3.1.1 统一执行接口设计

## 概念讲解：为什么需要统一接口？

### 接口碎片化的痛点

在LangChain引入Runnable之前，不同的AI组件调用方式完全不同：

```python
# ❌ OpenAI调用方式
response = openai_client.chat.completions.create(
    model="gpt-4", messages=[{"role": "user", "content": prompt}]
)
result = response.choices[0].message.content

# ❌ HuggingFace调用方式
outputs = model.generate(**tokenizer.encode(prompt))
result = tokenizer.decode(outputs[0])

# ❌ 自定义API调用方式
response = requests.post(url, json={"text": prompt})
result = response.json()["result"]
```

**问题**：每种组件都有不同的调用方式、参数格式、返回结构，代码难以复用和组合。

> **核心要点**：统一接口屏蔽底层差异，让开发者用一种方式调用所有组件。

## 核心要点：Runnable协议

### 六大执行方法

所有LangChain组件都实现相同的Runnable接口：

```python
# 所有组件都支持这6个方法
component.invoke(input)           # 同步调用
component.ainvoke(input)          # 异步调用
component.batch(inputs)           # 批量调用
component.abatch(inputs)          # 异步批量
component.stream(input)           # 流式调用
component.astream(input)          # 异步流式
```

### 实际效果

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# 三种不同类型的组件
prompt = ChatPromptTemplate.from_template("解释{topic}")
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

# 所有组件使用相同的方法调用
prompt.invoke({"topic": "AI"})    # ✅
llm.invoke("你好")                 # ✅
parser.invoke(ai_message)          # ✅
```

> **关键洞察**：学会一个接口，就学会所有组件的使用方式。

## 简单示例：传统代码 vs Runnable

### 场景：多模型切换

**传统方式（硬编码分支）**：

```python
# ❌ 每种模型不同的调用逻辑
def call_model(model_name, prompt):
    if model_name == "openai":
        return openai_client.chat.completions.create(...)
    elif model_name == "anthropic":
        return anthropic_client.messages.create(...)
    elif model_name == "local":
        return local_model.generate(prompt)
```

**Runnable方式（统一接口）**：

```python
# ✅ 所有模型使用相同接口
models = {
    "openai": ChatOpenAI(model="gpt-4o-mini"),
    "anthropic": ChatAnthropic(model="claude-3-haiku"),
    "local": Ollama(model="llama3"),
}

def call_model(model_name, prompt):
    return models[model_name].invoke(prompt)  # 一行搞定
```

## 进阶理解：管道操作符

### `|`操作符的本质

Runnable重载了`|`操作符，实现组件组合：

```python
# 管道组合
chain = prompt | llm | parser

# 等价于
from langchain_core.runnables import RunnableSequence
chain = RunnableSequence([prompt, llm, parser])

# 执行时：数据从左到右流动
result = chain.invoke({"topic": "AI"})
```

**执行流程**：

```
chain.invoke({"topic": "AI"})
    ↓
prompt → "请解释AI"
    ↓
llm → AIMessage(content="AI是...")
    ↓
parser → "AI是..."
    ↓
返回结果
```

## 常见问题

### Q1: Runnable和普通函数有什么区别？

| 特性 | 普通函数 | Runnable |
|------|---------|----------|
| 调用方式 | 各不相同 | 统一invoke/stream/batch |
| 组合能力 | 手动编排 | 管道符直接组合 |
| 流式支持 | 额外实现 | 内置stream方法 |
| 配置传递 | 手动处理 | with_config |

### Q2: 如何将传统函数转为Runnable？

```python
from langchain_core.runnables import RunnableLambda

def my_function(x):
    return x.upper()

# 一行包装
runnable = RunnableLambda(my_function)

# 可以直接放入管道
chain = prompt | llm | RunnableLambda(my_function)
```

### Q3: Runnable支持自定义配置吗？

```python
# with_config：动态修改行为
configured_llm = llm.with_config({"temperature": 0.7})

# with_retry：自动重试
retry_llm = llm.with_retry(stop_after_attempt=3)

# with_fallbacks：失败回退
fallback_llm = llm.with_fallbacks([backup_llm])
```

## 本节总结

### 核心认知
1. **Runnable是统一接口**：所有组件使用相同的6个执行方法
2. **管道是组合方式**：`|`操作符让组件像管道一样串联
3. **开闭原则**：对扩展开放，对修改封闭

### 关键启示
- 统一接口降低了学习成本和代码复杂度
- 学会invoke/batch/stream三种模式即可应对所有场景
- 管道组合让工作流声明式且易于修改
