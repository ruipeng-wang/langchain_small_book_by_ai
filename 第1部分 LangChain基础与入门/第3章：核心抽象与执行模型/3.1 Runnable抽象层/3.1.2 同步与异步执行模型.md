# 3.1.2 同步与异步执行模型

## 概念讲解：为什么需要异步执行？

### 同步执行的瓶颈

```
同步执行（阻塞等待）：
请求 → [LLM调用 3秒] → [向量搜索 1秒] → [缓存检查 0.5秒]
总时间 = 3 + 1 + 0.5 = 4.5秒 ❌

异步执行（并行等待）：
请求 → [LLM调用 3秒] ↘
       [向量搜索 1秒]   → 合并结果
       [缓存检查 0.5秒] ↗
总时间 ≈ 3秒（最慢的那个） ✅
```

> **核心要点**：异步执行让I/O密集型任务并行运行，大幅提升吞吐量。

## 核心要点：四种执行模式

### invoke / ainvoke / stream / astream

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")

# 1. invoke - 同步阻塞调用
result = llm.invoke("你好")

# 2. ainvoke - 异步非阻塞调用
result = await llm.ainvoke("你好")

# 3. stream - 同步流式输出
for chunk in llm.stream("讲个故事"):
    print(chunk, end="")

# 4. astream - 异步流式输出
async for chunk in llm.astream("讲个故事"):
    print(chunk, end="")
```

| 方法 | 适用场景 | 阻塞 | 并发 |
|------|---------|------|------|
| invoke | 简单脚本 | ✅ | ❌ |
| ainvoke | Web服务 | ❌ | ✅ |
| stream | 实时展示 | ✅ | ❌ |
| astream | Web SSE | ❌ | ✅ |

## 简单示例：批量任务对比

### 场景：处理多个查询

**同步方式（顺序执行）**：

```python
# ❌ 顺序执行，总时间 = 所有任务时间之和
for query in queries:
    result = llm.invoke(query)  # 每个都要等3秒
# 10个查询 ≈ 30秒
```

**异步方式（并发执行）**：

```python
import asyncio

# ✅ 并发执行，总时间 ≈ 最慢的那个任务
async def batch_process(queries):
    tasks = [llm.ainvoke(q) for q in queries]
    return await asyncio.gather(*tasks)
# 10个查询 ≈ 3秒
```

## 进阶理解：并行执行多个服务

### RunnableParallel的异步优势

```python
from langchain_core.runnables import RunnableParallel, RunnableLambda

# 构建并行管道
pipeline = RunnableParallel({
    "llm": llm,
    "search": RunnableLambda(lambda x: search_db(x)),  # 模拟I/O
    "cache": RunnableLambda(lambda x: check_cache(x)),
})

# 同步调用（内部按顺序执行）
result = pipeline.invoke("查询")

# 异步调用（内部真正并发执行）
result = await pipeline.ainvoke("查询")
```

**执行差异**：

```
sync invoke():
  → 先等LLM完成 → 再等search完成 → 再等cache完成
  总时间 = 3 + 1 + 0.5 = 4.5秒

async ainvoke():
  → 同时启动三个任务 → 等待最慢的那个
  总时间 ≈ 3秒
```

### Web框架中的异步实践

```python
from fastapi import FastAPI
from langchain_openai import ChatOpenAI

app = FastAPI()
llm = ChatOpenAI(model="gpt-4o-mini")

# ❌ 错误做法：同步调用阻塞事件循环
@app.post("/chat")
def chat_sync(message: str):
    return {"reply": llm.invoke(message).content}

# ✅ 正确做法：异步调用释放事件循环
@app.post("/chat")
async def chat_async(message: str):
    response = await llm.ainvoke(message)
    return {"reply": response.content}

# ✅ SSE流式输出
@app.get("/chat/stream")
async def chat_stream(message: str):
    async def event_generator():
        async for chunk in llm.astream(message):
            yield f"data: {chunk.content}\n\n"
    return StreamingResponse(event_generator())
```

## 常见问题

### Q1: 什么时候用invoke，什么时候用ainvoke？

| 场景 | 推荐 |
|------|------|
| Jupyter Notebook / 简单脚本 | invoke |
| FastAPI / Flask异步视图 | ainvoke |
| 批量数据处理 | ainvoke + gather |
| 实时聊天界面 | astream |

### Q2: 同步代码能调用异步组件吗？

```python
import asyncio

# 在同步环境中调用异步组件
def sync_wrapper():
    return asyncio.run(llm.ainvoke("你好"))

# 注意：不推荐在已有事件循环中使用asyncio.run()
```

### Q3: batch和多个ainvoke有什么区别？

```python
# batch - 针对LLM优化，批量发送请求
results = llm.batch(["问题1", "问题2", "问题3"])

# gather - 并发执行多个独立任务
tasks = [llm.ainvoke(q) for q in queries]
results = await asyncio.gather(*tasks)

# batch通常更快（一次HTTP请求）
```

## 本节总结

### 核心认知
1. **invoke/ainvoke**：同步阻塞 vs 异步非阻塞
2. **stream/astream**：逐步获取输出，适合实时展示
3. **Web服务优先使用异步**：提升并发处理能力

### 关键启示
- 简单脚本用invoke即可
- 高并发场景必须使用ainvoke
- 聊天界面配合astream实现流式输出
