# 8.2 内置链类型分析

## 概念讲解（文字+图示）

LangChain提供了多种**内置链类型**，覆盖常见的AI应用开发场景。了解这些链类型，能避免重复造轮子。

### 链类型的演进

LangChain的链类型经历了从**专用化**到**通用化**的演进：

```
早期版本：专用链类（LLMChain、SequentialChain等）
    ↓
现代版本：LCEL组合（推荐）
```

> **重要说明**：LangChain推荐使用LCEL组合替代专用链类。本节介绍历史链类型的设计思想，以及对应的LCEL替代方案。

### 为什么LCEL更优

| 特性 | 专用链类 | LCEL组合 |
|------|----------|----------|
| 灵活性 | 固定逻辑 | 自由组合 |
| 可读性 | 需要了解各类API | 管道符一目了然 |
| 维护性 | 多个类独立维护 | 统一Runnable接口 |
| 流式支持 | 部分支持 | 自动支持 |

## 核心要点

**🔑 常见链类型及其LCEL替代：**

| 历史链类型 | 用途 | LCEL替代方案 |
|-----------|------|-------------|
| LLMChain | 提示词+LLM调用 | `prompt | model` |
| SequentialChain | 多步骤顺序执行 | `step1 | step2 | step3` |
| TransformChain | 数据转换 | `RunnableLambda(func)` |
| RouterChain | 智能路由 | LangGraph条件边 |

## 简单示例

### LLMChain的LCEL替代

```python
# 旧方式：LLMChain
# from langchain.chains import LLMChain
# chain = LLMChain(llm=model, prompt=prompt)

# 新方式：LCEL
chain = prompt | model
result = chain.invoke({"question": "什么是AI？"})
```

### SequentialChain的LCEL替代

```python
# 旧方式：SequentialChain需要定义输入输出key
# chain = SequentialChain(chains=[chain1, chain2], ...)

# 新方式：直接用|连接
chain = (
    prompt1 | model | StrOutputParser()  # 步骤1
    | RunnableLambda(lambda x: {"summary": x})  # 数据转换
    | prompt2 | model  # 步骤2
)
```

### TransformChain的LCEL替代

```python
from langchain_core.runnables import RunnableLambda

# 旧方式：TransformChain
# chain = TransformChain(input_variables=["text"], ...)

# 新方式：RunnableLambda
def transform_func(text: str) -> dict:
    return {"cleaned": text.strip().lower()}

chain = RunnableLambda(transform_func)
```

### 路由链的LCEL替代

```python
# 旧方式：RouterChain
# 新方式：使用LangGraph的条件边（见第11章）

# 简单场景可用RunnableBranch（已弃用）或条件函数
def route_by_topic(input_data: dict) -> str:
    if "技术" in input_data.get("query", ""):
        return "technical"
    return "general"
```

## 进阶应用

### 多步骤数据处理链

```python
from langchain_core.runnables import RunnablePassthrough

# 构建完整数据处理流程
data_processing_chain = (
    # 步骤1：接收原始输入
    RunnablePassthrough.assign(
        cleaned=lambda x: x["raw_text"].strip(),
    )
    # 步骤2：提取关键信息
    | RunnableLambda(lambda x: {
        "text": x["cleaned"],
        "length": len(x["cleaned"]),
        "has_question": "?" in x["cleaned"],
    })
    # 步骤3：根据特征选择处理方式
    | RunnableLambda(
        lambda x: process_question(x) if x["has_question"] else process_statement(x)
    )
)
```

## 常见问题

### Q: 还需要学习旧的链类吗？
A: 了解设计思想即可，新项目直接使用LCEL。

### Q: 旧代码中的链类能继续用吗？
A: 可以，向后兼容，但建议逐步迁移到LCEL。

### Q: LCEL能实现所有链类的功能吗？
A: 绝大多数可以，复杂路由推荐用LangGraph。

## 本节总结

- 专用链类已被LCEL取代
- `prompt | model`替代LLMChain
- `|`连接替代SequentialChain
- `RunnableLambda`替代TransformChain
- 复杂路由使用LangGraph
