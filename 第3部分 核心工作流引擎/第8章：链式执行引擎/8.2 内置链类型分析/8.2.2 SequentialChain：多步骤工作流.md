# 8.2.2 SequentialChain：多步骤工作流

## 概念讲解

SequentialChain是LangChain框架中用于构建**多步骤、顺序执行工作流**的核心链类型。在LangChain v1.2.22版本中，SequentialChain虽然保留了其经典地位，但其设计哲学已经演变为**声明式工作流编排**的典范。

### 工作流编排的演进历程

从编程范式的发展来看，SequentialChain代表了工作流编排的三个阶段：

1. **命令式编排（传统方式）**：手动调用多个函数，显式传递数据
2. **声明式编排（SequentialChain）**：通过配置定义工作流，框架自动管理执行顺序和数据传递
3. **函数式编排（LCEL）**：使用管道运算符`|`实现更灵活的组合

在v1.2.22版本中，SequentialChain的核心价值在于**复杂工作流的标准化管理**。它特别适用于以下场景：
- 多步骤业务处理流程
- 数据转换流水线
- 企业级审批工作流
- 复杂的AI应用管道

### 架构设计

SequentialChain的架构设计体现了**关注点分离**原则：

```
┌─────────────────────────────────────────────────────────────┐
│                    SequentialChain                          │
├─────────────────────────────────────────────────────────────┤
│ 核心组件：                                                   │
│  1. 子链集合 (Chains) - 按顺序执行的组件                    │
│  2. 输入变量映射 (Input Variables) - 定义工作流输入          │
│  3. 输出变量映射 (Output Variables) - 定义工作流输出         │
│  4. 变量传递规则 (Variable Passing Rules) - 自动化数据流转   │
├─────────────────────────────────────────────────────────────┤
│ 执行流程：                                                   │
│  输入数据 → 链1处理 → 中间结果 → 链2处理 → ... → 最终输出    │
└─────────────────────────────────────────────────────────────┘
```

### SequentialChain家族

LangChain提供了两种主要的SequentialChain变体：

1. **SimpleSequentialChain**：最简单的顺序链，每个链只有一个输入和输出
2. **SequentialChain**：完整功能版，支持复杂的输入输出变量映射

### 在现代LangChain中的定位

在LCEL成为主流的今天，SequentialChain依然有其不可替代的价值：

1. **工作流标准化**：为复杂业务流程提供标准化的编排框架
2. **企业级集成**：更容易与企业现有系统集成
3. **调试友好性**：提供详细的执行日志和错误追踪
4. **向后兼容**：确保现有工作流代码的平稳运行

## 核心要点

### 1. 两种SequentialChain对比

```python
# SimpleSequentialChain - 最简单的工作流
from langchain.chains import SimpleSequentialChain

# 特点：
# 1. 每个子链必须只有一个输入和一个输出
# 2. 前一个链的输出自动作为后一个链的输入
# 3. 整个工作流只有一个最终输出
# 4. 适用于线性、简单的处理流程

# SequentialChain - 完整功能版
from langchain.chains import SequentialChain

# 特点：
# 1. 支持多个输入和输出变量
# 2. 可以显式定义变量映射关系
# 3. 支持并行处理和条件分支（通过组合）
# 4. 适用于复杂的业务流程
```

### 2. 关键特性

- **自动变量传递**：自动处理链之间的数据传递
- **错误传播**：任一链失败都会导致整个工作流终止
- **详细日志**：支持verbose模式，便于调试
- **内存管理**：支持在整个工作流中共享内存
- **配置继承**：运行时配置可以传递给所有子链
- **异步支持**：支持异步执行模式
- **批量处理**：支持批量输入的高效处理

### 3. 变量映射机制

SequentialChain的核心是**变量映射系统**，它定义了：
- 哪些变量是工作流的输入
- 每个链需要哪些变量
- 链的输出如何传递给后续链
- 哪些变量是工作流的最终输出

```python
# 变量映射示例
chain = SequentialChain(
    chains=[chain1, chain2, chain3],
    input_variables=["user_input", "context"],  # 工作流输入
    output_variables=["final_answer", "supporting_info"],  # 工作流输出
    # 变量自动传递规则：
    # chain1输出中的"intermediate_result"自动传递给chain2的"input_data"
    # chain2输出中的"processed_data"自动传递给chain3的"analysis_input"
)
```

### 4. 错误处理策略

SequentialChain提供了多层次错误处理：

```python
from langchain.chains import SequentialChain
from langchain.callbacks import get_openai_callback

# 策略1：全局错误捕获
try:
    result = chain.invoke(inputs)
except Exception as e:
    print(f"工作流执行失败：{e}")
    # 执行回退逻辑

# 策略2：链级重试
# 每个子链可以配置自己的重试策略

# 策略3：监控和追踪
with get_openai_callback() as cb:
    result = chain.invoke(inputs)
    print(f"Token使用统计：{cb}")
```

## 简单示例

### 示例1：简单文本处理流水线

```python
from langchain.chains import LLMChain, SimpleSequentialChain
from langchain.prompts import PromptTemplate
from langchain_openai import ChatOpenAI

# 初始化模型
llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.7)

# 步骤1：文本摘要
summary_template = """为以下文本生成简洁的摘要：
{text}

摘要要求：
1. 保留核心信息
2. 不超过100字
3. 使用中文

摘要："""

summary_prompt = PromptTemplate.from_template(summary_template)
summary_chain = LLMChain(llm=llm, prompt=summary_prompt)

# 步骤2：关键词提取
keyword_template = """从以下文本中提取3-5个关键词：
{text}

要求：
1. 关键词要具有代表性
2. 用逗号分隔
3. 使用中文

关键词："""

keyword_prompt = PromptTemplate.from_template(keyword_template)
keyword_chain = LLMChain(llm=llm, prompt=keyword_prompt)

# 步骤3：情感分析
sentiment_template = """分析以下文本的情感倾向：
{text}

请从以下选项中选择：
1. 积极 (Positive)
2. 消极 (Negative) 
3. 中性 (Neutral)
4. 混合 (Mixed)

并简要说明理由。

分析结果："""

sentiment_prompt = PromptTemplate.from_template(sentiment_template)
sentiment_chain = LLMChain(llm=llm, prompt=sentiment_prompt)

# 创建SimpleSequentialChain
text_processing_chain = SimpleSequentialChain(
    chains=[summary_chain, keyword_chain, sentiment_chain],
    verbose=True  # 显示执行详情
)

# 执行工作流
input_text = """
人工智能（AI）是计算机科学的一个分支，旨在创造能够执行通常需要人类智能的任务的机器。
近年来，深度学习技术的突破极大地推动了AI的发展。从图像识别到自然语言处理，AI正在改变我们的生活方式和工作方式。
虽然AI带来了巨大的机遇，但也引发了一些伦理和社会问题，如就业影响和隐私问题。
"""

result = text_processing_chain.invoke(input_text)
print(f"最终结果：{result['output']}")
```

### 示例2：复杂业务审批流程

```python
from langchain.chains import LLMChain, SequentialChain
from langchain.prompts import PromptTemplate
from langchain_openai import ChatOpenAI

# 初始化模型
llm = ChatOpenAI(model="gpt-4", temperature=0.3)

# 链1：需求分析
requirement_template = """分析以下业务需求：
{requirement}

请从以下维度分析：
1. 业务价值评估（高/中/低）
2. 技术可行性（高/中/低）
3. 预期工作量（人天估算）
4. 关键风险点

分析报告："""

requirement_prompt = PromptTemplate(
    input_variables=["requirement"],
    template=requirement_template
)
requirement_chain = LLMChain(
    llm=llm, 
    prompt=requirement_prompt,
    output_key="requirement_analysis"  # 指定输出键
)

# 链2：技术方案设计
design_template = """基于以下需求和分析，设计技术方案：
原始需求：{requirement}
需求分析：{requirement_analysis}

技术方案要求：
1. 架构设计思路
2. 技术选型建议
3. 实施步骤规划
4. 资源需求评估

技术方案："""

design_prompt = PromptTemplate(
    input_variables=["requirement", "requirement_analysis"],
    template=design_template
)
design_chain = LLMChain(
    llm=llm,
    prompt=design_prompt,
    output_key="technical_design"
)

# 链3：风险评估
risk_template = """评估以下技术方案的风险：
技术方案：{technical_design}

风险评估维度：
1. 技术风险（架构复杂性、技术债务等）
2. 时间风险（工期估算准确性）
3. 成本风险（预算控制）
4. 人员风险（团队能力匹配）

风险评估报告："""

risk_prompt = PromptTemplate(
    input_variables=["technical_design"],
    template=risk_template
)
risk_chain = LLMChain(
    llm=llm,
    prompt=risk_prompt,
    output_key="risk_assessment"
)

# 链4：审批建议
approval_template = """基于以下信息，给出审批建议：
原始需求：{requirement}
需求分析：{requirement_analysis}
技术方案：{technical_design}
风险评估：{risk_assessment}

审批建议选项：
1. 批准 (Approved)
2. 有条件批准 (Approved with Conditions)
3. 拒绝 (Rejected)
4. 需要更多信息 (Need More Information)

请给出建议并说明理由：
建议："""

approval_prompt = PromptTemplate(
    input_variables=[
        "requirement", "requirement_analysis", 
        "technical_design", "risk_assessment"
    ],
    template=approval_template
)
approval_chain = LLMChain(
    llm=llm,
    prompt=approval_prompt,
    output_key="approval_recommendation"
)

# 创建SequentialChain
approval_workflow = SequentialChain(
    chains=[
        requirement_chain,
        design_chain, 
        risk_chain,
        approval_chain
    ],
    input_variables=["requirement"],  # 工作流输入
    output_variables=[  # 工作流输出
        "requirement_analysis",
        "technical_design", 
        "risk_assessment",
        "approval_recommendation"
    ],
    verbose=True
)

# 执行业务审批工作流
business_requirement = """
我们需要开发一个智能客服系统，主要功能包括：
1. 自动回答常见问题
2. 处理用户投诉
3. 提供产品推荐
4. 集成到网站和微信小程序

项目预算：50万元
期望工期：3个月
团队规模：5人
"""

result = approval_workflow.invoke({"requirement": business_requirement})

print("=" * 60)
print("业务审批工作流结果：")
print("=" * 60)
for key, value in result.items():
    if key != "requirement":  # 不显示原始输入
        print(f"\n{key}：")
        print("-" * 40)
        print(value[:500] + "..." if len(value) > 500 else value)
```

### 示例3：多语言内容生产流水线

```python
from langchain.chains import LLMChain, SequentialChain
from langchain.prompts import PromptTemplate
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.7)

# 链1：中文内容生成
chinese_content_template = """用中文创作关于{theme}的内容：
目标读者：{audience}
内容类型：{content_type}
字数要求：约{word_count}字

要求：
1. 结构清晰，逻辑严谨
2. 语言生动，有感染力
3. 包含具体案例和数据支持

中文内容："""

chinese_chain = LLMChain(
    llm=llm,
    prompt=PromptTemplate.from_template(chinese_content_template),
    output_key="chinese_content"
)

# 链2：英文翻译
translation_template = """将以下中文内容翻译为英文：
{chinese_content}

翻译要求：
1. 保持原意准确
2. 符合英文表达习惯
3. 专业术语翻译准确
4. 保持相同的结构和格式

英文翻译："""

translation_chain = LLMChain(
    llm=llm,
    prompt=PromptTemplate.from_template(translation_template),
    output_key="english_translation"
)

# 链3：SEO优化
seo_template = """为以下英文内容进行SEO优化：
{english_translation}

目标关键词：{keywords}
优化要求：
1. 在标题、开头和结尾自然融入关键词
2. 优化元描述（不超过160字符）
3. 建议3-5个内部链接主题
4. 优化可读性（段落长度、句子结构）

SEO优化后的内容："""

seo_chain = LLMChain(
    llm=llm,
    prompt=PromptTemplate.from_template(seo_template),
    output_key="seo_optimized_content"
)

# 链4：社交媒体摘要
social_template = """为以下内容生成社交媒体摘要：
原始内容：{chinese_content}
英文内容：{english_translation}

要求生成：
1. 微博摘要（140字以内）
2. Twitter摘要（280字符以内，英文）
3. LinkedIn摘要（专业风格，200字以内）

社交媒体摘要："""

social_chain = LLMChain(
    llm=llm,
    prompt=PromptTemplate.from_template(social_template),
    output_key="social_media_summary"
)

# 创建内容生产工作流
content_pipeline = SequentialChain(
    chains=[chinese_chain, translation_chain, seo_chain, social_chain],
    input_variables=["theme", "audience", "content_type", "word_count", "keywords"],
    output_variables=[
        "chinese_content",
        "english_translation", 
        "seo_optimized_content",
        "social_media_summary"
    ],
    verbose=False
)

# 执行内容生产
content_result = content_pipeline.invoke({
    "theme": "人工智能在医疗健康领域的应用",
    "audience": "医疗行业从业者和技术爱好者",
    "content_type": "技术分析文章",
    "word_count": "1500",
    "keywords": "AI healthcare, medical AI, health tech"
})

print("内容生产流水线完成！")
print(f"中文内容长度：{len(content_result['chinese_content'])}字")
print(f"英文翻译长度：{len(content_result['english_translation'])}字符")
print(f"社交媒体摘要：{content_result['social_media_summary'][:200]}...")
```

        1. 去除无关信息（广告、垃圾内容等）
        2. 标准化格式（统一标点、大小写）

            llm=self.llm,

        清洗后的反馈：{cleaned_feedback}

        - 产品质量问题
        - 客户服务体验

        - 竞品对比
        
        分类结果："""
        
        classification_chain = LLMChain(
            llm=self.llm,
            prompt=PromptTemplate.from_template(classification_template),
            output_key="feedback_category"

        sentiment_template = """深度分析以下反馈的情感：
        
        反馈内容：{cleaned_feedback}

            llm=self.llm,

            output_key="sentiment_analysis"

        反馈内容：{cleaned_feedback}
        主题分类：{feedback_category}

        return SequentialChain(
            chains=[
                syntax_chain, 
                logic_chain, 
                security_chain, 
                refactor_chain, 
                scoring_chain
            ],
            input_variables=["code", "language"],
            output_variables=[
                "syntax_analysis",
                "logic_analysis",
                "security_report",
                "refactor_suggestions",
                "final_score_and_recommendations"
            ],
            verbose=True
        )
    
    def review_code(self, code: str, language: str = "python") -> Dict[str, Any]:
        """审查代码"""
        # 预处理代码
        cleaned_code = self._preprocess_code(code)
        
        # 执行审查工作流
        result = self.review_workflow.invoke({
            "code": cleaned_code,
            "language": language
        })
        
        # 解析评分
        score = self._parse_score(result["final_score_and_recommendations"])
        
        return {
            "code_stats": {
                "lines": len(code.split('\n')),
                "language": language,
                "review_timestamp": "当前时间"
            },
            "analysis": {
                "syntax": result["syntax_analysis"],
                "logic": result["logic_analysis"],
                "security": result["security_report"],
                "refactoring": result["refactor_suggestions"]
            },
            "score": score,
            "recommendations": result["final_score_and_recommendations"],
            "summary": self._generate_summary(result, score)
        }
    
    def _preprocess_code(self, code: str) -> str:
        """预处理代码"""
        # 移除多余的空行和尾随空格
        lines = [line.rstrip() for line in code.split('\n')]
        # 移除连续的空行，最多保留一个空行
        cleaned_lines = []
        prev_empty = False
        for line in lines:
            if line.strip() == "":
                if not prev_empty:
                    cleaned_lines.append(line)
                    prev_empty = True
            else:
                cleaned_lines.append(line)
                prev_empty = False
        
        return '\n'.join(cleaned_lines)
    
    def _parse_score(self, score_text: str) -> Dict[str, float]:
        """从文本中解析评分（简化实现）"""
        scores = {}
        
        # 实际实现中可能需要更复杂的NLP解析
        patterns = {
            "代码质量": r"代码质量[：:]\s*(\d+)",
            "安全性": r"安全性[：:]\s*(\d+)",
            "可维护性": r"可维护性[：:]\s*(\d+)",
            "性能": r"性能[：:]\s*(\d+)",
            "可读性": r"可读性[：:]\s*(\d+)"
        }
        
        for key, pattern in patterns.items():
            match = re.search(pattern, score_text)
            if match:
                try:
                    scores[key] = float(match.group(1)) / 10.0  # 转换为0-1范围
                except:
                    scores[key] = 0.5  # 默认值
        
        # 计算平均分
        if scores:
            scores["平均分"] = sum(scores.values()) / len(scores)
        else:
            scores["平均分"] = 0.5
        
        return scores
    
    def _generate_summary(self, result: Dict[str, str], score: Dict[str, float]) -> str:
        """生成审查摘要"""
        avg_score = score.get("平均分", 0.5)
        
        if avg_score >= 0.8:
            status = "优秀"
        elif avg_score >= 0.6:
            status = "良好"
        elif avg_score >= 0.4:
            status = "一般"
        else:
            status = "需要改进"
        
        summary = f"""
代码审查摘要：
- 综合评分：{avg_score:.2f}/1.0 ({status})
- 主要优点：代码结构清晰，逻辑正确
- 主要问题：安全性和性能有待优化
- 建议：根据详细报告进行针对性改进
"""
        return summary

## 常见问题

### Q1：SimpleSequentialChain和SequentialChain的主要区别是什么？

**A：** 主要区别如下：

| 特性 | SimpleSequentialChain | SequentialChain |
|------|----------------------|-----------------|
| 输入变量 | 单个输入 | 多个输入变量 |
| 输出变量 | 单个输出 | 多个输出变量 |
| 变量映射 | 自动传递 | 显式变量映射 |
| 链间通信 | 前一个输出作为下一个输入 | 灵活配置变量传递 |
| 适用场景 | 简单线性流程 | 复杂业务流程 |
| 配置复杂度 | 低 | 高 |

### Q2：SequentialChain中的变量冲突如何处理？

**A：** 变量冲突处理的策略：
1. **显式重命名**：使用`output_key`参数重命名输出变量
2. **变量隔离**：在不同链中使用不同的变量名
3. **命名空间**：为不同链的变量添加前缀
4. **自动处理**：部分版本的LangChain会自动处理冲突

```python
# 避免冲突的最佳实践
chain1 = LLMChain(..., output_key="chain1_output")
chain2 = LLMChain(..., output_key="chain2_output")
chain3 = LLMChain(..., output_key="chain3_output")

# 在SequentialChain中清晰定义变量传递
workflow = SequentialChain(
    chains=[chain1, chain2, chain3],
    input_variables=["input"],
    output_variables=["chain3_output"],  # 只暴露最终输出
    # chain1_output自动传递给chain2的对应输入
    # chain2_output自动传递给chain3的对应输入
)
```

### Q3：如何调试复杂的SequentialChain？

**A：** 调试策略：
1. **启用详细日志**：设置`verbose=True`
2. **分步测试**：先单独测试每个子链
3. **中间结果检查**：使用回调函数捕获中间结果
4. **变量追踪**：打印每个链的输入输出
5. **简化复现**：创建最小可复现示例

```python
# 调试技巧
chain = SequentialChain(
    chains=[chain1, chain2, chain3],
    input_variables=["input"],
    output_variables=["output"],
    verbose=True  # 显示详细执行信息
)

# 使用回调监控
from langchain.callbacks import StdOutCallbackHandler

callbacks = [StdOutCallbackHandler()]
result = chain.invoke({"input": "测试输入"}, {"callbacks": callbacks})
```

### Q4：SequentialChain的性能优化有哪些方法？

**A：** 性能优化建议：
1. **批量处理**：使用`batch()`方法处理多个输入
2. **异步执行**：使用`ainvoke()`提高并发性能
3. **缓存中间结果**：对相同输入使用缓存
4. **并行处理**：在可能的情况下使用`RunnableParallel`
5. **模型选择**：根据任务复杂度选择合适的模型

### Q5：SequentialChain与LCEL管道操作符`|`如何选择？

**A：** 选择建议：

**使用SequentialChain的情况**：
1. 需要详细的执行日志和错误追踪
2. 企业级工作流需要标准化管理
3. 现有代码已经使用SequentialChain
4. 需要复杂的变量映射和错误处理

**使用LCEL管道操作符的情况**：
1. 新项目开发
2. 需要更灵活的组合方式
3. 性能要求较高
4. 需要与其他Runnable组件无缝集成

```python
# LCEL方式（现代推荐）
lc_workflow = chain1 | chain2 | chain3

# SequentialChain方式（企业级工作流）
seq_workflow = SequentialChain(
    chains=[chain1, chain2, chain3],
    input_variables=["input"],
    output_variables=["output"]
)
```

## 本节总结

SequentialChain作为LangChain框架中的**工作流编排引擎**，在企业级AI应用中发挥着关键作用：

### 关键收获

1. **工作流标准化**：SequentialChain为复杂业务流程提供了标准化的编排框架
2. **变量管理**：强大的变量映射系统简化了链间数据传递
3. **错误处理**：内置的错误传播和重试机制提高了系统稳定性
4. **调试友好**：详细的执行日志和中间结果追踪便于调试

### 版本适配建议

- **新项目**：考虑使用LCEL管道操作符，但在需要复杂工作流管理时仍可使用SequentialChain
- **现有项目**：继续使用SequentialChain，逐步迁移到LCEL
- **企业应用**：优先使用SequentialChain，因其提供更好的管理和监控能力

### 最佳实践

1. **清晰变量命名**：使用有意义的变量名，避免冲突
2. **分步开发和测试**：先开发测试单个链，再组合成工作流
3. **合理设计工作流**：避免过长的链式调用，适当拆分
4. **监控和日志**：启用详细日志，监控工作流执行状态
5. **错误处理策略**：为每个链和工作流设计适当的错误处理

### 扩展学习方向

1. **高级工作流模式**：学习并行处理、条件分支、循环等高级模式
2. **与LangGraph集成**：探索使用LangGraph编排更复杂的工作流
3. **企业级监控**：学习如何监控和优化工作流性能
4. **容器化和部署**：学习如何将工作流部署到生产环境

SequentialChain代表了**声明式工作流编排**的最佳实践。它通过抽象复杂的数据流和执行业务逻辑，使开发者能够专注于业务需求而非实现细节。在构建企业级AI应用时，合理使用SequentialChain可以显著提高开发效率和应用质量。
