# 8.1.2 并行执行与结果合并

## 概念讲解

### 什么是并行执行

在实际应用中，我们经常需要同时执行多个独立任务。例如，同时搜索多个数据源、并行调用多个API、或者同时让多个模型处理不同子任务。串行执行会浪费大量等待时间，并行执行可以显著提升效率。

### RunnableParallel：LangChain的并行抽象

`RunnableParallel`是LangChain提供的并行执行组件。它接受一个字典，字典的每个值是一个Runnable，这些Runnable会并行执行，最终结果合并为一个字典。

```
输入 → ┌─ Runnable A → 结果A
      ├─ Runnable B → 结果B  ← 并行执行
      └─ Runnable C → 结果C
           ↓
      合并为 {A: 结果A, B: 结果B, C: 结果C}
```

## 核心要点

1. **RunnableParallel**：字典语法 `{key: runnable}` 自动创建并行链
2. **独立执行**：每个Runnable接收相同输入，独立运行
3. **结果合并**：所有结果合并为一个字典
4. **性能提升**：适合IO密集型任务的并行处理

## 简单示例

### 基本并行执行

```python
from langchain_core.runnables import RunnableParallel

# 定义多个并行任务
chain = RunnableParallel({
    "summary": summary_chain,      # 生成摘要
    "keywords": keyword_chain,     # 提取关键词
    "sentiment": sentiment_chain   # 情感分析
})

# 三个任务并行执行
result = chain.invoke({"text": "这是一篇文章..."})
# result = {"summary": "...", "keywords": [...], "sentiment": "positive"}
```

### 语法糖：字典即并行

```python
# 以下两种写法等价
chain1 = RunnableParallel({"a": chain_a, "b": chain_b})
chain2 = {"a": chain_a, "b": chain_b}  # 自动转换为RunnableParallel

# 使用
result = chain2.invoke(input_data)
```

### 并行 + 串联组合

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini")

# 先用不同提示词并行处理，然后合并
parallel_chain = {
    "translation": prompt1 | model,
    "summary": prompt2 | model,
}

# 整体还是一个Runnable，可以继续链式组合
full_chain = parallel_chain | merge_function
```

## 进阶应用

### 多模型并行对比

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

models = {
    "gpt": ChatOpenAI(model="gpt-4o-mini") | parser,
    "claude": ChatAnthropic(model="claude-sonnet-4-mini") | parser,
}

# 并行调用多个模型
results = models.invoke("解释量子计算")
# {"gpt": "...", "claude": "..."}
```

### 数据源并行查询

```python
from langchain_community.tools import TavilySearchResults

search_chain = {
    "web_search": tavily_chain,
    "database_query": db_chain,
    "api_data": api_chain,
}

# 同时查询多个数据源
results = search_chain.invoke({"query": "最新AI趋势"})
```

## 常见问题

**Q: 并行执行真的同时运行吗？**

A: 对于IO密集型任务（API调用），Python的asyncio可以实现真正的并发。对于CPU密集型任务，受GIL限制，可能更多是交错执行。

**Q: 如果一个Runnable失败了怎么办？**

A: 默认情况下，一个失败会导致整个链中断。可以使用错误处理组件添加重试逻辑。

## 本节总结

RunnableParallel是LangChain提供的并行执行组件。通过字典语法，可以轻松定义多个并行任务，并将结果合并。它与RunnableSequence配合使用，可以构建复杂的并行-串行混合流程。
