# 8.1.2 并行执行与结果合并

## 概念讲解（文字+图示）

并行执行是LangChain工作流引擎实现**高性能处理**的核心机制，它突破了传统顺序执行的性能瓶颈，通过**同时执行多个独立任务**来显著提升系统吞吐量。在AI应用开发中，许多处理任务（如多模型推理、多数据源查询、多维度分析）天然具有可并行性，LangChain的并行执行框架为开发者提供了简洁而强大的并发编程抽象。

### 设计哲学：声明式并发
LangChain v1.2.22采用**声明式并发模型**，开发者只需定义"要并行执行什么"，框架自动处理"如何并行执行"的复杂性：
1. **任务编排透明化**：自动调度并行任务，无需手动管理线程/协程
2. **资源感知调度**：智能分配计算资源，避免过载
3. **错误隔离**：单个任务失败不影响其他并行任务
4. **统一接口**：并行执行与顺序执行使用相同的`invoke`/`batch`/`stream`接口

### 并行执行架构
```mermaid
graph TB
    A[输入数据<br/>{"text": "example"}] --> B{并行执行引擎}
    
    B --> C[Runnable A<br/>任务1]
    B --> D[Runnable B<br/>任务2]
    B --> E[Runnable C<br/>任务3]
    
    C --> F[结果A]
    D --> G[结果B]
    E --> H[结果C]
    
    F --> I{结果合并器}
    G --> I
    H --> I
    
    I --> J[合并结果<br/>{"result_a": ..., "result_b": ..., "result_c": ...}]
    
    K[并发控制策略] --> B
    L[错误处理策略] --> I
    M[资源限制] --> B
```

### 框架为我们屏蔽的复杂性
1. **并发安全**：自动处理线程安全和数据竞争问题
2. **死锁预防**：智能任务调度避免死锁发生
3. **资源管理**：自动管理CPU/GPU/API调用等有限资源
4. **超时控制**：统一设置和管理并行任务超时
5. **结果排序**：保持结果顺序与定义顺序一致（可选）

## 核心要点（重点标记）

**🔑 RunnableParallel的核心特性：**

1. **字典字面量语法**：`{"key1": runnable1, "key2": runnable2}`自动创建并行任务
2. **统一输入**：所有并行任务接收相同的输入数据
3. **独立执行**：任务间无依赖，可同时执行
4. **自动合并**：结果自动合并为字典，键名对应任务键名
5. **批量优化**：`batch()`调用时自动合并批量处理，减少开销

**🚀 并行执行的构建方式：**

| 构建方式 | 语法 | 适用场景 | 优点 |
|---------|------|----------|------|
| 字典字面量 | `{ "task1": runnable1, "task2": runnable2 }` | 简单并行、快速原型 | 语法简洁，直观易用 |
| RunnableParallel类 | `RunnableParallel(tasks_dict)` | 动态构建、程序化控制 | 支持运行时动态配置 |
| 管道中的并行段 | `runnable1 | { "a": runnable2, "b": runnable3 }` | 混合工作流 | 灵活组合顺序与并行 |

**📊 结果合并策略：**

- **字典合并**：每个任务返回字典，自动合并到输出字典
- **键名映射**：通过任务键名控制结果字典的键名
- **冲突处理**：支持重命名、覆盖、合并等多种冲突解决策略
- **类型转换**：自动处理不同类型结果的兼容性
- **部分结果**：支持部分任务失败时的结果合并

```python
from langchain_openai import ChatOpenAI

from langchain_core.output_parsers import StrOutputParser
import os
import time

# 设置OpenAI API密钥
os.environ["OPENAI_API_KEY"] = "your-api-key-here"

# 创建基础处理函数
def analyze_length(text: str) -> dict:
    """分析文本长度"""
    time.sleep(0.1)  # 模拟处理延迟
    return {
        "char_count": len(text),
        "word_count": len(text.split()),
        "sentence_count": len([s for s in text.split('.') if s.strip()])
    }

def analyze_sentiment(text: str) -> dict:
    """分析文本情感"""
    time.sleep(0.15)  # 模拟处理延迟
    positive_words = ["好", "优秀", "满意", "推荐", "不错", "出色"]
    negative_words = ["差", "糟糕", "失望", "问题", "不好", "困难"]
    
    pos_score = sum(1 for word in positive_words if word in text)
    neg_score = sum(1 for word in negative_words if word in text)
    
    if pos_score > neg_score:
        sentiment = "积极"
        confidence = pos_score / (pos_score + neg_score + 1)
    elif neg_score > pos_score:
        sentiment = "消极"
        confidence = neg_score / (pos_score + neg_score + 1)
    else:
        sentiment = "中性"
        confidence = 0.5
    
    return {
        "sentiment": sentiment,
        "confidence": round(confidence, 2),
        "positive_words": pos_score,
        "negative_words": neg_score
    }

def extract_keywords(text: str) -> dict:
    """提取关键词（简化实现）"""
    time.sleep(0.2)  # 模拟处理延迟
    words = text.split()
    # 简单的关键词提取逻辑
    keywords = [word for word in words if len(word) > 2][:5]
    return {"keywords": keywords}

# 创建并行执行链
parallel_chain = RunnableParallel(
    length_analysis=RunnableLambda(lambda x: analyze_length(x["text"])),
    sentiment_analysis=RunnableLambda(lambda x: analyze_sentiment(x["text"])),
    keyword_extraction=RunnableLambda(lambda x: extract_keywords(x["text"]))
)

# 测试文本
test_text = """
LangChain是一个强大的AI应用开发框架。它提供了丰富的组件和工具，
简化了从原型到生产环境的开发过程。并行执行是框架的核心特性之一，
能够显著提升处理效率，特别适合处理大规模数据。
"""

# 执行并行分析
print("开始并行分析...")
start_time = time.perf_counter()
result = parallel_chain.invoke({"text": test_text})
execution_time = time.perf_counter() - start_time

print(f"并行分析完成，耗时: {execution_time:.3f}秒")
print("\n分析结果:")

# 显示分析结果
print("1. 长度分析:")
for key, value in result["length_analysis"].items():
    print(f"   {key}: {value}")

print("\n2. 情感分析:")
for key, value in result["sentiment_analysis"].items():
    print(f"   {key}: {value}")

print("\n3. 关键词提取:")
print(f"   关键词: {', '.join(result['keyword_extraction']['keywords'])}")

# 对比顺序执行时间
print("\n" + "="*50)
print("对比顺序执行性能:")

def sequential_analysis(text: str):
    """顺序执行相同的分析"""
    results = {}
    results.update(analyze_length(text))
    results.update(analyze_sentiment(text))
    results.update(extract_keywords(text))

start_time = time.perf_counter()
sequential_result = sequential_analysis(test_text)
sequential_time = time.perf_counter() - start_time

print(f"顺序执行耗时: {sequential_time:.3f}秒")
```

### 企业级并行执行管理器
```python
from typing import Dict, List, Any, Optional, Callable
from dataclasses import dataclass
from datetime import datetime
import concurrent.futures
import logging
from enum import Enum

# 配置日志
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class TaskPriority(Enum):
    """任务优先级"""
    HIGH = 3
    MEDIUM = 2
    LOW = 1

@dataclass
class ParallelTask:
    """并行任务定义"""
    name: str
    runnable: Any
    timeout_seconds: float = 30.0
    priority: TaskPriority = TaskPriority.MEDIUM
    retry_count: int = 2
    depends_on: List[str] = None
    
    def __post_init__(self):
        if self.depends_on is None:
            self.depends_on = []

class EnterpriseParallelExecutor:
    """企业级并行执行管理器"""
    
    def __init__(self, max_workers: int = 10, enable_async: bool = True):
        self.max_workers = max_workers
        self.enable_async = enable_async
        self.task_registry = {}
        self.execution_history = []
        
        # 性能统计
        self.stats = {
            "total_executions": 0,
            "successful_executions": 0,
            "failed_executions": 0,
            "total_execution_time": 0.0,
            "avg_execution_time": 0.0
        }
    
    def register_task(self, task: ParallelTask):
        """注册并行任务"""
        self.task_registry[task.name] = task
        logger.info(f"注册任务: {task.name} (超时: {task.timeout_seconds}s, 优先级: {task.priority.name})")
        return self
    
    def execute_parallel(self, input_data: Dict, task_names: List[str]) -> Dict[str, Any]:
        """执行并行任务"""
        execution_start = datetime.now()
        results = {}
        errors = {}
        
        # 筛选要执行的任务
        tasks_to_execute = []
        for name in task_names:
            if name in self.task_registry:
                tasks_to_execute.append(self.task_registry[name])
            else:
                logger.warning(f"任务 '{name}' 未注册，已跳过")
        
        # 检查依赖关系
        valid_tasks = self._validate_dependencies(tasks_to_execute)
        if not valid_tasks:
            raise ValueError("存在循环依赖或未满足的依赖关系")
        
        # 按优先级排序
        sorted_tasks = sorted(valid_tasks, key=lambda t: t.priority.value, reverse=True)
        
        # 执行任务
        with concurrent.futures.ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            # 提交任务
            future_to_task = {}
            for task in sorted_tasks:
                future = executor.submit(
                    self._execute_single_task,
                    task,
                    input_data
                )
                future_to_task[future] = task
            
            # 收集结果
            for future in concurrent.futures.as_completed(future_to_task):
                task = future_to_task[future]
                try:
                    task_result = future.result(timeout=task.timeout_seconds)
                    results[task.name] = task_result
                    logger.info(f"任务 '{task.name}' 执行成功")
                except concurrent.futures.TimeoutError:
                    error_msg = f"任务 '{task.name}' 执行超时 ({task.timeout_seconds}s)"
                    errors[task.name] = error_msg
                    logger.error(error_msg)
                except Exception as e:
                    error_msg = f"任务 '{task.name}' 执行失败: {str(e)}"
                    errors[task.name] = error_msg
                    logger.error(error_msg)
        
        # 记录执行历史
        execution_time = (datetime.now() - execution_start).total_seconds()
        execution_record = {
            "timestamp": execution_start,
            "duration": execution_time,
            "tasks_executed": len(sorted_tasks),
            "successful_tasks": len(results),
            "failed_tasks": len(errors),
            "input_data_keys": list(input_data.keys()),
            "results_keys": list(results.keys()),
            "errors": errors
        }
        self.execution_history.append(execution_record)
        
        # 更新统计
        self.stats["total_executions"] += 1
        self.stats["successful_executions"] += len(results)
        self.stats["failed_executions"] += len(errors)
        self.stats["total_execution_time"] += execution_time
        self.stats["avg_execution_time"] = (
            self.stats["total_execution_time"] / self.stats["total_executions"]
        )
        
        return {
            "results": results,
            "errors": errors,
            "execution_summary": execution_record
        }
    
    def _execute_single_task(self, task: ParallelTask, input_data: Dict) -> Any:
        """执行单个任务（带重试）"""
        last_error = None
        
        for attempt in range(task.retry_count + 1):
            try:
                start_time = time.perf_counter()
                
                # 执行任务
                if hasattr(task.runnable, "invoke"):
                    result = task.runnable.invoke(input_data)
                elif callable(task.runnable):
                    result = task.runnable(input_data)
                else:
                    raise ValueError(f"任务 '{task.name}' 的runnable不可执行")
                
                execution_time = time.perf_counter() - start_time
                logger.debug(f"任务 '{task.name}' 第{attempt+1}次尝试成功，耗时: {execution_time:.3f}s")
                
                return result
                
            except Exception as e:
                last_error = e
                if attempt < task.retry_count:
                    logger.warning(f"任务 '{task.name}' 第{attempt+1}次尝试失败，准备重试: {str(e)}")
                    time.sleep(1 * (attempt + 1))  # 指数退避
                else:
                    logger.error(f"任务 '{task.name}' 所有重试均失败")
        
        # 所有重试都失败
        raise last_error if last_error else RuntimeError(f"任务 '{task.name}' 执行失败")
    
    def _validate_dependencies(self, tasks: List[ParallelTask]) -> List[ParallelTask]:
        """验证任务依赖关系"""
        task_map = {task.name: task for task in tasks}
        valid_tasks = []
        
        # 检查循环依赖
        def has_cycle(task, visited=None, recursion_stack=None):
            if visited is None:
                visited = set()
            if recursion_stack is None:
                recursion_stack = set()
            
            if task.name in recursion_stack:
                return True
            if task.name in visited:
                return False
            
            visited.add(task.name)
            recursion_stack.add(task.name)
            
            for dep_name in task.depends_on:
                if dep_name in task_map:
                    if has_cycle(task_map[dep_name], visited, recursion_stack):
                        return True
            
            recursion_stack.remove(task.name)
            return False
        
        # 检查依赖是否满足
        for task in tasks:
            # 检查循环依赖
            if has_cycle(task):
                logger.error(f"任务 '{task.name}' 存在循环依赖")
                continue
            
            # 检查所有依赖是否都在要执行的任务列表中
            missing_deps = [dep for dep in task.depends_on if dep not in task_map]
            if missing_deps:
                logger.warning(f"任务 '{task.name}' 的依赖未满足: {missing_deps}")
                continue
            
            valid_tasks.append(task)
        
        return valid_tasks
    
    def get_statistics(self) -> Dict:
        """获取执行统计"""
        return {
            **self.stats,
            "registered_tasks": len(self.task_registry),
            "execution_history_count": len(self.execution_history),
            "current_max_workers": self.max_workers,
            "async_enabled": self.enable_async
        }
```

### 智能结果合并策略
```python
from typing import Dict, Any, List, Union
from functools import reduce
import json

class SmartResultMerger:
    """智能结果合并器"""
    
    @staticmethod
    def merge_dicts(dicts: List[Dict], strategy: str = "union") -> Dict:
        """合并多个字典"""
        if not dicts:
            return {}
        
        if strategy == "union":
            # 并集合并，冲突时覆盖
            result = {}
            for d in dicts:
                result.update(d)
            return result
            
        elif strategy == "intersection":
            # 交集合并，只保留所有字典都有的键
            if len(dicts) == 1:
                return dicts[0]
            
            common_keys = set(dicts[0].keys())
            for d in dicts[1:]:
                common_keys.intersection_update(d.keys())
            
            result = {}
            for key in common_keys:
                # 合并值（如果是字典则递归合并）
                values = [d[key] for d in dicts if key in d]
                if all(isinstance(v, dict) for v in values):
                    result[key] = SmartResultMerger.merge_dicts(values, strategy)
                else:
                    # 简单情况下取第一个值
                    result[key] = values[0]
            return result
            
        elif strategy == "weighted":
            # 加权合并（需要额外权重信息）
            # 简化实现：假设每个字典权重相等
            result = {}
            for d in dicts:
                for key, value in d.items():
                    if key not in result:
                        result[key] = []
                    result[key].append(value)
            
            # 对列表值进行聚合
            for key in list(result.keys()):
                values = result[key]
                if all(isinstance(v, (int, float)) for v in values):
                    result[key] = sum(values) / len(values)  # 平均值
                elif all(isinstance(v, str) for v in values):
                    result[key] = values[0]  # 取第一个字符串
                elif all(isinstance(v, list) for v in values):
                    # 合并列表
                    merged_list = []
                    for v in values:
                        merged_list.extend(v)
                    result[key] = merged_list
                else:
                    result[key] = values[0]
            
            return result
            
        else:
            raise ValueError(f"不支持的合并策略: {strategy}")
    
    @staticmethod
    def merge_parallel_results(parallel_results: Dict[str, Any], 
                               group_strategy: Dict[str, str] = None) -> Dict:
        """合并并行执行结果"""
        if group_strategy is None:
            group_strategy = {}
        
        result = {}
        
        for task_name, task_result in parallel_results.items():
            # 获取该任务的分组策略
            strategy = group_strategy.get(task_name, "union")
            
            if isinstance(task_result, dict):
                # 如果是字典，直接合并
                if strategy == "union":
                    result.update(task_result)
                elif strategy == "nested":
                    # 嵌套结构
                    result[task_name] = task_result
                elif strategy == "prefix":
                    # 添加前缀
                    prefixed = {f"{task_name}_{k}": v for k, v in task_result.items()}
                    result.update(prefixed)
                else:
                    result[task_name] = task_result
            else:
                # 非字典结果
                result[task_name] = task_result
        
        return result
    
    @staticmethod
    def resolve_conflicts(results: Dict[str, Any], 
                         conflict_rules: Dict[str, Callable] = None) -> Dict:
        """解决结果冲突"""
        if conflict_rules is None:
            conflict_rules = {}
        
        # 找出重复的键
        all_keys = []
        for value in results.values():
            if isinstance(value, dict):
                all_keys.extend(value.keys())
        
        from collections import Counter
        key_counts = Counter(all_keys)
        conflicting_keys = [key for key, count in key_counts.items() if count > 1]
        
        if not conflicting_keys:
            return results
        
        # 应用冲突解决规则
        resolved_results = results.copy()
        
        for conflict_key in conflicting_keys:
            # 收集所有冲突值
            conflicting_values = []
            sources = []
            
            for task_name, task_result in results.items():
                if isinstance(task_result, dict) and conflict_key in task_result:
                    conflicting_values.append(task_result[conflict_key])
                    sources.append(task_name)
            
            # 应用冲突解决规则
            if conflict_key in conflict_rules:
                # 使用自定义规则
                resolved_value = conflict_rules[conflict_key](conflicting_values, sources)
            else:
                # 默认规则：取第一个值
                resolved_value = conflicting_values[0]
            
            # 更新结果
            # 只保留一个副本，删除其他副本
            for task_name in sources:
                if isinstance(resolved_results[task_name], dict):
                    if task_name == sources[0]:
                        resolved_results[task_name][conflict_key] = resolved_value
                    else:
                        del resolved_results[task_name][conflict_key]
        
        return resolved_results
```

## 常见问题

### Q1: 并行执行与并发执行有什么区别？
**A:** 关键区别：
1. **并行执行**：多个任务**同时**在不同CPU核心上执行（真正的同时）
2. **并发执行**：多个任务**交替**在单个CPU核心上执行（看似同时）
3. **LangChain实现**：`RunnableParallel`支持真正并行（多线程/多进程）和并发（异步协程）
4. **选择依据**：CPU密集型任务适合并行，I/O密集型任务适合并发

### Q2: 如何控制并行执行的最大并发数？
**A:** 并发控制方法：
1. **线程池大小**：通过`ThreadPoolExecutor(max_workers=n)`控制
2. **信号量限制**：使用`asyncio.Semaphore(n)`限制异步并发数
3. **批量大小**：通过`batch()`的批量大小间接控制并发
4. **资源感知**：根据CPU核心数、内存大小、API限制动态调整

### Q3: 并行执行中的错误处理策略？
**A:** 错误处理策略：
1. **独立错误处理**：每个任务独立处理自己的错误，不影响其他任务
2. **部分结果返回**：允许部分任务失败，返回成功任务的结果
3. **重试机制**：为每个任务配置独立的重试策略
4. **断路器模式**：连续失败时暂时禁用任务，防止雪崩
5. **超时控制**：为每个任务设置独立超时，防止无限等待

### Q4: 如何调试并行执行的问题？
**A:** 调试技术：
1. **任务跟踪**：为每个任务添加唯一ID，跟踪执行过程
2. **性能分析**：使用`cProfile`或`py-spy`分析并行任务性能
3. **日志分离**：为每个任务生成独立日志，便于问题定位
4. **可视化工具**：使用`concurrent.futures`的回调可视化执行流程
5. **单元测试**：为每个并行任务编写独立的单元测试

### Q5: 并行执行的内存管理注意事项？
**A:** 内存管理建议：
1. **数据复制**：注意并行任务间的数据共享与复制开销
2. **批量处理**：对于大数据集，使用分批处理避免内存溢出
3. **结果积累**：及时清理不再需要的中间结果
4. **资源监控**：监控并行执行时的内存使用情况
5. **垃圾回收**：适当手动触发垃圾回收，释放不再使用的对象
