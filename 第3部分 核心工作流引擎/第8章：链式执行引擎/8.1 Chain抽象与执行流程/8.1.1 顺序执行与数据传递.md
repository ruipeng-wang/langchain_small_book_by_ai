# 8.1.1 顺序执行与数据传递

## 概念讲解

### 什么是链（Chain）

链（Chain）是LangChain中将多个组件按顺序连接执行的机制。它解决了AI应用中的一个核心问题：单个模型调用往往不够，需要将提示模板、模型、输出解析器等组件串联起来，形成完整的处理流程。

### Runnable接口：统一的执行抽象

在LangChain中，几乎所有组件都实现了**Runnable**接口。这意味着它们都可以用相同的方式被调用、流式输出和批处理。Runnable是链式执行的基础。

### RunnableSequence：使用 | 操作符链式组合

LangChain提供了简洁的语法来创建链：使用Python的 `|`（管道）操作符。这使得链的创建直观易懂。

```
输入 → 组件1 → 组件2 → ... → 组件N → 输出
```

## 核心要点

1. **| 操作符**：将多个Runnable按顺序连接
2. **数据流**：前一个组件的输出作为后一个组件的输入
3. **RunnablePassthrough**：透传数据，不修改内容
4. **字典传递**：链中可以使用字典传递多个值

## 简单示例

### 基础链：模板 + 模型

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 创建组件
prompt = ChatPromptTemplate.from_template("解释什么是{topic}")
model = ChatOpenAI(model="gpt-4o-mini")

# 组合成链
chain = prompt | model

# 执行
response = chain.invoke({"topic": "机器学习"})
print(response.content)
```

### 带输出解析的链

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_template("解释{topic}")
model = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()  # 将AIMessage转为字符串

# 三段式链
chain = prompt | model | parser

result = chain.invoke({"topic": "深度学习"})
print(type(result))  # <class 'str'>
```

### RunnablePassthrough：数据透传

当需要将原始输入同时传递给多个组件时，使用`RunnablePassthrough`：

```python
from langchain_core.runnables import RunnablePassthrough

# 透传原始输入
chain = {
    "topic": RunnablePassthrough(),  # 保持原始值
    "context": lambda x: f"关于{x['topic']}的补充信息"  # 转换值
} | prompt | model

result = chain.invoke({"topic": "AI"})
```

### 多步骤数据处理

```python
from langchain_core.runnables import RunnableLambda

def format_topic(topic: str) -> str:
    return f"请详细解释：{topic}"

def add_context(result: str) -> str:
    return f"以下是AI生成的内容：\n{result}"

# 数据处理链
chain = (
    RunnableLambda(format_topic)  # 格式化输入
    | prompt                       # 生成提示
    | model                        # 调用模型
    | parser                       # 解析输出
    | RunnableLambda(add_context)  # 添加上下文
)

result = chain.invoke({"topic": "自然语言处理"})
```

## 进阶应用

### 多输入链

```python
from operator import itemgetter
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个翻译助手。将以下{text}从{source_lang}翻译为{target_lang}。"),
    ("human", "{text}"),
])

# 使用itemgetter提取输入
chain = (
    {"text": itemgetter("text"), 
     "source_lang": itemgetter("source"), 
     "target_lang": itemgetter("target")}
    | prompt
    | ChatOpenAI(model="gpt-4o-mini")
)

result = chain.invoke({
    "text": "Hello World",
    "source": "English",
    "target": "Chinese"
})
```

### 链的链式组合

```python
# 创建子链
summarize_chain = prompt1 | model1
translate_chain = prompt2 | model2

# 组合子链
full_chain = summarize_chain | translate_chain
```

## 常见问题

**Q: | 操作符和普通函数调用有什么区别？**

A: `|` 创建的是一个惰性链对象，只有在调用`invoke()`时才执行。这支持了流式、批量和异步执行。

**Q: 数据在链中是如何传递的？**

A: 链中每个组件的输出会自动作为下一个组件的输入。如果输入是字典，它会原样传递；如果是其他类型，则需要确保下游组件能接受。

**Q: 如何在链中保留多个值？**

A: 使用字典格式 `{"key1": value1, "key2": value2}` 可以同时传递多个值。`RunnablePassthrough()` 可保留原始输入。

## 本节总结

顺序执行是链式执行最基础的用法。通过 `|` 操作符，可以将提示模板、模型、解析器等组件串联起来，形成清晰的数据处理流程。理解数据在链中的传递方式，是构建复杂AI应用的基础。
