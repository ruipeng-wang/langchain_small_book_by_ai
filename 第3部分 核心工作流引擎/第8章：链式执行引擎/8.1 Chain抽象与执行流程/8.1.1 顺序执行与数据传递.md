# 8.1.1 顺序执行与数据传递

## 概念讲解（文字+图示）

顺序执行是LangChain链式工作流最基础也是最核心的执行模式，它体现了**数据流驱动**的设计哲学。在顺序执行中，多个处理单元（Runnable）按照定义好的顺序依次执行，前一个单元的输出作为后一个单元的输入，形成一个**单向数据管道**。

### 设计哲学：函数式数据流
LangChain v1.2.22基于**函数式编程范式**重构了执行引擎，将每个处理单元抽象为`Runnable`接口。这种设计带来了几个关键优势：

1. **纯函数特性**：每个Runnable单元的输出只依赖于输入，不依赖外部状态
2. **易于组合**：通过管道操作符(`|`)或`.pipe()`方法可以轻松组合多个Runnable
3. **声明式编程**：定义"做什么"而不是"如何做"，简化复杂工作流构建
4. **自动错误处理**：内置重试、回退和超时机制

### 数据传递机制
```mermaid
graph LR
    A[输入字典<br/>{"input": "value"}] --> B[Runnable 1<br/>处理输入]
    B --> C[中间结果<br/>{"output1": "processed"}]
    C --> D[Runnable 2<br/>进一步处理]
    D --> E[最终结果<br/>{"output2": "final"}]
    
    F[输入键名映射] --> G{键名映射关系}
    H[输出键名映射] --> G
    
    G --> I[自动键名解析<br/>无需手动数据传递]
```

LangChain通过**智能键名映射**实现了自动数据传递：
- **输入映射**：自动从输入字典中选择正确的键值
- **输出合并**：多个Runnable的输出自动合并到结果字典
- **冲突解决**：同名键冲突时的处理策略
- **类型转换**：自动处理不同类型数据的兼容性

### 框架为我们屏蔽的复杂性
1. **异步处理同步化**：统一了同步(`invoke`)和异步(`ainvoke`)调用接口
2. **批量处理优化**：自动将单次调用转换为批量调用(`batch`)以提高效率
3. **流式输出处理**：支持逐步输出(`stream`)而无需手动管理中间状态
4. **错误传播**：自动将错误沿管道传递并收集上下文信息
5. **配置管理**：运行时配置（如模型参数）沿管道自动传递

## 核心要点（重点标记）

**🔑 Runnable接口的核心方法：**

1. **`invoke(input, config?)`**：同步执行，返回处理结果
2. **`ainvoke(input, config?)`**：异步执行，返回awaitable结果
3. **`batch(inputs, config?)`**：批量处理，优化性能
4. **`stream(input, config?)`**：流式处理，支持实时输出
5. **`transform(iterator, config?)`**：转换输入迭代器

**🚀 顺序执行的三种构建方式：**

| 构建方式 | 语法 | 适用场景 | 优点 |
|---------|------|----------|------|
| 管道操作符 | `runnable1 | runnable2 | runnable3` | 快速原型、简单工作流 | 简洁直观，类似Unix管道 |
| .pipe()方法 | `runnable1.pipe(runnable2).pipe(runnable3)` | 动态构建、条件组合 | 支持运行时动态组合 |
| SequentialChain | `SequentialChain(chains=[chain1, chain2])` | 传统API兼容、复杂映射 | 支持复杂的输入输出映射 |

**📊 数据传递的关键特性：**

- **自动键名解析**：根据`input_variables`自动匹配输入键
- **输出合并策略**：支持覆盖、合并、重命名等多种输出合并方式
- **类型安全**：支持Pydantic模型验证输入输出格式
- **调试友好**：每个步骤的执行详情可通过`verbose=True`查看

### 基础顺序链示例
```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser
import os

# 设置OpenAI API密钥（实际使用时应从环境变量读取）
os.environ["OPENAI_API_KEY"] = "your-api-key-here"

# 1. 创建提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位专业的文本编辑助手。"),
    ("user", "请优化以下文本，使其更加专业和流畅：\n\n{text}")
])

# 2. 创建语言模型
model = ChatOpenAI(model="gpt-4o", temperature=0.7)

# 3. 创建输出解析器
output_parser = StrOutputParser()

# 4. 使用管道操作符构建顺序链
chain = prompt | model | output_parser

# 5. 执行链
input_text = "我们公司的产品非常好用，用户都说很满意。"
result = chain.invoke({"text": input_text})
print("优化后的文本:")
print(result)
print("-" * 50)

# 6. 查看链的结构
print("链结构:")
print(f"输入模式: {chain.input_schema.schema()}")
print(f"输出模式: {chain.output_schema.schema()}")
```

```python
from langchain_core.runnables import RunnablePassthrough, RunnableLambda
from langchain_core.pydantic_v1 import BaseModel, Field
from typing import List, Dict, Any

# 定义数据结构
class TextAnalysis(BaseModel):
    """文本分析结果"""
    original_text: str = Field(description="原始文本")
    word_count: int = Field(description="单词数量")
    sentence_count: int = Field(description="句子数量")
    sentiment: str = Field(description="情感倾向", enum=["positive", "negative", "neutral"])
    key_phrases: List[str] = Field(description="关键短语列表")

# 创建自定义处理函数
def count_words(text: str) -> Dict[str, Any]:
    """统计单词数量"""
    words = text.split()
    return {"word_count": len(words)}

def count_sentences(text: str) -> Dict[str, Any]:
    """统计句子数量"""
    # 简单句子分割逻辑
    sentences = [s.strip() for s in text.replace('!', '.').replace('?', '.').split('.') if s.strip()]
    return {"sentence_count": len(sentences)}

def analyze_sentiment(text: str) -> Dict[str, Any]:
    """分析情感倾向（简化示例）"""
    positive_words = ["好", "优秀", "满意", "推荐", "不错"]
    negative_words = ["差", "糟糕", "失望", "问题", "不好"]
    
    pos_count = sum(1 for word in positive_words if word in text)
    neg_count = sum(1 for word in negative_words if word in text)
    
    if pos_count > neg_count:
        sentiment = "positive"
    elif neg_count > pos_count:
        sentiment = "negative"
    else:
        sentiment = "neutral"
    
    return {"sentiment": sentiment}

def extract_key_phrases(text: str) -> Dict[str, Any]:
    """提取关键短语（简化示例）"""
    # 实际应用中应使用NLP模型
    phrases = []
    words = text.split()
    if len(words) >= 3:
        for i in range(len(words) - 2):
            phrases.append(" ".join(words[i:i+3]))
    return {"key_phrases": phrases[:5]}  # 只返回前5个

# 构建顺序链
analysis_chain = (
    # 保留原始文本
    RunnablePassthrough.assign(original_text=lambda x: x["text"])
    # 并行执行多个分析
    | {
        "word_count": RunnableLambda(lambda x: count_words(x["text"])),
        "sentence_count": RunnableLambda(lambda x: count_sentences(x["text"])),
        "sentiment": RunnableLambda(lambda x: analyze_sentiment(x["text"])),
        "key_phrases": RunnableLambda(lambda x: extract_key_phrases(x["text"]))
    }
    # 合并结果
    | RunnableLambda(lambda x: {
        "original_text": x["original_text"],
        "word_count": x["word_count"]["word_count"],
        "sentence_count": x["sentence_count"]["sentence_count"],
        "sentiment": x["sentiment"]["sentiment"],
        "key_phrases": x["key_phrases"]["key_phrases"]
    })
)

# 执行分析
sample_text = "这款产品的用户体验非常出色！界面设计直观易用，功能强大且稳定。我们收到了大量用户的正面反馈。"
result = analysis_chain.invoke({"text": sample_text})

print("文本分析结果:")
for key, value in result.items():
    if isinstance(value, list):
        print(f"{key}: {', '.join(value[:3])}...")
    else:
        print(f"{key}: {value}")
```

### 使用.pipe()方法动态构建链
```python
from langchain_core.runnables import RunnableSequence

def create_processing_pipeline(steps: List[str]) -> RunnableSequence:
    """根据步骤列表动态构建处理管道"""
    
    # 步骤定义映射
    step_functions = {
        "clean": RunnableLambda(lambda x: {"cleaned_text": x["text"].strip()}),
        "normalize": RunnableLambda(lambda x: {
            "normalized_text": x.get("cleaned_text", x["text"]).lower()
        }),
        "tokenize": RunnableLambda(lambda x: {
            "tokens": x.get("normalized_text", x.get("cleaned_text", x["text"])).split()
        }),
        "count": RunnableLambda(lambda x: {
            "token_count": len(x["tokens"]) if "tokens" in x else 0
        })
    }
    
    # 构建链
    chain = RunnablePassthrough()
    
    for step in steps:
        if step in step_functions:
            chain = chain.pipe(step_functions[step])
        else:
            print(f"警告: 未知步骤 '{step}'，已跳过")
    
    return chain

# 动态构建管道
pipeline_steps = ["clean", "normalize", "tokenize", "count"]
dynamic_chain = create_processing_pipeline(pipeline_steps)

# 测试不同输入
test_inputs = [
    "  LangChain提供了强大的链式执行能力  ",
    "数据传递是顺序执行的核心机制",
    "  自动键名映射简化了开发工作  "
]

print("动态管道处理结果:")
for i, text in enumerate(test_inputs, 1):
    result = dynamic_chain.invoke({"text": text})
    print(f"\n输入 {i}: '{text}'")
    print(f"处理结果: {result}")
```

### 企业级顺序链设计模式
```python
from typing import Dict, Any, Optional
from dataclasses import dataclass
from datetime import datetime
import logging
from langchain_core.runnables import RunnableConfig

# 配置日志
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

@dataclass
class ProcessingContext:
    """处理上下文"""
    request_id: str
    start_time: datetime
    user_id: Optional[str] = None
    metadata: Dict[str, Any] = None
    
    def __post_init__(self):
        if self.metadata is None:
            self.metadata = {}

class EnterpriseSequentialChain:
    """企业级顺序链管理器"""
    
    def __init__(self, chain_name: str):
        self.chain_name = chain_name
        self.steps = []
        self.error_handlers = {}
        
    def add_step(self, name: str, runnable, retry_policy: Dict = None):
        """添加处理步骤"""
        step_info = {
            "name": name,
            "runnable": runnable,
            "retry_policy": retry_policy or {"max_retries": 3, "delay": 1}
        }
        self.steps.append(step_info)
        return self
    
    def add_error_handler(self, error_type, handler):
        """添加错误处理器"""
        self.error_handlers[error_type] = handler
    
    async def process(self, input_data: Dict, context: ProcessingContext) -> Dict:
        """执行处理流程"""
        current_data = input_data.copy()
        execution_log = []
        
        for i, step in enumerate(self.steps, 1):
            step_name = step["name"]
            step_start = datetime.now()
            
            try:
                logger.info(f"[{context.request_id}] 执行步骤 {i}/{len(self.steps)}: {step_name}")
                
                # 添加上下文到配置
                config = RunnableConfig(
                    callbacks=[],  # 可以添加自定义回调
                    tags=[self.chain_name, step_name],
                    metadata={
                        "request_id": context.request_id,
                        "step": step_name,
                        "user_id": context.user_id
                    }
                )
                
                # 执行步骤
                if hasattr(step["runnable"], "ainvoke"):
                    step_result = await step["runnable"].ainvoke(current_data, config=config)
                else:
                    step_result = step["runnable"].invoke(current_data, config=config)
                
                # 合并结果
                if isinstance(step_result, dict):
                    current_data.update(step_result)
                else:
                    current_data[f"step_{i}_result"] = step_result
                
                # 记录执行日志
                step_duration = (datetime.now() - step_start).total_seconds()
                execution_log.append({
                    "step": step_name,
                    "status": "success",
                    "duration": step_duration,
                    "timestamp": step_start.isoformat()
                })
                
                logger.info(f"[{context.request_id}] 步骤完成，耗时: {step_duration:.2f}s")
                
            except Exception as e:
                # 错误处理
                step_duration = (datetime.now() - step_start).total_seconds()
                error_msg = f"步骤 '{step_name}' 执行失败: {str(e)}"
                
                execution_log.append({
                    "step": step_name,
                    "status": "error",
                    "duration": step_duration,
                    "error": error_msg,
                    "timestamp": step_start.isoformat()
                })
                
                logger.error(f"[{context.request_id}] {error_msg}")
                
                # 尝试错误处理器
                error_type = type(e)
                if error_type in self.error_handlers:
                    try:
                        handler_result = self.error_handlers[error_type](e, current_data, context)
                        if handler_result is not None:
                            current_data.update(handler_result)
                        logger.info(f"[{context.request_id}] 错误处理器执行成功")
                    except Exception as handler_error:
                        logger.error(f"[{context.request_id}] 错误处理器失败: {handler_error}")
                
                # 根据重试策略决定是否继续
                if step["retry_policy"]["max_retries"] > 0:
                    logger.warning(f"[{context.request_id}] 准备重试步骤 '{step_name}'")
                    # 简化实现，实际应实现完整重试逻辑
                    step["retry_policy"]["max_retries"] -= 1
                    continue
                else:
                    # 返回部分结果和错误信息
                    current_data["_error"] = error_msg
                    current_data["_partial_results"] = True
                    current_data["_failed_step"] = step_name
                    break
        
        # 添加执行摘要
        current_data["_execution_summary"] = {
            "chain_name": self.chain_name,
            "request_id": context.request_id,
            "total_steps": len(self.steps),
            "completed_steps": len([log for log in execution_log if log["status"] == "success"]),
            "failed_steps": len([log for log in execution_log if log["status"] == "error"]),
            "total_duration": (datetime.now() - context.start_time).total_seconds(),
            "execution_log": execution_log
        }
        
        return current_data

# 使用示例

    # 步骤2: 摘要生成

        ("system", "你是一位专业的文档分析师。"),

    ])

    主要特性包括链式执行、智能代理、文档处理等。
    
    在v1.2.22版本中，框架引入了全新的Runnable接口，
    统一了同步和异步执行模型，提供了更好的类型安全和调试支持。
    
    顺序执行是LangChain的核心执行模式，支持复杂的数据传递和错误处理机制。
    """

    # 输出结果
    print("企业级链处理结果:")
    print(f"请求ID: {result['_execution_summary']['request_id']}")
    print(f"处理状态: {'部分成功' if result.get('_partial_results') else '完全成功'}")
    print(f"总耗时: {result['_execution_summary']['total_duration']:.2f}秒")
    
    if "summary" in result:
        print(f"\n文档摘要: {result['summary']}")
    elif "summary_fallback" in result:
        print(f"\n文档摘要（备用）: {result['summary_fallback']}")
    
    if "keywords" in result:
        print(f"关键词: {result['keywords']}")
    elif "keywords_fallback" in result:
        print(f"关键词（备用）: {', '.join(result['keywords_fallback'])}")
    
    # 显示执行日志
    print("\n执行日志:")
    for log in result["_execution_summary"]["execution_log"]:
        status_icon = "✅" if log["status"] == "success" else "❌"
```

### 数据传递的高级模式
```python
from langchain_core.runnables import RunnableParallel, RunnableBranch
from pydantic import BaseModel, ValidationError
import json

class AdvancedDataFlow:
    """高级数据流模式"""
    
    @staticmethod
    def conditional_routing():
        """条件路由数据流"""
        
        # 定义分支条件
        def route_by_length(input_dict):
            text = input_dict.get("text", "")
            if len(text) < 50:
                return "short"
            elif len(text) < 200:
                return "medium"
            else:
                return "long"
        
        # 定义不同长度的处理链
        short_processor = RunnableLambda(lambda x: {
            "processed": f"短文本处理: {x['text'][:20]}...",
            "strategy": "concise"
        })
        
        medium_processor = RunnableLambda(lambda x: {
            "processed": f"中等文本处理: 长度={len(x['text'])}",
            "strategy": "balanced"
        })
        
        long_processor = RunnableLambda(lambda x: {
            "processed": f"长文本深度处理: 分段分析",
            "strategy": "detailed"
        })
        
        # 创建条件路由链
        routing_chain = RunnableBranch(
            (lambda x: len(x.get("text", "")) < 50, short_processor),
            (lambda x: len(x.get("text", "")) < 200, medium_processor),
            long_processor
        )
        
        return routing_chain
    
    @staticmethod
    def parallel_processing():
        """并行处理数据流"""
        
        # 定义并行处理任务
        parallel_chain = RunnableParallel(
            {
                "word_count": RunnableLambda(lambda x: len(x["text"].split())),
                "char_count": RunnableLambda(lambda x: len(x["text"])),
                "line_count": RunnableLambda(lambda x: len(x["text"].split('\n'))),
                "uppercase_count": RunnableLambda(
                    lambda x: sum(1 for c in x["text"] if c.isupper())
                )
            }
        )
        
        return parallel_chain
    
    @staticmethod
    def validation_pipeline():
        """带验证的数据管道"""
        
        class ProcessedOutput(BaseModel):
            original: str
            processed: str
            metadata: Dict[str, Any]
            quality_score: float
        
        # 创建带验证的链
        validation_chain = (
            RunnablePassthrough()
            | RunnableLambda(lambda x: {
                "original": x["text"],
                "processed": x["text"].upper() if x.get("uppercase", False) else x["text"].lower(),
                "metadata": {"length": len(x["text"]), "words": len(x["text"].split())},
                "quality_score": min(1.0, len(x["text"]) / 1000)
            })
            | RunnableLambda(lambda x: ProcessedOutput(**x))
        )
        
        return validation_chain

# 使用示例
def demo_advanced_data_flow():
    print("=== 高级数据流模式演示 ===\n")
    
    # 1. 条件路由
    print("1. 条件路由示例:")
    routing_chain = AdvancedDataFlow.conditional_routing()
```

## 常见问题

### Q1: 顺序链中如何处理多个输出？
**A:** LangChain提供了多种处理多输出的策略：
1. **字典合并**：每个Runnable返回字典，自动合并到输出字典
2. **键名映射**：通过`input_variables`和`output_keys`控制数据流
3. **RunnableParallel**：并行执行多个任务，结果合并到字典
4. **选择器**：使用`.pick()`方法选择特定输出
5. **转换器**：使用RunnableLambda自定义输出格式

### Q2: 顺序执行与并行执行如何选择？
**A:** 选择依据：
1. **数据依赖**：步骤间有数据依赖 → 顺序执行
2. **性能要求**：无依赖且需要高性能 → 并行执行
3. **资源限制**：考虑CPU/GPU/API调用限制
4. **错误隔离**：需要错误隔离 → 适当并行化
5. **业务逻辑**：按业务需求选择执行模式

### Q3: 如何处理链中的错误和异常？
**A:** LangChain内置错误处理机制：
1. **自动重试**：通过`max_retries`参数配置
2. **回退策略**：主模型失败时使用备用模型
3. **超时控制**：通过`timeout`参数设置执行超时
4. **错误传播**：可配置错误是否传播到后续步骤
5. **自定义处理器**：通过`try/except`包装或错误处理器

### Q4: 如何调试复杂的顺序链？
**A:** 调试工具和技术：
1. **verbose模式**：设置`verbose=True`查看执行详情
2. **回调系统**：使用callbacks记录执行过程
3. **中间结果**：通过RunnablePassthrough保留中间结果
4. **单元测试**：为每个Runnable编写独立测试
5. **可视化工具**：使用`draw_ascii()`查看链结构

### Q5: 顺序链的性能优化策略？
**A:** 性能优化建议：
1. **批量处理**：使用`batch()`替代循环调用`invoke()`
2. **缓存中间结果**：对计算密集型步骤添加缓存
3. **异步执行**：使用`ainvoke()`提高并发性能
4. **合理分片**：将长链分解为可并行执行的子链
5. **资源池**：对模型调用使用连接池
