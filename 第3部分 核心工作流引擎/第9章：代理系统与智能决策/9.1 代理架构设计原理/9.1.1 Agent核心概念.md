# 9.1.1 Agent核心概念

## 概念讲解

### 什么是Agent（代理）

在LangChain中，Agent是一个智能决策系统，它能够根据用户输入自主决定采取哪些行动。与固定链不同，Agent不是预先定义好执行步骤，而是动态规划行动方案。

**核心区别**：
- **传统链**：开发者定义执行流程（A→B→C）
- **Agent**：模型自己决定使用哪些工具、以什么顺序执行

### Agent的工作原理

```
用户输入 → Agent
              ↓
         思考：需要什么工具？
              ↓
    ┌──── 调用工具A ────┐
    ↓                   ↓
   观察结果          思考下一步
    ↓                   ↓
    └──── 继续或结束 ←──┘
              ↓
         返回最终结果
```

这个过程被称为 **ReAct循环**（Reasoning + Acting）。

## 核心要点

1. **Agent = LLM + 工具 + 循环**：模型决定何时调用工具
2. **工具是Agent的手脚**：定义Agent能做什么
3. **AgentExecutor**：执行Agent决策的引擎
4. **create_agent**：LangGraph推荐的Agent创建方式

## 简单示例

### 使用LangGraph创建Agent

```python
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool

@tool
def search(query: str) -> str:
    """搜索互联网获取信息"""
    return "搜索结果..."

@tool
def calculator(expression: str) -> str:
    """执行数学计算"""
    return str(eval(expression))

# 创建Agent
model = ChatOpenAI(model="gpt-4o-mini")
agent = create_agent(
    model=model,
    tools=[search, calculator],
    system_prompt="你是一个有用的助手，可以使用工具帮助用户。"
)

# 执行
result = agent.invoke({
    "messages": [("human", "北京今天天气如何？如果温度是华氏度，帮我转成摄氏度。")]
})
```

### Agent的执行流程

```python
# Agent自动完成以下流程：
# 1. 理解用户请求
# 2. 决定需要调用哪些工具
# 3. 调用工具并获取结果
# 4. 根据结果决定下一步
# 5. 重复直到完成任务
```

## 进阶应用

### 自定义Agent提示

```python
agent = create_agent(
    model=model,
    tools=[search, calculator],
    system_prompt="""你是一个研究助手。
当用户需要事实性信息时，使用search工具。
当需要计算时，使用calculator工具。
始终先搜索，再计算。"""
)
```

### 检查点：可中断和恢复

```python
from langgraph.checkpoint.memory import MemorySaver

# 添加检查点支持
checkpointer = MemorySaver()

agent = create_agent(
    model=model,
    tools=[search, calculator],
    checkpointer=checkpointer,
    system_prompt="你是一个有用的助手。"
)

# 支持中断和恢复
config = {"configurable": {"thread_id": "session-1"}}
result = agent.invoke(
    {"messages": [("human", "帮我研究一个话题")]},
    config=config
)
```

## 常见问题

**Q: Agent和Chain有什么区别？**

A: Chain的执行流程是固定的，Agent根据输入动态决定执行路径。Agent更灵活但更复杂。

**Q: Agent一定会调用工具吗？**

A: 不一定。如果模型判断直接回答即可满足需求，可能不调用任何工具。

**Q: 如何限制Agent的行为？**

A: 通过工具定义（只给必要的工具）和系统提示（说明使用规则）来约束Agent的行为。

## 本节总结

Agent是LangChain中的智能决策系统。它通过LLM推理决定使用哪些工具、以什么顺序执行，实现了从"流程驱动"到"智能驱动"的转变。create_agent是推荐的Agent创建方式，支持工具绑定、自定义提示和检查点。
