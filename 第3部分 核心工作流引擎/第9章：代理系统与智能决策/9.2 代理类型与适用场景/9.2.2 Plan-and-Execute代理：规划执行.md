# 9.2.2 Plan-and-Execute代理：规划执行

## 概念讲解

Plan-and-Execute（规划执行）代理是一种更高级的智能代理范式，它明确地将问题解决过程分为两个阶段：**规划阶段**和**执行阶段**。与ReAct代理的即时决策不同，Plan-and-Execute代理先制定完整的行动计划，然后再按计划执行。

### 规划执行范式的核心思想

Plan-and-Execute模式的核心优势在于其结构化的思维方式：

1. **规划阶段（Planning）**：LLM分析任务，分解为子任务序列，制定详细行动计划
2. **执行阶段（Execution）**：按计划逐步执行子任务，调用相应工具
3. **监控与调整**：监控执行进度，必要时调整计划
4. **结果汇总**：整合各步骤结果，生成最终答案

这种"先谋后动"的策略特别适合复杂、多步骤、需要协调多个工具的任务。

### LangChain v1.2.22中的实现演进

在最新版本中，Plan-and-Execute代理主要通过以下机制实现：

**核心机制：**
- **TodoListMiddleware**：自动任务规划和跟踪中间件
- **CompiledSubAgent**：将工作流包装为可调用子代理
- **StateGraph分层设计**：多级状态图实现规划与执行的分离

**架构对比：**
```
ReAct代理：
┌─────────┐   ┌─────────┐   ┌─────────┐
│  思考   │ → │  行动   │ → │  观察   │ → (循环)
└─────────┘   └─────────┘   └─────────┘

Plan-and-Execute代理：
┌─────────────────┐
│   规划阶段      │ ← 一次性完整规划
│ - 任务分解      │
│ - 制定计划      │
└────────┬────────┘
         │
┌────────▼────────┐
│   执行阶段      │ ← 按计划执行
│ - 逐步执行子任务│
│ - 监控与调整    │
└─────────────────┘
```

### 示例2：自定义Plan-and-Execute工作流

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict, List, Optional
from langchain.tools import tool
from langchain.chat_models import init_chat_model

# 1. 定义状态结构
class PlanExecuteState(TypedDict):
    """规划执行状态"""
    original_task: str
    plan: List[dict]  # 计划步骤
    current_step_index: int
    step_results: dict
    final_answer: Optional[str]
    error_message: Optional[str]

# 2. 定义工具
@tool
def research_topic(topic: str) -> str:
    """研究特定主题"""
    research_db = {
        "机器学习": "机器学习是人工智能的核心技术，包括监督学习、无监督学习和强化学习。",
        "深度学习": "深度学习是基于神经网络的机器学习方法，在图像识别、自然语言处理等领域有广泛应用。",
        "强化学习": "强化学习通过试错学习最优策略，在游戏AI、机器人控制等领域表现出色。"
    }
    return research_db.get(topic, f"未找到关于'{topic}'的详细研究资料")

@tool
def compare_technologies(tech1: str, tech2: str) -> str:
    """比较两种技术"""
    comparisons = {
        ("机器学习", "深度学习"): "深度学习是机器学习的子集，专门使用深度神经网络。机器学习更广泛，包括多种算法。",
        ("监督学习", "无监督学习"): "监督学习使用标记数据，无监督学习发现数据中的模式而不需要标记。"
    }
    key = (tech1, tech2) if (tech1, tech2) in comparisons else (tech2, tech1)
    return comparisons.get(key, f"无法比较{tech1}和{tech2}")

@tool
def synthesize_findings(topics: List[str], findings: List[str]) -> str:
    """综合研究发现"""
    summary = "研究发现总结：\n\n"
    for topic, finding in zip(topics, findings):
        summary += f"• {topic}: {finding}\n"
    
    summary += "\n综合结论：这些技术相互补充，共同推动人工智能发展。"
    return summary

# 3. 定义节点函数
def planning_node(state: PlanExecuteState) -> dict:
    """规划节点：制定执行计划"""
    model = init_chat_model("gpt-4-turbo-preview")
    
    # 基于原始任务创建计划
    task = state["original_task"]
    
    # 简化的计划制定逻辑
    if "比较" in task and "机器学习" in task:
        plan = [
            {"type": "research", "topic": "机器学习", "description": "研究机器学习基础"},
            {"type": "research", "topic": "深度学习", "description": "研究深度学习原理"},
            {"type": "compare", "tech1": "机器学习", "tech2": "深度学习", "description": "比较两种技术"},
            {"type": "synthesize", "description": "综合研究发现"}
        ]
    else:
        # 通用计划
        plan = [
            {"type": "research", "topic": "人工智能", "description": "研究基础概念"},
            {"type": "synthesize", "description": "总结研究发现"}
        ]
    
    return {
        "plan": plan,
        "current_step_index": 0,
        "step_results": {}
    }

def execution_node(state: PlanExecuteState) -> dict:
    """执行节点：执行当前步骤"""
    if not state["plan"] or state["current_step_index"] >= len(state["plan"]):
        return {"error_message": "无效的计划或步骤索引"}
    
    current_step = state["plan"][state["current_step_index"]]
    step_type = current_step["type"]
    step_results = state.get("step_results", {}).copy()
    
    result = ""
    
    # 根据步骤类型执行
    if step_type == "research":
        result = research_topic.invoke(current_step["topic"])
    elif step_type == "compare":
        result = compare_technologies.invoke(
            current_step["tech1"], 
            current_step["tech2"]
        )
    elif step_type == "synthesize":
        # 收集之前的研究结果
        research_results = []
        for step in state["plan"]:
            if step["type"] == "research" and step["topic"] in step_results:
                research_results.append(step_results[step["topic"]])
        
        topics = [step["topic"] for step in state["plan"] if step["type"] == "research"]
        result = synthesize_findings.invoke(topics, research_results)
    
    # 记录结果
    step_key = f"step_{state['current_step_index']}"
    step_results[step_key] = {
        "description": current_step["description"],
        "result": result,
        "completed": True
    }
    
    return {
        "step_results": step_results,
        "current_step_index": state["current_step_index"] + 1
    }

def check_completion(state: PlanExecuteState) -> str:
    """检查是否完成所有步骤"""
    if not state["plan"]:
        return "error"
    
    if state["current_step_index"] >= len(state["plan"]):
        # 所有步骤完成，提取最终答案
        last_step_key = f"step_{len(state['plan'])-1}"
        if last_step_key in state.get("step_results", {}):
            final_answer = state["step_results"][last_step_key]["result"]
            return "complete"
    
    return "continue"

def completion_node(state: PlanExecuteState) -> dict:
    """完成节点：生成最终答案"""
    last_step_key = f"step_{len(state['plan'])-1}"
    final_answer = state["step_results"][last_step_key]["result"]
    
    return {
        "final_answer": final_answer,
        "error_message": None
    }

def error_node(state: PlanExecuteState) -> dict:
    """错误处理节点"""
    return {
        "final_answer": None,
        "error_message": "执行过程中出现错误"
    }

# 4. 构建状态图
workflow = StateGraph(PlanExecuteState)

# 添加节点
workflow.add_node("planning", planning_node)
workflow.add_node("execution", execution_node)
workflow.add_node("completion", completion_node)
workflow.add_node("error", error_node)

# 添加边
workflow.add_edge(START, "planning")
workflow.add_edge("planning", "execution")

# 条件边：检查执行状态
workflow.add_conditional_edges(
    "execution",
    check_completion,
    {
        "continue": "execution",  # 继续执行
        "complete": "completion", # 完成
        "error": "error"          # 错误
    }
)

workflow.add_edge("completion", END)
workflow.add_edge("error", END)

# 5. 编译工作流
app = workflow.compile()

# 6. 执行复杂任务
initial_state = {
    "original_task": "比较机器学习与深度学习的区别，并给出综合结论",
    "plan": [],
    "current_step_index": 0,
    "step_results": {},
    "final_answer": None,
    "error_message": None
}

result = app.invoke(initial_state)

print("原始任务:", result["original_task"])
print("\n执行计划:")
for i, step in enumerate(result.get("plan", [])):
    print(f"{i+1}. {step['description']}")

print("\n执行结果:")
for key, step_result in result.get("step_results", {}).items():
    print(f"{key}: {step_result['description']}")
    print(f"   结果: {step_result['result'][:100]}...")

if result.get("final_answer"):
    print("\n最终答案:")
    print(result["final_answer"])
```

## 进阶应用

### 企业级多代理规划系统

对于复杂的企业应用，需要更高级的规划执行系统：

```python
from typing import Dict, List, Any, Optional
from dataclasses import dataclass
from enum import Enum
import asyncio
from langgraph.graph import StateGraph
from langchain.tools import BaseTool
import logging

class TaskPriority(Enum):
    """任务优先级"""
    CRITICAL = 1
    HIGH = 2
    MEDIUM = 3
    LOW = 4

class TaskStatus(Enum):
    """任务状态"""
    PENDING = "pending"
    PLANNED = "planned"
    EXECUTING = "executing"
    COMPLETED = "completed"
    FAILED = "failed"

@dataclass
class SubTask:
    """子任务定义"""
    id: str
    description: str
    required_tools: List[str]
    dependencies: List[str]  # 依赖的其他子任务ID
    estimated_duration: int  # 估计耗时（秒）
    priority: TaskPriority
    assigned_agent: Optional[str] = None
    status: TaskStatus = TaskStatus.PENDING
    result: Optional[str] = None
    error: Optional[str] = None

class EnterprisePlanExecuteSystem:
    """企业级规划执行系统"""
    
    def __init__(
        self,
        available_agents: Dict[str, Any],
        tool_registry: Dict[str, BaseTool],
        max_parallel_tasks: int = 3
    ):
        self.available_agents = available_agents
        self.tool_registry = tool_registry
        self.max_parallel_tasks = max_parallel_tasks
        self.logger = logging.getLogger(__name__)
        
        # 任务队列和状态跟踪
        self.task_queue = []
        self.executing_tasks = {}
        self.completed_tasks = {}
        self.failed_tasks = {}
        
    async def plan_complex_task(self, task_description: str) -> List[SubTask]:
        """规划复杂任务"""
        
        # 使用LLM进行任务分解（简化实现）
        subtasks = []
        
        # 示例：分析性研究任务
        if "研究" in task_description or "分析" in task_description:
            subtasks = [
                SubTask(
                    id="t1",
                    description="收集相关文献和资料",
                    required_tools=["web_search", "database_query"],
                    dependencies=[],
                    estimated_duration=60,
                    priority=TaskPriority.HIGH
                ),
                SubTask(
                    id="t2",
                    description="分析收集到的数据",
                    required_tools=["data_analysis", "statistical_analysis"],
                    dependencies=["t1"],
                    estimated_duration=120,
                    priority=TaskPriority.HIGH
                ),
                SubTask(
                    id="t3",
                    description="生成初步结论",
                    required_tools=["report_generation"],
                    dependencies=["t2"],
                    estimated_duration=45,
                    priority=TaskPriority.MEDIUM
                ),
                SubTask(
                    id="t4",
                    description="验证结论和添加建议",
                    required_tools=["validation", "suggestion_generation"],
                    dependencies=["t3"],
                    estimated_duration=90,
                    priority=TaskPriority.MEDIUM
                )
            ]
        
        # 建立任务依赖图
        self._build_dependency_graph(subtasks)
        
        return subtasks
    
    def _build_dependency_graph(self, subtasks: List[SubTask]):
        """构建任务依赖图"""
        self.dependency_graph = {}
        for task in subtasks:
            self.dependency_graph[task.id] = {
                "task": task,
                "dependencies": task.dependencies,
                "dependents": []  # 反向依赖
            }
        
        # 填充反向依赖
        for task_id, task_info in self.dependency_graph.items():
            for dep_id in task_info["dependencies"]:
                if dep_id in self.dependency_graph:
                    self.dependency_graph[dep_id]["dependents"].append(task_id)
    
    async def execute_plan(self, subtasks: List[SubTask]) -> Dict[str, Any]:
        """执行任务计划"""
        
        # 按优先级排序
        subtasks.sort(key=lambda x: x.priority.value)
        
        results = {}
        
        # 使用信号量控制并行度
        semaphore = asyncio.Semaphore(self.max_parallel_tasks)
        
        async def execute_task(task: SubTask):
            async with semaphore:
                try:
                    self.logger.info(f"开始执行任务: {task.id} - {task.description}")
                    task.status = TaskStatus.EXECUTING
                    
                    # 检查依赖是否完成
                    for dep_id in task.dependencies:
                        if dep_id not in results or results[dep_id]["status"] != TaskStatus.COMPLETED:
                            raise Exception(f"依赖任务 {dep_id} 未完成")
                    
                    # 执行任务（简化实现）
                    # 实际应用中会调用相应的代理和工具
                    await asyncio.sleep(task.estimated_duration)  # 模拟执行时间
                    
                    # 模拟任务结果
                    task.result = f"任务 {task.id} 执行成功: {task.description}"
                    task.status = TaskStatus.COMPLETED
                    
                    self.logger.info(f"任务完成: {task.id}")
                    
                except Exception as e:
                    task.status = TaskStatus.FAILED
                    task.error = str(e)
                    self.logger.error(f"任务失败: {task.id} - {str(e)}")
        
        # 创建并执行所有任务
        tasks = [execute_task(task) for task in subtasks]
        await asyncio.gather(*tasks, return_exceptions=True)
        
        # 收集结果
        for task in subtasks:
            results[task.id] = {
                "status": task.status,
                "result": task.result,
                "error": task.error,
                "description": task.description
            }
        
        return results
    
    async def synthesize_results(self, task_results: Dict[str, Any]) -> str:
        """综合所有任务结果"""
        
        successful_tasks = [
            result for result in task_results.values()
            if result["status"] == TaskStatus.COMPLETED
        ]
        
        failed_tasks = [
            result for result in task_results.values()
            if result["status"] == TaskStatus.FAILED
        ]
        
        # 构建综合报告
        report = "任务执行综合报告\n"
        report += "=" * 50 + "\n\n"
        
        report += "成功完成的任务:\n"
        for i, task in enumerate(successful_tasks, 1):
            report += f"{i}. {task['description']}\n"
            if task["result"]:
                report += f"   结果: {task['result'][:100]}...\n"
        
        if failed_tasks:
            report += "\n失败的任务:\n"
            for i, task in enumerate(failed_tasks, 1):
                report += f"{i}. {task['description']}\n"
                if task["error"]:
                    report += f"   错误: {task['error']}\n"
        
        # 计算成功率
        total_tasks = len(task_results)
        success_count = len(successful_tasks)
        success_rate = (success_count / total_tasks * 100) if total_tasks > 0 else 0
        
        report += f"\n执行统计: {success_count}/{total_tasks} 任务成功 ({success_rate:.1f}%)\n"
        
        return report
```

### 智能动态规划调整

```python
class AdaptivePlanner:
    """自适应规划器：根据执行情况动态调整计划"""
    
    def __init__(self, learning_rate: float = 0.1):
        self.learning_rate = learning_rate
        self.task_history = []  # 历史任务记录
        self.tool_performance = {}  # 工具性能统计
        
    def adapt_plan_based_on_feedback(
        self, 
        original_plan: List[dict],
        execution_feedback: Dict[str, Any]
    ) -> List[dict]:
        """基于执行反馈调整计划"""
        
        adjusted_plan = original_plan.copy()
        
        # 分析反馈
        failed_steps = [
            step for step in execution_feedback.get("steps", [])
            if step.get("status") == "failed"
        ]
        
        # 如果有失败步骤，调整计划
        if failed_steps:
            for failed_step in failed_steps:
                step_id = failed_step.get("id")
                
                # 找到对应步骤并调整
                for i, step in enumerate(adjusted_plan):
                    if step.get("id") == step_id:
                        # 添加重试或替代方案
                        retry_step = {
                            **step,
                            "retry_count": step.get("retry_count", 0) + 1,
                            "alternative_approach": self._suggest_alternative(step)
                        }
                        
                        if step.get("retry_count", 0) < 3:  # 最多重试3次
                            # 在失败步骤后插入重试
                            adjusted_plan.insert(i + 1, retry_step)
                        else:
                            # 重试次数过多，标记为跳过
                            adjusted_plan[i]["status"] = "skipped"
                            adjusted_plan[i]["reason"] = "max_retries_exceeded"
                        
                        break
        
        # 基于性能优化调整顺序
        if self.tool_performance:
            # 将性能较差的工具对应的步骤后移
            adjusted_plan.sort(
                key=lambda step: self._calculate_step_priority(step)
            )
        
        return adjusted_plan
    
    def _suggest_alternative(self, step: dict) -> dict:
        """为失败步骤建议替代方案"""
        step_type = step.get("type")
        
        alternatives = {
            "web_search": {
                "type": "database_query",
                "description": f"使用数据库查询替代网络搜索: {step.get('query', '')}"
            },
            "complex_calculation": {
                "type": "simplified_estimation",
                "description": "使用简化估算替代复杂计算"
            },
            "api_call": {
                "type": "fallback_data",
                "description": "使用备用数据替代API调用"
            }
        }
        
        return alternatives.get(step_type, {"type": "manual_review", "description": "需要人工审核"})
    
    def _calculate_step_priority(self, step: dict) -> float:
        """计算步骤优先级（越低越优先）"""
        base_priority = 1.0
        
        # 考虑工具性能
        tool_name = step.get("tool")
        if tool_name in self.tool_performance:
            success_rate = self.tool_performance[tool_name]["success_rate"]
            # 成功率越高，优先级越低（更早执行）
            base_priority -= success_rate / 100
        
        # 考虑依赖关系
        if step.get("dependencies"):
            base_priority += 0.5  # 有依赖的步骤优先级提高
        
        return base_priority
    
    def update_performance_stats(self, tool_name: str, success: bool):
        """更新工具性能统计"""
        if tool_name not in self.tool_performance:
            self.tool_performance[tool_name] = {
                "total_calls": 0,
                "successful_calls": 0,
                "success_rate": 0.0
            }
        
        stats = self.tool_performance[tool_name]
        stats["total_calls"] += 1
        if success:
            stats["successful_calls"] += 1
        
        # 计算成功率
        stats["success_rate"] = (
            stats["successful_calls"] / stats["total_calls"] * 100
            if stats["total_calls"] > 0 else 0.0
        )
```

## 常见问题

### Q1: Plan-and-Execute代理与ReAct代理的主要区别是什么？

**A:** 核心区别在于决策时机和结构：

| 特性 | ReAct代理 | Plan-and-Execute代理 |
|------|-----------|----------------------|
| 决策时机 | 即时决策，每步动态选择 | 预先规划，按计划执行 |
| 适用场景 | 动态变化环境，简单到中等复杂度 | 复杂结构化任务，需要协调多个步骤 |
| 资源消耗 | 每个决策都调用LLM，Token消耗较高 | 规划阶段消耗较大，执行阶段可能较高效 |
| 可预测性 | 较低，依赖于每一步的即时决策 | 较高，有明确的执行计划 |
| 错误恢复 | 通过条件边处理局部错误 | 可以重新规划或调整计划 |

### Q2: 如何确定使用哪种代理模式？

**A:** 根据任务特性选择：

**使用ReAct代理的场景：**
- 任务步骤不明确，需要探索性解决
- 环境动态变化，需要实时适应
- 任务相对简单，决策开销不大
- 需要快速原型开发

**使用Plan-and-Execute代理的场景：**
- 任务可以明确分解为子任务
- 需要协调多个工具或服务
- 执行过程需要可预测和可审计
- 有严格的时间或资源约束
- 任务失败成本高，需要详细规划

### Q3: Plan-and-Execute代理如何处理计划变更？

**A:** 多层次变更处理策略：

```python
class PlanMonitor:
    """计划监控器，检测和处理计划变更"""
    
    def monitor_execution(self, original_plan: list, current_state: dict) -> dict:
        """监控执行状态，检测是否需要变更计划"""
        
        alerts = []
        
        # 1. 检查时间偏差
        elapsed_time = current_state.get("elapsed_time", 0)
        planned_time = sum(step.get("estimated_time", 0) for step in original_plan)
        
        if elapsed_time > planned_time * 1.5:  # 超时50%
            alerts.append({
                "type": "timeout",
                "message": f"执行超时: 已用{elapsed_time}s，计划{planned_time}s",
                "severity": "warning"
            })
        
        # 2. 检查资源使用
        resource_usage = current_state.get("resource_usage", {})
        for resource, usage in resource_usage.items():
            if usage > 0.8:  # 资源使用超过80%
                alerts.append({
                    "type": "resource",
                    "message": f"资源{resource}使用率过高: {usage:.1%}",
                    "severity": "critical"
                })
        
        # 3. 检查外部变化
        external_changes = self.detect_external_changes(current_state)
        if external_changes:
            alerts.append({
                "type": "environment",
                "message": "检测到外部环境变化",
                "severity": "info",
                "changes": external_changes
            })
        
        return {
            "alerts": alerts,
            "needs_replanning": any(a["severity"] in ["critical", "warning"] for a in alerts)
        }
```

1. **结构化思维**：先谋后动，减少盲目探索

3. **可预测性**：明确的执行计划便于监控和审计

3. **性能监控**：跟踪规划质量和执行效率
4. **渐进增强**：从简单规划开始，逐步增加复杂性
