# 9.2.1 ReAct代理：推理与行动

## 概念讲解

ReAct（Reasoning and Acting，推理与行动）是当前最主流的智能代理范式，它通过交替进行"推理"和"行动"来解决问题。在LangChain v1.2.22中，ReAct代理的实现经历了重要的架构演进，从传统的AgentExecutor模式升级为基于LangGraph的状态机模式。

### ReAct范式的核心思想

ReAct模式的核心创新在于将大型语言模型的推理能力与外部工具的行动能力有机结合：

1. **推理（Reasoning）**：LLM分析当前状态，决定下一步行动
2. **行动（Acting）**：调用工具执行具体操作，获取结果
3. **观察（Observing）**：观察工具执行结果，更新状态
4. **循环迭代**：重复上述过程直至问题解决

这种"思考-行动-观察"的循环模式模仿了人类解决问题的认知过程，使代理能够处理复杂、多步骤的任务。

### LangChain v1.2.22中的架构演进

在最新版本中，ReAct代理的实现发生了重要变化：

**传统架构（v0.x）：**
```python
from langchain.agents import create_react_agent
# 基于AgentExecutor的复杂配置
```

**现代架构（v1.2.22）：**
```python
from langchain.agents import create_agent
# 基于LangGraph的状态机，支持中间件
```

关键改进：
- **声明式状态管理**：使用StateGraph定义代理工作流
- **中间件系统**：可插拔的扩展机制
- **简化API**：`create_agent`统一接口
- **更好的错误处理**：结构化异常处理

### 2. 代理工作流的关键节点

现代ReAct代理包含以下关键节点：

1. **Agent节点**：调用LLM进行推理和决策
2. **Tools节点**：执行工具调用
3. **Conditional Edges**：条件边，决定下一步流向
4. **Memory集成**：可选的记忆系统

## 简单示例

### 示例1：基础ReAct代理

```python
from langchain.agents import create_agent
from langchain.tools import tool
from langchain.chat_models import init_chat_model

# 1. 定义工具
@tool
def calculator(expression: str) -> str:
    """计算数学表达式"""
    try:
        # 安全地评估表达式
        result = eval(expression)
        return f"计算结果: {result}"
    except Exception as e:
        return f"计算错误: {str(e)}"

@tool  
def web_search(query: str) -> str:
    """搜索网络信息（模拟实现）"""
    # 实际应用中会调用搜索引擎API
    knowledge_base = {
        "北京天气": "北京今天晴转多云，气温15-25°C",
        "Python装饰器": "装饰器是Python的高阶函数，用于修改其他函数的行为",
        "机器学习": "机器学习是人工智能的分支，通过算法从数据中学习模式"
    }
    
    for key, value in knowledge_base.items():
        if key.lower() in query.lower():
            return value
    
    return f"未找到'{query}'的相关信息"

# 2. 初始化模型
model = init_chat_model("gpt-4-turbo-preview")

# 3. 创建ReAct代理
tools = [calculator, web_search]
agent = create_agent(
    model=model,
    tools=tools,
    system_prompt="你是一个智能助手，可以计算数学表达式和搜索网络信息。",
    max_iterations=5  # 限制最大迭代次数
)

# 4. 运行代理
question = "请计算(25 + 37) * 2的结果，然后告诉我机器学习的基本概念"
response = agent.invoke({"messages": [{"role": "user", "content": question}]})

print("用户问题:", question)
print("\n代理响应:")
for message in response["messages"]:
    if message.type == "ai":
        print(f"AI: {message.content}")
    elif message.type == "tool":
        print(f"工具调用: {message.name} -> {message.content[:50]}...")
```

### 示例2：自定义状态和条件逻辑

```python
from typing import Literal
from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import ToolNode
from langchain.tools import tool

# 1. 自定义状态结构
from typing import TypedDict, Annotated
from langgraph.graph.message import add_messages

class ReActState(TypedDict):
    messages: Annotated[list, add_messages]
    reasoning_chain: list  # 推理链记录
    tools_called: list     # 已调用工具
    error_count: int       # 错误计数

# 2. 定义工具
@tool
def get_stock_price(symbol: str) -> str:
    """获取股票价格（模拟）"""
    prices = {
        "AAPL": "172.35美元",
        "GOOGL": "152.78美元", 
        "TSLA": "175.79美元"
    }
    return prices.get(symbol.upper(), f"未找到股票{symbol}的价格")

@tool
def calculate_investment(amount: float, years: int, rate: float) -> str:
    """计算投资回报"""
    future_value = amount * ((1 + rate/100) ** years)
    return f"投资{amount}美元，{years}年后价值{future_value:.2f}美元"

# 3. 初始化模型和工具
from langchain.chat_models import init_chat_model
model = init_chat_model("gpt-4-turbo-preview")
tools = [get_stock_price, calculate_investment]
tool_node = ToolNode(tools)
model_with_tools = model.bind_tools(tools)

# 4. 定义代理节点函数
def agent_node(state: ReActState):
    """代理推理节点"""
    # 记录推理步骤
    current_reasoning = {
        "step": len(state.get("reasoning_chain", [])) + 1,
        "input": state["messages"][-1].content if state["messages"] else "初始状态"
    }
    
    # 调用模型
    response = model_with_tools.invoke(state["messages"])
    
    # 更新推理链
    reasoning_chain = state.get("reasoning_chain", []) + [current_reasoning]
    
    return {
        "messages": [response],
        "reasoning_chain": reasoning_chain
    }

# 5. 定义条件边函数
def should_continue(state: ReActState) -> Literal["tools", END]:
    """决定下一步行动"""
    messages = state["messages"]
    if not messages:
        return END
    
    last_message = messages[-1]
    
    # 如果模型调用了工具，转到工具节点
    if hasattr(last_message, 'tool_calls') and last_message.tool_calls:
        # 更新已调用工具记录
        tools_called = state.get("tools_called", [])
        for tool_call in last_message.tool_calls:
            tools_called.append(tool_call["name"])
        
        return "tools"
    
    # 否则结束
    return END

# 6. 构建状态图
workflow = StateGraph(ReActState)
workflow.add_node("agent", agent_node)
workflow.add_node("tools", tool_node)

workflow.add_edge(START, "agent")
workflow.add_conditional_edges("agent", should_continue)
workflow.add_edge("tools", "agent")

# 7. 编译和执行
app = workflow.compile()

# 运行代理
initial_state = {
    "messages": [{"role": "user", "content": "苹果公司的股票价格是多少？如果投资1000美元，5年后的价值是多少？"}],
    "reasoning_chain": [],
    "tools_called": [],
    "error_count": 0
}

result = app.invoke(initial_state)

print("最终响应:", result["messages"][-1].content)
print("推理链长度:", len(result.get("reasoning_chain", [])))
print("调用的工具:", result.get("tools_called", []))
```

### 企业级ReAct代理框架

对于生产环境，建议使用以下增强框架：

```python
from typing import List, Dict, Any, Optional
from langchain.agents import create_agent
from langchain.tools import BaseTool
from langgraph.checkpoint.postgres import PostgresSaver
import logging

class EnterpriseReActAgent:
    """企业级ReAct代理框架"""
    
    def __init__(
        self,
        model,
        tools: List[BaseTool],
        system_prompt: str,
        max_iterations: int = 10,
        db_url: Optional[str] = None,
        enable_monitoring: bool = True
    ):
        self.model = model
        self.tools = tools
        self.system_prompt = system_prompt
        self.max_iterations = max_iterations
        self.logger = logging.getLogger(__name__)
        
        # 初始化检查点（状态持久化）
        if db_url:
            self.checkpointer = PostgresSaver.from_conn_string(db_url)
        else:
            from langgraph.checkpoint.memory import InMemorySaver
            self.checkpointer = InMemorySaver()
        
        # 创建代理
        self.agent = create_agent(
            model=model,
            tools=tools,
            system_prompt=system_prompt,
            max_iterations=max_iterations
        )
        
        # 监控指标
        self.metrics = {
            "total_invocations": 0,
            "successful_completions": 0,
            "average_iterations": 0,
            "tool_usage_stats": {}
        }
    
    def invoke_with_monitoring(
        self, 
        input_data: Dict[str, Any],
        user_id: str,
        session_id: str
    ) -> Dict[str, Any]:
        """带监控的代理调用"""
        
        config = {
            "configurable": {
                "thread_id": f"{user_id}_{session_id}",
                "user_id": user_id
            }
        }
        
        try:
            # 记录开始时间
            import time
            start_time = time.time()
            
            # 调用代理
            result = self.agent.invoke(input_data, config)
            
            # 计算执行时间
            execution_time = time.time() - start_time
            
            # 更新监控指标
            self.metrics["total_invocations"] += 1
            
            # 分析迭代次数
            messages = result.get("messages", [])
            tool_calls = sum(1 for msg in messages if hasattr(msg, 'tool_calls') and msg.tool_calls)
            self.metrics["average_iterations"] = (
                (self.metrics["average_iterations"] * (self.metrics["total_invocations"] - 1) + tool_calls)
                / self.metrics["total_invocations"]
            )
            
            # 记录工具使用统计
            for msg in messages:
                if hasattr(msg, 'tool_calls') and msg.tool_calls:
                    for tool_call in msg.tool_calls:
                        tool_name = tool_call["name"]
                        self.metrics["tool_usage_stats"][tool_name] = \
                            self.metrics["tool_usage_stats"].get(tool_name, 0) + 1
            
            # 记录成功
            self.metrics["successful_completions"] += 1
            
            # 日志记录
            self.logger.info(
                f"代理执行完成 - "
                f"用户: {user_id}, "
                f"会话: {session_id}, "
                f"耗时: {execution_time:.2f}s, "
                f"工具调用: {tool_calls}次"
            )
            
            # 添加监控数据到结果
            result["monitoring"] = {
                "execution_time": execution_time,
                "iterations": tool_calls,
                "success": True
            }
            
            return result
            
        except Exception as e:
            self.logger.error(f"代理执行失败: {str(e)}", exc_info=True)
            
            # 返回错误信息
            return {
                "messages": [{"role": "assistant", "content": f"执行失败: {str(e)}"}],
                "monitoring": {
                    "execution_time": time.time() - start_time,
                    "iterations": 0,
                    "success": False,
                    "error": str(e)
                }
            }
    
    def get_performance_report(self) -> Dict[str, Any]:
        """获取性能报告"""
        success_rate = 0
        if self.metrics["total_invocations"] > 0:
            success_rate = (
                self.metrics["successful_completions"] / 
                self.metrics["total_invocations"] * 100
            )
        
        return {
            "total_invocations": self.metrics["total_invocations"],
            "success_rate": f"{success_rate:.1f}%",
            "average_iterations": f"{self.metrics['average_iterations']:.2f}",
            "tool_usage": self.metrics["tool_usage_stats"],
            "checkpoint_enabled": hasattr(self, 'checkpointer') and self.checkpointer is not None
        }
```

from typing import Callable, Dict, Any
from langchain.agents.middleware import AgentMiddleware

class ValidationMiddleware(AgentMiddleware):
    """验证中间件：检查工具调用参数"""
    
    def __init__(self, validation_rules: Dict[str, Callable]):
        self.validation_rules = validation_rules
    
    def on_tool_call(self, tool_name: str, tool_args: Dict[str, Any]) -> Dict[str, Any]:
        """工具调用前的验证"""
        
        if tool_name in self.validation_rules:
            validation_func = self.validation_rules[tool_name]
            try:
                # 执行验证
                validation_func(tool_args)
                self.logger.info(f"工具 {tool_name} 参数验证通过")
            except Exception as e:
                self.logger.warning(f"工具 {tool_name} 参数验证失败: {str(e)}")
                # 可以修改参数或抛出异常
                raise ValueError(f"参数验证失败: {str(e)}")
        
        return tool_args
    
    def on_tool_result(self, tool_name: str, result: Any) -> Any:
        """工具结果返回前的处理"""
        # 可以在这里对结果进行清理、格式化等操作
        if isinstance(result, str) and len(result) > 1000:
            result = result[:1000] + "...[截断]"
        
        return result

## 常见问题

### Q1: ReAct代理与普通链式调用有什么区别？

**A:** 主要区别在于决策动态性：
- **链式调用**：固定的执行流程，预定义的步骤顺序
- **ReAct代理**：基于当前状态动态决定下一步行动，支持条件分支和循环

例如，处理"查询天气然后决定是否带伞"的任务：
- 链式调用：先查询天气，然后固定执行后续步骤
- ReAct代理：查询天气 → 分析结果 → 如果下雨则建议带伞，否则跳过

### Q3: ReAct代理如何处理工具调用失败？

**A:** 实现健壮的错误处理：

```python
class ResilientReActAgent:
    """具有容错能力的ReAct代理"""
    
    def tool_execution_with_retry(self, tool_func, args, max_retries=3):
        """带重试的工具执行"""
        for attempt in range(max_retries):
            try:
                return tool_func(**args)
            except Exception as e:
                if attempt == max_retries - 1:
                    return f"工具执行失败: {str(e)} (尝试{max_retries}次)"
                
                # 指数退避重试
                import time
                time.sleep(2 ** attempt)
        
        return "工具执行失败"
    
    def handle_tool_error(self, error: Exception, state: dict) -> dict:
        """工具错误处理"""
        # 记录错误
        error_count = state.get("error_count", 0) + 1
        
        # 构建错误消息
        error_message = f"工具执行出错: {str(error)}"
        
        # 如果错误过多，提前终止
        if error_count > 3:
            return {
                "messages": [{"role": "assistant", "content": "多次尝试失败，请简化您的问题。"}],
                "should_stop": True
            }
        
        return {
            "messages": [{"role": "assistant", "content": error_message}],
            "error_count": error_count,
            "should_stop": False
        }
```
