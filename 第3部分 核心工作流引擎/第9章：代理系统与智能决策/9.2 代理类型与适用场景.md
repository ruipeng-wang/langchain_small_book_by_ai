# 9.2 代理类型与适用场景

## 概念讲解（文字+图示）

LangChain提供了多种Agent类型，每种类型针对不同的应用场景进行了优化。理解这些类型的差异，有助于选择最合适的方案。

### Agent类型演进

```
早期版本：多种Agent类型（ReAct、Plan-and-Execute等）
    ↓
现代版本：统一使用 create_agent（基于LangGraph）
```

> **重要说明**：LangChain已统一使用`create_agent`作为标准Agent创建方式。旧的Agent类型（如ReAct Agent、Plan-and-Execute Agent）已被整合到统一架构中。

### 为什么统一架构更优

| 特性 | 旧多类型方案 | 统一create_agent |
|------|-------------|------------------|
| 学习成本 | 需掌握多种API | 一个函数搞定 |
| 灵活性 | 类型固定 | 通过提示词和工具控制 |
| 维护性 | 多套代码 | 统一底层实现 |
| 扩展性 | 难以自定义 | 基于LangGraph易扩展 |

## 核心要点

**🔑 通过提示词实现不同Agent行为：**

| 场景 | 实现方式 |
|------|----------|
| ReAct（推理+行动） | 默认行为，模型自动推理 |
| 规划执行 | 提示词要求先规划再执行 |
| 对话交互 | 提示词强调对话风格 |
| 代码执行 | 添加代码工具+规则提示 |

## 简单示例

### ReAct风格Agent（推理+行动）

```python
from langchain.agents import create_agent
from langchain_core.tools import tool

@tool
def search(query: str) -> str:
    """搜索网络获取信息"""
    return "搜索结果..."

@tool
def calculator(expr: str) -> str:
    """计算数学表达式"""
    return str(eval(expr))

# ReAct风格：模型自动推理并决定行动
agent = create_agent(
    model=model,
    tools=[search, calculator],
    system_prompt="你是一个研究助手。先搜索信息，再分析结论，最后给出答案。",
)
```

### 规划执行风格

```python
# 通过提示词实现规划执行
agent = create_agent(
    model=model,
    tools=[search, calculator],
    system_prompt="""你是一个规划型助手。
遵循以下步骤：
1. 理解问题并制定计划
2. 按计划逐步执行
3. 根据结果调整计划
4. 完成所有步骤后给出总结""",
)
```

## 进阶应用

### 代码执行Agent

```python
import subprocess

@tool
def run_python(code: str) -> str:
    """执行Python代码并返回结果"""
    try:
        result = subprocess.run(
            ["python", "-c", code],
            capture_output=True, text=True, timeout=30
        )
        return result.stdout or result.stderr
    except Exception as e:
        return f"执行失败: {e}"

code_agent = create_agent(
    model=model,
    tools=[run_python],
    system_prompt="""你是代码助手。
- 先分析问题
- 编写简洁的代码
- 执行并解释结果""",
)
```

### 多工具协作Agent

```python
from langchain_core.tools import tool

@tool
def read_file(path: str) -> str:
    """读取文件内容"""
    with open(path) as f:
        return f.read()

@tool
def analyze_text(text: str) -> str:
    """分析文本特征"""
    return f"字数: {len(text)}, 行数: {text.count(chr(10))}"

@tool
def save_report(content: str, path: str) -> str:
    """保存报告到文件"""
    with open(path, "w") as f:
        f.write(content)
    return f"报告已保存到 {path}"

# 多工具协作：读取→分析→保存
analysis_agent = create_agent(
    model=model,
    tools=[read_file, analyze_text, save_report],
    system_prompt="你是文件分析助手。读取文件、分析特征、保存报告。",
)
```

## 常见问题

### Q: 如何选择Agent类型？
A: 现在统一使用`create_agent`，通过提示词和工具组合控制行为。

### Q: 旧版Agent类型还能用吗？
A: 部分旧类型已废弃，建议迁移到`create_agent`。

### Q: 如何实现复杂的规划逻辑？
A: 推荐使用LangGraph（见第11章），提供更精细的流程控制。

## 本节总结

- LangChain已统一使用`create_agent`创建Agent
- 通过提示词实现不同风格：ReAct、规划执行、对话等
- 工具定义决定Agent能力范围
- 复杂流程控制推荐用LangGraph
