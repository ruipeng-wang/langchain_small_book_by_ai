# 9.3.1 内置工具类型分析

## 概念讲解

### 什么是工具（Tool）

在LangChain中，**工具（Tool）**是代理系统与外部世界交互的桥梁。简单来说，工具就是模型可以调用的外部功能。大语言模型本身只能生成文本，但通过工具，模型可以：

- **获取信息**：搜索互联网、查询数据库
- **执行操作**：发送邮件、更新文档
- **计算处理**：数学运算、代码执行
- **调用API**：天气查询、股票价格等

### 工具在代理系统中的角色

```
模型大脑（思考决策） → 工具手脚（执行操作）
```

| 能力 | 模型本身 | 模型 + 工具 |
|------|---------|------------|
| 知识范围 | 训练数据截止 | 实时互联网信息 |
| 计算能力 | 文本生成 | 数学计算、代码执行 |
| 行动能力 | 无 | 调用API、操作系统 |
| 数据访问 | 无 | 数据库、文件系统 |

## 核心要点

1. **@tool装饰器**：最简便的工具定义方式
2. **BaseTool基类**：灵活的工具定义
3. **参数Schema**：使用Pydantic定义参数类型
4. **内置工具**：Tavily Search、Python REPL等

## 简单示例

### @tool装饰器：最简便的定义

```python
from langchain_core.tools import tool

@tool
def get_weather(location: str) -> str:
    """获取指定城市的天气信息"""
    # 实际实现中调用天气API
    return f"{location}: 晴，25°C"

# 工具属性
print(get_weather.name)        # get_weather
print(get_weather.description) # 获取指定城市的天气信息
```

### 带复杂参数的工具

```python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class SearchInput(BaseModel):
    query: str = Field(description="搜索关键词")
    max_results: int = Field(default=5, description="最大结果数")

@tool("search", args_schema=SearchInput)
def search_documents(query: str, max_results: int = 5) -> str:
    """搜索相关文档"""
    # 实际实现中调用搜索API
    return f"找到{max_results}个关于'{query}'的结果"
```

### BaseTool基类：更灵活的定义

```python
from langchain_core.tools import BaseTool
from pydantic import BaseModel, Field

class CalculatorInput(BaseModel):
    expression: str = Field(description="数学表达式，如 '2 + 3 * 4'")

class CalculatorTool(BaseTool):
    name: str = "calculator"
    description: str = "执行数学计算"
    args_schema: type[BaseModel] = CalculatorInput
    
    def _run(self, expression: str) -> str:
        """执行计算"""
        try:
            result = eval(expression)  # 注意：生产环境需要安全限制
            return str(result)
        except Exception as e:
            return f"计算错误: {e}"

calculator = CalculatorTool()
```

## 内置工具

### Tavily搜索

```python
from langchain_community.tools.tavily_search import TavilySearchResults

search = TavilySearchResults(
    max_results=3,
    search_depth="advanced"  # 深度搜索
)

# 模型可以直接调用
# model.bind_tools([search])
```

### Python REPL

```python
from langchain_experimental.tools import PythonREPLTool

python_tool = PythonREPLTool()
result = python_tool.invoke("print(2 + 3 * 4)")
# 输出: 14
```

## 进阶应用

### 工具绑定到模型

```python
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool

@tool
def get_current_time() -> str:
    """获取当前时间"""
    from datetime import datetime
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

@tool
def calculate(a: int, b: int) -> int:
    """计算两个数的和"""
    return a + b

# 绑定工具到模型
model = ChatOpenAI(model="gpt-4o-mini")
model_with_tools = model.bind_tools([get_current_time, calculate])

# 模型会自动决定何时调用工具
response = model_with_tools.invoke("现在几点了？")
```

### 工具集合管理

```python
from langchain_core.tools import tool

@tool
def add(a: int, b: int) -> int:
    """加法"""
    return a + b

@tool
def multiply(a: int, b: int) -> int:
    """乘法"""
    return a * b

# 工具集合
tools = [add, multiply]
tool_map = {tool.name: tool for tool in tools}

def execute_tool(name: str, **kwargs):
    """执行指定工具"""
    tool = tool_map.get(name)
    if tool:
        return tool.invoke(kwargs)
    return f"工具 '{name}' 不存在"
```

## 常见问题

**Q: 工具描述怎么写才好？**

A: 描述应该清楚说明工具的**功能**和**使用场景**，而不是技术实现。模型根据描述决定是否调用工具。

**Q: 工具可以返回复杂对象吗？**

A: 推荐返回字符串，因为模型处理的是文本。复杂对象可以先序列化为JSON字符串。

**Q: 工具执行失败怎么处理？**

A: 在工具内部捕获异常并返回错误信息，让模型了解问题并尝试修复。

## 本节总结

工具系统是代理系统与外部世界交互的关键。通过`@tool`装饰器和`BaseTool`基类，开发者可以轻松定义模型可调用的工具。LangChain提供了丰富的内置工具，覆盖搜索、计算、代码执行等常见需求。
