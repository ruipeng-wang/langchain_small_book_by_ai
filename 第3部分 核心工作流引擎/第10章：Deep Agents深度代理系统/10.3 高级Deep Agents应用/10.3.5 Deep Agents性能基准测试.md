# 10.3.5 Deep Agents性能基准测试

## 概念讲解

性能基准测试是评估 Deep Agents 在不同场景下的表现，帮助开发者选择合适的配置和优化策略。与 LangGraph 相比，Deep Agents 由于内置了智能优化，在某些场景下表现更好，但在需要精细控制的场景下可能不如 LangGraph 灵活。

### 性能测试的关键维度

| 维度 | 说明 | 测试方法 |
|------|------|---------|
| 启动延迟 | 从创建到可执行的时间 | 记录 `create_deep_agent()` 到首次响应的时间 |
| 单轮响应 | 简单查询的响应时间 | 测量单次 `invoke()` 的延迟 |
| 多轮对话 | 长对话的响应时间 | 测量多次 `invoke()` 的平均延迟 |
| 上下文压缩 | 长对话的压缩效果 | 比较压缩前后的 Token 使用量 |
| 工具调用 | 带工具调用的延迟 | 测量工具调用链路的总时间 |
| 子代理 | 子代理创建的开销 | 比较有/无子代理的延迟差异 |

### 性能优化的关键因素

1. **模型选择**：不同模型的延迟和成本差异显著
2. **上下文管理**：智能压缩可减少 Token 使用和延迟
3. **工具缓存**：缓存工具结果可避免重复调用
4. **中间件优化**：精简中间件可减少处理开销

## 核心要点

- **基准测试需建立基线**：先测量基础性能，再测试优化效果
- **对比不同配置**：模型、工具数量、上下文长度等
- **关注实际场景**：测试应反映真实的使用模式
- **Deep Agents 智能优化**：自动压缩和缓存在长对话中优势明显

## 简单示例

### 基础性能测试

```python
import time
import statistics
from deepagents import create_deep_agent

def benchmark_agent(agent, inputs: list, runs: int = 5) -> dict:
    """性能测试函数"""
    latencies = []
    
    for input_text in inputs:
        run_times = []
        for _ in range(runs):
            start = time.perf_counter()
            agent.invoke(input_text)
            end = time.perf_counter()
            run_times.append(end - start)
        latencies.append(statistics.mean(run_times))
    
    return {
        "avg": statistics.mean(latencies),
        "min": min(latencies),
        "max": max(latencies),
        "p95": sorted(latencies)[int(len(latencies) * 0.95)]
    }

# 测试不同模型的性能
models = {
    "claude-sonnet": "anthropic:claude-sonnet-4-6",
    "gpt-4o": "openai:gpt-4o",
    "gemini": "google:gemini-2.5-flash"
}

test_inputs = [
    "你好",
    "请简要介绍一下人工智能",
    "详细解释深度学习的工作原理"
]

for name, model_id in models.items():
    agent = create_deep_agent(model=model_id, tools=[])
    results = benchmark_agent(agent, test_inputs)
    print(f"{name}: 平均延迟 {results['avg']:.2f}s")
```

### 上下文压缩效果测试

```python
from deepagents import create_deep_agent

# 创建带上下文管理的代理
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[],
    config={"context_compression": True}
)

# 模拟长对话
def test_context_compression(agent, num_turns: int = 20):
    """测试上下文压缩效果"""
    latencies = []
    token_usage = []
    
    for i in range(num_turns):
        start = time.perf_counter()
        response = agent.invoke(f"这是第 {i+1} 轮对话，请总结之前的内容")
        end = time.perf_counter()
        
        latencies.append(end - start)
        token_usage.append(response.get("token_usage", {}))
    
    # 分析延迟趋势
    first_half_avg = statistics.mean(latencies[:num_turns//2])
    second_half_avg = statistics.mean(latencies[num_turns//2:])
    
    print(f"前半段平均延迟: {first_half_avg:.2f}s")
    print(f"后半段平均延迟: {second_half_avg:.2f}s")
    print(f"延迟增长: {(second_half_avg/first_half_avg-1)*100:.1f}%")
    
    return latencies, token_usage

latencies, tokens = test_context_compression(agent)
```

### Deep Agents vs LangGraph 性能对比

```python
import time
from deepagents import create_deep_agent
from langgraph.graph import StateGraph, START, END

# Deep Agents 测试
def test_deep_agents():
    agent = create_deep_agent(
        model="anthropic:claude-sonnet-4-6",
        tools=["web_search"],
        system_prompt="你是一个研究助手"
    )
    
    start = time.perf_counter()
    result = agent.invoke("请研究量子计算的最新进展")
    end = time.perf_counter()
    
    return end - start

# LangGraph 测试
def test_langgraph():
    from langchain.messages import HumanMessage
    
    class State(TypedDict):
        messages: list
        research_result: str
    
    def research_node(state: State):
        response = model.invoke(state["messages"])
        return {"research_result": response.content}
    
    builder = StateGraph(State)
    builder.add_node("research", research_node)
    builder.add_edge(START, "research")
    builder.add_edge("research", END)
    
    graph = builder.compile()
    
    start = time.perf_counter()
    result = graph.invoke({
        "messages": [HumanMessage(content="研究量子计算")],
        "research_result": ""
    })
    end = time.perf_counter()
    
    return end - start

# 对比测试
print("Deep Agents 延迟:", test_deep_agents())
print("LangGraph 延迟:", test_langgraph())
```

### 工具调用性能测试

```python
def tool_performance_test():
    """测试不同工具数量对性能的影响"""
    tools_config = {
        "no_tools": [],
        "few_tools": ["search"],
        "many_tools": ["search", "calculator", "file_reader", "web_scraper"]
    }
    
    results = {}
    
    for config_name, tools in tools_config.items():
        agent = create_deep_agent(
            model="anthropic:claude-sonnet-4-6",
            tools=tools
        )
        
        # 测量 10 次调用的平均延迟
        latencies = []
        for _ in range(10):
            start = time.perf_counter()
            agent.invoke("请帮我搜索相关信息")
            end = time.perf_counter()
            latencies.append(end - start)
        
        results[config_name] = statistics.mean(latencies)
    
    return results

perf_results = tool_performance_test()
for config, latency in perf_results.items():
    print(f"{config}: {latency:.2f}s")
```

## 常见问题

### Q1: 为什么 Deep Agents 的启动延迟比 LangGraph 高？

**A**: Deep Agents 需要初始化更多组件（上下文管理器、虚拟文件系统等），导致启动延迟稍高。但首次响应后，性能差异通常不明显。

### Q2: 长对话中 Deep Agents 的性能优势是什么？

**A**: Deep Agents 的智能上下文压缩可以显著减少 Token 使用和延迟，在长对话中优势明显。

### Q3: 如何优化 Deep Agents 的性能？

**A**: 1）选择更快的模型；2）减少不必要的工具；3）启用上下文压缩；4）使用中间件缓存。

### Q4: 子代理会影响性能吗？

**A**: 子代理创建有一定开销，但对于复杂任务，并行执行子代理可以减少总体延迟。

## 本节总结

Deep Agents 性能基准测试帮助理解不同配置下的表现：

1. 建立性能基线，识别瓶颈
2. 对比不同模型和工具配置
3. 分析上下文压缩的效果
4. 与 LangGraph 进行性能对比

根据测试结果选择合适的配置，优化生产环境的性能表现。
