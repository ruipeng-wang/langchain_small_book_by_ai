# 10.1.4 LangChain v1新特性与选择指南

## 概念讲解

LangChain v1.2.22 引入了两个并行但定位不同的框架来构建 AI 代理：**Deep Agents** 和 **LangGraph**。理解两者的定位差异和适用场景，是技术选型的关键。

### 框架定位对比

**Deep Agents** 是"电池内置"（batteries-included）的高级代理框架。它提供了开箱即用的完整解决方案：智能上下文管理、虚拟文件系统、子代理系统、会话压缩等。开发者只需最少的配置就能构建生产级应用。

**LangGraph** 是低级别的状态编排框架。它提供了对执行流程的精细控制，通过 StateGraph 定义节点、边和状态转换。适合需要完全定制工作流的复杂场景。

### 核心差异对比

| 维度 | Deep Agents | LangGraph |
|------|-------------|-----------|
| 抽象层级 | 高级（声明式） | 低级（命令式） |
| 配置复杂度 | 一行创建 | 需要定义图结构 |
| 上下文管理 | 自动压缩和摘要 | 手动管理 |
| 虚拟文件系统 | 内置支持 | 需要自己实现 |
| 子代理 | 动态自动创建 | 手动定义子图 |
| 控制粒度 | 粗粒度 | 细粒度 |
| 学习曲线 | 平缓 | 较陡峭 |

### LangChain v1.2.22 的新特性

LangChain v1.2.22 带来了以下重要更新：

1. **统一模型接口**：通过 `init_chat_model()` 简化多模型切换
2. **中间件系统**：支持请求/响应拦截和处理
3. **增强的工具调用**：改进的工具绑定和并行调用
4. **改进的检查点**：更灵活的持久化配置

## 核心要点

- **Deep Agents 适合快速开发**：一行创建，智能默认
- **LangGraph 适合精细控制**：完全自定义工作流
- **两者可以结合使用**：Deep Agents 作为 LangGraph 的节点
- **选择取决于需求**：简单任务用 Deep Agents，复杂编排用 LangGraph

## 简单示例

### Deep Agents：一行创建代理

```python
from deepagents import create_deep_agent

# 一行配置即可创建
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=["web_search", "file_reader"],
    system_prompt="你是一个研究助手"
)

# 直接使用
result = agent.invoke("请帮我研究量子计算的最新进展")
```

### LangGraph：精细控制工作流

```python

    messages: list

def llm_node(state: AgentState):

    return {"messages": [response], "step_count": state["step_count"] + 1}

def tool_node(state: AgentState):
    # 处理工具调用
    ...

def should_continue(state: AgentState) -> str:
    if state["step_count"] < 5:
        return "tool_node"
    return END

# 构建图
builder = StateGraph(AgentState)
builder.add_node("llm", llm_node)
builder.add_node("tools", tool_node)
builder.add_edge(START, "llm")
builder.add_conditional_edges("llm", should_continue, ["tools", END])
builder.add_edge("tools", "llm")

graph = builder.compile()
```

## 进阶应用

### 结合使用：Deep Agents 作为 LangGraph 节点

```python
from langgraph.graph import StateGraph, START, END

# 创建 Deep Agents
research_agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=["web_search"],
    name="researcher"
)

write_agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=["file_writer"],
    name="writer"
)

# 在 LangGraph 中编排
class WorkflowState(TypedDict):
    topic: str
    research_result: str
    final_report: str

def research_node(state: WorkflowState):
    result = research_agent.invoke(f"研究主题：{state['topic']}")
    return {"research_result": result}

def write_node(state: WorkflowState):
    report = write_agent.invoke(f"基于以下研究撰写报告：{state['research_result']}")
    return {"final_report": report}

builder = StateGraph(WorkflowState)
builder.add_node("research", research_node)
builder.add_node("write", write_node)
builder.add_edge(START, "research")
builder.add_edge("research", "write")
builder.add_edge("write", END)

graph = builder.compile()
```

## 常见问题

### Q1: 我应该选择 Deep Agents 还是 LangGraph？

**A**: 如果你需要快速构建一个功能完整的代理，选择 Deep Agents。如果你需要对执行流程进行精细控制，选择 LangGraph。

### Q2: 两者可以同时使用吗？

**A**: 可以。Deep Agents 可以作为 LangGraph 的节点，两者结合使用能发挥各自优势。

### Q3: Deep Agents 的性能比 LangGraph 差吗？

**A**: 不一定。Deep Agents 的自动优化在很多场景下性能表现良好。对于极端性能要求，LangGraph 提供了更多优化空间。

### Q4: 从旧版本迁移应该选哪个？

**A**: 如果是从旧的 LangChain agents 迁移，Deep Agents 提供了更好的兼容性和更低的迁移成本。

## 本节总结

LangChain v1.2.22 提供了两个互补的代理构建方案：

1. **Deep Agents**：快速开发，智能默认，适合大多数场景
2. **LangGraph**：精细控制，高度定制，适合复杂工作流
3. **结合使用**：发挥各自优势，构建复杂系统

选择哪个框架取决于具体的项目需求和团队经验。
