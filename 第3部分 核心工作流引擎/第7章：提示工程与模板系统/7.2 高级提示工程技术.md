# 7.2 高级提示工程技术

## 概念讲解

### 思维链（Chain-of-Thought）

思维链提示技术引导模型逐步推理，而不是直接给出答案。研究表明，让模型"一步一步思考"可以显著提升复杂问题的准确率。

### 少样本学习（Few-Shot Learning）

通过在提示中提供少量示例，模型可以学习特定任务的格式、风格或逻辑。这比零样本（Zero-Shot）效果更好，尤其对于复杂任务。

### 结构化输出

企业应用需要将模型的自由文本输出转换为结构化数据（JSON、Pydantic对象等），以便下游系统处理。LangChain提供了多种结构化输出方案。

## 核心要点

1. **思维链**：引导模型逐步推理，提升复杂问题准确率
2. **少样本学习**：通过示例教会模型特定格式和逻辑
3. **with_structured_output**：ChatModel内置的结构化输出方法
4. **PydanticOutputParser**：兼容所有模型的解析器方案
5. **示例选择器**：根据语义相似度动态选择最佳示例

## 简单示例

### 思维链提示

```python
from langchain_core.prompts import ChatPromptTemplate

# 思维链模板
cot_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个逻辑清晰的思考者。请一步步分析你的推理过程。"),
    ("human", """问题：{question}

请按以下格式回答：
1. 问题分析
2. 推理过程
3. 最终结论""")
])

# 使用
chain = cot_prompt | llm
result = chain.invoke({"question": "如果一个商店打八折，原价100元的商品实际支付多少？"})
```

### 少样本学习

```python
from langchain_core.prompts import FewShotChatMessagePromptTemplate, ChatPromptTemplate

# 定义示例
examples = [
    {"input": "开心", "output": "😊 积极"},
    {"input": "糟糕", "output": "😞 消极"},
    {"input": "还行", "output": "😐 中性"},
]

# 创建示例提示
example_prompt = ChatPromptTemplate.from_messages([
    ("human", "这个词 '{input}' 的情感是什么？"),
    ("ai", "{output}")
])

# 创建少样本提示
few_shot_prompt = FewShotChatMessagePromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
)

# 组合完整提示
final_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个情感分析助手。请参考以下示例的风格。"),
    few_shot_prompt,
    ("human", "请分析 '{text}' 的情感。"),
])

chain = final_prompt | llm
result = chain.invoke({"text": "太棒了"})
```

### 结构化输出 - with_structured_output

```python
from pydantic import BaseModel, Field
from typing import List, Optional
from langchain_openai import ChatOpenAI

# 定义输出结构
class PersonInfo(BaseModel):
    name: str = Field(description="姓名")
    age: int = Field(description="年龄")
    hobbies: List[str] = Field(description="爱好列表")
    email: Optional[str] = Field(default=None, description="邮箱")

# 创建支持结构化输出的模型
llm = ChatOpenAI(model="gpt-4o-mini")
structured_llm = llm.with_structured_output(PersonInfo)

# 使用
result = structured_llm.invoke("我叫张三，今年28岁，喜欢编程和跑步。邮箱是zhangsan@example.com")
print(result.name)   # 张三
print(result.age)    # 28
print(result.hobbies)  # ['编程', '跑步']
```

## 进阶应用

### 动态示例选择器

```python
from langchain_core.prompts import FewShotPromptTemplate, PromptTemplate
from langchain_core.example_selectors import SemanticSimilarityExampleSelector
from langchain_openai import OpenAIEmbeddings

# 示例库
examples = [
    {"input": "Python是什么", "output": "Python是一种高级编程语言..."},
    {"input": "Java和Python区别", "output": "主要区别在于类型系统和语法..."},
    {"input": "如何学习编程", "output": "建议从基础语法开始，多做练习..."},
]

# 创建语义示例选择器
selector = SemanticSimilarityExampleSelector.from_examples(
    examples=examples,
    embeddings=OpenAIEmbeddings(),  # 使用Embeddings计算相似度
    k=2  # 选择最相似的2个示例
)

# 动态少样本提示
dynamic_prompt = FewShotPromptTemplate(
    example_selector=selector,
    example_prompt=PromptTemplate.from_template("Q: {input}\nA: {output}"),
    prefix="请参考以下示例：",
    suffix="问题：{question}\n回答：",
    input_variables=["question"],
)
```

### TypedDict轻量级结构化输出

```python
from typing import TypedDict
from langchain_openai import ChatOpenAI

# 轻量级类型定义
class TaskResult(TypedDict):
    task: str      # 任务描述
    status: str    # 状态：done/pending
    priority: int  # 优先级：1-5

llm = ChatOpenAI(model="gpt-4o-mini")
structured_llm = llm.with_structured_output(TaskResult)

result = structured_llm.invoke("完成数据库迁移，优先级高")
# 返回：{"task": "数据库迁移", "status": "done", "priority": 5}
```

## 常见问题

**Q: with_structured_output和PydanticOutputParser有什么区别？**

A: `with_structured_output`是ChatModel的内置方法，直接返回Pydantic对象，使用简单。`PydanticOutputParser`是独立的输出解析器，兼容更多模型（包括不支持结构化输出的模型）。优先使用`with_structured_output`。

**Q: 结构化输出失败怎么办？**

A: 1) 确保模型版本支持（如gpt-3.5-turbo-1106+）；2) 在Pydantic字段中添加详细description；3) 设置temperature=0提高稳定性；4) 添加重试逻辑。

**Q: 思维链会增加Token消耗吗？**

A: 会。思维链要求模型输出推理过程，增加了输出token。但对于复杂问题，它提高了准确率，减少了重试次数，总体可能更节省成本。

## 本节总结

高级提示工程技术包括思维链、少样本学习和结构化输出。思维链提升推理准确率，少样本学习教会模型特定格式，结构化输出确保下游系统可处理。`with_structured_output`是当前最推荐的结构化输出方案。
