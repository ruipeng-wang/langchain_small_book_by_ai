# 7.3 输出解析器设计

## 概念讲解

### 什么是输出解析器

输出解析器（Output Parser）负责将模型的原始输出转换为结构化数据。模型返回的是纯文本或消息对象，而应用需要的是Pydantic对象、字典或自定义类型。

### 解析器的核心价值

1. **类型安全**：解析后得到类型明确的对象，IDE可提供自动补全
2. **数据验证**：通过Pydantic验证器确保数据质量
3. **错误处理**：标准化的异常处理机制
4. **格式指令**：自动生成提示词中的格式说明

LangChain的解析器都继承自`BaseOutputParser`基类，核心方法是`parse(text: str) -> T`。

## 核心要点

1. **StrOutputParser**：提取文本内容，最基础的解析器
2. **JsonOutputParser**：解析JSON输出，返回字典
3. **PydanticOutputParser**：解析为Pydantic对象，带验证
4. **CommaSeparatedListOutputParser**：解析逗号分隔列表
5. **自定义解析器**：继承`BaseOutputParser`实现特殊逻辑

## 简单示例

### JsonOutputParser

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import PromptTemplate

# 创建解析器
parser = JsonOutputParser()

# 在提示中包含格式指令
prompt = PromptTemplate.from_template(
    "分析以下文本的情感，返回JSON格式：\n{text}\n\n{format_instructions}"
)

# 组合链
chain = prompt | llm | parser

result = chain.invoke({
    "text": "这个产品太棒了，物超所值！",
    "format_instructions": parser.get_format_instructions()
})
# 返回：{"sentiment": "positive", "score": 0.9}
```

### PydanticOutputParser

```python
from langchain_core.output_parsers import PydanticOutputParser
from langchain_core.prompts import PromptTemplate
from pydantic import BaseModel, Field
from typing import List

# 定义输出结构
class Recipe(BaseModel):
    name: str = Field(description="菜名")
    ingredients: List[str] = Field(description="食材列表")
    steps: List[str] = Field(description="制作步骤")
    time_minutes: int = Field(description="制作时间（分钟）")

# 创建解析器
parser = PydanticOutputParser(pydantic_object=Recipe)

# 提示模板
prompt = PromptTemplate.from_template(
    "请提供{dish_name}的食谱。\n\n{format_instructions}"
)

# 组合链
chain = prompt | llm | parser

result = chain.invoke({
    "dish_name": "番茄炒蛋",
    "format_instructions": parser.get_format_instructions()
})
print(result.name)        # 番茄炒蛋
print(result.ingredients) # ['番茄', '鸡蛋', ...]
```

## 进阶应用

### 自定义解析器

```python
from langchain_core.output_parsers import BaseOutputParser
import re

class PhoneNumberParser(BaseOutputParser[dict]):
    """电话号码解析器"""
    
    def parse(self, text: str) -> dict:
        # 提取电话号码
        pattern = r'(\d{3})[-\s]?(\d{4})[-\s]?(\d{4})'
        match = re.search(pattern, text)
        
        if not match:
            raise ValueError(f"未找到电话号码: {text}")
        
        return {
            "full": match.group(0),
            "area": match.group(1),
            "middle": match.group(2),
            "last": match.group(3)
        }
    
    def get_format_instructions(self) -> str:
        return "请提供电话号码，格式：XXX-XXXX-XXXX"

# 使用
parser = PhoneNumberParser()
chain = prompt_template | llm | parser
result = chain.invoke({"query": "联系方式"})
```

## 常见问题

**Q: 模型输出不是合法JSON怎么办？**

A: 1) 在提示中强调"只返回JSON，不要添加其他内容"；2) 使用`JsonOutputParser`的容错模式；3) 在自定义解析器中用正则提取JSON块。

**Q: PydanticOutputParser与with_structured_output选哪个？**

A: 如果模型支持（如OpenAI），优先用`with_structured_output`，更简洁可靠。对于不支持的模型，使用`PydanticOutputParser`作为兼容方案。

**Q: 解析器可以在链的中间使用吗？**

A: 可以。LangChain的`|`运算符允许在链的任意位置插入解析器。例如：`prompt | llm | parser | next_step`。

## 本节总结

输出解析器是连接LLM自由文本输出与下游结构化数据的桥梁。从简单的`JsonOutputParser`到强大的`PydanticOutputParser`，再到自定义解析器，LangChain提供了完整的解析工具链。结合结构化输出方法，可以确保应用获得可靠、可验证的数据。
