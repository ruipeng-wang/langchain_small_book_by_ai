# 7.1.3 模板复用与继承机制

## 模板复用的核心价值：代码重用与一致性保证

在企业级AI应用开发中，模板复用（Template Reusability）和继承机制（Inheritance Mechanism）是提升开发效率、确保代码质量和维护一致性的关键技术。其核心价值体现在：

1. **代码重用性**：避免重复编写相似的提示模板，减少代码冗余
2. **一致性保证**：确保相同功能在不同场景下使用统一的模板结构
3. **维护便捷性**：中央化模板管理，一处修改全局生效
4. **扩展灵活性**：通过继承机制轻松创建变体模板，适应新需求

## 传统模板复用的复杂性：硬编码与散弹式修改

在LangChain出现之前，开发人员需要手动管理模板复用，导致大量重复代码和难以维护的模板系统。让我们通过一个企业知识库问答系统来展示传统实现的复杂性：

### 场景需求
企业知识库问答系统需要为不同部门提供定制化问答服务：
- 技术部门：需要详细的技术文档解答
- 销售部门：需要简洁的产品功能介绍  
- 客服部门：需要标准的服务流程指导
- 所有部门：共享基础的系统指令和格式要求

### 传统实现的问题分析

1. **代码重复严重**：每个部门的模板函数包含大量重复的格式化逻辑
2. **维护成本高昂**：修改基础指令需要修改所有模板函数
3. **缺乏继承机制**：无法创建模板基类实现代码重用
4. **扩展性差**：新增部门需要从头编写完整模板函数
5. **一致性难保证**：相似的格式化逻辑在不同部门可能有细微差异
6. **测试负担重**：每个模板都需要独立测试，无法测试共享逻辑

## LangChain v1.2.22模板复用解决方案：面向对象的模板继承

LangChain通过面向对象的模板系统，提供了强大的模板复用和继承机制。基于`langchain_core.prompts`模块，让我们探索LangChain v1.2.22中的多种模板复用模式。

### 核心继承体系分析

通过分析LangChain源代码，我们发现了以下核心继承结构：

```python
# LangChain v1.2.22核心模板继承体系（简化示意）
"""
BasePromptTemplate (抽象基类)
    ├── StringPromptTemplate (字符串模板基类)
    │       ├── PromptTemplate (标准提示模板)
    │       ├── FewShotPromptTemplate (少样本提示模板)
    │       └── ... (其他字符串模板)
    │
    └── ChatPromptTemplate (聊天提示模板基类)
            ├── ... (各种聊天模板变体)
            └── ... (支持消息序列的模板)
            
所有模板都继承自RunnableSerializable，支持序列化和链式执行。
"""
```

### 方案一：模板继承与扩展 - 创建可复用的模板基类

```python
from langchain_core.prompts import PromptTemplate, BasePromptTemplate
from typing import Dict, Any, List
import json

# 1. 定义企业级模板基类
class EnterprisePromptTemplate:
    """企业级提示模板基类 - 封装通用逻辑"""
    
    def __init__(self, department: str, format_style: str = "标准格式"):
        self.department = department
        self.format_style = format_style
        self.base_instructions = self._load_base_instructions()
    
    def _load_base_instructions(self) -> Dict[str, str]:
        """加载基础指令配置"""
        return {
            "技术部门": "你是一个技术专家助手。请提供详细、准确的技术解答。",
            "销售部门": "你是一个销售专家助手。请提供简洁、有吸引力的产品介绍。",
            "客服部门": "你是一个客服专家助手。请提供友好、专业的服务指导。",
            "人力资源部门": "你是一个HR专家助手。请提供专业、合规的人力资源建议。",
            "默认": "你是一个专业助手。请提供准确、有用的回答。"
        }
    
    def _get_formatting_rules(self) -> str:
        """根据格式风格获取格式化规则"""
        formatting_rules = {
            "标准格式": """请按以下格式回答：
1. 问题分析
2. 核心要点
3. 具体建议
4. 后续步骤""",
            "简洁格式": """请提供简洁的回答，包含：
• 关键信息
• 主要建议""",
            "详细格式": """请提供详细的回答，包含：
一、问题背景分析
二、解决方案详细说明
三、实施步骤详解
四、注意事项与风险提示
五、相关资源推荐"""
        }
        return formatting_rules.get(self.format_style, formatting_rules["标准格式"])
    
    def create_template(self) -> PromptTemplate:
        """创建具体的LangChain PromptTemplate"""
        template_str = f"""{self.base_instructions.get(self.department, self.base_instructions["默认"])}

{{context_prefix}}用户问题：{{question}}

{self._get_formatting_rules()}

{{additional_instructions}}"""
        
        return PromptTemplate.from_template(
            template=template_str,
            template_format="jinja2",  # 支持多种模板引擎
            input_variables=["question", "context_prefix", "additional_instructions"]
        )

# 2. 创建具体的部门模板类
```

### 方案二：模板组合与拼接 - 使用+运算符重载

LangChain的模板系统支持使用`+`运算符进行模板组合，这是实现模板复用的强大机制：

```python

)

    """请按以下格式回答：
{format_structure}

{additional_requirements}"""

# 4. 动态模板组合
def create_dynamic_template(requirements: Dict[str, Any]) -> PromptTemplate:
    """根据需求动态组合模板"""
    template_parts = []
    
    # 基础部分
    template_parts.append(base_instruction.partial(
        role=requirements.get("role", "专业"),
        quality=requirements.get("quality", "准确"),
        domain=requirements.get("domain", "相关")
    ))
    
    template_parts.append(PromptTemplate.from_template("\n\n"))
    template_parts.append(context_handler)
    
    # 条件性添加格式化规则
    if requirements.get("include_formatting", True):
        format_structure = requirements.get("format_structure", "标准格式")
        template_parts.append(formatting_rules.partial(
            format_structure=format_structure,
            additional_requirements=requirements.get("additional_requirements", "")
        ))
    
    # 条件性添加特定领域内容
    if requirements.get("domain") == "技术":
        template_parts.append(PromptTemplate.from_template("\n\n"))
        template_parts.append(tech_specific.partial(
            tech_standard=requirements.get("tech_standard", "行业标准"),
            code_examples=requirements.get("code_examples", "示例"),
            performance_metrics=requirements.get("performance_metrics", "性能")
        ))
    
    # 组合所有部分
    dynamic_template = template_parts[0]
    for part in template_parts[1:]:
        dynamic_template = dynamic_template + part
    
    return dynamic_template

# 测试动态模板组合
```

### 方案三：FewShotPromptTemplate的示例复用机制

`FewShotPromptTemplate`提供了基于示例的模板复用机制，特别适用于需要上下文学习的场景：

```python
from langchain_core.prompts import FewShotPromptTemplate, PromptTemplate
from langchain_core.example_selectors import SemanticSimilarityExampleSelector
from langchain_openai import OpenAIEmbeddings
import numpy as np

# 1. 创建可复用的示例库
class ExampleRepository:
    """示例库管理器 - 支持示例复用"""
    
    def __init__(self):

    def get_examples(self, category: str, max_examples: int = 3) -> list:
        """获取指定类别的示例"""
        return self.examples_by_category.get(category, [])[:max_examples]
    
    def add_example(self, category: str, example: dict):
        """添加新示例"""
        if category not in self.examples_by_category:
            self.examples_by_category[category] = []
        self.examples_by_category[category].append(example)
```

```python
from abc import ABC, abstractmethod
from typing import Dict, List, Optional
import yaml

class TemplateRegistry(ABC):
    """模板注册表抽象类"""
    
    @abstractmethod
    def register_template(self, template_id: str, template: BasePromptTemplate):
        """注册模板"""
        pass
    
    @abstractmethod
    def get_template(self, template_id: str) -> Optional[BasePromptTemplate]:
        """获取模板"""
        pass
    
    @abstractmethod
    def find_templates_by_tag(self, tag: str) -> List[BasePromptTemplate]:
        """根据标签查找模板"""
        pass

class VersionedTemplate:
    """版本化模板"""
    
    def __init__(self, template_id: str, version: str, template: BasePromptTemplate):
        self.template_id = template_id
        self.version = version
        self.template = template
        self.metadata = {
            "created_at": "2024-01-01",
            "author": "system",
            "tags": [],
            "dependencies": []
        }
    
    def to_dict(self) -> Dict:
        """序列化为字典"""
        return {
            "id": self.template_id,
            "version": self.version,
            "template": self.template.dict(),
            "metadata": self.metadata
        }

class EnterpriseTemplateManager:
    """企业级模板管理器"""
    
    def __init__(self):
        self.templates: Dict[str, List[VersionedTemplate]] = {}
        self.template_index = {}  # 快速索引
        self.dependency_graph = {}  # 依赖关系图
    
    def register_template(self, template: VersionedTemplate):
        """注册版本化模板"""
        if template.template_id not in self.templates:
            self.templates[template.template_id] = []
        
        self.templates[template.template_id].append(template)
        
        # 按版本排序（语义化版本）
        self.templates[template.template_id].sort(
            key=lambda x: [int(num) for num in x.version.split('.')],
            reverse=True
        )
        
        # 更新索引
        for tag in template.metadata.get("tags", []):
            if tag not in self.template_index:
                self.template_index[tag] = []
            self.template_index[tag].append(template.template_id)
    
    def get_latest_template(self, template_id: str) -> Optional[VersionedTemplate]:
        """获取最新版本的模板"""
        if template_id in self.templates and self.templates[template_id]:
            return self.templates[template_id][0]
        return None
    
    def get_template_by_version(self, template_id: str, version: str) -> Optional[VersionedTemplate]:
        """获取指定版本的模板"""
        if template_id in self.templates:
            for template in self.templates[template_id]:
                if template.version == version:
                    return template
        return None
    
    def create_template_variant(self, base_template_id: str, variant_name: str, 
                               modifications: Dict) -> VersionedTemplate:
        """创建模板变体（继承）"""
        base_template = self.get_latest_template(base_template_id)
        if not base_template:
            raise ValueError(f"基础模板不存在: {base_template_id}")
        
        # 克隆并修改模板
        variant_template = self._apply_modifications(base_template.template, modifications)
        
        variant = VersionedTemplate(
            template_id=f"{base_template_id}_{variant_name}",
            version="1.0.0",
            template=variant_template
        )
        
        # 记录依赖关系
        variant.metadata["dependencies"] = [{
            "template_id": base_template_id,
            "version": base_template.version,
            "modifications": modifications
        }]
        
        return variant
    
    def _apply_modifications(self, template: BasePromptTemplate, modifications: Dict) -> BasePromptTemplate:
        """应用模板修改"""
        # 这里简化实现，实际应根据修改类型应用不同操作
        # 例如：修改模板字符串、添加/删除变量、更改格式等
        return template
    
    def export_templates(self, format: str = "yaml") -> str:
        """导出所有模板"""
        export_data = {}
        
        for template_id, versions in self.templates.items():
            export_data[template_id] = [
                version.to_dict() for version in versions
            ]
        
        if format == "yaml":
            return yaml.dump(export_data, allow_unicode=True)
        else:
            return json.dumps(export_data, indent=2, ensure_ascii=False)
    
    def import_templates(self, data: str, format: str = "yaml"):
        """导入模板"""
        if format == "yaml":
            import_data = yaml.safe_load(data)
        else:
            import_data = json.loads(data)
        
        for template_id, versions in import_data.items():
            for version_data in versions:
                # 这里简化，实际需要反序列化模板对象
                template = PromptTemplate.from_template(version_data["template"]["template"])
                versioned_template = VersionedTemplate(
                    template_id=template_id,
                    version=version_data["version"],
                    template=template
                )
                versioned_template.metadata = version_data["metadata"]
                self.register_template(versioned_template)
```

## 最佳实践与模式总结

### 1. 模板复用设计原则

1. **单一职责原则**：每个模板应只负责一个特定功能
2. **开闭原则**：模板应对扩展开放，对修改封闭
3. **里氏替换原则**：子类模板应能替换父类模板而不影响功能
4. **接口隔离原则**：定义小而专的模板接口，而非大而全的通用接口
5. **依赖倒置原则**：高层模板不应依赖低层模板，两者都应依赖抽象

### 2. LangChain模板复用优势

1. **面向对象设计**：基于类的继承体系，支持真正的代码复用
2. **运算符重载**：使用`+`运算符实现直观的模板组合
3. **部分填充**：`partial()`方法支持模板预填充和参数化
4. **序列化支持**：所有模板都支持JSON/YAML序列化，便于存储和传输
5. **类型安全**：输入变量类型检查，减少运行时错误

### 3. 企业级实施建议

1. **建立模板库**：集中管理所有可复用模板
2. **版本控制**：为重要模板添加版本号，支持回滚和AB测试
3. **依赖管理**：记录模板间的依赖关系，避免循环依赖
4. **质量门禁**：为模板添加质量标准（如变量覆盖率、复杂度指标）
5. **使用统计**：监控模板使用频率和效果，持续优化

### 4. 性能优化策略

1. **模板缓存**：对高频使用的模板进行缓存
2. **懒加载**：大型模板库支持按需加载
3. **预编译**：对稳定模板进行预编译优化
4. **增量更新**：只更新变化的模板部分
