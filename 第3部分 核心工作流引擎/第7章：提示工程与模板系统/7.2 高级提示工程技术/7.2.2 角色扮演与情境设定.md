# 7.2.2 角色扮演与情境设定

## 角色扮演与情境设定的核心价值：从通用助手到专业专家

在AI应用开发中，角色扮演（Role-Playing）和情境设定（Scenario Setting）是提升模型专业性、适应性和用户体验的关键技术。其核心价值体现在：

1. **专业性增强**：通过特定角色设定（如医生、律师、教师），模型能够提供更专业的回答
2. **情境适应性**：根据用户场景（如商务会议、教育辅导、客户服务）调整响应风格和内容
3. **个性化体验**：针对不同用户群体（如儿童、专家、初学者）提供定制化交互
4. **一致性维护**：确保在同一角色或情境下，模型行为保持一致性
5. **可信度提升**：专业的角色设定增强用户对模型回答的信任度

## 传统角色扮演实现的复杂性：硬编码角色与情境切换

在LangChain出现之前，开发人员需要手动管理角色设定和情境切换，导致代码重复且难以维护。让我们通过一个多角色客服系统来展示传统实现的复杂性：

### 场景需求
企业智能客服系统需要支持多种角色和情境：
1. **技术支持专家**：解决技术问题，提供详细解决方案
2. **销售顾问**：介绍产品优势，处理购买咨询
3. **客户服务代表**：处理投诉和售后服务
4. **培训导师**：提供产品使用培训
5. **紧急情况处理员**：处理紧急问题和投诉

```python
                "tone": "专业、迅速",
                "urgency": "高",

            },

                "context": "客户投诉场景",
                "tone": "同理心、专业",
                "urgency": "中高",

                "tone": "耐心、详细",
                "urgency": "低",
                "formality": "适中",
                "detail_level": "非常详细"
            }
        }
    
    def generate_prompt(self, user_query: str, role: str, scenario: str = "normal") -> str:
        """生成角色扮演提示 - 复杂的条件逻辑"""
        
        if role not in self.role_profiles:
            raise ValueError(f"角色不存在: {role}")
        
        if scenario not in self.scenario_templates:
            raise ValueError(f"情境不存在: {scenario}")
        
        role_profile = self.role_profiles[role]
        scenario_template = self.scenario_templates[scenario]
        
        # 构建复杂的提示
        prompt_parts = []
        
        # 1. 角色设定部分
        prompt_parts.append(f"你是一个{role_profile['name']}，角色描述：{role_profile['description']}")
        prompt_parts.append(f"性格特点：{role_profile['personality']}")
        prompt_parts.append(f"专业知识领域：{', '.join(role_profile['knowledge_domains'])}")
        prompt_parts.append(f"回答风格：{role_profile['response_style']}")
        
        # 2. 情境设定部分
        prompt_parts.append(f"\n当前情境：{scenario_template['context']}")
        prompt_parts.append(f"语气要求：{scenario_template['tone']}")
        prompt_parts.append(f"紧急程度：{scenario_template['urgency']}")
        
        if scenario == "urgent":
            prompt_parts.append("注意：这是一个紧急问题，需要优先处理！")
        elif scenario == "complaint":
            prompt_parts.append("注意：这是一个投诉场景，需要特别耐心和处理技巧。")
        
        # 3. 对话历史处理（简化）
        prompt_parts.append(f"\n{role_profile['greeting']}")
        
        # 4. 用户查询
        prompt_parts.append(f"\n用户咨询：{user_query}")
        
        # 5. 响应要求
        prompt_parts.append(f"\n请以{role_profile['name']}的身份回答：")
        
        # 根据情境调整响应要求
        if scenario_template['detail_level'] == "非常详细":
            prompt_parts.append("请提供详细的步骤说明和解释。")
        elif scenario_template['detail_level'] == "关键信息":
            prompt_parts.append("请直接提供关键解决方案，避免不必要的细节。")
        
        # 6. 格式要求
        prompt_parts.append(f"\n回答格式要求：")
        prompt_parts.append("1. 首先确认问题理解")
        prompt_parts.append("2. 提供解决方案或建议")
        prompt_parts.append("3. 如有必要，提供后续步骤")
        prompt_parts.append(f"4. 结束语：{role_profile['closing']}")
        
        return "\n".join(prompt_parts)
    
    def process_response(self, response: str, role: str, scenario: str) -> dict:
        """处理响应 - 手动验证角色一致性"""
        
        role_profile = self.role_profiles[role]
        
        # 检查响应是否包含角色特定的关键词
        role_keywords = {
            "tech_support": ["步骤", "解决方案", "配置", "调试"],
            "sales_consultant": ["优势", "特点", "价格", "购买"],
            "customer_service": ["理解", "抱歉", "解决", "反馈"]
        }
        
        keywords = role_keywords.get(role, [])
        contains_keywords = any(keyword in response for keyword in keywords)
        
        # 检查响应长度是否符合情境要求
        scenario_template = self.scenario_templates[scenario]
        response_length = len(response)
        
        length_appropriate = True
        if scenario_template['detail_level'] == "非常详细" and response_length < 200:
            length_appropriate = False
        elif scenario_template['detail_level'] == "关键信息" and response_length > 500:
            length_appropriate = False
        
        # 检查是否包含角色名称
        contains_role_name = role_profile['name'] in response
        
        return {
            "response": response,
            "role_consistency": contains_keywords and contains_role_name,
            "scenario_appropriateness": length_appropriate,
            "response_length": response_length,
            "analysis": {
                "contains_role_keywords": contains_keywords,
                "contains_role_name": contains_role_name,
                "length_appropriate": length_appropriate
            }
        }
```

### 传统实现的问题分析

1. **配置硬编码**：所有角色和情境配置都硬编码在代码中，难以扩展
2. **逻辑复杂性**：角色和情境的组合导致复杂的条件判断逻辑
3. **维护困难**：修改一个角色需要更新多个相关函数和配置文件
4. **缺乏动态性**：无法根据对话上下文动态调整角色或情境
5. **验证薄弱**：简单的关键词匹配无法全面评估角色扮演质量
6. **复用性差**：角色模板无法在不同场景中复用

## LangChain v1.2.22角色扮演解决方案：动态系统提示与情境管理

LangChain通过ChatPromptTemplate和动态系统提示中间件，提供了强大的角色扮演和情境设定支持。让我们探索LangChain v1.2.22中的多种角色扮演实现方式。

### 核心角色扮演模式分析

通过分析LangChain文档，我们发现了以下核心角色扮演模式：

### 方案二：动态系统提示中间件 - 基于上下文的角色调整

```python
from typing import Dict, Any, Optional
from pydantic import BaseModel, Field
from enum import Enum

# 1. 定义上下文模型
class UserContext(BaseModel):
    """用户上下文"""
    user_role: str = Field(description="用户角色", default="general")

    def generate_system_prompt(self, 
                              role: str,
                              scenario: ScenarioType,
                              user_context: UserContext,

        # 获取基础模板
        base_template = self.role_base_templates[role]
        
        # 填充基础模板
        filled_template = base_template.format(
            expertise_level=expertise_level,
            domain=domain,
            expertise_areas=expertise_areas,
            response_requirements=response_requirements,
            style_guidelines=style_guidelines
        )
        
        # 添加情境修饰

{scenario_modifier['urgency_note']}

请根据以上设定回答用户问题。"""
        
        # 如果有对话历史，添加到提示中
        if user_context.conversation_history:
            final_prompt += f"\n\n对话历史摘要：{user_context.conversation_history}"
        
        return final_prompt

        """从上下文创建提示"""
        
        # 从上下文中提取参数
        role = context.get("role", "tech_expert")
        scenario_str = context.get("scenario", "normal")
        user_context = context.get("user_context", UserContext())
        domain = context.get("domain", "相关")
        
        # 转换情境字符串为枚举
        try:
            scenario = ScenarioType(scenario_str)
        except ValueError:
            scenario = ScenarioType.NORMAL
        
        # 生成系统提示
        system_prompt = generator.generate_system_prompt(
            role=role,
            scenario=scenario,
            user_context=user_context,
            domain=domain
        )
        
        # 创建聊天提示模板
        messages = [
            ("system", system_prompt),
            ("human", "{user_query}")
        ]
        
        return ChatPromptTemplate.from_messages(messages)
    
    # 创建链
    chain = (
        RunnablePassthrough()  # 传递所有输入
        | RunnableLambda(lambda x: create_prompt_from_context(x))  # 创建提示
        | ChatOpenAI(model="gpt-3.5-turbo", temperature=0.7)  # LLM
    )
    
    return chain
```

class RoleSwitchingMechanism:
    """角色切换机制"""
    
    def __init__(self, collaboration_system: MultiRoleCollaborationSystem):
        self.collaboration_system = collaboration_system
        self.conversation_history = []
    
    def analyze_conversation_for_role_switch(self, conversation: List[Dict]) -> Dict[str, Any]:
        """分析对话，决定是否需要切换角色"""
        
        if not conversation:
            return {"switch_needed": False, "recommended_role": None}
        
        # 简单的关键词分析（实际应使用更复杂的NLP）
        last_user_message = conversation[-1].get("content", "").lower()
        
        # 检测技术关键词
        tech_keywords = ["代码", "bug", "错误", "崩溃", "性能", "配置", "服务器", "数据库"]
        if any(keyword in last_user_message for keyword in tech_keywords):
            return {
                "switch_needed": True,
                "recommended_role": RoleType.TECHNICAL,
                "reason": "检测到技术问题"
            }
        
        # 检测商业关键词
        business_keywords = ["成本", "收益", "市场", "策略", "投资", "风险", "竞争"]
        if any(keyword in
