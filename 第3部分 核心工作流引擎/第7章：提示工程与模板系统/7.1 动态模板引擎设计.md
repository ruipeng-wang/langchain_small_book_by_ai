# 7.1 动态模板引擎设计

## 概念讲解

### 什么是提示模板

提示模板（Prompt Template）是LLM应用的"脚本框架"。它定义了模型接收什么输入、以什么角色回答、输出什么格式。没有模板，每次调用都需要手动拼接字符串，既容易出错又难以维护。

### 模板引擎的核心价值

1. **变量绑定**：将动态数据注入固定模板结构
2. **条件渲染**：根据不同场景生成不同提示内容
3. **模板复用**：一次定义，多处使用
4. **格式控制**：确保输出符合下游系统要求

LangChain的模板系统基于`BasePromptTemplate`抽象类，所有模板都继承自这个基类，保证了接口的一致性。

## 核心要点

1. **PromptTemplate**：基础字符串模板，使用`{variable}`语法
2. **ChatPromptTemplate**：聊天模型专用，支持消息序列
3. **FewShotPromptTemplate**：少样本学习，自动注入示例
4. **partial()方法**：预填充部分变量，实现模板定制
5. **模板组合**：使用`+`运算符组合多个模板片段

## 简单示例

### 基础模板与变量绑定

```python
from langchain_core.prompts import PromptTemplate

# 创建模板
template = PromptTemplate.from_template(
    "你是一个{role}助手。请回答关于{topic}的问题。\n\n用户问题：{question}"
)

# 渲染模板
prompt = template.format(
    role="技术专家",
    topic="Python编程",
    question="什么是装饰器？"
)
print(prompt)
```

### ChatPromptTemplate与消息序列

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

# 创建聊天模板
chat_template = ChatPromptTemplate.from_messages([
    ("system", "你是一个{role}，专注于{domain}领域。"),
    MessagesPlaceholder("history", optional=True),  # 可选的历史消息
    ("human", "{question}")
])

# 渲染
messages = chat_template.invoke({
    "role": "编程助手",
    "domain": "Web开发",
    "question": "如何用FastAPI创建REST API？"
})
```

### 使用partial预填充变量

```python
from langchain_core.prompts import PromptTemplate

# 创建基础模板
base = PromptTemplate.from_template("{role}\n\n{content}")

# 预填充角色
expert_prompt = base.partial(role="你是一位资深数据科学家。")

# 使用时只需提供内容
result = expert_prompt.format(content="请解释交叉验证的原理。")
```

## 进阶应用

### 条件模板

```python
from langchain_core.prompts import PromptTemplate

# 根据是否有上下文生成不同模板
def create_conditional_prompt(question: str, context: str = None):
    if context:
        template = """基于以下上下文回答问题：
{context}

问题：{question}"""
    else:
        template = "问题：{question}"
    
    return PromptTemplate.from_template(template)

# 使用
prompt_with_context = create_conditional_prompt(
    "性能如何优化？",
    context="系统使用Python编写，数据库为PostgreSQL"
)
```

### 模板组合

```python
from langchain_core.prompts import PromptTemplate

# 定义可复用的模板片段
role_def = PromptTemplate.from_template("你是{role}。")
task_def = PromptTemplate.from_template("任务：{task}")
format_req = PromptTemplate.from_template("输出格式：{format}")

# 组合模板
full_prompt = role_def + task_def + format_req

# 渲染
result = full_prompt.format(
    role="代码审查员",
    task="检查以下代码的潜在问题",
    format="列出问题及修复建议"
)
```

### FewShotPromptTemplate示例学习

```python
from langchain_core.prompts import FewShotPromptTemplate, PromptTemplate

# 定义示例格式
example_prompt = PromptTemplate.from_template(
    "问题：{input}\n回答：{output}"
)

# 准备示例
examples = [
    {"input": "什么是API？", "output": "API是应用程序编程接口..."},
    {"input": "什么是REST？", "output": "REST是一种架构风格..."},
]

# 创建少样本模板
few_shot = FewShotPromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
    prefix="请参考以下示例回答用户问题：",
    suffix="问题：{question}\n回答：",
    input_variables=["question"],
    example_separator="\n\n"
)

# 渲染
prompt = few_shot.format(question="什么是GraphQL？")
```

## 常见问题

**Q: PromptTemplate和ChatPromptTemplate有什么区别？**

A: `PromptTemplate`生成纯文本字符串，适用于LLM（文本补全模型）。`ChatPromptTemplate`生成消息序列（system/human/ai），适用于ChatModel（聊天模型）。现代应用推荐使用ChatPromptTemplate。

**Q: 如何在模板中使用Jinja2语法？**

A: LangChain默认使用f-string语法。如需Jinja2，设置`template_format="jinja2"`：
```python
PromptTemplate.from_template("你好{{ name }}", template_format="jinja2")
```

**Q: 模板中的变量不存在会怎样？**

A: 调用`format()`时会抛出`KeyError`。建议使用`partial()`预填充默认值，或在调用前检查变量完整性。

## 本节总结

LangChain的模板系统通过变量绑定、partial预填充、模板组合等机制，将硬编码的提示词转变为可复用、可配置的模板引擎。掌握这些技术，可以大幅减少重复代码，提高应用的可维护性。
