# 6.3 流式输出与Token控制

## 概念讲解

### 为什么需要流式输出

在大语言模型应用中，流式输出是提升用户体验的关键技术。传统的一次性等待模式存在以下问题：

- **用户焦虑**：长时间等待无反馈，产生不确定性
- **无法中断**：发现方向错误时已无法挽回
- **资源浪费**：不合适的生成无法及时停止

流式输出通过实时返回生成的token，让用户能够即时看到进展，大幅改善交互体验。

### 流式输出的实现原理

LangChain的流式输出基于回调机制。模型在生成每个token后，会通过回调函数通知调用方：

```
用户请求 → LangChain → 模型API → 逐个token返回
                              ↓
                         回调函数处理 → 实时显示
```

核心方法：
- `model.stream()`：返回迭代器，逐个获取token
- `StreamingStdOutCallbackHandler`：内置的标准输出回调处理器
- 自定义`BaseCallbackHandler`：实现更复杂的进度展示

### Token预算管理的必要性

Token作为LLM处理的基本单位，直接影响：

1. **API成本**：云服务按Token计费，无限制使用会导致预算超支
2. **上下文限制**：每个模型都有最大上下文窗口
3. **性能表现**：长上下文增加处理延迟

## 核心要点

1. **流式输出**：使用`stream()`方法实现实时token返回
2. **回调系统**：`CallbackHandler`提供丰富的生命周期事件
3. **消息截断**：`trim_messages()`智能控制消息长度
4. **Token估算**：`count_tokens_approximately()`快速估算token数

## 简单示例

### 基本流式输出

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini")

for chunk in model.stream("讲一个关于AI的故事"):
    print(chunk.content, end="", flush=True)
```

### 带回调的流式输出

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks.streaming_stdout import StreamingStdOutCallbackHandler

model = ChatOpenAI(
    model="gpt-4o-mini",
    streaming=True,
    callbacks=[StreamingStdOutCallbackHandler()]

```

### 自定义进度回调

```python
from langchain_core.callbacks import BaseCallbackHandler
from langchain_openai import ChatOpenAI
from typing import Any, Dict, List
import time

class ProgressCallback(BaseCallbackHandler):
    """自定义进度回调"""
    
    def __init__(self):
        self.token_count = 0
        self.start_time = None
    
    def on_llm_start(self, serialized: Dict[str, Any], prompts: List[str], **kwargs):
        self.start_time = time.time()
        print("[开始] LLM开始处理...")
    
    def on_llm_new_token(self, token: str, **kwargs):
        self.token_count += 1
        if self.token_count % 10 == 0:  # 每10个token显示一次
            elapsed = time.time() - self.start_time
            speed = self.token_count / elapsed if elapsed > 0 else 0
            print(f"\r进度: {self.token_count} tokens | 速度: {speed:.1f} tokens/s", end="")
    
    def on_llm_end(self, response, **kwargs):
        elapsed = time.time() - self.start_time
        print(f"\n[完成] 总token: {self.token_count}, 总时间: {elapsed:.2f}s")

# 使用自定义回调
model = ChatOpenAI(model="gpt-4o-mini", streaming=True, callbacks=[ProgressCallback()])
response = model.invoke("解释机器学习的基本概念")
```

## 进阶应用

### 带进度条的流式输出

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import BaseCallbackHandler
from typing import Dict, Any, List

class ProgressBarCallback(BaseCallbackHandler):
    """带进度条的回调"""
    
    def __init__(self, estimated_tokens: int = 200):
        self.estimated_tokens = estimated_tokens
        self.token_count = 0
    
    def on_llm_new_token(self, token: str, **kwargs):
        self.token_count += 1
        progress = min(100, (self.token_count / self.estimated_tokens) * 100)
        bar_len = 30
        filled = int(bar_len * progress / 100)
        bar = "█" * filled + "░" * (bar_len - filled)
        print(f"\r[{bar}] {progress:.1f}%", end="", flush=True)
    
    def on_llm_end(self, response, **kwargs):
        print("\n[完成]")

model = ChatOpenAI(model="gpt-4o-mini", streaming=True, callbacks=[ProgressBarCallback(estimated_tokens=300)])
response = model.invoke("写一段500字的技术说明")
```

### 多轮对话的Token管理

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import trim_messages, HumanMessage, AIMessage

class ConversationManager:
    """对话管理器"""
    
    def __init__(self, max_tokens: int = 4000):
        self.model = ChatOpenAI(model="gpt-4o-mini")
        self.max_tokens = max_tokens
        self.history = []
    
    def add_message(self, role: str, content: str):
        self.history.append((role, content))
    
    def invoke(self, user_input: str) -> str:
        # 添加用户输入
        self.history.append(("human", user_input))
        
        # 裁剪历史消息
        trimmed_history = trim_messages(
            self.history,
            max_tokens=self.max_tokens,
            strategy="last",
            token_counter=self.model,
            include_system=True
        )
        
        # 调用模型
        response = self.model.invoke(trimmed_history)
        
        # 保存助手回复
        self.history.append(("ai", response.content))
        
        return response.content

# 使用示例
chat = ConversationManager(max_tokens=4000)
chat.add_message("system", "你是一个专业的编程助手")
response = chat.invoke("解释什么是闭包")
print(response)
```

## 常见问题

**Q: 流式输出会增加API成本吗？**

A: 不会。流式输出只是改变了数据返回方式，token数量与一次性返回相同。

**Q: 如何估算输出token数？**

A: 使用`model.get_num_tokens()`可以估算输入token数。输出token数较难预测，可设置`max_tokens`参数限制。

**Q: trim_messages的strategy参数有什么区别？**

A: `"last"`保留最后的消息（适合对话），`"first"`保留开头的消息（适合文档摘要）。

## 本节总结

流式输出和Token控制是构建优质AI应用的关键技术。通过`stream()`方法实现实时反馈，通过回调系统监控进度，通过`trim_messages()`管理上下文长度。掌握这些技术，可以构建响应迅速、成本可控的AI应用。
