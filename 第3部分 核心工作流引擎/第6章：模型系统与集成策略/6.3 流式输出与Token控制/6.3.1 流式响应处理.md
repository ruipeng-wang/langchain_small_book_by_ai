# 6.3.1 流式响应处理

## 概念讲解

### 为什么需要流式输出

传统的模型调用是"等待-返回"模式：用户发送请求后，必须等待模型完成全部内容生成才能看到结果。对于长内容生成，这可能导致数秒甚至数十秒的等待。

流式输出解决了这个问题：**模型每生成一个token就立即发送**，为用户提供实时反馈。

| 维度 | 非流式 | 流式 |
|------|--------|------|
| 首字延迟 | 需等待全部内容 | 首个token即可展示 |
| 用户体验 | 长时间空白等待 | 实时打字机效果 |
| 资源占用 | 响应全部加载到内存 | 按需处理，内存友好 |
| 错误发现 | 生成完毕后才能发现 | 可及早发现并中断 |

### LangChain的流式抽象

LangChain为所有模型提供了统一的流式接口：

```
模型提供商API → LangChain适配器 → 统一流式接口
├── OpenAI SSE       ├── 格式标准化    ├── stream() 同步流式
├── Anthropic流式    ├── 错误处理包装  ├── astream() 异步流式
├── 本地模型         └── 统一Chunk格式
```

开发者只需学习一套API，即可操作所有模型的流式功能。

## 核心要点

1. **stream()**：返回生成器（Generator），每次迭代产生一个AIMessageChunk
2. **惰性计算**：只在需要时生成下一个token，无需预加载
3. **可中断**：随时可以停止迭代，释放资源
4. **格式统一**：所有模型返回相同的AIMessageChunk格式

## 简单示例

### 基本流式输出

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini", streaming=True)

for chunk in model.stream("写一个关于AI的短篇故事"):
    print(chunk.content, end="", flush=True)
```

### 异步流式

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini", streaming=True)

async for chunk in model.astream("讲一个技术故事"):
    print(chunk.content, end="", flush=True)
```

### 多模型流式对比

```python
from langchain.chat_models import init_chat_model

models = {
    "openai": init_chat_model("gpt-4o-mini"),
    "anthropic": init_chat_model("claude-sonnet-4-mini"),
}

prompt = "用一句话介绍你自己"

for name, model in models.items():
    print(f"\n{name}: ", end="")
    for chunk in model.stream(prompt):
        print(chunk.content, end="", flush=True)
```

## 进阶应用

### Web应用中的流式输出

在FastAPI中实现SSE（Server-Sent Events）流式响应：

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from langchain_openai import ChatOpenAI

app = FastAPI()
model = ChatOpenAI(model="gpt-4o-mini", streaming=True)

@app.get("/chat")
async def chat(prompt: str):
    def event_stream():
        for chunk in model.stream(prompt):
            yield f"data: {chunk.content}\n\n"
    
    return StreamingResponse(event_stream(), media_type="text/event-stream")
```

### 流式输出中加入中断逻辑

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini", streaming=True)

def stream_with_interrupt(prompt: str, max_chunks: int = 100):
    """带中断的流式输出"""
    count = 0
    for chunk in model.stream(prompt):
        print(chunk.content, end="", flush=True)
        count += 1
        if count >= max_chunks:
            print("\n[达到最大chunk数，已中断]")
            break

stream_with_interrupt("写一个很长的故事", max_chunks=50)
```

### 累积完整响应

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini", streaming=True)

def get_full_response(prompt: str) -> str:
    """流式输出的同时累积完整响应"""
    full_response = ""
    for chunk in model.stream(prompt):
        full_response += chunk.content
    return full_response

result = get_full_response("总结机器学习的主要流派")
print(f"完整响应长度: {len(result)} 字符")
```

## 常见问题

**Q: 流式输出和非流式输出结果会不同吗？**

A: 理论上相同，但实际可能因实现细节有微小差异。核心内容应一致。

**Q: 所有模型都支持流式输出吗？**

A: 主流模型（OpenAI、Anthropic、Google、Ollama）都支持。部分模型可能需要设置`streaming=True`。

**Q: 流式输出会影响性能吗？**

A: 总体生成时间相同，但首字延迟大幅降低，用户体验更好。

## 本节总结

流式输出是提升用户体验的关键技术。通过`stream()`方法，LangChain为所有模型提供了统一的流式接口，使实时交互变得简单。在Web应用中，结合SSE技术，可以构建流畅的打字机效果，大幅改善用户等待体验。
