# 6.2.4 多模型混合使用策略

## 概念讲解

### 为什么需要多模型策略

在实际生产环境中，依赖单一模型存在诸多风险：

- **服务中断**：提供商API故障导致业务停摆
- **成本不可控**：模型价格调整难以应对
- **能力局限**：没有模型在所有任务上都表现最优
- **合规要求**：某些场景必须使用特定模型

多模型策略通过组合不同提供商的模型，实现成本、性能和可靠性的最佳平衡。

### 多模型架构的核心组件

```
请求 → 路由器 → [模型选择] → 执行 → 结果
         ↑
    [策略引擎]
    - 任务类型
    - 成本预算
    - 性能要求
    - 可用性检查
```

## 核心要点

1. **智能路由**：根据任务复杂度、类型自动选择最优模型
2. **故障转移**：主模型不可用时自动切换到备用模型
3. **成本控制**：简单任务用低成本模型，复杂任务用高性能模型
4. **A/B测试**：同时使用多个模型，对比效果

## 简单示例

### 基于复杂度的路由

```python
from langchain.chat_models import init_chat_model

class ComplexityRouter:
    """基于复杂度的模型路由器"""
    
    def __init__(self):
        self.fast_model = init_chat_model("gpt-4o-mini")
        self.smart_model = init_chat_model("gpt-4o")
    
    def invoke(self, task: str) -> str:
        # 简单判断任务复杂度
        if len(task) < 100 and not any(k in task for k in ["分析", "比较", "设计"]):
            return self.fast_model.invoke(task).content
        else:
            return self.smart_model.invoke(task).content

router = ComplexityRouter()
router.invoke("翻译'Hello World'")       # 用快速模型
router.invoke("分析这篇文章的核心观点")   # 用智能模型
```

### 故障转移

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

class FallbackChain:
    """带故障转移的调用链"""
    
    def __init__(self):
        self.primary = ChatOpenAI(model="gpt-4o-mini")
        self.backup = ChatAnthropic(model="claude-sonnet-4-mini")
    
    def invoke(self, prompt: str, max_retries: int = 2) -> str:
        try:
            return self.primary.invoke(prompt).content
        except Exception as e:
            print(f"主模型失败: {e}，切换到备用模型")
            return self.backup.invoke(prompt).content

chain = FallbackChain()
```

## 进阶应用

### 成本优化路由

```python
from langchain.chat_models import init_chat_model

class CostOptimizer:
    """成本优化器：在预算内选择最佳模型"""
    
    def __init__(self, budget_per_task: float = 0.01):
        self.budget = budget_per_task
        self.models = {
            "cheap": init_chat_model("gpt-4o-mini"),      # ~$0.00015/1K tokens
            "medium": init_chat_model("claude-sonnet-4-mini"),
            "premium": init_chat_model("gpt-4o"),         # ~$0.0025/1K tokens
        }
    
    def select_model(self, estimated_tokens: int) -> str:
        """根据预估token数和预算选择模型"""
        estimated_cost = estimated_tokens * 0.00015 / 1000
        if estimated_cost <= self.budget:
            return "cheap"
        elif estimated_cost <= self.budget * 3:
            return "medium"
        else:
            return "premium"
```

### A/B测试框架

```python
import random
from langchain.chat_models import init_chat_model

class ABTester:
    """A/B测试框架"""
    
    def __init__(self):
        self.model_a = init_chat_model("gpt-4o-mini")
        self.model_b = init_chat_model("claude-sonnet-4-mini")
        self.results = {"a": [], "b": []}
    
    def invoke(self, prompt: str) -> tuple:
        """随机选择模型并记录结果"""
        if random.random() < 0.5:
            result = self.model_a.invoke(prompt).content
            self.results["a"].append({"prompt": prompt, "result": result})
            return "A", result
        else:
            result = self.model_b.invoke(prompt).content
            self.results["b"].append({"prompt": prompt, "result": result})
            return "B", result
    
    def get_stats(self) -> dict:
        return {
            "model_a_calls": len(self.results["a"]),
            "model_b_calls": len(self.results["b"])
        }
```

## 常见问题

**Q: 如何评估哪个模型更好？**

A: 建议对典型任务样本进行测试，从准确性、速度、成本三个维度比较。也可以在生产环境做A/B测试。

**Q: 多模型策略会增加代码复杂度吗？**

A: 通过良好的抽象（如统一的路由接口），可以增加有限的复杂度，但换来更大的灵活性和可靠性。

## 本节总结

多模型混合使用策略是构建可靠、经济AI应用的关键。通过智能路由、故障转移和成本优化，可以在不同场景下自动选择最优模型，平衡性能、成本和可靠性。
