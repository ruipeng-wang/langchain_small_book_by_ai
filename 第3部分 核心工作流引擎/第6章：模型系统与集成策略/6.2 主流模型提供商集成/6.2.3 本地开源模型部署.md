# 6.2.3 本地开源模型部署

## 概念讲解

### 为什么选择本地部署

与云端API相比，本地部署开源模型有独特优势：

- **数据隐私**：敏感数据不出本地网络，符合GDPR等合规要求
- **成本控制**：高频调用场景下，一次性硬件投资比持续API费用更低
- **网络独立**：不依赖外部网络，离线环境也能工作
- **完全定制**：可以微调模型适应特定领域

### 主流本地部署方案

| 方案 | 适合场景 | 硬件要求 |
|------|---------|---------|
| **Ollama** | 快速上手、开发测试 | 8GB+内存 |
| **vLLM** | 高并发生产环境 | GPU 16GB+ |
| **HuggingFace Transformers** | 精细控制、微调 | 灵活 |

## 核心要点

1. **Ollama**是最简单的本地部署方案，一键安装运行
2. **ChatOllama**是LangChain的Ollama集成类，接口与其他ChatModel一致
3. **vLLM**适合高性能生产环境，支持连续批处理和量化
4. 本地模型支持工具调用、流式输出等高级功能

## 简单示例

### Ollama部署

首先安装Ollama并拉取模型：

```bash
# 安装后拉取模型
ollama pull llama3.2
```

LangChain集成：

```python
from langchain_ollama import ChatOllama

# 创建本地模型实例
model = ChatOllama(model="llama3.2", temperature=0.7)

# 调用方式与云端模型完全相同
response = model.invoke("解释什么是递归")
print(response.content)
```

### 中文模型部署

```python
from langchain_ollama import ChatOllama

# 中文优化模型
qwen = ChatOllama(model="qwen2.5:7b")
response = qwen.invoke("用中文介绍一下你自己")
```

### 工具调用

```python
from langchain_ollama import ChatOllama
from langchain_core.tools import tool

@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    return str(eval(expression))

model = ChatOllama(model="llama3.2")
model_with_tools = model.bind_tools([calculate])

response = model_with_tools.invoke("计算 3.14 乘以 100")
```

## 进阶应用

### 流式输出

```python
model = ChatOllama(model="llama3.2")

for chunk in model.stream("写一个短篇故事"):
    print(chunk.content, end="", flush=True)
```

### 批量处理

```python
questions = [
    "什么是机器学习？",
    "什么是深度学习？", 
    "什么是强化学习？"
]

model = ChatOllama(model="llama3.2")
responses = model.batch(questions)
for q, r in zip(questions, responses):
    print(f"Q: {q}\nA: {r.content}\n")
```

### 本地模型路由

```python
from langchain_ollama import ChatOllama
from langchain_openai import ChatOpenAI

class HybridRouter:
    """本地+云端混合路由"""
    
    def __init__(self):
        self.local_model = ChatOllama(model="llama3.2")
        self.cloud_model = ChatOpenAI(model="gpt-4o-mini")
    
    def invoke(self, task: str, use_local: bool = True):
        model = self.local_model if use_local else self.cloud_model
        return model.invoke(task)

router = HybridRouter()
# 简单任务用本地模型
router.invoke("翻译这句话", use_local=True)
# 复杂任务用云端模型
router.invoke("分析这篇论文", use_local=False)
```

## 常见问题

**Q: 本地模型和云端API性能差距大吗？**

A: 取决于硬件配置。高端GPU（RTX 4090）上，7B-13B模型的性能接近云端小模型。70B模型需要多GPU。

**Q: 最低硬件要求是什么？**

A: 7B模型约需8GB显存/内存，13B需16GB，70B需48GB+。CPU推理也能工作但速度较慢。

**Q: 如何选择本地模型？**

A: 聊天对话选`llama3.2`，中文任务选`qwen2.5`，代码生成选`codellama`，资源有限选`gemma2:2b`。

## 本节总结

本地开源模型部署提供了数据隐私、成本控制和网络独立性。Ollama是最简单的上手方案，vLLM适合高性能场景。通过LangChain的统一接口，本地模型与云端模型可以无缝混用，构建灵活可靠的AI应用。
