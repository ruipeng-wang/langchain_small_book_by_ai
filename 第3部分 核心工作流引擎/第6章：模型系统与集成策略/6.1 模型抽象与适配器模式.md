# 6.1 模型抽象与适配器模式

## 概念讲解

### 为什么需要模型抽象

在AI应用开发中，我们面临一个现实问题：市面上有数十种语言模型提供商（OpenAI、Anthropic、Google、Mistral等），每家都有独特的API接口、参数格式和响应结构。如果直接在代码中调用这些API，会导致：

1. **代码耦合严重**：业务逻辑与特定模型API绑定
2. **切换成本高昂**：更换模型需要重写大量代码
3. **维护困难**：每种模型需要单独的错误处理和重试逻辑

LangChain通过`BaseLanguageModel`抽象层解决了这些问题。这个设计模式被称为**适配器模式**——它将不同提供商的API差异封装在内部，对外提供统一的接口。

### BaseLanguageModel继承体系

LangChain的模型抽象采用继承树设计：

```
BaseLanguageModel (基础抽象类)
├── BaseLLM (传统文本模型)
│   ├── OpenAI
│   └── Cohere
└── BaseChatModel (对话模型)
    ├── ChatOpenAI
    ├── ChatAnthropic
    ├── ChatGoogle
    └── ChatMistralAI
```

所有模型类都实现了**Runnable接口**，提供统一的调用方法：

| 方法 | 说明 |
|------|------|
| `invoke()` | 单次调用，等待完整响应 |
| `stream()` | 流式调用，实时获取token |
| `batch()` | 批量调用，处理多个输入 |
| `ainvoke()` | 异步调用版本 |

### LLM与ChatModel的本质区别

LangChain将语言模型分为两类，理解它们的差异是正确选型的基础：

**核心区别一句话概括**：LLM是"字符串进字符串出"，ChatModel是"消息列表进消息出"。

```
LLM (BaseLLM):
  输入: "写一篇关于春天的诗"  →  输出: "春风拂面花开..."

ChatModel (BaseChatModel):
  输入: [SystemMessage, HumanMessage]  →  输出: AIMessage
```

ChatModel支持结构化的消息类型，这是LLM不具备的：

| 消息类型 | 角色 | 用途 |
|---------|------|------|
| `SystemMessage` | system | 设定模型行为和上下文 |
| `HumanMessage` | user | 用户输入 |
| `AIMessage` | assistant | 模型回复 |
| `ToolMessage` | tool | 工具执行结果 |

**何时选择LLM**：
- 简单的一次性文本生成
- 文本补全、摘要、翻译
- 成本敏感，不需要多轮对话

**何时选择ChatModel**：
- 多轮对话系统
- 需要工具调用（Function Calling）
- 需要结构化输出
- 需要系统提示控制模型行为

> **趋势提醒**：大多数新模型都是ChatModel，传统LLM正在逐渐被淘汰。除非你有明确的成本考虑，否则推荐使用ChatModel。

## 核心要点

1. **统一接口**：所有模型继承自`BaseLanguageModel`，提供一致的调用方法
2. **Runnable集成**：模型支持链式组合、配置传递和中间件
3. **LLM vs ChatModel**：前者适合简单文本生成，后者适合对话和复杂任务
4. **可扩展性**：通过社区包添加新模型

## 简单示例

### 统一调用

无论使用哪个提供商的模型，调用方式保持一致：

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

# OpenAI模型
openai_model = ChatOpenAI(model="gpt-4o-mini")

# Anthropic模型  
anthropic_model = ChatAnthropic(model="claude-sonnet-4-20250514")

# 调用方式完全相同
response1 = openai_model.invoke("你好")
response2 = anthropic_model.invoke("你好")
```

### 流式输出

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini")

for chunk in model.stream("讲一个关于AI的故事"):
    print(chunk.content, end="", flush=True)
```

### 便捷写法

LangChain支持给ChatModel直接传字符串，会自动包装为`HumanMessage`：

```python
chat = ChatOpenAI(model="gpt-4o-mini")
# 等价于 chat.invoke([HumanMessage(content="你好")])
response = chat.invoke("你好")
```

## 常见问题

**Q: LLM和ChatModel可以互换使用吗？**

A: 可以。由于都继承自`BaseLanguageModel`，它们提供相同的调用方法。切换时主要调整输入格式和输出解析即可。

**Q: 如何切换模型提供商？**

A: 只需更换模型类的导入和实例化，业务逻辑无需修改。确保安装对应的包（如`langchain-anthropic`）。

## 本节总结

`BaseLanguageModel`是LangChain模型系统的核心抽象，它通过统一接口屏蔽了不同模型提供商的API差异。理解LLM与ChatModel的区别是正确选型的基础，在现代AI应用开发中，ChatModel是更通用的选择。
