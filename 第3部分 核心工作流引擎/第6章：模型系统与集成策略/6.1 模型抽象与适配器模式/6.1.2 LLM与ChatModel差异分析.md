# 6.1.2 LLM与ChatModel差异分析

## 概念讲解

### 两种模型接口的本质区别

LangChain将语言模型分为两类，理解它们的差异是正确选型的基础。

**核心区别一句话概括**：LLM是"字符串进字符串出"，ChatModel是"消息列表进消息出"。

#### 输入输出对比

```
LLM (BaseLLM):
  输入: "写一篇关于春天的诗"  →  输出: "春风拂面花开..."

ChatModel (BaseChatModel):
  输入: [SystemMessage, HumanMessage]  →  输出: AIMessage
```

### 消息类型系统

ChatModel支持结构化的消息类型，这是LLM不具备的：

| 消息类型 | 角色 | 用途 |
|---------|------|------|
| `SystemMessage` | system | 设定模型行为和上下文 |
| `HumanMessage` | user | 用户输入 |
| `AIMessage` | assistant | 模型回复 |
| `ToolMessage` | tool | 工具执行结果 |

### 何时选择LLM

- 简单的一次性文本生成
- 文本补全、摘要、翻译
- 成本敏感，不需要多轮对话
- 不需要工具调用或结构化输出

### 何时选择ChatModel

- 多轮对话系统
- 需要工具调用（Function Calling）
- 需要结构化输出
- 需要系统提示控制模型行为

## 核心要点

1. **LLM**：简单文本生成，成本低，适合单轮任务
2. **ChatModel**：支持消息角色、多轮对话、工具调用，是现代应用的首选
3. **统一接口**：两者都提供`invoke()`、`stream()`、`batch()`方法
4. **趋势**：大多数新模型都是ChatModel，传统LLM正在逐渐被淘汰

## 简单示例

### LLM：简单文本生成

```python
from langchain_openai import OpenAI

llm = OpenAI(model="gpt-3.5-turbo-instruct")
# 直接传入字符串，返回字符串
poem = llm.invoke("写一首关于春天的诗")
print(poem)
```

### ChatModel：多轮对话

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage

chat = ChatOpenAI(model="gpt-4o-mini")

messages = [
    SystemMessage(content="你是一个专业的翻译助手。"),
    HumanMessage(content="将'Hello world'翻译成中文")
]

response = chat.invoke(messages)
print(response.content)  # 从AIMessage中提取内容
```

### 便捷写法：直接传字符串

LangChain也支持给ChatModel直接传字符串，会自动包装为`HumanMessage`：

```python
chat = ChatOpenAI(model="gpt-4o-mini")
# 等价于 chat.invoke([HumanMessage(content="你好")])
response = chat.invoke("你好")
```

## 常见问题

**Q: 我的项目应该选LLM还是ChatModel？**

A: 除非你有明确的成本考虑或只需要简单文本补全，否则推荐使用ChatModel。它功能更全面，且是当前主流模型的标准接口。

**Q: 两者可以互换使用吗？**

A: 可以。由于都继承自`BaseLanguageModel`，它们提供相同的调用方法。切换时主要调整输入格式和输出解析即可。

**Q: 为什么传统LLM正在被淘汰？**

A: 因为现代模型（GPT-4、Claude、Gemini等）都基于对话接口设计，纯文本补全接口已不再是主流。

## 本节总结

LLM和ChatModel的核心差异在于**输入输出的结构化程度**。LLM适合简单文本生成，ChatModel适合需要角色区分和多轮交互的场景。在现代AI应用开发中，ChatModel是更通用的选择。
