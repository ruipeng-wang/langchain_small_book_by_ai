# 6.1.1 BaseLanguageModel核心接口

## 概念讲解

### 为什么需要模型抽象

在AI应用开发中，我们面临一个现实问题：市面上有数十种语言模型提供商（OpenAI、Anthropic、Google、Mistral等），每家都有独特的API接口、参数格式和响应结构。如果直接在代码中调用这些API，会导致：

1. **代码耦合严重**：业务逻辑与特定模型API绑定
2. **切换成本高昂**：更换模型需要重写大量代码
3. **维护困难**：每种模型需要单独的错误处理和重试逻辑
4. **无法复用**：相同的调用模式需要反复编写

LangChain通过`BaseLanguageModel`抽象层解决了这些问题。

### BaseLanguageModel继承体系

LangChain的模型抽象采用继承树设计：

```
BaseLanguageModel (基础抽象类)
├── BaseLLM (传统文本模型)
│   ├── OpenAI
│   └── Cohere
└── BaseChatModel (对话模型)
    ├── ChatOpenAI
    ├── ChatAnthropic
    ├── ChatGoogle
    └── ChatMistralAI
```

所有模型类都实现了**Runnable接口**，提供统一的调用方法：

| 方法 | 说明 |
|------|------|
| `invoke()` | 单次调用，等待完整响应 |
| `stream()` | 流式调用，实时获取token |
| `batch()` | 批量调用，处理多个输入 |
| `ainvoke()` | 异步调用版本 |

### 统一调用示例

无论使用哪个提供商的模型，调用方式保持一致：

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

# OpenAI模型
openai_model = ChatOpenAI(model="gpt-4.1-mini")

# Anthropic模型  
anthropic_model = ChatAnthropic(model="claude-sonnet-4-20250514")

# 调用方式完全相同
response1 = openai_model.invoke("你好")
response2 = anthropic_model.invoke("你好")
```

这种设计使得更换模型只需修改一行代码。

## 核心要点

1. **统一接口**：所有模型继承自`BaseLanguageModel`，提供一致的调用方法
2. **Runnable集成**：模型支持链式组合、配置传递和中间件
3. **可扩展性**：通过社区包（`langchain_openai`、`langchain_anthropic`等）添加新模型
4. **类型安全**：完整的类型提示和IDE支持

## 简单示例

### 流式输出

流式输出可以在生成过程中实时获取token，提升用户体验：

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4.1-mini")

for chunk in model.stream("讲一个关于AI的故事"):
    print(chunk.content, end="", flush=True)
```

### 批量处理

当需要处理多个输入时，`batch()`方法比多次调用更高效：

```python
questions = [
    "什么是机器学习？",
    "什么是深度学习？",
    "什么是强化学习？"
]

responses = model.batch(questions)
for q, r in zip(questions, responses):
    print(f"Q: {q}")
    print(f"A: {r.content}\n")
```

## 常见问题

**Q: LLM和ChatModel有什么区别？**

A: `LLM`处理纯文本输入输出（字符串→字符串），适用于简单的补全任务。`ChatModel`处理消息列表（Message列表→Message），支持系统消息、用户消息、助手消息的角色区分，更适合对话场景。

**Q: 如何切换模型提供商？**

A: 只需更换模型类的导入和实例化，业务逻辑无需修改。确保安装对应的包（如`langchain-anthropic`）。

**Q: 流式输出和批量调用的区别？**

A: 流式输出（`stream`）用于单次调用时实时获取结果；批量调用（`batch`）用于并行处理多个独立输入，提高吞吐量。

## 本节总结

`BaseLanguageModel`是LangChain模型系统的核心抽象，它通过统一接口屏蔽了不同模型提供商的API差异，使开发者能够专注于业务逻辑而非集成细节。这一设计是实现可插拔模型架构的基础。
