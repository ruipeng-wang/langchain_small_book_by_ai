# 6.2 主流模型提供商集成

## 概念讲解

### 多模型集成的现实挑战

在实际AI应用开发中，企业往往需要同时使用多个模型提供商的服务：

- **成本优化**：简单任务使用低价模型，复杂任务使用高价模型
- **容灾备份**：当某个提供商服务不可用时自动切换
- **合规要求**：某些数据必须使用本地部署的模型
- **功能互补**：不同模型各有擅长

### init_chat_model：统一初始化方案

LangChain提供了`init_chat_model`函数，提供了一种标准化的方式来初始化和切换不同提供商的模型。

**核心价值**：
- 一处代码，多提供商支持
- 自动推断提供商：根据模型名称自动识别
- 运行时可配置：支持动态切换模型
- 参数统一管理：所有模型共享通用的参数接口

**模型名称规范**：
```
格式1: "提供商:模型ID"    例如 "openai:gpt-4o"
格式2: "模型ID"          自动推断提供商，例如 "gpt-4o"
```

常见模型前缀自动推断规则：

| 模型前缀 | 推断提供商 | 所需包 |
|---------|-----------|--------|
| `gpt-*`, `o1*`, `o3*` | `openai` | `langchain-openai` |
| `claude*` | `anthropic` | `langchain-anthropic` |
| `gemini*` | `google_vertexai` | `langchain-google-vertexai` |
| `mistral*` | `mistralai` | `langchain-mistralai` |

## 核心要点

1. **OpenAI GPT**：生态最完善，工具调用和结构化输出支持最好
2. **Anthropic Claude**：长上下文（200K）和安全性强，系统提示有独特处理方式
3. **本地模型（Ollama）**：数据隐私、成本控制、网络独立
4. **统一初始化**：`init_chat_model()`是推荐的标准初始化方式

## 简单示例

### 统一初始化

```python
from langchain.chat_models import init_chat_model

# 指定提供商
openai_model = init_chat_model("openai:gpt-4o-mini")
anthropic_model = init_chat_model("anthropic:claude-sonnet-4-20250514")

# 自动推断提供商
gpt_model = init_chat_model("gpt-4o-mini")  # 自动识别为openai
claude_model = init_chat_model("claude-sonnet-4-20250514")  # 自动识别为anthropic

# 调用方式完全相同
response1 = openai_model.invoke("你好")
response2 = anthropic_model.invoke("你好")
```

### OpenAI：工具调用与结构化输出

```python
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from pydantic import BaseModel, Field

@tool
def get_weather(location: str) -> str:
    """获取指定城市的天气"""
    return f"{location}: 晴, 25°C"

class Person(BaseModel):
    name: str = Field(description="姓名")
    age: int = Field(description="年龄")

model = ChatOpenAI(model="gpt-4o-mini")

# 工具调用
model_with_tools = model.bind_tools([get_weather])
response = model_with_tools.invoke("北京今天天气怎么样？")

# 结构化输出
structured_model = model.with_structured_output(Person)
result = structured_model.invoke("马云，1964年生，企业家")
print(result.name)  # 马云
```

### Anthropic：系统提示与长文档

Claude有独特的系统提示处理方式：

```python
from langchain_anthropic import ChatAnthropic
from langchain_core.messages import HumanMessage

model = ChatAnthropic(model="claude-sonnet-4-20250514")

# 方式1：使用system参数（推荐）
response = model.invoke(
    [HumanMessage(content="帮我写一个快速排序")],
    system="你是一个Python编程专家，代码要简洁高效"
)
```

### 本地模型：Ollama部署

## 进阶应用

### 运行时动态切换模型

```python
from langchain.chat_models import init_chat_model

# 创建可配置模型
configurable_model = init_chat_model(
    temperature=0.7,
    configurable_fields=("model", "model_provider")
)

# 运行时通过config切换
response1 = configurable_model.invoke(
    "解释量子计算",
    config={"configurable": {"model": "gpt-4o-mini"}}
)

response2 = configurable_model.invoke(
    "解释量子计算", 
    config={"configurable": {"model": "claude-sonnet-4-20250514"}}
)
```

### 基于复杂度的模型路由

```python
from langchain.chat_models import init_chat_model

class ComplexityRouter:
    """基于复杂度的模型路由器"""
    
    def __init__(self):
        self.fast_model = init_chat_model("gpt-4o-mini")
        self.smart_model = init_chat_model("gpt-4o")
    
    def invoke(self, task: str) -> str:
        # 简单判断任务复杂度
        if len(task) < 100 and not any(k in task for k in ["分析", "比较", "设计"]):
            return self.fast_model.invoke(task).content
        else:
            return self.smart_model.invoke(task).content

router = ComplexityRouter()
router.invoke("翻译'Hello World'")       # 用快速模型
router.invoke("分析这篇文章的核心观点")   # 用智能模型
```

### 故障转移

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

class FallbackChain:
    """带故障转移的调用链"""
    
    def __init__(self):
        self.primary = ChatOpenAI(model="gpt-4o-mini")
        self.backup = ChatAnthropic(model="claude-sonnet-4-mini")
    
    def invoke(self, prompt: str, max_retries: int = 2) -> str:
        try:
            return self.primary.invoke(prompt).content
        except Exception as e:
            print(f"主模型失败: {e}，切换到备用模型")
            return self.backup.invoke(prompt).content
```

## 模型选择指南

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 日常对话 | gpt-4o-mini | 成本低、速度快 |
| 复杂推理 | gpt-4o / claude-sonnet-4 | 推理能力强 |
| 长文档分析 | claude-sonnet-4 | 200K上下文 |
| 数据敏感 | 本地Ollama | 数据不出本地 |
| 批量处理 | gpt-4o-mini | 成本最低 |

## 常见问题

**Q: 使用`init_chat_model`需要安装哪些依赖？**

A: 需要安装对应提供商的包。例如`openai:gpt-4o`需要`langchain-openai`，`anthropic:claude-*`需要`langchain-anthropic`。

**Q: Claude和GPT哪个更好？**

A: 各有所长。Claude在安全性和长文本处理上更优，GPT在工具生态和集成便利性上更好。建议根据具体任务测试选择。

**Q: 本地模型和云端API性能差距大吗？**

A: 取决于硬件配置。高端GPU（RTX 4090）上，7B-13B模型的性能接近云端小模型。7B模型约需8GB显存/内存。

## 本节总结

通过`init_chat_model`统一初始化接口，LangChain使多模型集成变得简单。理解各模型的特点和适用场景，结合智能路由和故障转移策略，可以构建灵活、可靠的AI应用。
