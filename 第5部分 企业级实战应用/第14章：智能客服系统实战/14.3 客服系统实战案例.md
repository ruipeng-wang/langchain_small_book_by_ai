# 14.3 客服系统实战案例

## 概念讲解

前面两节介绍了架构设计和对话管理，本节将通过一个完整的实战案例，展示如何构建一个具备意图识别、知识检索、多轮对话能力的电商客服系统。

### 实战案例：电商订单查询客服

这个案例涵盖以下核心功能：

1. **意图识别**：判断用户是查询订单、申请退款还是其他咨询
2. **知识检索**：从知识库中检索相关政策（如退货政策）
3. **多轮对话**：在对话中保持上下文，追问订单号等必要信息
4. **人机协作**：复杂情况转人工处理

### 系统工作流程

## 核心要点

### 1. 状态设计原则

客服系统的状态应该包含：
- **对话历史**：使用 `MessagesState` 自动管理
- **业务状态**：意图、实体、当前操作阶段
- **辅助状态**：置信度、是否需要人工介入

### 2. 节点设计原则

每个节点应该：
- 职责单一，只做一件事
- 返回状态增量，而非全量覆盖
- 具备错误处理能力

### 3. 条件路由策略

使用条件边实现灵活的业务流程：
- 基于意图路由到不同处理节点
- 基于置信度决定是否需要人工审核
- 基于对话阶段决定下一步操作

## 完整示例

以下是一个完整的电商客服系统实现：

```python
from typing import Literal
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.checkpoint.memory import InMemorySaver
from langchain.chat_models import init_chat_model
from langchain_core.messages import HumanMessage

model = init_chat_model("gpt-4o-mini")

# 定义业务状态
class CustomerServiceState(MessagesState):
    intent: str = ""                    # 用户意图
    order_id: str = ""                  # 订单号
    needs_human: bool = False           # 是否需要人工
    confidence: float = 1.0             # 置信度
    response_type: str = ""             # 响应类型

# === 节点函数 ===

def classify_intent(state: CustomerServiceState):
    """识别用户意图"""
    last_msg = state["messages"][-1].content
    
    prompt = f"""分析用户意图，只返回JSON格式：{{"intent": "order_query|refund|general", "confidence": 0.0-1.0}}
用户输入：{last_msg}"""
    
    response = model.invoke([{"role": "user", "content": prompt}])
    # 解析响应获取意图
    import json
    try:
        result = json.loads(response.content)
        return {"intent": result["intent"], "confidence": result["confidence"]}
    except:
        return {"intent": "general", "confidence": 0.5}

def check_order_id(state: CustomerServiceState):
    """检查并提取订单号"""
    last_msg = state["messages"][-1].content
    order_id = state.get("order_id", "")
    
    # 尝试从消息中提取订单号
    import re
    match = re.search(r"(?:订单号|ORD)[：:]?\s*(\w+)", last_msg, re.IGNORECASE)
    if match:
        order_id = match.group(1)
    
    return {"order_id": order_id}

def route_intent(state: CustomerServiceState) -> str:
    """根据意图路由"""
    intent = state.get("intent", "general")
    confidence = state.get("confidence", 0)
    
    if confidence < 0.6:
        return "human_handoff"
    elif intent == "order_query":
        if not state.get("order_id"):
            return "ask_order_id"
        return "query_order"
    elif intent == "refund":
        return "refund_policy"
    else:
        return "general_response"

def ask_order_id(state: CustomerServiceState):
    """追问订单号"""
    return {"messages": [{"role": "assistant", 
                          "content": "请问您的订单号是多少？我来帮您查询。"}]}

def query_order(state: CustomerServiceState):
    """查询订单状态（模拟）"""
    order_id = state["order_id"]
    # 实际应用中这里调用订单API
    order_status = "正在配送，预计明天送达"
    return {"messages": [{"role": "assistant", 
                          "content": f"订单 {order_id} 当前状态：{order_status}"}]}

def refund_policy(state: CustomerServiceState):
    """退货政策（模拟RAG检索）"""
    policy = "我们提供7天无理由退货服务。请确保商品未使用且包装完整。"
    return {"messages": [{"role": "assistant", 
                          "content": f"退货政策：{policy}"}]}

def general_response(state: CustomerServiceState):
    """通用回复"""
    return {"messages": [{"role": "assistant", 
                          "content": "请问有什么我可以帮到您的？"}]}

def human_handoff(state: CustomerServiceState):
    """转人工处理"""
    return {"needs_human": True, 
            "messages": [{"role": "assistant", 
                          "content": "正在为您转接人工客服，请稍候..."}]}

# === 构建状态图 ===

builder = StateGraph(CustomerServiceState)

# 添加节点
builder.add_node("classify", classify_intent)
builder.add_node("check_order", check_order_id)
builder.add_node("ask_order_id", ask_order_id)
builder.add_node("query_order", query_order)
builder.add_node("refund_policy", refund_policy)
builder.add_node("general_response", general_response)
builder.add_node("human_handoff", human_handoff)

# 添加边
builder.add_edge(START, "classify")
builder.add_edge("classify", "check_order")

# 添加条件路由
builder.add_conditional_edges("check_order", route_intent, {
    "ask_order_id": "ask_order_id",
    "query_order": "query_order",
    "refund_policy": "refund_policy",
    "general_response": "general_response",
    "human_handoff": "human_handoff"
})

# 各处理节点最终都结束
builder.add_edge("ask_order_id", END)
builder.add_edge("query_order", END)
builder.add_edge("refund_policy", END)
builder.add_edge("general_response", END)
builder.add_edge("human_handoff", END)

# 编译
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)
```

### 结合 RAG 的知识检索

将知识库检索集成到客服系统中：

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables import RunnablePassthrough
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

# 创建知识库
def build_knowledge_base():
    """构建知识库"""
    documents = [
        "退货政策：我们提供7天无理由退货服务，商品需保持原样。",
        "物流时效：标准配送3-5个工作日，加急配送1-2个工作日。",
        "质量保证：所有商品享受30天质量保证，质量问题免费退换。",
        "支付方式：支持支付宝、微信、银行卡、信用卡。",
        "发票说明：可在订单完成后7天内申请开具电子发票。"
    ]
    
    embeddings = OpenAIEmbeddings()
    vectorstore = FAISS.from_texts(documents, embeddings)
    return vectorstore.as_retriever()

retriever = build_knowledge_base()

def retrieve_knowledge(state: CustomerServiceState):
    """从知识库检索相关信息"""
    query = state["messages"][-1].content
    docs = retriever.invoke(query)
    context = "\n".join([doc.page_content for doc in docs[:2]])
    return {"context": context}

def generate_with_context(state: CustomerServiceState):
    """基于检索到的知识生成回答"""
    context = state.get("context", "暂无相关信息")
    query = state["messages"][-1].content
    
    prompt = ChatPromptTemplate.from_messages([
        ("system", f"""你是电商客服助手。请根据以下知识回答问题。
如果知识中没有，请礼貌地告知用户无法提供相关信息。

知识：{context}"""),
        MessagesPlaceholder(variable_name="messages")
    ])
    
    chain = prompt | model
    response = chain.invoke({"messages": state["messages"]})
    return {"messages": [response]}
```

## 常见问题

### Q1: 如何处理复杂的多轮业务逻辑？

**A**: 将复杂流程拆分为多个节点，通过状态字段追踪当前阶段。例如订单退款可以分为：确认意图 → 收集信息 → 验证条件 → 执行退款 → 确认结果，每个步骤作为一个独立节点。

### Q2: RAG 检索不准确怎么办？

**A**: 三种优化方向：
1. **查询重写**：在检索前用 LLM 优化用户查询
2. **多路检索**：同时使用多种检索策略（关键词+向量）
3. **反馈优化**：收集用户反馈持续优化知识库

### Q3: 如何评估客服系统效果？

**A**: 关注以下指标：
- **意图识别准确率**：分类正确的比例
- **首次解决率**：一轮对话内解决问题的比例
- **人工转接率**：需要人工介入的比例
- **用户满意度**：对话结束后的评分
