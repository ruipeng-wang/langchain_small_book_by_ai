# 15.2 监控与可观测性

## 概念讲解

在开发环境中，我们可以通过打印日志和调试来定位问题。但在生产环境中，系统运行在用户的真实流量下，问题往往难以复现。可观测性（Observability）帮助我们理解系统内部状态，而不需要修改代码或重新部署。

### 可观测性的三大支柱

```
┌──────────────────────────────────────────────────────┐
│                  可观测性                              │
│                                                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │ Metrics  │  │  Traces  │  │  Logs    │            │
│  │  指标    │  │  追踪    │  │  日志    │            │
│  └──────────┘  └──────────┘  └──────────┘            │
│       │              │              │                 │
│       ▼              ▼              ▼                 │
│   系统健康度    请求执行路径    详细事件记录            │
│   聚合统计      性能瓶颈定位    问题根因分析            │
└──────────────────────────────────────────────────────┘
```

### LangSmith 在 LangChain 生态中的角色

LangSmith 是 LangChain 官方提供的可观测性平台，专门为 LLM 应用设计。它能够：

1. **自动追踪**：记录 LLM 调用的输入输出、Token 消耗、响应时间
2. **图可视化**：以图的形式展示 LangGraph 节点执行流程
3. **评估框架**：对 AI 输出质量进行自动化评估
4. **问题标注**：在追踪记录上添加人工标注，持续改进模型

### LangGraph 特有的监控需求

| 监控维度 | 传统应用 | LangGraph 应用 |
|----------|----------|----------------|
| 响应时间 | 接口响应时间 | 每个节点的执行时间 |
| 错误追踪 | HTTP 错误码 | Agent 执行异常、工具调用失败 |
| 成本控制 | 服务器成本 | LLM Token 消耗、API 调用次数 |
| 质量保证 | 单元测试通过率 | AI 输出相关性、准确性 |

## 核心要点

### 1. LangSmith 集成方式

最简单的方式是设置环境变量：

```bash
export LANGSMITH_TRACING=true
export LANGSMITH_API_KEY="your-api-key"
export LANGSMITH_PROJECT="my-production-app"
```

或者在代码中配置：

```python
import langsmith
from langsmith import tracing_context

# 全局配置
langsmith.configure(api_key="your-api-key", project_name="my-app")

# 或者使用上下文管理器选择性追踪
with tracing_context(enabled=True, tags=["production"]):
    result = graph.invoke({"messages": [...]})
```

### 2. 关键监控指标

| 指标类别 | 关键指标 | 告警阈值建议 |
|----------|----------|-------------|
| 性能指标 | 平均响应时间、P95/P99 延迟 | P95 > 10s 告警 |
| 质量指标 | 工具调用成功率、Agent 完成率 | 成功率 < 90% 告警 |
| 成本指标 | 每次请求 Token 消耗、日消费额 | 日消费超限告警 |
| 系统指标 | 内存使用率、Checkpointer 延迟 | 内存 > 80% 告警 |

### 3. 追踪策略

- **全量追踪**：开发测试环境，追踪所有请求
- **采样追踪**：生产环境，按百分比采样（如 10%）
- **异常追踪**：只追踪失败和超时的请求

## 简单示例

### 使用 LangSmith 追踪 LangGraph 执行

```python
import os
from langgraph.graph import StateGraph, MessagesState, START
from langgraph.checkpoint.memory import InMemorySaver
from langchain.chat_models import init_chat_model
from langsmith import Client, traceable

# 配置 LangSmith（设置环境变量）
os.environ["LANGSMITH_TRACING"] = "true"
os.environ["LANGSMITH_API_KEY"] = "your-api-key"
os.environ["LANGSMITH_PROJECT"] = "customer-service-prod"

model = init_chat_model("gpt-4o-mini")

class AgentState(MessagesState):
    intent: str = ""

def classify_intent(state: AgentState):
    """意图识别节点"""
    last_msg = state["messages"][-1].content
    prompt = f"判断意图：{last_msg}，返回：order/refund/other"
    response = model.invoke([{"role": "user", "content": prompt}])
    return {"intent": response.content.strip()}

def handle_request(state: AgentState):
    """处理请求节点"""
    intent = state.get("intent", "other")
    responses = {
        "order": "正在为您查询订单信息...",
        "refund": "正在处理退款事宜...",
        "other": "请问有什么可以帮到您？"
    }
    return {"messages": [{"role": "assistant", "content": responses.get(intent, responses["other"])}]}

# 构建图
builder = StateGraph(AgentState)
builder.add_node("classify", classify_intent)
builder.add_node("handle", handle_request)
builder.add_edge(START, "classify")
builder.add_edge("classify", "handle")

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 执行时自动记录追踪信息
config = {"configurable": {"thread_id": "trace-demo"}}
result = graph.invoke(
    {"messages": [{"role": "user", "content": "我想查一下订单"}]},
    config
)

print(result["messages"][-1].content)
```

### 使用 @traceable 装饰器自定义追踪

```python
from langsmith import traceable, tracing_context

@traceable(run_type="chain", name="custom_intent_classifier")
def classify_with_metadata(user_input: str) -> dict:
    """带元数据的意图分类追踪"""
    model = init_chat_model("gpt-4o-mini")
    response = model.invoke([{"role": "user", "content": f"分类：{user_input}"}])
    return {
        "intent": response.content,
        "model_used": "gpt-4o-mini",
        "input_length": len(user_input)
    }

# 使用上下文添加自定义标签和元数据
with tracing_context(
    tags=["production", "intent-analysis"],
    metadata={"user_id": "user_123", "session_id": "session_456"}
):
    result = classify_with_metadata("我的订单到哪了？")
    print(result)
```

### 选择性追踪

```python
import langsmith as ls

# 这段代码会被追踪
with ls.tracing_context(enabled=True):
```

## 进阶应用

### 1. 自定义回调监控

除了 LangSmith，还可以添加自定义的回调来收集特定指标：

```python
from langchain_core.callbacks import BaseCallbackHandler
from langchain_core.messages import LLMMessage
import time

class MetricsCallbackHandler(BaseCallbackHandler):
    """自定义指标收集回调"""
    
    def __init__(self):
        self.metrics = {
            "llm_calls": 0,
            "total_tokens": 0,
            "total_latency": 0
        }
    
    def on_llm_start(self, serialized, prompts, **kwargs):
        self.start_time = time.time()
    
    def on_llm_end(self, response, **kwargs):
        latency = time.time() - self.start_time
        self.metrics["llm_calls"] += 1
        self.metrics["total_latency"] += latency
        
        if response.llm_output and "token_usage" in response.llm_output:
            tokens = response.llm_output["token_usage"]["total_tokens"]
            self.metrics["total_tokens"] += tokens
    
    def get_average_latency(self):
        if self.metrics["llm_calls"] == 0:
            return 0
        return self.metrics["total_latency"] / self.metrics["llm_calls"]

# 在 LangGraph 中使用
handler = MetricsCallbackHandler()
result = graph.invoke(
    {"messages": [{"role": "user", "content": "你好"}]},
    config={"callbacks": [handler]}
)
print(f"平均延迟: {handler.get_average_latency():.2f}s")
```

### 2. 生产环境告警

结合 Prometheus 等工具，将 LangGraph 运行指标导出到监控平台：

```python
from prometheus_client import Counter, Histogram
import time

REQUEST_COUNT = Counter("langgraph_requests_total", "Total requests", ["status"])
REQUEST_DURATION = Histogram("langgraph_duration_seconds", "Request duration")

async def monitored_invoke(graph, input_data, config):
    start = time.time()
    try:
        result = await graph.ainvoke(input_data, config)
        REQUEST_COUNT.labels(status="success").inc()
        return result
    except Exception as e:
        REQUEST_COUNT.labels(status="error").inc()
        raise
    finally:
        REQUEST_DURATION.observe(time.time() - start)
```

## 常见问题

### Q1: LangSmith 追踪会影响性能吗？

**A**: 追踪是异步的，对主流程影响很小（通常 < 10ms）。但如果网络不稳定，可以设置 `LANGSMITH_TRACING=false` 关闭追踪。

### Q2: 如何追踪 LangGraph 的每个节点执行？

**A**: 设置 `LANGSMITH_TRACING=true` 后，LangGraph 的每个节点执行都会自动记录到 LangSmith，包括节点名称、输入输出、执行时间。在 LangSmith 的 Trace 页面可以看到完整的执行树。

### Q3: 如何在生产环境控制追踪数据量？

**A**: 使用 `tracing_context(enabled=...)` 控制追踪范围，或设置采样率。也可以在 LangSmith 控制台配置数据保留策略。

### Q4: LangSmith 和其他监控工具如何配合？

**A**: LangSmith 专注于 LLM 应用层面的追踪。系统层面（CPU、内存）仍使用 Prometheus/Grafana 等传统工具。两者互补，共同构成完整的可观测性体系。

## 本节总结

LangGraph 应用的可观测性核心要点：

1. **LangSmith 集成**：设置环境变量即可启用自动追踪
2. **图可视化**：自动展示 LangGraph 节点执行流程
3. **选择性追踪**：使用 `tracing_context` 控制追踪范围
4. **自定义回调**：收集特定的业务指标
5. **质量评估**：自动化评估 AI 输出质量

可观测性不是事后的附加功能，而是应该在开发初期就设计好的基础设施。
