# 15.3 安全与成本控制

## 概念讲解

将 LangGraph 应用部署到生产环境，安全和成本是两个不可忽视的方面。LLM 应用有其独特的安全挑战和成本特征，需要专门的防护和优化策略。

### 安全挑战

LLM 应用面临的独特安全威胁：

```
┌─────────────────────────────────────────────────────┐
│                LLM 安全威胁矩阵                       │
│                                                     │
│  提示词注入    │  数据泄露    │  模型滥用              │
│  用户通过精心  │  敏感信息    │  恶意用户消耗大量      │
│  设计的输入绕  │  在提示词或  │  Token 造成成本损失    │
│  过安全限制    │  输出中暴露  │                      │
│               │             │                      │
│  越权访问      │  输出污染    │  供应链风险            │
│  通过对话获取  │  模型产生    │  第三方工具/插件        │
│  未授权的数据  │  有害或偏见  │  可能引入漏洞          │
│  或执行操作    │  的内容      │                      │
└─────────────────────────────────────────────────────┘
```

### 成本挑战

LLM API 调用成本是 LangGraph 应用的主要运营成本：

| 成本因素 | 说明 | 优化方向 |
|----------|------|----------|
| Token 数量 | 输入输出 Token 总数 | 精简提示词、压缩上下文 |
| 模型选择 | 不同模型定价差异大 | 场景化选择合适模型 |
| 调用频率 | 用户请求量 | 缓存、批量处理 |
| 工具调用 | Agent 多步工具调用 | 减少不必要的调用 |

## 核心要点

### 1. 输入安全防护

对用户输入进行验证和过滤，防止提示词注入：

```python
import re

def validate_input(user_input: str) -> dict:
    """输入验证"""
    result = {
        "valid": True,
        "sanitized": user_input,
        "issues": []
    }
    
    # 检查长度限制
    if len(user_input) > 2000:
        result["valid"] = False
        result["issues"].append("输入过长")
    
    # 检测潜在的提示词注入模式
    injection_patterns = [
        r"ignore\s+previous",
        r"forget\s+all",
        r"system\s*:",
        r"as\s+an\s+ai",
        r"dan\s+mode",
    ]
    
    for pattern in injection_patterns:
        if re.search(pattern, user_input, re.IGNORECASE):
            result["valid"] = False
            result["issues"].append("检测到潜在注入尝试")
            break
    
    return result
```

### 2. 输出安全过滤

对模型输出进行检查，防止敏感信息泄露：

```python
import re

def sanitize_output(model_output: str) -> str:
    """输出过滤"""
    # 移除潜在的敏感信息模式
    patterns = [
        (r'(?:api[_-]?key|secret|token)\s*[:=]\s*\S+', '[REDACTED]'),
        (r'\b\d{3}-\d{2}-\d{4}\b', '[SSN REDACTED]'),  # 社保号
        (r'\b(?:\d{4}[-\s]?){3}\d{4}\b', '[CARD REDACTED]'),  # 信用卡
    ]
    
    sanitized = model_output
    for pattern, replacement in patterns:
        sanitized = re.sub(pattern, replacement, sanitized, flags=re.IGNORECASE)
    
    return sanitized
```

### 3. 成本控制策略

| 策略 | 实现方式 | 预期节省 |
|------|----------|----------|
| 响应缓存 | 相同问题返回缓存结果 | 30-50% |
| 模型降级 | 简单问题用便宜模型 | 40-70% |
| 提示词优化 | 精简不必要的提示词 | 10-20% |
| 批量处理 | 合并相似请求 | 15-30% |

### 智能路由：根据复杂度选择模型

```python
from langchain.chat_models import init_chat_model

# 初始化不同成本的模型
cheap_model = init_chat_model("gpt-4o-mini")   # 低成本
expensive_model = init_chat_model("gpt-4o")    # 高质量

def estimate_complexity(user_input: str) -> str:
    """估算问题复杂度"""
    complex_indicators = [
        "分析", "比较", "评价", "解释", "为什么",
        "how", "analyze", "compare", "evaluate"
    ]
    
    score = 0
    for indicator in complex_indicators:
        if indicator in user_input.lower():
            score += 1
    
    if score >= 2:
        return "complex"
    elif len(user_input) > 100:
        return "complex"
    else:
        return "simple"

def smart_response(user_input: str) -> str:
    """智能响应：根据复杂度选择模型"""
    complexity = estimate_complexity(user_input)
    
    if complexity == "complex":
        model = expensive_model
    else:
        model = cheap_model
    
    response = model.invoke([{"role": "user", "content": user_input}])
    return response.content
```

### Token 使用监控

```python
from langchain_core.callbacks import BaseCallbackHandler

class TokenUsageTracker(BaseCallbackHandler):
    """Token 使用追踪器"""
    
    def __init__(self, budget_limit: float = 10.0):
        self.total_tokens = 0
        self.total_cost = 0.0
        self.budget_limit = budget_limit
        # 各模型每 1000 Token 的价格（示例）
        self.pricing = {
            "gpt-4o-mini": 0.00015,
            "gpt-4o": 0.0025,
        }
    
    def on_llm_end(self, response, **kwargs):
        if response.llm_output and "token_usage" in response.llm_output:
            tokens = response.llm_output["token_usage"]["total_tokens"]
            model = response.llm_output.get("model_name", "unknown")
            
            self.total_tokens += tokens
            cost = tokens * self.pricing.get(model, 0.001) / 1000
            self.total_cost += cost
            
            # 检查预算
            if self.total_cost > self.budget_limit:
                raise BudgetExceededException(
                    f"预算已超出: ${self.total_cost:.2f} > ${self.budget_limit:.2f}"
                )

class BudgetExceededException(Exception):
    """预算超出异常"""
    pass

# 使用示例
tracker = TokenUsageTracker(budget_limit=5.0)
try:
    result = graph.invoke(
        {"messages": [{"role": "user", "content": "你好"}]},
        config={"callbacks": [tracker]}
    )
except BudgetExceededException as e:
    print(f"预算告警: {e}")
```

### 2. 缓存优化

```python
import hashlib
from functools import lru_cache

class ResponseCache:
    """响应缓存：减少重复请求的成本"""
    
    def __init__(self, ttl: int = 3600):
        self.cache = {}
        self.ttl = ttl
    
    def _get_key(self, user_input: str) -> str:
        """生成缓存键"""
        return hashlib.md5(user_input.lower().encode()).hexdigest()
    
    def get(self, user_input: str) -> str | None:
        """获取缓存"""
        key = self._get_key(user_input)
        if key in self.cache:
            entry = self.cache[key]
            import time
            if time.time() - entry["timestamp"] < self.ttl:
                return entry["response"]
            else:
                del self.cache[key]
        return None
    
    def set(self, user_input: str, response: str):
        """设置缓存"""
        key = self._get_key(user_input)
        import time
        self.cache[key] = {
            "response": response,
            "timestamp": time.time()
        }
    
    def clear(self):
        """清空缓存"""
        self.cache.clear()

# 在 LangGraph 中使用
cache = ResponseCache(ttl=1800)

def cached_chat(state: MessagesState):
    user_input = state["messages"][-1].content
    
    # 检查缓存
    cached_response = cache.get(user_input)
    if cached_response:
        return {"messages": [{"role": "assistant", "content": cached_response}]}
    
    # 调用模型
    model = init_chat_model("gpt-4o-mini")
    response = model.invoke(state["messages"])
    
    # 存储缓存
    cache.set(user_input, response.content)
    
    return {"messages": [response]}
```

### 3. 成本监控仪表盘

结合 LangSmith 和自定义监控，建立完整的成本视图：

```python
class CostDashboard:
    """成本监控仪表盘"""
    
    def __init__(self):
        self.daily_usage = {}
    
    def record_usage(self, date: str, model: str, tokens: int, cost: float):
        """记录使用情况"""
        if date not in self.daily_usage:
            self.daily_usage[date] = {
                "models": {},
                "total_cost": 0,
                "total_tokens": 0
            }
        
        if model not in self.daily_usage[date]["models"]:
            self.daily_usage[date]["models"][model] = {
                "tokens": 0, "cost": 0
            }
        
        self.daily_usage[date]["models"][model]["tokens"] += tokens
        self.daily_usage[date]["models"][model]["cost"] += cost
        self.daily_usage[date]["total_tokens"] += tokens
        self.daily_usage[date]["total_cost"] += cost
    
    def get_daily_report(self, date: str) -> dict:
        """获取日报"""
        return self.daily_usage.get(date, {})
    
    def get_alerts(self, daily_budget: float) -> list:
        """生成告警"""
        import datetime
        today = str(datetime.date.today())
        usage = self.daily_usage.get(today, {})
        
        alerts = []
        if usage.get("total_cost", 0) > daily_budget * 0.8:
            alerts.append("⚠️ 已使用日预算的 80%")
        if usage.get("total_cost", 0) > daily_budget:
            alerts.append("🚨 已超出日预算!")
        
        return alerts
```

## 常见问题

### Q1: 如何防止用户通过提示词注入获取系统权限？

**A**: 多层防护：
1. 输入验证：检测常见的注入模式
2. 系统提示词保护：使用 `{"type": "system"}` 消息类型，部分模型对系统消息有保护
3. 输出过滤：检查模型输出是否包含敏感信息
4. 最小权限原则：Agent 只拥有必要的工具权限

### Q2: 如何有效控制 LLM API 成本？

**A**: 综合策略：
1. **缓存**：相同问题返回缓存结果
2. **模型分级**：简单问题用便宜模型，复杂问题用高质量模型
3. **提示词精简**：去除不必要的提示词
4. **限流**：控制每秒/每分钟请求数
5. **预算告警**：设置预算上限，超出时自动降级或告警

### Q3: 如何保护用户隐私数据？

**A**: 
1. 不在提示词中传递敏感信息（如密码、身份证号）
2. 如需使用，在传递前进行脱敏处理
3. 使用私有部署的模型或支持数据隐私保护的 API
4. 定期清理日志和追踪数据中的敏感信息

## 本节总结

安全与成本控制是 LangGraph 生产部署的重要保障：

1. **输入验证**：防止提示词注入和恶意输入
2. **输出过滤**：防止敏感信息泄露
3. **智能路由**：根据复杂度选择模型，平衡成本和质量
4. **缓存优化**：减少重复请求的 Token 消耗
5. **预算监控**：实时追踪成本，及时告警

安全和成本优化不是一次性工作，需要持续监控和改进。
