# 13.2 高级检索策略

## 概念讲解

### 为什么需要高级检索？

基础相似度搜索在实际应用中常常遇到以下问题：
1. **检索不精准**：简单向量相似度可能忽略语义细节
2. **上下文丢失**：分块检索可能丢失全局信息
3. **查询理解不足**：单一查询可能无法覆盖所有相关信息

### 高级检索策略概览

| 策略 | 原理 | 适用场景 |
|------|------|----------|
| 上下文压缩 | 检索后重排序/压缩 | 需要高精度回答 |
| 多查询检索 | LLM生成多个查询角度 | 复杂问题理解 |
| 父子文档 | 检索小片段，返回大上下文 | 保持上下文完整性 |
| 混合检索 | 向量+关键词组合 | 专业术语查询 |

## 核心要点

### ContextualCompressionRetriever

LangChain的上下文压缩检索器，在检索后进行重排序和压缩：

```python
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import FlashrankRerank

# 压缩器（重排序）
compressor = FlashrankRerank()

# 压缩检索器
compression_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=vectorstore.as_retriever()
)
```

### MultiQueryRetriever

使用LLM从不同角度生成多个查询，提升召回率：

```python
from langchain.retrievers.multi_query import MultiQueryRetriever

multi_query_retriever = MultiQueryRetriever.from_llm(
    retriever=vectorstore.as_retriever(),
    llm=llm
)
```

## 简单示例

### 上下文压缩检索

```python
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import FlashrankRerank

# 基础检索器
base_retriever = vectorstore.as_retriever(search_kwargs={"k": 10})

# 重排序压缩器
compressor = FlashrankRerank()

# 压缩检索器
compression_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=base_retriever
)

# 使用 - 返回最相关的压缩文档
docs = compression_retriever.invoke("LangChain的核心组件是什么？")
```

### 多查询检索

```python
from langchain.retrievers.multi_query import MultiQueryRetriever
from langchain.chat_models import init_chat_model

llm = init_chat_model("gpt-4o-mini")

# 多查询检索器
multi_query_retriever = MultiQueryRetriever.from_llm(
    retriever=vectorstore.as_retriever(),
    llm=llm
)

# 构建RAG链
rag_chain = (
    {"context": multi_query_retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

result = rag_chain.invoke("LangChain在企业中有哪些应用场景？")
```

## 进阶应用

### 混合检索（向量+关键词）

```python
from langchain.retrievers import EnsembleRetriever
from langchain.retrievers import BM25Retriever

# BM25关键词检索
bm25_retriever = BM25Retriever.from_texts(texts)
bm25_retriever.k = 5

# 向量检索
vector_retriever = vectorstore.as_retriever(search_kwargs={"k": 5})

# 组合检索器
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.3, 0.7]  # 权重
)
```

### 父子文档检索

```python
from langchain.retrievers import ParentDocumentRetriever
from langchain.storage import InMemoryStore
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 父文档分割器（大块）
parent_splitter = RecursiveCharacterTextSplitter(chunk_size=2000)
# 子文档分割器（小块）
child_splitter = RecursiveCharacterTextSplitter(chunk_size=400)

# 存储
store = InMemoryStore()

# 父子文档检索器
retriever = ParentDocumentRetriever(
    vectorstore=vectorstore,
    docstore=store,
    child_splitter=child_splitter,
    parent_splitter=parent_splitter,
)

# 添加文档 - 自动分块
retriever.add_documents(documents)

# 检索 - 返回父文档（完整上下文）
docs = retriever.invoke("查询")
```

## 常见问题

### Q: 什么时候使用压缩检索？

**A:** 当检索结果数量多且质量参差时，使用重排序提升精准度。

### Q: 多查询检索会增加成本吗？

**A:** 会，因为需要LLM生成多个查询。可以通过缓存查询结果来优化。

## 本节总结

高级检索策略：
- `ContextualCompressionRetriever`：检索后重排序
- `MultiQueryRetriever`：多角度查询提升召回率
- `EnsembleRetriever`：向量+关键词混合检索
- `ParentDocumentRetriever`：父子文档保持上下文
