# 13.1.1 RAG架构概述

## 概念讲解

### 什么是RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合检索和生成的AI架构。它的核心思想是：在模型生成回答之前，先从外部知识库检索相关信息，然后将检索结果作为上下文提供给模型。

### 为什么需要RAG

大语言模型存在三个固有限制：

| 限制 | 说明 | RAG如何解决 |
|------|------|------------|
| 知识时效性 | 训练数据有截止时间 | 检索最新信息 |
| 幻觉问题 | 可能生成不准确的内容 | 基于事实回答 |
| 私有知识 | 无法访问企业内部数据 | 接入私有知识库 |

### RAG的核心流程

```
用户问题
    ↓
[检索器] → 从知识库找到相关片段
    ↓
[增强提示] → 问题 + 检索结果组合
    ↓
[生成器] → LLM基于上下文回答
    ↓
最终回答
```

## 核心要点

1. **检索（Retrieval）**：从知识库中找出与问题相关的文档片段
2. **增强（Augmentation）**：将检索结果与问题组合成提示
3. **生成（Generation）**：LLM基于增强后的提示生成回答
4. **三阶段解耦**：检索和生成可以独立优化

## 简单示例

### 最小RAG实现

```python
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_community.vectorstores import FAISS
from langchain_core.prompts import ChatPromptTemplate

# 1. 检索
embeddings = OpenAIEmbeddings()
vectorstore = FAISS.load_local("knowledge_base", embeddings)

def retrieve(query: str, k: int = 3) -> str:
    docs = vectorstore.similarity_search(query, k=k)
    return "\n".join(doc.page_content for doc in docs)

# 2. 增强 + 3. 生成
prompt = ChatPromptTemplate.from_messages([
    ("system", "基于以下上下文回答问题：\n{context}"),
    ("human", "{question}")
])

def rag(query: str) -> str:
    context = retrieve(query)
    chain = prompt | ChatOpenAI(model="gpt-4o-mini")
    return chain.invoke({"context": context, "question": query}).content
```

## 常见问题

**Q: RAG和微调有什么区别？**

A: RAG通过检索外部信息增强生成，适合知识更新和私有数据；微调改变模型权重，适合领域适配。

**Q: 检索质量如何保证？**

A: 关键在文档分块策略和嵌入模型选择。分块太大会丢失细节，太小会丢失上下文。

**Q: 如何处理检索不到的情况？**

A: 设置相关性阈值，低于阈值的文档不加入上下文，并告知模型"未找到相关信息"。

## 本节总结

RAG通过检索外部知识库增强LLM的生成能力，有效解决了知识时效性、幻觉和私有数据等问题。它由检索、增强、生成三个阶段组成，是当前企业级AI应用的核心架构。
