# 11.1.3 图遍历与路径搜索

## 概念讲解

图遍历与路径搜索是LangGraph图计算引擎的核心算法，负责在复杂的工作流图中寻找最优执行路径、检测循环依赖、优化执行顺序等。LangGraph v1.2.22提供了多种高级图算法，帮助开发者构建高效、可靠的工作流系统。

### 图遍历算法

LangGraph支持以下图遍历算法：

1. **广度优先搜索（BFS）**：按层次遍历图，适用于最短路径查找
2. **深度优先搜索（DFS）**：深入遍历分支，适用于拓扑排序和循环检测
3. **优先级遍历（Priority Traversal）**：基于权重的最优路径搜索
4. **并行遍历（Parallel Traversal）**：多路径并发探索

### 路径搜索策略

在复杂工作流中，LangGraph使用智能路径搜索策略：

1. **最短路径搜索**：寻找节点间的最少步骤路径
2. **最优权重路径**：基于权重成本的最优路径
3. **条件路径搜索**：满足特定条件的路径查找
4. **动态路径规划**：运行时根据状态调整路径

### 循环检测与处理

LangGraph内置了循环检测机制，防止工作流陷入无限循环：

1. **静态循环检测**：编译时检测明显的循环结构
2. **动态循环检测**：运行时监测执行次数和状态变化
3. **循环中断策略**：最大迭代次数、超时控制、状态变化监测

## 核心要点

- **图遍历是工作流执行的基础**：决定了节点的执行顺序
- **路径搜索优化执行效率**：寻找最优执行路径，减少不必要的节点执行
- **循环检测保障系统可靠性**：防止无限循环和资源耗尽
- **权重感知的路径规划**：考虑节点成本、优先级等权重因素
- **动态路径调整**：根据运行时状态调整执行路径

## 简单示例

### 基础图遍历示例

```python
# 示例：工作流图的遍历与路径搜索
from typing import TypedDict, List
from langgraph.graph import StateGraph, END, START
from langgraph.algorithms import find_shortest_path, detect_cycles

# 1. 定义状态模式
class TraversalState(TypedDict):
    """图遍历状态"""
    current_path: List[str]  # 当前执行路径
    visited_nodes: List[str]  # 已访问节点
    execution_cost: float  # 执行成本

# 2. 构建复杂工作流图
workflow = StateGraph(TraversalState)

# 添加多个节点
def node_a(state: TraversalState):
    """节点A"""
    path = state.get("current_path", []) + ["A"]
    visited = state.get("visited_nodes", []) + ["A"]
    return {"current_path": path, "visited_nodes": visited, "execution_cost": state.get("execution_cost", 0) + 1}

def node_b(state: TraversalState):
    """节点B"""
    path = state.get("current_path", []) + ["B"]
    visited = state.get("visited_nodes", []) + ["B"]
    return {"current_path": path, "visited_nodes": visited, "execution_cost": state.get("execution_cost", 0) + 2}

def node_c(state: TraversalState):
    """节点C"""
    path = state.get("current_path", []) + ["C"]
    visited = state.get("visited_nodes", []) + ["C"]
    return {"current_path": path, "visited_nodes": visited, "execution_cost": state.get("execution_cost", 0) + 3}

def node_d(state: TraversalState):
    """节点D"""
    path = state.get("current_path", []) + ["D"]
    visited = state.get("visited_nodes", []) + ["D"]
    return {"current_path": path, "visited_nodes": visited, "execution_cost": state.get("execution_cost", 0) + 1}

# 添加节点到工作流
workflow.add_node("A", node_a)
workflow.add_node("B", node_b)
workflow.add_node("C", node_c)
workflow.add_node("D", node_d)

# 构建复杂图结构
workflow.add_edge(START, "A")
workflow.add_edge("A", "B")
workflow.add_edge("A", "C")  # 分支路径
workflow.add_edge("B", "D")
workflow.add_edge("C", "D")
workflow.add_edge("D", END)

# 3. 编译工作流
graph = workflow.compile()

# 4. 执行图遍历
initial_state = {
    "current_path": [],
    "visited_nodes": [],
    "execution_cost": 0
}

print("=== 执行图遍历 ===")
result = graph.invoke(initial_state)
print(f"最终路径: {result['current_path']}")
print(f"执行成本: {result['execution_cost']}")
print(f"访问节点: {result['visited_nodes']}")

# 5. 路径搜索示例
print("\n=== 路径搜索分析 ===")
# 查找从START到END的所有可能路径
def enumerate_paths():
    """枚举所有可能路径"""
    paths = [
        ["A", "B", "D"],  # 路径1: A→B→D
        ["A", "C", "D"]   # 路径2: A→C→D
    ]
    
    # 计算各路径成本
    for i, path in enumerate(paths, 1):
        cost = 0
        if "A" in path: cost += 1
        if "B" in path: cost += 2
        if "C" in path: cost += 3
        if "D" in path: cost += 1
        print(f"路径{i}: {path}, 成本: {cost}")
    
    # 返回最优路径
    optimal_path = min(paths, key=lambda p: (
        1 if "A" in p else 0 +
        2 if "B" in p else 0 +
        3 if "C" in p else 0 +
        1 if "D" in p else 0
    ))
    print(f"最优路径: {optimal_path}")

enumerate_paths()
```

### 循环检测与处理示例

```python
# 示例：循环检测与处理
from langgraph.graph import StateGraph, START
from langgraph.checkpoint.memory import MemorySaver

class LoopState(TypedDict):
    """循环检测状态"""
    iteration_count: int
    max_iterations: int
    should_stop: bool

def loop_node(state: LoopState):
    """可能产生循环的节点"""
    count = state.get("iteration_count", 0) + 1
    max_iter = state.get("max_iterations", 5)
    
    # 检查是否达到最大迭代次数
    if count >= max_iter:
        return {
            "iteration_count": count,
            "should_stop": True
        }
    
    # 模拟一些处理逻辑
    print(f"迭代 {count}/{max_iter}")
    
    return {
        "iteration_count": count,
        "should_stop": False
    }

def should_continue(state: LoopState) -> str:
    """决定是否继续循环"""
    if state.get("should_stop", False):
        return "stop"
    else:
        return "continue"

# 构建带循环的工作流
loop_workflow = StateGraph(LoopState)
loop_workflow.add_node("process", loop_node)

# 添加条件边实现循环
loop_workflow.add_edge(START, "process")
loop_workflow.add_conditional_edges(
    "process",
    should_continue,
    {
        "continue": "process",  # 循环回到自身
        "stop": END
    }
)

# 配置检查点（支持循环中的状态持久化）
memory = MemorySaver()
loop_graph = loop_workflow.compile(checkpointer=memory)

print("\n=== 循环执行演示 ===")
initial_loop_state = {
    "iteration_count": 0,
    "max_iterations": 3,
    "should_stop": False
}

result = loop_graph.invoke(initial_loop_state)
print(f"循环执行完成，总迭代次数: {result['iteration_count']}")
```

from typing import Optional
from dataclasses import dataclass

@dataclass
class PathMetrics:
    """路径度量指标"""
    execution_time: float
    resource_usage: float
    success_rate: float
    cost: float

def calculate_path_score(metrics: PathMetrics) -> float:
    """计算路径综合得分"""
    # 加权综合评分
    weights = {
        "time": 0.3,
        "resource": 0.2,
        "success": 0.4,
        "cost": 0.1
    }
    
    # 归一化处理（这里简化处理）
    score = (
        (1 / max(metrics.execution_time, 0.1)) * weights["time"] +
        (1 / max(metrics.resource_usage, 0.1)) * weights["resource"] +
        metrics.success_rate * weights["success"] +
        (1 / max(metrics.cost, 0.1)) * weights["cost"]
    )
    
    return score

def select_optimal_path(path_options: List[str], state: dict) -> str:

# 智能循环优化
class SmartLoopController:
    """智能循环控制器"""
    
    def __init__(self, max_iterations: int = 10, timeout: float = 30.0):
        self.max_iterations = max_iterations
        self.timeout = timeout
        self.iteration_history = []
    
    def should_continue(self, state: dict, start_time: float) -> bool:
        """智能判断是否继续循环"""
        import time
        
        # 检查迭代次数
        if len(self.iteration_history) >= self.max_iterations:
            print(f"达到最大迭代次数: {self.max_iterations}")
            return False
        
        # 检查超时
        if time.time() - start_time > self.timeout:
            print(f"循环超时: {self.timeout}秒")
            return False
        
        # 检查状态收敛（连续3次状态变化小于阈值）
        if len(self.iteration_history) >= 3:
            recent_changes = self.iteration_history[-3:]
            if max(recent_changes) - min(recent_changes) < 0.01:
                print("状态已收敛，停止循环")
                return False
        
        return True
    
    def record_iteration(self, state_value: float):
        """记录迭代状态"""
        self.iteration_history.append(state_value)

## 常见问题

### Q1: LangGraph如何避免无限循环？
**A**: LangGraph通过三种机制避免无限循环：1) 编译时的静态循环检测；2) 运行时的最大迭代次数限制；3) 检查点机制支持手动中断和恢复。

### Q2: 如何优化工作流的执行路径？
**A**: 可以使用权重系统定义节点成本，让LangGraph自动选择最优路径。也可以使用条件边实现动态路径选择。

### Q3: 图遍历算法对性能有什么影响？
**A**: 简单的BFS/DFS算法对小型图影响很小。对于大型复杂图，建议使用优先级遍历或预计算的路径缓存。

### Q4: 如何调试图遍历问题？
**A**: 可以使用LangSmith可视化工具查看执行路径，或在节点中添加日志输出。也可以使用检查点机制分析中间状态。

### Q5: 循环检测会影响正常的工作流循环吗？
**A**: 不会。LangGraph能区分"有意义的循环"（如迭代处理）和"有害的无限循环"。开发者可以设置合理的循环边界条件。
