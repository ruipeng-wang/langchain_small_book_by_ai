# 11.1.1 有向图与状态机

## 概念讲解

LangGraph图计算引擎的核心建立在有向图（Directed Graph）和状态机（State Machine）两大基础概念之上。有向图是一种由节点（Nodes）和边（Edges）组成的图结构，其中边具有方向性，表示数据或控制流的传递方向。在LangGraph中，每个节点代表一个具体的处理函数（如调用模型、执行工具等），边则定义了节点之间的执行顺序和条件转移路径。

状态机是LangGraph工作流的运行时表现形式，它维护着工作流的当前状态（State），并根据状态变化驱动节点间的转移。LangGraph的StateGraph将两者完美结合，形成了一种**声明式的工作流编排范式**——开发者只需定义状态模式、节点函数和边条件，LangGraph会自动处理状态传递、条件分支、循环控制等复杂逻辑。

### 有向图在LangGraph中的应用

1. **节点（Node）**：表示工作流中的一个处理单元，可以是：
   - 模型调用节点：执行LLM推理
   - 工具调用节点：执行外部工具操作
   - 条件判断节点：根据状态决定下一步走向
   - 数据处理节点：对状态进行转换或增强

2. **边（Edge）**：定义节点间的连接关系，分为：
   - 无条件边（Unconditional Edge）：总是执行的下一个节点
   - 条件边（Conditional Edge）：根据条件决定下一个节点
   - 开始边（START → Node）：工作流的入口
   - 结束边（Node → END）：工作流的出口

### 状态机的工作原理

LangGraph的状态机基于**状态快照**和**检查点**机制工作：
- 每次节点执行后，系统会生成状态快照
- 检查点机制允许工作流在任意时刻暂停和恢复
- 状态是可序列化的，支持持久化存储
- 支持并发执行和状态共享（在多代理场景中）

## 核心要点

- **有向图是LangGraph的结构基础**：定义了工作流的静态结构
- **状态机是LangGraph的执行引擎**：驱动工作流的动态执行
- **StateGraph是两者的统一抽象**：简化了复杂工作流的编排
- **状态是可序列化的数据结构**：支持检查点、持久化和恢复
- **边条件支持复杂控制流**：实现分支、循环、条件跳转等逻辑

## 简单示例

下面是一个简单的LangGraph有向图状态机示例，演示了基本的节点和边定义：

```python
# 示例：简单的对话状态机
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, END, START
from langgraph.checkpoint.memory import MemorySaver

# 1. 定义状态模式
class ChatState(TypedDict):
    """聊天状态定义"""
    messages: list  # 消息历史
    user_intent: Literal["greeting", "question", "farewell", "unknown"]  # 用户意图
    response: str  # 模型响应

# 2. 定义节点函数
def analyze_intent(state: ChatState):
    """分析用户意图的节点"""
    last_message = state["messages"][-1] if state["messages"] else ""
    
    # 简单的意图分析逻辑
    if "hello" in last_message.lower():
        intent = "greeting"
    elif "?" in last_message:
        intent = "question"
    elif "bye" in last_message.lower():
        intent = "farewell"
    else:
        intent = "unknown"
    
    return {"user_intent": intent}

def generate_response(state: ChatState):
    """生成响应的节点"""
    intent = state["user_intent"]
    
    # 根据意图生成响应
    if intent == "greeting":
        response = "Hello! How can I help you today?"
    elif intent == "question":
        response = "I'd be happy to help with your question."
    elif intent == "farewell":
        response = "Goodbye! Have a great day!"
    else:
        response = "I'm not sure how to respond to that."
    
    return {"response": response}

def log_conversation(state: ChatState):
    """记录对话的节点"""
    print(f"Intent: {state['user_intent']}, Response: {state['response']}")
    return {}

# 3. 构建有向图状态机
workflow = StateGraph(ChatState)

# 添加节点
workflow.add_node("analyze", analyze_intent)
workflow.add_node("generate", generate_response)
workflow.add_node("log", log_conversation)

# 添加边（定义执行流程）
workflow.add_edge(START, "analyze")  # 从开始到意图分析
workflow.add_edge("analyze", "generate")  # 从分析到生成响应
workflow.add_edge("generate", "log")  # 从生成到记录
workflow.add_edge("log", END)  # 从记录到结束

# 4. 编译并配置检查点
memory = MemorySaver()
graph = workflow.compile(checkpointer=memory)

# 5. 执行工作流
initial_state = {
    "messages": ["Hello, how are you?"],
    "user_intent": "unknown",
    "response": ""
}

result = graph.invoke(initial_state)
print(f"Final state: {result}")
```

## 进阶应用

### 多分支状态机设计

对于更复杂的业务场景，可以设计支持多分支的状态机：

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, END

class DecisionState(TypedDict):
    """决策状态"""
    input_data: dict
    decision_path: list[str]
    result: Annotated[list, lambda left, right: left + right]

def decision_node(state: DecisionState):
    """决策节点示例"""
    # 根据输入数据决定分支
    if state["input_data"].get("priority") == "high":
        return {"decision_path": ["urgent_processing"]}
    else:
        return {"decision_path": ["normal_processing"]}
```

### 有向图的循环控制

LangGraph支持通过条件边实现循环：

```python
# 循环控制示例
def should_continue(state: dict) -> str:
    """决定是否继续循环的条件函数"""
    if state.get("iteration_count", 0) < 3:
        return "continue"
    else:
        return "end"

# 添加条件边
workflow.add_conditional_edges(
    "processing_node",
    should_continue,
    {
        "continue": "processing_node",  # 循环回到自身
        "end": END
    }
)
```

## 常见问题

### Q1: 有向图和状态机有什么区别？
**A**: 有向图是静态结构，描述节点间的关系；状态机是动态执行，管理状态转移。LangGraph的StateGraph将两者结合，用有向图定义结构，用状态机管理执行。

### Q2: 如何设计复杂的控制流？
**A**: 可以使用条件边（`add_conditional_edges`）实现分支，使用循环边实现迭代，使用嵌套图实现层次化工作流。

### Q3: 状态持久化有什么好处？
**A**: 状态持久化支持：1）长时间运行工作流的暂停/恢复；2）故障恢复；3）审计和调试；4）多实例间状态共享。

### Q4: 如何调试LangGraph工作流？
**A**: 可以使用LangSmith进行可视化调试，或通过检查点机制查看中间状态。也可以在节点函数中添加日志输出。

## 本节总结

本节介绍了LangGraph图计算引擎的两个核心基础概念：有向图和状态机。有向图提供了工作流的静态结构描述，状态机提供了动态执行引擎。通过StateGraph抽象，开发者可以以声明式的方式编排复杂工作流，而无需关注底层状态传递和控制流细节。

关键收获：
1. LangGraph的有向图由节点和边组成，支持复杂的控制流模式
2. 状态机管理着工作流的运行时状态和转移逻辑
3. StateGraph统一了两者，提供了简洁的API
4. 检查点机制支持状态持久化和工作流恢复
5. 条件边和循环边支持实现复杂业务逻辑
