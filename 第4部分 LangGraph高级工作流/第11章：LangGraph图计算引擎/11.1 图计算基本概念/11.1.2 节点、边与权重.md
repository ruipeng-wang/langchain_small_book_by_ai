# 11.1.2 节点、边与权重

## 概念讲解

在LangGraph图计算引擎中，**节点（Node）**、**边（Edge）**和**权重（Weight）**是构建工作流的三个核心要素。它们共同定义了工作流的执行逻辑、数据流向和优先级控制。

### 节点（Node）

节点是工作流中的基本处理单元，每个节点对应一个具体的处理函数。在LangGraph中，节点不仅仅是简单的函数调用，它还包含以下特性：

1. **状态感知**：节点函数接收当前状态作为输入，返回状态更新
2. **副作用管理**：节点可以执行有副作用的操作（如调用外部API、写入数据库）
3. **错误处理**：节点可以定义自己的错误处理策略
4. **并发支持**：多个节点可以并行执行（通过并行边定义）

### 边（Edge）

边定义了节点间的连接关系，控制着工作流的执行顺序。LangGraph支持多种边类型：

1. **无条件边（Unconditional Edge）**：总是执行的下一个节点
2. **条件边（Conditional Edge）**：根据条件决定下一个节点
3. **开始边（START → Node）**：工作流的入口
4. **结束边（Node → END）**：工作流的出口
5. **并行边（Parallel Edge）**：多个节点并行执行

### 权重（Weight）

权重是LangGraph v1.2.22引入的高级特性，用于控制边的优先级和执行概率。权重系统支持：
- **优先级权重**：决定并行边中节点的执行顺序
- **概率权重**：在条件边中决定分支的选择概率
- **成本权重**：考虑执行成本的路由决策
- **资源权重**：基于资源可用性的动态调度

## 核心要点

- **节点是工作流的处理单元**：每个节点封装特定业务逻辑
- **边是工作流的控制流**：定义了节点的执行顺序和条件转移
- **权重是高级调度策略**：支持优先级、概率、成本等复杂调度逻辑
- **节点函数必须返回状态更新**：遵循函数式编程范式
- **边条件支持复杂业务逻辑**：实现动态工作流路由
- **权重系统支持智能调度**：优化工作流执行效率和资源利用率

## 简单示例

### 基础节点和边定义

```python
# 示例：订单处理工作流
from typing import TypedDict
from langgraph.graph import StateGraph, END, START

# 1. 定义状态模式
class OrderState(TypedDict):
    """订单处理状态"""
    order_id: str
    items: list[dict]
    status: str  # "pending", "processing", "shipped", "delivered"
    total_price: float
    customer_info: dict

# 2. 定义节点函数
def validate_order(state: OrderState):
    """验证订单节点"""
    # 检查订单项是否有效
    valid_items = []
    for item in state["items"]:
        if item.get("quantity", 0) > 0 and item.get("price", 0) > 0:
            valid_items.append(item)
    
    if len(valid_items) == 0:
        return {"status": "cancelled", "items": []}
    
    # 计算总价
    total = sum(item["quantity"] * item["price"] for item in valid_items)
    
    return {
        "items": valid_items,
        "total_price": total,
        "status": "validated"
    }

def process_payment(state: OrderState):
    """处理支付节点"""
    # 模拟支付处理
    print(f"Processing payment for order {state['order_id']}: ${state['total_price']}")
    
    # 假设支付成功
    return {"status": "paid"}

def ship_order(state: OrderState):
    """发货节点"""
    print(f"Shipping order {state['order_id']} to {state['customer_info'].get('address')}")
    return {"status": "shipped"}

def notify_customer(state: OrderState):
    """通知客户节点"""
    print(f"Notifying customer {state['customer_info'].get('email')} about order status: {state['status']}")
    return {}

# 3. 构建工作流
workflow = StateGraph(OrderState)

# 添加节点
workflow.add_node("validate", validate_order)
workflow.add_node("pay", process_payment)
workflow.add_node("ship", ship_order)
workflow.add_node("notify", notify_customer)

# 添加边（定义执行流程）
workflow.add_edge(START, "validate")  # 从开始到验证
workflow.add_edge("validate", "pay")   # 从验证到支付
workflow.add_edge("pay", "ship")       # 从支付到发货
workflow.add_edge("ship", "notify")    # 从发货到通知
workflow.add_edge("notify", END)       # 从通知到结束

# 4. 编译工作流
graph = workflow.compile()

# 5. 执行工作流
initial_state = {
    "order_id": "ORD-001",
    "items": [
        {"name": "Product A", "quantity": 2, "price": 29.99},
        {"name": "Product B", "quantity": 1, "price": 49.99}
    ],
    "status": "pending",
    "total_price": 0.0,
    "customer_info": {
        "name": "John Doe",
        "email": "john@example.com",
        "address": "123 Main St"
    }
}

result = graph.invoke(initial_state)
print(f"Order processing completed: {result}")
```

### 带权重的条件边示例

```python
# 示例：基于权重的智能路由
from typing import Literal
from langgraph.graph import StateGraph, END

class RoutingState(TypedDict):
    """路由状态"""
    request_type: Literal["simple", "complex", "urgent"]
    priority: int  # 1-10，数字越大优先级越高
    processing_path: list[str]

def route_request(state: RoutingState):
    """路由决策节点"""
    # 根据请求类型和优先级决定路由
    if state["request_type"] == "urgent" or state["priority"] >= 8:
        return {"processing_path": ["high_priority_queue"]}
    elif state["request_type"] == "complex":
        return {"processing_path": ["expert_processing"]}
    else:
        return {"processing_path": ["standard_processing"]}

# 构建带权重的条件边
workflow = StateGraph(RoutingState)
workflow.add_node("route", route_request)

# 添加条件边，根据状态决定下一步
workflow.add_conditional_edges(
    "route",
    lambda state: state["processing_path"][0] if state["processing_path"] else "standard_processing",
    {
        "high_priority_queue": "process_urgent",
        "expert_processing": "process_complex", 
        "standard_processing": "process_standard"
    }
)
```

### 并行节点与权重调度

```python
# 并行处理示例
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class ParallelState(TypedDict):
    """并行处理状态"""
    data: dict
    results: Annotated[dict, lambda left, right: {**left, **right}]
    processing_time: float

def process_task_a(state: ParallelState):
    """任务A处理节点"""
    import time
    start = time.time()
    # 模拟处理时间
    time.sleep(0.1)
    processing_time = time.time() - start
    
    return {
        "results": {"task_a": "completed"},
        "processing_time": state.get("processing_time", 0) + processing_time
    }

def process_task_b(state: ParallelState):
    """任务B处理节点"""
    import time
    start = time.time()
    # 模拟处理时间
    time.sleep(0.2)
    processing_time = time.time() - start
    
    return {
        "results": {"task_b": "completed"},
        "processing_time": state.get("processing_time", 0) + processing_time
    }

# 构建并行工作流
workflow = StateGraph(ParallelState)
workflow.add_node("task_a", process_task_a)
workflow.add_node("task_b", process_task_b)

# 并行执行：从开始同时执行task_a和task_b
workflow.add_edge(START, "task_a")
workflow.add_edge(START, "task_b")

# 收集结果
def collect_results(state: ParallelState):
    """收集结果节点"""
    print(f"All tasks completed. Results: {state['results']}")
    print(f"Total processing time: {state['processing_time']:.2f}s")
    return {}

workflow.add_node("collect", collect_results)
workflow.add_edge("task_a", "collect")
workflow.add_edge("task_b", "collect")
workflow.add_edge("collect", END)
```

### 动态权重调整

```python
# 动态权重示例
class DynamicWeightState(TypedDict):
    """动态权重状态"""
    workload: dict[str, int]  # 各节点的工作负载
    execution_history: list[dict]  # 执行历史

def calculate_weights(state: DynamicWeightState) -> dict[str, float]:
    """根据工作负载动态计算权重"""
    weights = {}
    total_load = sum(state["workload"].values())
    
    for node, load in state["workload"].items():
        # 负载越轻的节点权重越高（优先分配任务）
        if total_load > 0:
            weights[node] = (total_load - load) / total_load
        else:
            weights[node] = 1.0
    
    return weights
```

## 常见问题

### Q1: 节点函数可以返回什么类型的数据？
**A**: 节点函数必须返回一个字典，其中的键对应状态字段名，值将用于更新状态。返回空字典表示不更新状态。

### Q2: 条件边和权重有什么区别？
**A**: 条件边决定执行哪个节点，权重决定执行顺序或概率。条件边是二元的（去A或去B），权重是连续的（80%去A，20%去B）。

### Q3: 如何实现节点的错误处理？
**A**: 可以在节点函数中使用try-catch，或使用LangGraph的`try_except`装饰器。也可以在状态中定义错误字段，由后续节点处理。

### Q4: 并行边如何保证数据一致性？
**A**: 并行节点应处理不同的状态字段，或使用状态归约器（Reducer）合并结果。避免多个并行节点修改同一字段。

### Q5: 权重系统会影响性能吗？
**A**: 权重计算是轻量级的，但复杂的权重函数可能影响性能。建议使用缓存或预计算的权重。

## 本节总结

本节深入探讨了LangGraph图计算引擎的三个核心要素：节点、边和权重。节点是工作流的处理单元，封装业务逻辑；边控制执行流程，支持复杂控制流；权重提供高级调度能力，优化资源利用。

关键收获：
1. 节点函数遵循函数式编程范式，接收状态、返回状态更新
2. 边类型丰富，支持线性、条件、并行等多种控制流模式
3. 权重系统支持智能调度，包括优先级、概率、成本等维度
4. 并行边支持并发执行，提高工作流效率
5. 动态权重支持自适应调度，应对变化的执行环境
