# 11.3.3 循环工作流设计

## 概念讲解

循环工作流是LangGraph中处理重复任务和迭代计算的核心设计模式。与线性和分支工作流不同，循环工作流允许节点**重复执行**，直到满足特定终止条件，实现迭代处理、状态收敛、批量操作等场景。

### 循环工作流的特点

1. **重复执行**：同一节点或节点序列可以多次执行
2. **条件终止**：基于状态值决定是否继续循环
3. **状态迭代**：每次循环都更新状态，逐步接近目标
4. **安全边界**：最大迭代次数、超时控制防止无限循环
5. **检查点支持**：支持循环中的状态持久化和恢复

### 循环类型

LangGraph支持多种循环类型：

1. **简单循环**：固定次数的重复执行
2. **条件循环**：基于状态的动态循环（while-loop）
3. **收敛循环**：迭代直到状态变化小于阈值
4. **嵌套循环**：循环内部包含子循环
5. **并行循环**：循环内节点并行执行

### 核心机制

循环工作流的关键机制：
- **条件边回环**：`add_conditional_edges()`实现节点到自身的循环
- **状态监测**：检查状态变化决定是否继续
- **迭代计数器**：跟踪循环次数，防止无限循环
- **收敛检测**：监测状态变化率，智能终止

## 核心要点

- **循环通过条件边实现**：节点返回自身形成循环
- **终止条件必须明确**：避免无限循环，设置最大迭代次数
- **状态是循环的驱动因素**：每次循环更新状态，直到满足条件
- **检查点支持长循环**：长时间循环可暂停、恢复
- **性能需要考虑**：循环可能成为性能瓶颈，需要优化

## 简单示例

### 基础条件循环

```python
# 示例：条件循环工作流 - 迭代优化
from typing import TypedDict
from langgraph.graph import StateGraph, END, START

class LoopState(TypedDict):
    """循环状态"""
    iteration: int
    max_iterations: int
    value: float
    target: float
    tolerance: float
    converged: bool

def process_iteration(state: LoopState):
    """迭代处理节点"""
    iteration = state["iteration"] + 1
    current_value = state["value"]
    
    # 模拟迭代优化（简单的梯度下降）
    gradient = 0.1 * (state["target"] - current_value)
    new_value = current_value + gradient
    
    # 检查收敛
    error = abs(new_value - state["target"])
    converged = error < state["tolerance"]
    
    print(f"迭代 {iteration}: 值={new_value:.4f}, 误差={error:.4f}, 收敛={converged}")
    
    return {
        "iteration": iteration,
        "value": new_value,
        "converged": converged
    }

def should_continue(state: LoopState) -> str:
    """决定是否继续循环的条件函数"""
    if state["converged"]:
        return "end"
    elif state["iteration"] >= state["max_iterations"]:
        print(f"达到最大迭代次数: {state['max_iterations']}")
        return "end"
    else:
        return "continue"

# 构建循环工作流
workflow = StateGraph(LoopState)
workflow.add_node("process", process_iteration)

# 循环结构：process → (条件判断) → process 或 END
workflow.add_edge(START, "process")
workflow.add_conditional_edges(
    "process",
    should_continue,
    {
        "continue": "process",  # 循环回自身
        "end": END
    }
)

# 编译并测试
graph = workflow.compile()

print("=== 条件循环测试 ===")
initial_state = {
    "iteration": 0,
    "max_iterations": 20,
    "value": 0.0,
    "target": 1.0,
    "tolerance": 0.01,
    "converged": False
}

result = graph.invoke(initial_state)
print(f"\n最终结果: 值={result['value']:.4f}, 迭代次数={result['iteration']}, 收敛={result['converged']}")
```

### 批量处理循环

```python
# 示例：批量处理循环
from typing import List
import time

class BatchState(TypedDict):
    """批量处理状态"""
    items: List[str]
    processed_items: List[str]
    current_index: int
    batch_size: int
    total_processed: int

def process_batch(state: BatchState):
    """批量处理节点"""
    items = state["items"]
    index = state["current_index"]
    batch_size = state["batch_size"]
    
    # 获取当前批次
    end_index = min(index + batch_size, len(items))
    batch = items[index:end_index]
    
    # 模拟处理（实际中可能是API调用、数据库操作等）
    print(f"处理批次 {index//batch_size + 1}: {batch}")
    time.sleep(0.1)  # 模拟处理时间
    
    processed = state.get("processed_items", []) + batch
    new_index = end_index
    
    return {
        "processed_items": processed,
        "current_index": new_index,
        "total_processed": len(processed)
    }

def batch_complete(state: BatchState) -> str:
    """检查批次是否完成"""
    if state["current_index"] >= len(state["items"]):
        return "complete"
    else:
        return "continue"

# 构建批量处理循环
batch_workflow = StateGraph(BatchState)
batch_workflow.add_node("process_batch", process_batch)

batch_workflow.add_edge(START, "process_batch")
batch_workflow.add_conditional_edges(
    "process_batch",
    batch_complete,
    {
        "continue": "process_batch",  # 继续下一批次
        "complete": END
    }
)

print("\n=== 批量处理循环测试 ===")
test_items = [f"item_{i}" for i in range(10)]
initial_batch_state = {
    "items": test_items,
    "processed_items": [],
    "current_index": 0,
    "batch_size": 3,
    "total_processed": 0
}

result = batch_workflow.compile().invoke(initial_batch_state)
print(f"\n批量处理完成: 共处理 {result['total_processed']} 项")
print(f"处理结果: {result['processed_items']}")
```

## 进阶应用

### 智能循环控制器

```python
# 智能循环控制器
class SmartLoopController:
    """智能循环控制器"""
    
    def __init__(self, max_iterations: int = 100, timeout: float = 30.0, 
                 convergence_threshold: float = 0.001):
        self.max_iterations = max_iterations
        self.timeout = timeout
        self.convergence_threshold = convergence_threshold
        self.iteration_history = []
    
    def should_continue(self, state: dict, start_time: float) -> bool:
        """智能判断是否继续循环"""
        import time
        
        # 检查迭代次数
        if len(self.iteration_history) >= self.max_iterations:
            return False
        
        # 检查超时
        if time.time() - start_time > self.timeout:
            return False
        
        # 检查收敛（最近3次变化小于阈值）
        if len(self.iteration_history) >= 3:
            recent_values = self.iteration_history[-3:]
            changes = [abs(recent_values[i] - recent_values[i-1]) 
                      for i in range(1, len(recent_values))]
            
            if max(changes) < self.convergence_threshold:
                return False
        
        return True
    
    def record_iteration(self, state_value: float):
        """记录迭代状态"""
        self.iteration_history.append(state_value)
    
    def get_stats(self):
        """获取统计信息"""
        if not self.iteration_history:
            return {}
        
        return {
            "total_iterations": len(self.iteration_history),
            "final_value": self.iteration_history[-1],
            "initial_value": self.iteration_history[0],
            "total_change": self.iteration_history[-1] - self.iteration_history[0]
        }
```

### 循环工作流优化

```python
# 循环工作流优化技巧
from langgraph.compiler import CompilerOptions, OptimizationLevel

def create_optimized_loop_workflow(state_schema, loop_node_func, 
                                   condition_func, max_cache_size: int = 50):
    """创建优化的循环工作流"""
    from langgraph.graph import StateGraph, START, END
    
    workflow = StateGraph(state_schema)
    workflow.add_node("loop_node", loop_node_func)
    
    workflow.add_edge(START, "loop_node")
    workflow.add_conditional_edges(
        "loop_node",
        condition_func,
        {
            "continue": "loop_node",
            "end": END
        }
    )
    
    # 启用编译优化
    options = CompilerOptions(
        optimization_level=OptimizationLevel.AGGRESSIVE,
        enable_caching=True,
        cache_size=max_cache_size,
        enable_lazy_loading=True  # 懒加载减少内存使用
    )
    
    return workflow.compile(options=options)
```

## 常见问题

### Q1: 如何避免无限循环？
**A**: 三种机制：1) 设置最大迭代次数；2) 添加超时控制；3) 监测状态收敛。LangGraph也会在编译时检测明显的无限循环模式。

### Q2: 循环工作流可以暂停和恢复吗？
**A**: 可以，通过检查点机制。LangGraph支持在循环任意迭代保存状态，后续从该点恢复执行。

### Q3: 循环性能如何优化？
**A**: 优化策略：1) 缓存确定性节点结果；2) 使用JIT编译热点循环；3) 并行化独立迭代；4) 预取下一次迭代所需数据。

### Q4: 如何处理循环中的错误？
**A**: 错误处理策略：1) 节点级try-catch；2) 循环级错误恢复；3) 优雅降级；4) 检查点支持错误后恢复。

### Q5: 循环工作流适合什么场景？
**A**: 适用场景：1) 迭代优化算法；2) 批量数据处理；3) 状态收敛任务；4) 轮询等待；5) 渐进式增强。

## 本节总结

循环工作流扩展了LangGraph处理重复任务的能力，通过条件边回环实现灵活的迭代逻辑。合理设计的循环工作流可以处理复杂的迭代算法、批量操作和状态收敛任务。

关键收获：
1. 循环通过条件边实现，支持灵活的中止条件
2. 安全边界（最大迭代、超时）防止无限循环
3. 状态迭代是循环的核心机制
4. 检查点支持长时间循环的持久化和恢复
5. 性能优化对循环工作流尤为重要
