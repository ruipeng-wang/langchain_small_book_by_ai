# 11.3.2 分支工作流设计

## 概念讲解

分支工作流是LangGraph中处理条件逻辑的核心设计模式，它允许工作流根据运行时状态选择不同的执行路径。与线性工作流不同，分支工作流引入了**决策点**，在这些点上工作流会根据条件判断走向不同的分支，实现动态的业务逻辑。

### 分支工作流的特点

1. **条件执行**：基于状态值决定执行哪个分支
2. **多路径选择**：一个决策点可以通向多个不同的节点
3. **动态路由**：执行路径在运行时确定，而非编译时
4. **路径收敛**：不同分支最终可能汇聚到同一个节点
5. **业务规则编码**：将复杂的业务规则转化为条件逻辑

### 分支类型

LangGraph支持多种分支类型：

1. **二元分支**：if-else逻辑，两个互斥的路径
2. **多路分支**：switch-case逻辑，多个可选路径
3. **权重分支**：基于概率的随机选择（v1.2.22新增）
4. **动态分支**：运行时计算目标节点
5. **嵌套分支**：分支中包含子分支

### 核心组件

分支工作流的关键组件：
- **条件函数**：接收状态，返回下一个节点名称
- **条件边**：`add_conditional_edges()`方法定义
- **分支节点**：执行特定分支逻辑的处理节点
- **汇聚节点**：不同分支汇聚的公共节点

## 核心要点

- **条件函数决定执行路径**：基于状态值返回目标节点名称
- **分支路径互斥**：同一时刻只执行一个分支
- **状态是所有分支的共享上下文**：分支节点可以访问完整状态
- **汇聚点简化复杂逻辑**：不同分支可以汇聚到公共处理节点
- **权重分支支持概率路由**：实现A/B测试、负载均衡等场景

## 简单示例

### 基础二元分支工作流

```python
# 示例：二元分支工作流 - 用户请求路由
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, END, START

# 定义状态
class RequestState(TypedDict):
    """请求处理状态"""
    request_type: Literal["query", "command", "admin"]
    user_role: Literal["user", "admin", "guest"]
    request_data: dict
    processing_path: list[str]
    result: str

# 定义条件函数
def route_request(state: RequestState) -> str:
    """路由请求的条件函数"""
    request_type = state["request_type"]
    user_role = state["user_role"]
    
    # 基于请求类型和用户角色决定路由
    if request_type == "admin":
        if user_role == "admin":
            return "admin_processor"
        else:
            return "unauthorized_handler"
    elif request_type == "command":
        return "command_processor"
    else:  # query
        return "query_processor"

# 定义分支节点函数
def process_query(state: RequestState):
    """查询处理器节点"""
    data = state["request_data"]
    query = data.get("query", "")
    
    # 模拟查询处理
    result = f"查询结果: {query.upper()}"
    path = state.get("processing_path", []) + ["query_processed"]
    
    return {
        "result": result,
        "processing_path": path
    }

def process_command(state: RequestState):
    """命令处理器节点"""
    data = state["request_data"]
    command = data.get("command", "")
    
    # 模拟命令处理
    result = f"命令执行: {command}"
    path = state.get("processing_path", []) + ["command_processed"]
    
    return {
        "result": result,
        "processing_path": path
    }

def process_admin(state: RequestState):
    """管理员处理器节点"""
    data = state["request_data"]
    admin_cmd = data.get("admin_command", "")
    
    # 模拟管理员操作
    result = f"管理员操作: {admin_cmd}"
    path = state.get("processing_path", []) + ["admin_processed"]
    
    return {
        "result": result,
        "processing_path": path
    }

def handle_unauthorized(state: RequestState):
    """未授权处理器节点"""
    path = state.get("processing_path", []) + ["unauthorized"]
    
    return {
        "result": "错误: 未授权访问",
        "processing_path": path
    }

def log_result(state: RequestState):
    """日志记录节点（汇聚点）"""
    print(f"请求处理完成: {state['result']}")
    print(f"处理路径: {state.get('processing_path', [])}")
    return {}

# 构建分支工作流
workflow = StateGraph(RequestState)

# 添加节点
workflow.add_node("router", lambda s: s)  # 路由节点（不改变状态）
workflow.add_node("query_processor", process_query)
workflow.add_node("command_processor", process_command)
workflow.add_node("admin_processor", process_admin)
workflow.add_node("unauthorized_handler", handle_unauthorized)
workflow.add_node("logger", log_result)

# 添加边
workflow.add_edge(START, "router")

# 添加条件边：基于路由结果选择分支
workflow.add_conditional_edges(
    "router",
    route_request,
    {
        "query_processor": "query_processor",
        "command_processor": "command_processor",
        "admin_processor": "admin_processor",
        "unauthorized_handler": "unauthorized_handler"
    }
)

# 所有分支汇聚到日志节点
workflow.add_edge("query_processor", "logger")
workflow.add_edge("command_processor", "logger")
workflow.add_edge("admin_processor", "logger")
workflow.add_edge("unauthorized_handler", "logger")
workflow.add_edge("logger", END)

# 编译工作流
graph = workflow.compile()

# 测试分支工作流
print("=== 分支工作流测试 ===")
test_cases = [
    {"request_type": "query", "user_role": "user", "request_data": {"query": "hello"}},
    {"request_type": "command", "user_role": "user", "request_data": {"command": "restart"}},
    {"request_type": "admin", "user_role": "admin", "request_data": {"admin_command": "shutdown"}},
    {"request_type": "admin", "user_role": "user", "request_data": {"admin_command": "shutdown"}},
]

for i, test_case in enumerate(test_cases, 1):
    print(f"\n测试用例 {i}: {test_case}")
    state = {
        "request_type": test_case["request_type"],
        "user_role": test_case["user_role"],
        "request_data": test_case["request_data"],
        "processing_path": [],
        "result": ""
    }
    
    result = graph.invoke(state)
    print(f"最终结果: {result['result']}")
```

### 多路分支与权重选择

```python
# 示例：多路分支与权重选择
from typing import TypedDict
from langgraph.graph import StateGraph, END
import random

class MultiBranchState(TypedDict):
    """多分支状态"""
    priority: int  # 1-10，优先级
    workload: float  # 0-1，系统负载
    selected_worker: str
    processing_time: float

def select_worker(state: MultiBranchState) -> str:
    """选择工作节点的条件函数（带权重）"""
    priority = state["priority"]
    workload = state["workload"]
    
    # 基于优先级和负载计算权重
    if priority >= 8:  # 高优先级
        weights = {"fast_worker": 0.7, "medium_worker": 0.3}
    elif priority >= 5:  # 中优先级
        if workload < 0.5:
            weights = {"fast_worker": 0.4, "medium_worker": 0.4, "slow_worker": 0.2}
        else:
            weights = {"fast_worker": 0.3, "medium_worker": 0.5, "slow_worker": 0.2}
    else:  # 低优先级
        weights = {"medium_worker": 0.5, "slow_worker": 0.5}
    
    # 基于权重随机选择
    workers = list(weights.keys())
    probabilities = list(weights.values())
    return random.choices(workers, probabilities)[0]

def fast_worker(state: MultiBranchState):
    """快速工作节点"""
    import time
    start = time.time()
    time.sleep(0.1)  # 快速处理
    processing_time = time.time() - start
    
    return {
        "selected_worker": "fast",
        "processing_time": state.get("processing_time", 0) + processing_time
    }

def medium_worker(state: MultiBranchState):
    """中等工作节点"""
    import time
    start = time.time()
    time.sleep(0.3)  # 中等处理
    processing_time = time.time() - start
    
    return {
        "selected_worker": "medium",
        "processing_time": state.get("processing_time", 0) + processing_time
    }

def slow_worker(state: MultiBranchState):
    """慢速工作节点"""
    import time
    start = time.time()
    time.sleep(0.5)  # 慢速处理
    processing_time = time.time() - start
    
    return {
        "selected_worker": "slow",
        "processing_time": state.get("processing_time", 0) + processing_time
    }

def collect_results(state: MultiBranchState):
    """收集结果节点"""
    print(f"工作节点: {state['selected_worker']}, 处理时间: {state['processing_time']:.3f}s")
    return {}

# 构建多分支工作流
multi_workflow = StateGraph(MultiBranchState)

multi_workflow.add_node("selector", lambda s: s)
multi_workflow.add_node("fast_worker", fast_worker)
multi_workflow.add_node("medium_worker", medium_worker)
multi_workflow.add_node("slow_worker", slow_worker)
multi_workflow.add_node("collector", collect_results)

multi_workflow.add_edge(START, "selector")
multi_workflow.add_conditional_edges(
    "selector",
    select_worker,
    {
        "fast_worker": "fast_worker",
        "medium_worker": "medium_worker",
        "slow_worker": "slow_worker"
    }
)

# 所有工作节点汇聚到收集器
multi_workflow.add_edge("fast_worker", "collector")
multi_workflow.add_edge("medium_worker", "collector")
multi_workflow.add_edge("slow_worker", "collector")
multi_workflow.add_edge("collector", END)

print("\n=== 多路分支权重选择测试 ===")
# 测试不同优先级和负载情况
test_scenarios = [
    {"priority": 9, "workload": 0.3},
    {"priority": 6, "workload": 0.7},
    {"priority": 3, "workload": 0.5},
]

for scenario in test_scenarios:
    print(f"\n场景: 优先级={scenario['priority']}, 负载={scenario['workload']}")
    
    # 多次运行观察权重选择
    selections = {"fast": 0, "medium": 0, "slow": 0}
    for _ in range(100):
        state = {
            "priority": scenario["priority"],
            "workload": scenario["workload"],
            "selected_worker": "",
            "processing_time": 0
        }
        result = multi_workflow.compile().invoke(state)
        selections[result["selected_worker"]] += 1
    
    print(f"选择分布: {selections}")
```

### 动态分支生成

```python
# 动态分支生成示例
from typing import Dict, Callable, List

class DynamicBranchWorkflow:
    """动态分支工作流生成器"""
    
    def __init__(self):
        self.branch_handlers: Dict[str, Callable] = {}
        self.condition_logic: List[Callable] = []
    
    def add_branch(self, name: str, handler: Callable, condition: Callable = None):
        """添加分支"""
        self.branch_handlers[name] = handler
        
        if condition:
            self.condition_logic.append((name, condition))
        
        return self
    
    def create_dynamic_router(self, state: Dict) -> str:
        """动态路由函数"""
        # 按顺序评估条件
        for branch_name, condition_func in self.condition_logic:
            if condition_func(state):
                return branch_name
        
        # 默认分支
        return "default"
    
    def build(self, state_schema):
        """构建动态分支工作流"""
        from langgraph.graph import StateGraph, START, END
        
        workflow = StateGraph(state_schema)
        
        # 添加路由节点
        workflow.add_node("router", lambda s: s)
        
        # 添加所有分支节点
        for name, handler in self.branch_handlers.items():
            workflow.add_node(name, handler)
        
        # 添加条件边
        workflow.add_edge(START, "router")
        workflow.add_conditional_edges(
            "router",
            self.create_dynamic_router,
            {name: name for name in self.branch_handlers}
        )
        
        # 添加汇聚节点
        def converge(state: Dict):
            print(f"所有分支汇聚，最终状态: {state}")
            return {}
        
        workflow.add_node("converge", converge)
        for name in self.branch_handlers:
            workflow.add_edge(name, "converge")
        workflow.add_edge("converge", END)
        
        return workflow.compile()
```

## 常见问题

### Q1: 条件函数应该放在哪里？
**A**: 条件函数应该：1) 保持简洁，只做路由决策；2) 避免副作用；3) 可测试；4) 与业务规则分离。建议将复杂条件逻辑提取到单独模块。

### Q2: 如何处理分支间的数据共享？
**A**: 策略包括：1) 使用状态作为共享数据源；2) 通过汇聚节点合并结果；3) 使用消息传递模式；4) 避免分支间直接数据依赖。

### Q3: 分支工作流会影响性能吗？
**A**: 影响很小。条件判断是轻量级的，但分支数量过多可能增加复杂度。建议：1) 限制分支数量（3-5个）；2) 使用权重分支减少条件判断；3) 预编译热点路径。

### Q4: 如何测试分支工作流？
**A**: 测试策略：1) 单元测试条件函数；2) 集成测试每个分支路径；3) 使用模拟状态测试边界条件；4) 覆盖率测试确保所有分支都被执行。

### Q5: 分支工作流可以嵌套吗？
**A**: 可以，但需要谨慎设计。嵌套分支会增加复杂度，建议：1) 深度不超过2层；2) 使用状态标志管理嵌套层次；3) 考虑拆分为子工作流。

## 本节总结

分支工作流是LangGraph中处理条件逻辑的强大工具，它将复杂的业务规则转化为清晰的条件执行路径。通过合理的分支设计，可以构建出灵活、可维护的动态工作流系统。

关键收获：
1. 条件函数是分支决策的核心，应保持简洁和可测试
2. 分支路径应该清晰明确，避免过度复杂的条件逻辑
3. 汇聚节点简化了分支结果的合并处理
4. 权重分支支持概率路由，适用于A/B测试、负载均衡等场景
5. 动态分支生成提供了高度灵活的路由能力

分支工作流扩展了线性工作流的能力，使LangGraph能够处理真实世界中复杂的业务逻辑。在下一节中，我们将探讨循环工作流设计，学习如何处理重复和迭代任务。
