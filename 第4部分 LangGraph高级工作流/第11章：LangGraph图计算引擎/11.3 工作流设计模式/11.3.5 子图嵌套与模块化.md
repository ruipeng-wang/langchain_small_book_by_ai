# 11.3.5 子图嵌套与模块化

## 概念讲解

随着工作流复杂度的增加，将所有逻辑放在单个图中会导致代码难以维护和复用。**子图（Subgraph）**机制允许我们将复杂工作流拆分为多个独立的、可复用的模块，每个模块可以独立开发、测试，然后像搭积木一样组合在一起。

LangGraph 的子图嵌套遵循**层次化设计**原则：一个图（父图）可以将另一个图（子图）作为节点添加。子图拥有独立的状态空间和执行逻辑，同时可以与父图共享检查点和状态。

### 子图的核心优势

1. **模块化**：将复杂流程拆分为可管理的独立模块
2. **可复用性**：同一子图可以在多个父图中使用
3. **隔离性**：子图内部状态与父图隔离，减少耦合
4. **可测试性**：子图可以独立测试，确保正确性后再集成

### 子图的工作模式

子图有两种主要使用方式：

- **静态子图**：在编译时确定的固定子图
- **动态子图**：根据运行时条件动态选择要执行的子图

## 核心要点

- **子图作为节点添加**：将编译后的子图直接作为父图的节点
- **状态传递**：父图和子图通过共享的状态模式进行数据交换
- **检查点传播**：父图的检查点器会自动传递给子图
- **子图可独立编译**：子图可以有自己的检查点配置
- **嵌套深度无限制**：理论上支持任意深度的子图嵌套

## 简单示例

### 基础子图使用

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver
from typing import TypedDict

# 定义共享状态
class State(TypedDict):
    topic: str
    result: str

# === 子图定义 ===
def sub_step_one(state: State):
    return {"result": f"子图处理: {state['topic']} - 步骤1"}

def sub_step_two(state: State):
    return {"result": state["result"] + " - 步骤2"}

# 构建子图
sub_builder = StateGraph(State)
sub_builder.add_node("sub_one", sub_step_one)
sub_builder.add_node("sub_two", sub_step_two)
sub_builder.add_edge(START, "sub_one")
sub_builder.add_edge("sub_one", "sub_two")
sub_builder.add_edge("sub_two", END)
subgraph = sub_builder.compile()  # 子图编译

# === 父图定义 ===
def parent_init(state: State):
    return {"result": "父图初始化完成"}

def parent_finalize(state: State):
    return {"result": state["result"] + " - 父图完成"}

# 构建父图，将子图作为节点
builder = StateGraph(State)
builder.add_node("init", parent_init)
builder.add_node("sub_process", subgraph)  # 子图作为节点
builder.add_node("finalize", parent_finalize)

builder.add_edge(START, "init")
builder.add_edge("init", "sub_process")
builder.add_edge("sub_process", "finalize")
builder.add_edge("finalize", END)

# 编译父图（检查点会传播到子图）
checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 执行
config = {"configurable": {"thread_id": "subgraph-demo"}}
result = graph.invoke({"topic": "机器学习", "result": ""}, config)
print(result)
```

### 查看子图状态

```python
# 获取当前状态（包含子图信息）
current_state = graph.get_state(config, subgraphs=True)

# 访问子图任务状态
for task in current_state.tasks:
    print(f"子图节点: {task.name}")
    print(f"子图状态: {task.state}")
```

## 进阶应用

### 多子图协作模式

多个子图协作完成复杂任务：

```python
# 定义三个独立的子图
def create_research_subgraph():
    """研究子图"""
    def search(state: State):
        return {"result": f"搜索关于 {state['topic']} 的资料"}
    
    def analyze(state: State):
        return {"result": state["result"] + "，分析内容"}
    
    builder = StateGraph(State)
    builder.add_node("search", search)
    builder.add_node("analyze", analyze)
    builder.add_edge(START, "search")
    builder.add_edge("search", "analyze")
    builder.add_edge("analyze", END)
    return builder.compile()

def create_writing_subgraph():
    """写作子图"""
    def outline(state: State):
        return {"result": state["result"] + "，生成大纲"}
    
    def draft(state: State):
        return {"result": state["result"] + "，撰写草稿"}
    
    builder = StateGraph(State)
    builder.add_node("outline", outline)
    builder.add_node("draft", draft)
    builder.add_edge(START, "outline")
    builder.add_edge("outline", "draft")
    builder.add_edge("draft", END)
    return builder.compile()

# 主图组合子图
main_builder = StateGraph(State)
main_builder.add_node("research", create_research_subgraph())
main_builder.add_node("write", create_writing_subgraph())

main_builder.add_edge(START, "research")
main_builder.add_edge("research", "write")
main_builder.add_edge("write", END)

graph = main_builder.compile()
result = graph.invoke({"topic": "人工智能", "result": ""})
```

### 动态子图选择

根据条件动态选择执行的子图：

```python
from langgraph.types import Command
from typing import Literal

def router_node(state: State) -> Command[Literal["simple_flow", "complex_flow"]]:
    """根据主题复杂度路由到不同子图"""
    if "基础" in state["topic"]:
        return Command(goto="simple_flow")
    else:
        return Command(goto="complex_flow")

# 构建动态路由图
builder = StateGraph(State)
builder.add_node("router", router_node)
builder.add_node("simple_flow", create_research_subgraph())
builder.add_node("complex_flow", create_writing_subgraph())

builder.add_edge(START, "router")
builder.add_edge("simple_flow", END)
builder.add_edge("complex_flow", END)

graph = builder.compile()
```

## 常见问题

### Q1: 父图和子图的状态是如何共享的？

**A**: 父图和子图使用相同的状态模式（State Schema）。子图可以读取和更新父图状态中的任何字段，父图也能看到子图的更新。

### Q2: 子图可以有独立的检查点器吗？

**A**: 可以。子图编译时可以设置 `checkpointer=True` 来启用独立的检查点，或者不设置来使用父图的检查点器。

### Q3: 子图的深度有限制吗？

**A**: 理论上没有深度限制，但过深的嵌套会影响性能和可维护性。建议控制在 3 层以内。

### Q4: 如何在子图中实现人工介入？

**A**: 子图中的节点同样可以使用 `interrupt()` 函数。中断时会返回到最外层调用方，恢复时从父图传入 `Command(resume=...)`。

## 本节总结

子图嵌套与模块化是 LangGraph 管理复杂工作流的关键机制。通过将工作流拆分为独立的子图模块，可以实现：

1. 模块化的设计思路，降低系统复杂度
2. 子图的可复用性，提高开发效率
3. 状态和检查点的层次化管理
4. 灵活的组合方式，支持静态和动态子图

掌握子图嵌套模式，是构建企业级复杂工作流的必备技能。
