# 11.3.1 线性工作流设计

## 概念讲解

线性工作流是LangGraph中最基础、最常用的设计模式，它模拟了传统的顺序执行流程。在线性工作流中，节点按照严格的顺序依次执行，前一个节点的输出作为后一个节点的输入，形成一个**单向的数据流水线**。

### 线性工作流的特点

1. **顺序执行**：节点A → 节点B → 节点C，严格按顺序执行
2. **数据依赖**：后序节点依赖前序节点的处理结果
3. **无分支**：执行路径是确定的，没有条件判断或并行分支
4. **简单可控**：易于理解、调试和测试
5. **错误传播**：任意节点失败会导致整个工作流终止

### 适用场景

线性工作流适用于以下场景：
- **数据处理流水线**：数据清洗 → 转换 → 分析的连续处理
- **审批流程**：提交 → 审核 → 批准的层级审批
- **订单处理**：验证 → 支付 → 发货 → 通知的顺序流程
- **模型调用链**：预处理 → 模型推理 → 后处理的AI工作流

### LangGraph中的线性实现

在LangGraph中，线性工作流通过`add_edge()`方法连接节点，形成单向链：

```python
workflow.add_edge("node_a", "node_b")  # A → B
workflow.add_edge("node_b", "node_c")  # B → C
workflow.add_edge("node_c", END)       # C → 结束
```

## 核心要点

- **节点顺序决定执行顺序**：边的添加顺序定义了工作流的执行路径
- **状态传递是隐式的**：LangGraph自动管理状态在节点间的传递
- **错误处理需要显式设计**：线性工作流需要额外的错误处理节点
- **性能优化机会有限**：由于严格的顺序依赖，并行化困难
- **适合初学者上手**：线性工作流是学习LangGraph的最佳起点

## 简单示例

### 基础线性工作流

```python
# 示例：简单的线性工作流 - 文本处理流水线
from typing import TypedDict
from langgraph.graph import StateGraph, END, START

# 定义状态
class TextProcessingState(TypedDict):
    """文本处理状态"""
    original_text: str
    cleaned_text: str
    analyzed_result: dict
    final_output: str

# 定义节点函数
def clean_text(state: TextProcessingState):
    """清理文本节点"""
    text = state["original_text"]
    
    # 简单的文本清理
    cleaned = text.strip().lower()
    print(f"清理文本: '{text}' -> '{cleaned}'")
    
    return {"cleaned_text": cleaned}

def analyze_text(state: TextProcessingState):
    """分析文本节点"""
    text = state["cleaned_text"]
    
    # 简单的文本分析
    word_count = len(text.split())
    char_count = len(text)
    has_special = any(c in text for c in "!@#$%^&*")
    
    result = {
        "word_count": word_count,
        "char_count": char_count,
        "has_special_chars": has_special
    }
    
    print(f"分析文本: {result}")
    return {"analyzed_result": result}

def generate_output(state: TextProcessingState):
    """生成输出节点"""
    analysis = state["analyzed_result"]
    text = state["cleaned_text"]
    
    # 基于分析生成输出
    if analysis["word_count"] < 5:
        output = f"文本过短: {text}"
    elif analysis["has_special_chars"]:
        output = f"包含特殊字符: {text}"
    else:
        output = f"正常文本: {text} (字数: {analysis['word_count']})"
    
    print(f"生成输出: {output}")
    return {"final_output": output}

# 构建线性工作流
workflow = StateGraph(TextProcessingState)

# 添加节点（顺序很重要）
workflow.add_node("clean", clean_text)
workflow.add_node("analyze", analyze_text)
workflow.add_node("generate", generate_output)

# 创建线性连接：START → clean → analyze → generate → END
workflow.add_edge(START, "clean")
workflow.add_edge("clean", "analyze")
workflow.add_edge("analyze", "generate")
workflow.add_edge("generate", END)

# 编译工作流
graph = workflow.compile()

# 测试执行
print("=== 线性工作流测试 ===")
test_cases = [
    "Hello, World! This is a test.",
    "Short text",
    "Text with @special chars"
]

for i, text in enumerate(test_cases, 1):
    print(f"\n测试用例 {i}: '{text}'")
    result = graph.invoke({"original_text": text})
    print(f"最终输出: {result['final_output']}")
```

### 带错误处理的线性工作流

```python
# 示例：带错误处理的线性工作流
from typing import Optional

class RobustProcessingState(TypedDict):
    """健壮处理状态"""
    input_data: str
    processing_stage: str
    result: Optional[str]
    error_message: Optional[str]
    retry_count: int

def validate_input(state: RobustProcessingState):
    """验证输入节点"""
    data = state["input_data"]
    
    if not data:
        return {
            "error_message": "输入数据不能为空",
            "processing_stage": "failed"
        }
    
    if len(data) < 3:
        return {
            "error_message": "输入数据太短",
            "processing_stage": "failed"
        }
    
    return {"processing_stage": "validated"}

def process_data(state: RobustProcessingState):
    """处理数据节点"""
    if state.get("error_message"):
        return {}  # 跳过处理
    
    data = state["input_data"]
    
    try:
        # 模拟可能失败的处理
        if "error" in data:
            raise ValueError("模拟处理错误")
        
        result = data.upper()
        return {
            "result": result,
            "processing_stage": "processed"
        }
    except Exception as e:
        return {
            "error_message": f"处理失败: {e}",
            "processing_stage": "failed"
        }

def handle_result(state: RobustProcessingState):
    """处理结果节点"""
    if state.get("error_message"):
        error_msg = state["error_message"]
        
        # 检查重试次数
        if state.get("retry_count", 0) < 3:
            print(f"错误处理: {error_msg}，准备重试")
            return {
                "retry_count": state.get("retry_count", 0) + 1,
                "processing_stage": "retry"
            }
        else:
            print(f"错误处理: {error_msg}，达到最大重试次数")
            return {"processing_stage": "failed_final"}
    else:
        print(f"处理成功: {state['result']}")
        return {"processing_stage": "completed"}

# 构建带错误处理的线性工作流
robust_workflow = StateGraph(RobustProcessingState)

robust_workflow.add_node("validate", validate_input)
robust_workflow.add_node("process", process_data)
robust_workflow.add_node("handle", handle_result)

# 线性连接
robust_workflow.add_edge(START, "validate")
robust_workflow.add_edge("validate", "process")
robust_workflow.add_edge("process", "handle")

# 条件边处理重试
def decide_next_step(state: RobustProcessingState) -> str:
    """决定下一步"""
    stage = state.get("processing_stage", "")
    
    if stage == "retry":
        return "validate"  # 重试
    elif stage in ["failed_final", "completed"]:
        return END  # 结束
    else:
        return "handle"  # 继续处理

robust_workflow.add_conditional_edges(
    "handle",
    decide_next_step,
    {
        "validate": "validate",
        "handle": "handle",
        END: END
    }
)

print("\n=== 带错误处理的线性工作流测试 ===")
test_inputs = ["", "hi", "normal data", "data with error"]

for input_data in test_inputs:
    print(f"\n处理输入: '{input_data}'")
    state = {
        "input_data": input_data,
        "processing_stage": "start",
        "result": None,
        "error_message": None,
        "retry_count": 0
    }
    
    result = robust_workflow.compile().invoke(state)
    print(f"最终状态: {result['processing_stage']}")
```

## 进阶应用

### 模块化线性工作流

```python
# 模块化线性工作流设计
from typing import Callable, List

class LinearWorkflowBuilder:
    """线性工作流构建器"""
    
    def __init__(self, state_schema):
        self.state_schema = state_schema
        self.nodes: List[tuple[str, Callable]] = []
    
    def add_step(self, name: str, func: Callable):
        """添加处理步骤"""
        self.nodes.append((name, func))
        return self
    
    def build(self):
        """构建线性工作流"""
        from langgraph.graph import StateGraph, START, END
        
        workflow = StateGraph(self.state_schema)
        
        # 添加所有节点
        for name, func in self.nodes:
            workflow.add_node(name, func)
        
        # 创建线性连接
        if self.nodes:
            # 第一个节点从START开始
            workflow.add_edge(START, self.nodes[0][0])
            
            # 中间节点线性连接
            for i in range(len(self.nodes) - 1):
                workflow.add_edge(self.nodes[i][0], self.nodes[i+1][0])
            
            # 最后一个节点连接到END
            workflow.add_edge(self.nodes[-1][0], END)
        
        return workflow.compile()

# 使用构建器创建线性工作流
class PipelineState(TypedDict):
    """流水线状态"""
    data: str
    step_results: List[str]

builder = LinearWorkflowBuilder(PipelineState)

# 链式调用添加步骤
workflow_app = (
    builder
    .add_step("step1", lambda s: {"step_results": ["step1_done"]})
    .add_step("step2", lambda s: {"step_results": s.get("step_results", []) + ["step2_done"]})
    .add_step("step3", lambda s: {"step_results": s.get("step_results", []) + ["step3_done"]})
    .build()
)

print("\n=== 模块化线性工作流 ===")
result = workflow_app.invoke({"data": "test", "step_results": []})
print(f"步骤结果: {result['step_results']}")
```

### 性能优化技巧

```python
# 线性工作流性能优化
import time
from functools import lru_cache

class OptimizedLinearWorkflow:
    """优化线性工作流"""
    
    @staticmethod
    @lru_cache(maxsize=128)
    def cached_processing(input_data: str) -> str:
        """可缓存的处理函数"""
        # 模拟耗时计算
        time.sleep(0.1)
        return input_data.upper()
    
    def process_with_cache(self, state: dict):
        """带缓存的处理节点"""
        data = state.get("data", "")
        
        if not data:
            return {"result": ""}
        
        # 使用缓存结果
        cached_result = self.cached_processing(data)
        return {"result": cached_result}
    
    def create_optimized_workflow(self, state_schema):
        """创建优化的工作流"""
        from langgraph.graph import StateGraph, START, END
        from langgraph.compiler import CompilerOptions, OptimizationLevel
        
        workflow = StateGraph(state_schema)
        
        # 添加优化节点
        workflow.add_node("cached_process", self.process_with_cache)
        
        # 线性工作流
        workflow.add_edge(START, "cached_process")
        workflow.add_edge("cached_process", END)
        
        # 启用编译优化
        options = CompilerOptions(
            optimization_level=OptimizationLevel.AGGRESSIVE,
            enable_caching=True
        )
        
        return workflow.compile(options=options)
```

## 常见问题

### Q1: 线性工作流和传统函数调用有什么区别？
**A**: 线性工作流提供了：1) 状态自动管理；2) 错误处理框架；3) 执行可视化；4) 检查点支持；5) 性能监控。而传统函数调用需要手动处理这些。

### Q2: 如何处理线性工作流中的节点失败？
**A**: 策略包括：1) 添加错误处理节点；2) 使用try-catch包装节点函数；3) 实现重试机制；4) 提供降级方案。

### Q3: 线性工作流可以转换为并行工作流吗？
**A**: 可以，但需要满足：1) 节点间无数据依赖；2) 节点函数线程安全。LangGraph支持自动检测并行机会。

### Q4: 如何调试线性工作流？
**A**: 调试方法：1) 使用LangSmith可视化执行；2) 添加日志节点；3) 使用检查点查看中间状态；4) 逐步执行测试。

### Q5: 线性工作流适合多长的工作流？
**A**: 建议保持工作流在5-15个节点之间。如果更长，应考虑：1) 拆分子工作流；2) 使用嵌套工作流；3) 模块化设计。

## 本节总结

线性工作流是LangGraph中最基础、最重要的设计模式，它提供了简单直观的顺序执行模型。虽然结构简单，但通过合理的错误处理、模块化设计和性能优化，可以构建出健壮高效的生产级工作流。

关键收获：
1. 线性工作流适合顺序依赖的业务流程
2. 状态自动传递简化了数据管理
3. 错误处理需要显式设计，保障系统健壮性
4. 模块化设计提高工作流的可维护性和可重用性
5. 性能优化技巧可以显著提升执行效率
