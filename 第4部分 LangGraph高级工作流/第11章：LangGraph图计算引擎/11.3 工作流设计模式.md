# 11.3 工作流设计模式

## 概念讲解

### 常见工作流模式

基于StateGraph可以构建多种工作流模式：

```mermaid
graph TD
    A[线性工作流] --> A1[START → A → B → C → END]
    B[分支工作流] --> B1[START → A → {条件} → B/C → END]
    C[循环工作流] --> C1[START → A → {条件} → A/END]
    D[扇出扇入] --> D1[START → A → B,C,D → 聚合 → END]
```

### 流式输出

LangGraph支持多种流式模式，实时获取工作流执行状态：

| 模式 | 说明 |
|------|------|
| `updates` | 获取每个节点的输出更新 |
| `values` | 获取完整状态快照 |
| `messages` | 流式LLM消息内容 |
| `checkpoints` | 获取检查点事件 |

## 核心要点

### 线性工作流

最简单的模式，节点按固定顺序执行：

```python
from langgraph.graph import StateGraph, START, END

class LinearState(TypedDict):
    data: str
    result: str

def step1(state): return {"data": "处理1"}
def step2(state): return {"data": state["data"] + " 处理2"}
def step3(state): return {"result": state["data"] + " 完成"}

workflow = StateGraph(LinearState)
workflow.add_node("step1", step1)
workflow.add_node("step2", step2)
workflow.add_node("step3", step3)

workflow.add_edge(START, "step1")
workflow.add_edge("step1", "step2")
workflow.add_edge("step2", "step3")
workflow.add_edge("step3", END)

app = workflow.compile()
```

### 分支工作流

基于条件动态选择执行路径：

```python
def route(state) -> Literal["path_a", "path_b"]:
    if "keyword" in state.get("data", ""):
        return "path_a"
    return "path_b"

branch = StateGraph(LinearState)
branch.add_node("classifier", lambda s: {"data": "classified"})
branch.add_node("path_a", lambda s: {"result": "路径A结果"})
branch.add_node("path_b", lambda s: {"result": "路径B结果"})

branch.add_edge(START, "classifier")
branch.add_conditional_edges("classifier", route)
branch.add_edge("path_a", END)
branch.add_edge("path_b", END)
```

### 循环工作流

支持重试、迭代等场景：

```python
class LoopState(TypedDict):
    count: int
    done: bool

def process(state: LoopState) -> dict:
    new_count = state["count"] + 1
    return {
        "count": new_count,
        "done": new_count >= 3  # 最多循环3次
    }

def check_done(state: LoopState) -> Literal["process", "done"]:
    return "done" if state["done"] else "process"

loop = StateGraph(LoopState)
loop.add_node("process", process)
loop.add_edge(START, "process")
loop.add_conditional_edges("process", check_done, {
    "process": "process",  # 循环回自己
    "done": END
})

app = loop.compile()
result = app.invoke({"count": 0, "done": False})
```

## 简单示例

### 流式输出示例

```python
from langgraph.checkpoint.memory import MemorySaver

app = workflow.compile(checkpointer=MemorySaver())

# 方式1：获取节点更新
for event in app.stream({"data": "初始值"}):
    print(event)
    # 输出: {"step1": {"data": "处理1"}}
    #       {"step2": {"data": "处理1 处理2"}}
    #       {"step3": {"result": "处理1 处理2 完成"}}

# 方式2：获取完整状态
for event in app.stream({"data": "初始值"}, stream_mode="values"):
    print(event)
    # 输出每个节点执行后的完整状态
```

### 人工介入（Human-in-the-loop）

```python
from langgraph.types import interrupt

def human_review(state: dict) -> dict:
    """人工审核节点"""
    answer = interrupt({
        "action": "请审核内容",
        "content": state.get("draft", "")
    })
    return {"approved": answer}

workflow = StateGraph(ReviewState)
workflow.add_node("draft", create_draft)
workflow.add_node("review", human_review)
workflow.add_node("publish", publish_content)

workflow.add_edge(START, "draft")
workflow.add_edge("draft", "review")
workflow.add_edge("review", "publish")
workflow.add_edge("publish", END)

app = workflow.compile(checkpointer=MemorySaver())

# 第一次执行 - 会在review节点暂停
config = {"configurable": {"thread_id": "review-1"}}
for event in app.stream({"draft": ""}, config):
    print(event)

# 获取人工审核结果并继续
for event in app.stream(Command(resume=True), config):
    print(event)
```

## 进阶应用

### 扇出扇入模式（并行处理）

```python
# 多个处理节点并行执行，结果聚合
from typing import Annotated

def reducer(left: list, right: list) -> list:
    return left + right

class FanState(TypedDict):
    results: Annotated[list, reducer]

def worker_a(state): return {"results": ["A的结果"]}
def worker_b(state): return {"results": ["B的结果"]}
def worker_c(state): return {"results": ["C的结果"]}
def aggregator(state): return {"final": state["results"]}

fan = StateGraph(FanState)
fan.add_node("a", worker_a)
fan.add_node("b", worker_b)
fan.add_node("c", worker_c)
fan.add_node("aggregate", aggregator)

fan.add_edge(START, "a")
fan.add_edge(START, "b")
fan.add_edge(START, "c")
fan.add_edge("a", "aggregate")
fan.add_edge("b", "aggregate")
fan.add_edge("c", "aggregate")
fan.add_edge("aggregate", END)
```

### 检查点与恢复

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = workflow.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "session-1"}}

# 执行工作流
app.invoke({"data": "test"}, config)

# 查看历史记录
states = list(app.get_state_history(config))
for state in states:
    print(state.values)

# 恢复到特定状态
latest = app.get_state(config)
app.update_state(config, {"data": "修改后的数据"})
```

## 常见问题

### Q: 如何实现并行节点执行？

**A:** 当多个节点都从START出发，且没有相互依赖时，LangGraph会自动并行执行（异步模式下）。

### Q: 流式输出如何获取LLM的token？

**A:** 使用`stream_mode="messages"`，可以实时获取LLM生成的token：

```python
for event in app.stream(input, stream_mode="messages"):
    message, metadata = event
    if isinstance(message, AIMessageChunk) and message.content:
        print(message.content, end="", flush=True)
```

### Q: 如何限制循环次数？

**A:** 在状态中添加计数器，条件边检查计数：

```python
def check_loop(state) -> Literal["continue", "end"]:
    if state.get("loop_count", 0) >= max_loops:
        return "end"
    return "continue"
```

## 本节总结

工作流设计模式：
- **线性**：最简单的顺序执行
- **分支**：基于条件动态路由
- **循环**：支持重试和迭代
- **扇出扇入**：并行处理聚合结果
- **人工介入**：通过interrupt实现人工审核点

核心API：
- `stream()`：获取执行事件
- `checkpointer`：状态持久化
- `interrupt/Command`：人工介入
- `stream_mode`：控制输出格式
