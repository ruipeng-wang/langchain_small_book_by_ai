# 11.2.2 节点函数与边条件

## 概念讲解

节点函数与边条件是LangGraph StateGraph架构的核心组成部分，它们共同定义了工作流的业务逻辑和控制流。节点函数封装了具体的处理逻辑，边条件决定了工作流的执行路径。

### 节点函数（Node Functions）

节点函数是工作流中的基本处理单元，具有以下特性：

1. **纯函数设计**：节点函数接收当前状态，返回状态更新，尽量保持无副作用
2. **类型安全**：函数签名与状态类型保持一致，支持编译时类型检查
3. **错误处理**：节点函数可以抛出异常，由工作流错误处理机制捕获
4. **并发安全**：节点函数应设计为可重入和线程安全的
5. **幂等性**：理想情况下，节点函数应具有幂等性，支持重试和恢复

### 边条件（Edge Conditions）

边条件决定了工作流的控制流，支持复杂的业务逻辑：

1. **无条件边**：简单的线性执行顺序
2. **条件边**：基于状态的条件分支
3. **条件函数**：接收状态并返回下一个节点名称的函数
4. **权重边**：带权重的概率分支（v1.2.22新增）
5. **动态边**：运行时根据状态动态计算的目标节点

### 节点与边的协作模式

节点函数和边条件共同构成了工作流的执行引擎：
- 节点函数处理业务逻辑，更新状态
- 边条件根据更新后的状态决定下一步执行哪个节点
- 这种分离使得业务逻辑和控制逻辑可以独立设计和测试

## 核心要点

- **节点函数应保持简洁专注**：每个节点负责单一职责
- **边条件实现业务规则**：将复杂的业务规则编码为条件逻辑
- **类型系统保障正确性**：节点函数输入输出类型与状态定义一致
- **条件函数支持动态路由**：基于运行时状态智能选择执行路径
- **错误处理集成到工作流**：节点异常可以通过边条件路由到错误处理节点

## 简单示例

### 基础节点函数示例

```python
# 示例：节点函数与边条件基础
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, END, START

# 定义状态
class ProcessingState(TypedDict):
    """处理状态"""
    input_data: str
    processing_stage: Literal["start", "processing", "validating", "completed"]
    result: str
    errors: list[str]
    should_retry: bool

# 定义节点函数
def start_processing(state: ProcessingState):
    """开始处理节点"""
    print(f"开始处理数据: {state['input_data']}")
    return {"processing_stage": "processing"}

def process_data(state: ProcessingState):
    """数据处理节点"""
    input_data = state["input_data"]
    
    # 简单的处理逻辑
    if len(input_data) < 5:
        return {
            "result": f"处理失败: 数据太短",
            "errors": ["数据长度不足"],
            "processing_stage": "completed"
        }
    
    processed = input_data.upper()
    return {
        "result": f"处理成功: {processed}",
        "processing_stage": "validating"
    }

def validate_result(state: ProcessingState):
    """验证结果节点"""
    result = state["result"]
    
    if "失败" in result:
        return {
            "errors": state.get("errors", []) + ["验证失败"],
            "should_retry": True,
            "processing_stage": "completed"
        }
    
    return {
        "processing_stage": "completed",
        "should_retry": False
    }

def handle_error(state: ProcessingState):
    """错误处理节点"""
    print(f"处理过程中发生错误: {state['errors']}")
    
    if state.get("should_retry", False):
        return {"processing_stage": "processing"}  # 重试处理
    else:
        return {"processing_stage": "completed"}

# 定义条件函数
def decide_next_step(state: ProcessingState) -> str:
    """决定下一步的条件函数"""
    stage = state.get("processing_stage", "start")
    
    if stage == "start":
        return "process"
    elif stage == "processing":
        return "validate"
    elif stage == "validating":
        if state.get("errors"):
            return "error_handler"
        else:
            return "end"
    elif stage == "completed":
        return "end"
    else:
        return "error_handler"

# 构建工作流
workflow = StateGraph(ProcessingState)

# 添加节点
workflow.add_node("start", start_processing)
workflow.add_node("process", process_data)
workflow.add_node("validate", validate_result)
workflow.add_node("error_handler", handle_error)

# 添加条件边
workflow.add_edge(START, "start")
workflow.add_conditional_edges(
    "start",
    lambda state: "process",
    {"process": "process"}
)
workflow.add_conditional_edges(
    "process",
    lambda state: "validate",
    {"validate": "validate"}
)
workflow.add_conditional_edges(
    "validate",
    decide_next_step,
    {
        "error_handler": "error_handler",
        "end": END
    }
)
workflow.add_conditional_edges(
    "error_handler",
    lambda state: "process" if state.get("should_retry") else "end",
    {
        "process": "process",
        "end": END
    }
)

# 编译工作流
graph = workflow.compile()

# 测试执行
print("=== 测试1: 正常流程 ===")
result1 = graph.invoke({
    "input_data": "hello world",
    "processing_stage": "start",
    "result": "",
    "errors": [],
    "should_retry": False
})
print(f"结果: {result1['result']}")

print("\n=== 测试2: 错误流程 ===")
result2 = graph.invoke({
    "input_data": "hi",
    "processing_stage": "start",
    "result": "",
    "errors": [],
    "should_retry": False
})
print(f"结果: {result2['result']}")
print(f"错误: {result2['errors']}")
```

### 复杂边条件示例

```python
# 示例：复杂边条件与权重
from typing import TypedDict
from langgraph.graph import StateGraph, END
import random

class DecisionState(TypedDict):
    """决策状态"""
    request_type: str
    priority: int
    processing_cost: float
    selected_path: str

def analyze_request(state: DecisionState):
    """分析请求节点"""
    # 基于请求类型和优先级计算处理成本
    cost_multiplier = 1.0
    
    if state["request_type"] == "complex":
        cost_multiplier = 2.0
    elif state["request_type"] == "simple":
        cost_multiplier = 0.5
    
    if state["priority"] >= 8:
        cost_multiplier *= 1.5  # 高优先级增加成本
    
    cost = 10.0 * cost_multiplier
    return {"processing_cost": cost}

def weighted_decision(state: DecisionState) -> str:
    """带权重的决策函数"""
    # 基于处理成本决定路径
    cost = state.get("processing_cost", 0)
    
    if cost > 20:
        # 高成本请求，80%概率走优化路径，20%走标准路径
        return random.choices(
            ["optimized_path", "standard_path"],
            weights=[0.8, 0.2]
        )[0]
    elif cost > 10:
        # 中等成本，50/50
        return random.choices(
            ["optimized_path", "standard_path"],
            weights=[0.5, 0.5]
        )[0]
    else:
        # 低成本，走标准路径
        return "standard_path"

def process_optimized(state: DecisionState):
    """优化处理路径"""
    print(f"使用优化路径处理，成本: {state['processing_cost']}")
    return {"selected_path": "optimized"}

def process_standard(state: DecisionState):
    """标准处理路径"""
    print(f"使用标准路径处理，成本: {state['processing_cost']}")
    return {"selected_path": "standard"}

# 构建带权重决策的工作流
decision_workflow = StateGraph(DecisionState)
decision_workflow.add_node("analyze", analyze_request)
decision_workflow.add_node("optimized", process_optimized)
decision_workflow.add_node("standard", process_standard)

decision_workflow.add_edge(START, "analyze")
decision_workflow.add_conditional_edges(
    "analyze",
    weighted_decision,
    {
        "optimized_path": "optimized",
        "standard_path": "standard"
    }
)
decision_workflow.add_edge("optimized", END)
decision_workflow.add_edge("standard", END)

# 测试权重决策
print("\n=== 权重决策测试 ===")
test_cases = [
    {"request_type": "complex", "priority": 9},
    {"request_type": "simple", "priority": 3},
    {"request_type": "normal", "priority": 5}
]

for i, test in enumerate(test_cases, 1):
    print(f"\n测试用例 {i}: {test}")
    state = {
        "request_type": test["request_type"],
        "priority": test["priority"],
        "processing_cost": 0,
        "selected_path": ""
    }
    result = decision_workflow.compile().invoke(state)
    print(f"选择路径: {result['selected_path']}")
```

### 动态节点生成

```python
# 动态节点生成示例
from typing import Callable, Dict, Any

class DynamicWorkflow:
    """动态工作流生成器"""
    
    def __init__(self):
        self.node_registry: Dict[str, Callable] = {}
    
    def register_node(self, name: str, func: Callable):
        """注册节点函数"""
        self.node_registry[name] = func
    
    def create_dynamic_edge(self, state: Dict[str, Any]) -> str:
        """动态边条件：根据状态选择节点"""
        # 基于状态特征动态选择节点
        request_type = state.get("request_type", "")
        
        # 查找匹配的节点
        for node_name in self.node_registry:
            if request_type in node_name:
                return node_name
        
        # 默认节点
        return "default_processor"
    
    def build_workflow(self, state_schema):
        """构建动态工作流"""
        from langgraph.graph import StateGraph, START, END
        
        workflow = StateGraph(state_schema)
        
        # 添加所有注册的节点
        for name, func in self.node_registry.items():
            workflow.add_node(name, func)
        
        # 添加动态边
        workflow.add_edge(START, "router")
        workflow.add_conditional_edges(
            "router",
            self.create_dynamic_edge,
            {name: name for name in self.node_registry}
        )
        
        # 所有节点都连接到结束
        for name in self.node_registry:
            workflow.add_edge(name, END)
        
        return workflow.compile()
```

### 节点函数装饰器

```python
# 节点函数装饰器
import time
from functools import wraps
from typing import Callable

def measure_performance(func: Callable):
    """性能测量装饰器"""
    @wraps(func)
    def wrapper(state: dict):
        start_time = time.time()
        
        try:
            result = func(state)
            execution_time = time.time() - start_time
            
            # 添加性能指标到结果
            if isinstance(result, dict):
                result["_performance"] = {
                    "execution_time": execution_time,
                    "node_name": func.__name__
                }
            
            return result
        except Exception as e:
            execution_time = time.time() - start_time
            print(f"节点 {func.__name__} 执行失败，耗时 {execution_time:.2f}s: {e}")
            raise
    
    return wrapper

def validate_input(schema: dict):
    """输入验证装饰器"""
    def decorator(func: Callable):
        @wraps(func)
        def wrapper(state: dict):
            # 验证输入字段
            for field, expected_type in schema.items():
                if field in state and not isinstance(state[field], expected_type):
                    raise TypeError(
                        f"字段 {field} 类型错误，期望 {expected_type}，实际 {type(state[field])}"
                    )
            
            return func(state)
        return wrapper
    return decorator

# 使用装饰器的节点函数
@measure_performance
@validate_input({"input_data": str, "priority": int})
def enhanced_processor(state: dict):
    """增强的处理器节点"""
    # 业务逻辑
    return {"result": f"处理完成: {state['input_data']}"}
```

## 常见问题

### Q1: 节点函数应该多大合适？
**A**: 节点函数应保持简洁，通常不超过50-100行。如果函数太大，应考虑拆分为多个节点或提取辅助函数。

### Q2: 条件函数可以访问哪些数据？
**A**: 条件函数接收完整的状态字典，可以访问所有状态字段。但应避免修改状态，只用于读取和决策。

### Q3: 如何处理节点函数的副作用？
**A**: 如果节点必须有副作用（如调用外部API），应确保：1) 记录副作用到状态；2) 支持幂等重试；3) 提供补偿机制。

### Q4: 边条件可以嵌套吗？
**A**: 是的，可以通过在条件函数中实现复杂逻辑来模拟嵌套条件。也可以使用多个条件边串联实现嵌套效果。

### Q5: 如何测试节点函数和边条件？
**A**: 测试策略：1) 单元测试节点函数；2) 集成测试边条件逻辑；3) 使用模拟状态测试完整工作流；4) 利用LangSmith进行可视化测试。

关键收获：
1. 节点函数应设计为纯函数，保持简洁和专注
2. 边条件实现了复杂的业务规则和控制逻辑
3. 条件函数支持基于运行时状态的智能路由
4. 权重边提供了概率性的分支选择
5. 装饰器和动态生成支持高级节点功能
