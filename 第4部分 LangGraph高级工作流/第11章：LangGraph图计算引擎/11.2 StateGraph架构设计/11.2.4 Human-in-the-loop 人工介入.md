# 11.2.4 Human-in-the-loop 人工介入

## 概念讲解

**Human-in-the-loop（HITL）**是指在工作流执行过程中引入人工判断和决策的设计模式。在 AI 应用开发中，这种模式尤为重要——某些关键操作需要人工审核、批准或修改后才能继续执行。

LangGraph 通过**中断机制（Interrupt）**实现人工介入。当工作流执行到需要人工判断的节点时，可以调用 `interrupt()` 函数暂停执行，等待外部输入后通过 `Command(resume=...)` 恢复执行。

### 人工介入的典型场景

1. **审批流程**：关键操作（如发送邮件、修改数据库）需要人工审批
2. **内容审核**：AI 生成的内容需要人工审核和修改
3. **工具调用确认**：敏感操作需要用户确认后再执行
4. **异常处理**：遇到不确定情况时请求人工指导

### 中断机制的工作原理

LangGraph 的中断机制遵循"暂停-等待-恢复"模型：

1. **暂停**：节点执行到 `interrupt()` 时，工作流暂停并保存当前状态
2. **等待**：返回中断信息给调用方，等待人工输入
3. **恢复**：调用方通过 `Command(resume=...)` 提供输入，工作流从暂停点继续

## 核心要点

- **interrupt() 函数暂停执行**：将控制权交还给调用方
- **Command(resume=...) 恢复执行**：将人工输入传递回中断点
- **中断信息可自定义**：可以传递任意结构化数据说明需要人工处理的内容
- **状态自动保存**：中断时自动创建检查点，支持长时间等待
- **支持条件路由**：根据人工输入决定后续执行路径

## 简单示例

### 基础人工审核

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command
from typing import TypedDict

class ReviewState(TypedDict):
    content: str
    review_status: str

def review_node(state: ReviewState):
    # 暂停执行，请求人工审核
    edited = interrupt({
        "instruction": "请审核并修改以下内容",
        "content": state["content"]
    })
    return {"content": edited, "review_status": "approved"}

# 构建审核工作流
builder = StateGraph(ReviewState)
builder.add_node("review", review_node)
builder.add_edge(START, "review")
builder.add_edge("review", END)

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "review-001"}}

# 首次执行，会触发中断
result = graph.invoke({"content": "这是一段待审核的内容", "review_status": ""}, config)
print(result.get("__interrupt__"))  
# 输出: [Interrupt(value={'instruction': '请审核...', 'content': '...'})]

# 人工审核后，恢复执行
final = graph.invoke(Command(resume="审核通过的内容"), config)
print(final)  
# 输出: {'content': '审核通过的内容', 'review_status': 'approved'}
```

### 审批决策模式

根据人工输入决定后续流程：

```python
from typing import Literal

def approval_node(state: ReviewState) -> Command[Literal["proceed", "cancel"]]:
    """审批节点，根据人工决策决定后续流程"""
    decision = interrupt({
        "question": "是否批准此操作？",
        "details": state["content"]
    })
    
    if decision == "approve":
        return Command(goto="proceed")
    else:
        return Command(goto="cancel")

def proceed_node(state: ReviewState):
    return {"review_status": "已批准并继续"}

def cancel_node(state: ReviewState):
    return {"review_status": "已拒绝并取消"}

# 构建带分支的审批流
builder = StateGraph(ReviewState)
builder.add_node("approval", approval_node)
builder.add_node("proceed", proceed_node)
builder.add_node("cancel", cancel_node)

builder.add_edge(START, "approval")
builder.add_edge("proceed", END)
builder.add_edge("cancel", END)

graph = builder.compile(checkpointer=MemorySaver())
```

## 进阶应用

### 工具调用前的中断

在实际应用中，经常需要在调用敏感工具前请求人工确认：

```python
from langchain_core.tools import tool

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """发送邮件工具，发送前需要人工确认"""
    response = interrupt({
        "action": "send_email",
        "to": to,
        "subject": subject,
        "body": body,
        "message": "确认发送此邮件？"
    })
    
    if response.get("approved"):
        # 实际发送邮件逻辑
        return f"邮件已发送至 {to}"
    return "邮件发送已取消"
```

### 带内容编辑的人工审核

允许人工在审核时修改内容：

```python
def content_review(state: ReviewState):
    """内容审核节点，允许人工编辑"""
    # 展示待审核内容
    review_result = interrupt({
        "action": "review",
        "current_content": state["content"],
        "instruction": "请编辑内容或原样返回"
    })
    
    # review_result 包含人工编辑后的内容
    edited_content = review_result.get("edited_content", state["content"])
    return {"content": edited_content}

# 恢复时传入编辑后的内容
graph.invoke(
    Command(resume={"edited_content": "编辑后的内容"}),
    config
)
```

## 常见问题

### Q1: interrupt() 和条件边有什么区别？

**A**: 条件边是在图结构层面定义分支逻辑，在编译时确定；`interrupt()` 是在运行时动态暂停执行，等待外部输入。两者配合使用可以实现复杂的 HITL 流程。

### Q2: 中断后状态会丢失吗？

**A**: 不会。配置检查点器后，中断时的状态会自动保存。即使进程重启，也能通过 `thread_id` 恢复并继续等待人工输入。

### Q3: 如何取消一个已中断的工作流？

**A**: 可以通过 `Command(goto="cancel_node")` 将工作流引导至取消节点，或者直接不调用 `resume` 让工作流保持暂停状态。

### Q4: 中断可以同时等待多个人工输入吗？

**A**: 每个中断点是独立的，但可以在不同节点设置多个中断点。工作流会在每个中断点依次暂停，等待人工输入后继续。

## 本节总结

Human-in-the-loop 模式是 LangGraph 的核心特性之一，通过中断机制实现了优雅的人工介入流程。关键要点包括：

1. `interrupt()` 函数暂停执行并等待人工输入
2. `Command(resume=...)` 恢复执行并传递人工决策
3. 支持审批、审核、编辑等多种人工介入模式
4. 配合检查点机制，支持长时间等待和进程恢复
5. 可以与条件边结合实现复杂的 HITL 分支逻辑
