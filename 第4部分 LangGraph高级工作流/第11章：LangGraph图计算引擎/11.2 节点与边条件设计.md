# 11.2 节点与边条件设计

## 概念讲解

### 节点函数（Node Functions）

节点函数是工作流中的基本处理单元，接收当前状态，返回状态更新。

```mermaid
graph LR
    A[输入状态] --> B[节点函数]
    B --> C[返回状态更新]
    C --> D[合并到全局状态]
```

**关键特性：**
- **纯函数设计**：接收状态，返回更新，避免副作用
- **类型安全**：函数签名与状态类型一致
- **专注单一职责**：每个节点只处理一个任务

### 边（Edge）的类型

| 类型 | API | 说明 |
|------|-----|------|
| 无条件边 | `add_edge()` | 固定流转到下一个节点 |
| 条件边 | `add_conditional_edges()` | 基于状态动态选择下一个节点 |

### 条件边工作原理

```python
def route(state: State) -> Literal["node_a", "node_b", "node_c"]:
    # 根据状态决定下一个节点
    if condition_a:
        return "node_a"
    elif condition_b:
        return "node_b"
    else:
        return "node_c"

graph.add_conditional_edges("current_node", route)
```

## 核心要点

### 条件边使用

`add_conditional_edges`是LangGraph实现分支逻辑的核心API：

```python
from typing import Literal
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    aggregate: Annotated[list, operator.add]
    which: str

def a(state: State):
    return {"aggregate": ["A"], "which": "c"}

def b(state: State):
    return {"aggregate": ["B"]}

def c(state: State):
    return {"aggregate": ["C"]}

builder = StateGraph(State)
builder.add_node(a)
builder.add_node(b)
builder.add_node(c)
builder.add_edge(START, "a")
builder.add_edge("b", END)
builder.add_edge("c", END)

# 条件边：根据state["which"]决定下一步
def conditional_edge(state: State) -> Literal["b", "c"]:
    return state["which"]

builder.add_conditional_edges("a", conditional_edge)
graph = builder.compile()
```

### 节点返回值规范

节点函数返回一个字典，只包含需要更新的字段：

```python
def process_node(state: State) -> dict:
    # 只返回需要更新的字段
    return {
        "result": "处理完成",
        "next_step": "validate"
    }
```

## 简单示例

### 条件分支工作流

```python
from typing import Literal
from langgraph.graph import StateGraph, START, END

class RouterState(TypedDict):
    topic: str
    response: str

def classify(state: RouterState) -> dict:
    """分类节点：判断问题类型"""
    topic = state["topic"].lower()
    if "code" in topic or "python" in topic:
        return {"response": "coding"}
    elif "weather" in topic:
        return {"response": "weather"}
    else:
        return {"response": "general"}

def handle_coding(state: RouterState) -> dict:
    return {"response": "这里是编程助手..."}

def handle_weather(state: RouterState) -> dict:
    return {"response": "天气查询中..."}

def handle_general(state: RouterState) -> dict:
    return {"response": "通用助手回复..."}

# 路由函数
def router(state: RouterState) -> Literal["coding", "weather", "general"]:
    return state["response"]

# 构建图
graph = StateGraph(RouterState)
graph.add_node("classify", classify)
graph.add_node("coding", handle_coding)
graph.add_node("weather", handle_weather)
graph.add_node("general", handle_general)

graph.add_edge(START, "classify")
graph.add_conditional_edges("classify", router)
graph.add_edge("coding", END)
graph.add_edge("weather", END)
graph.add_edge("general", END)

app = graph.compile()
```

### 错误处理与重试

```python
def process_data(state: ProcessingState) -> dict:
    try:
        result = heavy_computation(state["input"])
        return {"result": result, "status": "success"}
    except Exception as e:
        return {"error": str(e), "status": "failed"}

def handle_error(state: ProcessingState) -> dict:
    if state.get("retry_count", 0) < 3:
        return {"retry_count": state.get("retry_count", 0) + 1, 
                "status": "retry"}
    return {"status": "exhausted"}

def decide_next(state: ProcessingState) -> Literal["success", "retry", "end"]:
    if state["status"] == "success":
        return "success"
    elif state["status"] == "failed":
        return "retry"
    return "end"

graph = StateGraph(ProcessingState)
graph.add_node("process", process_data)
graph.add_node("error_handler", handle_error)
graph.add_node("success_node", lambda s: {"final": s["result"]})

graph.add_edge(START, "process")
graph.add_conditional_edges("process", decide_next, {
    "success": "success_node",
    "retry": "error_handler",
    "end": END
})
graph.add_edge("error_handler", "process")
graph.add_edge("success_node", END)
```

## 进阶应用

### 子图嵌套

```python
# 子图
subgraph = StateGraph(SubState)
subgraph.add_node("step1", step1_func)
subgraph.add_node("step2", step2_func)
subgraph.add_edge(START, "step1")
subgraph.add_edge("step1", "step2")
subgraph.add_edge("step2", END)
subgraph_app = subgraph.compile()

# 主图中使用子图
main_graph = StateGraph(MainState)
main_graph.add_node("subgraph", subgraph_app)
main_graph.add_edge(START, "subgraph")
main_graph.add_edge("subgraph", END)
```

### 动态节点选择

```python
def dynamic_router(state: State) -> str:
    """根据运行时状态动态选择节点"""
    available_nodes = ["fast_path", "normal_path", "deep_path"]
    
    # 根据复杂度选择

        return "deep_path"

        return "normal_path"
    return "fast_path"

graph.add_conditional_edges("router", dynamic_router, {
```

## 常见问题

### Q: 节点函数应该多大合适？

**A:** 节点函数应保持简洁，通常不超过50-100行。如果函数太大，应拆分为多个节点。

### Q: 条件函数可以修改状态吗？

**A:** 条件函数只用于读取状态和决策，不应修改状态。状态修改应在节点函数中进行。

### Q: 如何处理节点函数的错误？

**A:** 推荐模式：
1. 节点函数捕获异常并返回错误状态
2. 条件边根据错误状态路由到错误处理节点
3. 错误处理节点决定重试或终止

## 本节总结

节点与边是StateGraph的两大核心组件：
- **节点函数**：封装业务逻辑，接收状态返回更新
- **条件边**：实现分支逻辑，基于状态动态路由
- **错误处理**：通过条件边路由到错误处理节点
- **子图嵌套**：支持模块化设计和代码复用
