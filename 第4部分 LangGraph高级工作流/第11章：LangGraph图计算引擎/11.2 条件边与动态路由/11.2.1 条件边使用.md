# 11.2.1 条件边使用

## 概念讲解

### 什么是条件边

条件边（Conditional Edge）是LangGraph中实现动态路由的核心机制。它根据运行时状态，决定下一步执行哪个节点。与固定边不同，条件边的目标节点不是预先确定的，而是由路由函数动态计算。

```
节点A
  ↓
[条件函数] → 返回 "B" 或 "C"
  ↓
节点B 或 节点C  ← 动态选择
```

### 为什么需要条件边

在实际业务中，很多流程需要根据条件分支：
- 用户意图不同，走不同处理路径
- 搜索结果质量不同，决定是否需要人工介入
- 任务复杂度不同，选择简单或复杂处理流程

## 核心要点

1. **add_conditional_edges**：添加条件边
2. **路由函数**：返回目标节点名称的字符串或列表
3. **path_map**：可选，限定合法的目标节点
4. **多目标**：返回列表可触发并行执行

## 简单示例

### 基本条件边

```python
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    intent: str
    response: str

graph = StateGraph(State)

# 定义节点
def classify(state: State) -> State:
    # 模拟意图分类
    return {"intent": "greeting"}

def handle_greeting(state: State) -> State:
    return {"response": "你好！有什么可以帮助你的？"}

def handle_question(state: State) -> State:
    return {"response": "正在查找答案..."}

# 添加节点
graph.add_node("classify", classify)
graph.add_node("greeting", handle_greeting)
graph.add_node("question", handle_question)

# 路由函数
def route_by_intent(state: State) -> str:
    if state["intent"] == "greeting":
        return "greeting"
    else:
        return "question"

# 添加条件边
graph.add_edge(START, "classify")
graph.add_conditional_edges(
    "classify",
    route_by_intent,
    {"greeting": "greeting", "question": "question"}  # path_map
)
graph.add_edge("greeting", END)
graph.add_edge("question", END)

app = graph.compile()
```

### 简化写法

```python
# 路由函数直接返回节点名
def route(state: State) -> str:
    return "greeting" if state["intent"] == "greeting" else "question"

# 不需要path_map，LangGraph自动推断
graph.add_conditional_edges("classify", route)
```

## 进阶应用

### 多目标并行

```python
def route_parallel(state: State) -> list:
    """返回多个节点名，触发并行执行"""
    return ["search", "analyze"]

graph.add_conditional_edges("classify", route_parallel)
```

### 结束条件

```python
def check_done(state: State) -> str:
    """判断是否需要继续"""
    if state.get("response"):
        return END  # 直接结束
    return "retry"

graph.add_conditional_edges("check", check_done)
```

## 常见问题

**Q: 条件边和固定边可以同时使用吗？**

A: 可以。固定边用于确定的流转，条件边用于动态分支。

**Q: path_map是必须的吗？**

A: 不是必须的。但如果路由函数可能返回不合法的节点名，使用path_map可以做验证。

**Q: 条件边支持嵌套吗？**

A: 支持。一个节点可以有多条条件边，形成复杂的路由树。

## 本节总结

条件边是LangGraph实现动态路由的关键。通过路由函数根据状态决定下一步，可以灵活应对各种分支场景。结合固定边使用，能构建复杂的业务流程图。
