# 13.1.4 多代理通信协议

## 概念讲解

在多代理协作系统中，**通信协议**定义了代理之间交换信息和协调行为的方式。良好的通信协议是多代理系统高效协作的基础。

LangGraph 的多代理通信基于**消息传递（Message Passing）**模式。每个代理可以发送消息、接收消息，并根据接收到的消息做出决策。代理间的通信通过**共享状态**和**消息队列**两种机制实现。

### 通信模式

多代理系统中有三种主要通信模式：

1. **直接通信**：代理 A 直接向代理 B 发送消息
2. **发布-订阅**：代理发布消息到主题，订阅该主题的代理接收消息
3. **请求-响应**：代理 A 发送请求，等待代理 B 的响应后继续

### LangGraph 中的消息传递

LangGraph 使用 `messages` 字段作为代理间通信的载体。每条消息包含：
- **角色（role）**：标识消息来源（user、assistant、tool 等）
- **内容（content）**：消息的具体内容
- **元数据（metadata）**：附加信息（如代理名称、任务 ID 等）

## 核心要点

- **消息是代理间通信的载体**：通过共享的 messages 状态传递信息
- **支持同步和异步通信**：可以等待响应或继续执行
- **消息路由通过图结构定义**：边和条件决定了消息流向
- **子图天然支持代理隔离**：每个子图有独立的消息空间
- **Command 支持跨代理控制**：可以动态引导消息到其他代理

## 简单示例

### 基础双代理通信

```python
from langgraph.graph import StateGraph, START, END
from langchain_core.messages import HumanMessage, AIMessage
from typing import TypedDict, Annotated
import operator

class AgentState(TypedDict):
    messages: Annotated[list, operator.add]
    current_agent: str

# 代理 A：负责分析任务
def analyst_agent(state: AgentState):
    """分析代理"""
    last_msg = state["messages"][-1].content if state["messages"] else ""
    analysis = f"分析结果：{last_msg} 的关键点是..."
    return {
        "messages": [AIMessage(content=analysis, name="analyst")],
        "current_agent": "writer"
    }

# 代理 B：负责撰写报告
def writer_agent(state: AgentState):
    """写作代理"""
    analysis = state["messages"][-1].content
    report = f"基于分析：{analysis}\n\n生成完整报告..."
    return {
        "messages": [AIMessage(content=report, name="writer")],
        "current_agent": "end"
    }

# 路由函数
def route_agent(state: AgentState) -> str:
    return state.get("current_agent", "end")

builder = StateGraph(AgentState)
builder.add_node("analyst", analyst_agent)
builder.add_node("writer", writer_agent)

builder.add_edge(START, "analyst")
builder.add_conditional_edges("analyst", route_agent, {
    "writer": "writer",
    "end": END
})
builder.add_edge("writer", END)

graph = builder.compile()

# 执行
initial_messages = [HumanMessage(content="请分析人工智能对教育的影响")]
result = graph.invoke({
    "messages": initial_messages,
    "current_agent": "analyst"
})

for msg in result["messages"]:
    print(f"[{msg.name}] {msg.content}")
```

### 请求-响应模式

```python
from langgraph.types import Command

def requester_agent(state: AgentState):
    """请求代理：发送请求并等待响应"""
    request = "请帮我查询以下数据..."
    return {
        "messages": [AIMessage(content=request, name="requester")],
        "current_agent": "responder"
    }

def responder_agent(state: AgentState):
    """响应代理：处理请求并返回结果"""
    request = state["messages"][-1].content
    response = f"查询结果：数据已找到..."
    return {
        "messages": [AIMessage(content=response, name="responder")],
        "current_agent": "requester"
    }

def finalize(state: AgentState):
    """汇总结果"""
    return {"current_agent": "end"}
```

## 进阶应用

### 多代理广播模式

一个代理向多个代理广播消息：

```python
from langgraph.types import Send

class BroadcastState(TypedDict):
    task: str
    results: Annotated[list, operator.add]

def broadcaster(state: BroadcastState):
    """广播任务给多个代理"""
    return {"results": []}  # 初始化结果列表

def worker_a(state: BroadcastState):
    return {"results": [f"Worker A 完成: {state['task']}"]}

def worker_b(state: BroadcastState):
    return {"results": [f"Worker B 完成: {state['task']}"]}

def worker_c(state: BroadcastState):
    return {"results": [f"Worker C 完成: {state['task']}"]}

def collector(state: BroadcastState):
    """收集所有结果"""
    return {"results": state["results"]}

def fan_out(state: BroadcastState):
    """广播到多个 worker"""
    return [
        Send("worker_a", state),
        Send("worker_b", state),
        Send("worker_c", state)
    ]

builder = StateGraph(BroadcastState)
builder.add_node("broadcaster", broadcaster)
builder.add_node("worker_a", worker_a)
builder.add_node("worker_b", worker_b)
builder.add_node("worker_c", worker_c)
builder.add_node("collector", collector)

builder.add_edge(START, "broadcaster")
builder.add_conditional_edges("broadcaster", fan_out, ["worker_a", "worker_b", "worker_c"])
builder.add_edge("worker_a", "collector")
builder.add_edge("worker_b", "collector")
builder.add_edge("worker_c", "collector")
builder.add_edge("collector", END)
```

### 子图代理通信

每个子图作为一个独立代理：

```python
def create_research_agent():
    """研究代理（子图）"""
    def research(state: AgentState):
        return {"messages": [AIMessage(content="研究完成", name="researcher")]}
    
    builder = StateGraph(AgentState)
    builder.add_node("research", research)
    builder.add_edge(START, "research")
    builder.add_edge("research", END)
    return builder.compile()

def create_summary_agent():
    """总结代理（子图）"""
    def summary(state: AgentState):
        return {"messages": [AIMessage(content="总结完成", name="summarizer")]}
    
    builder = StateGraph(AgentState)
    builder.add_node("summary", summary)
    builder.add_edge(START, "summary")
    builder.add_edge("summary", END)
    return builder.compile()

# 主图编排代理通信
research_agent = create_research_agent()
summary_agent = create_summary_agent()

main_builder = StateGraph(AgentState)
main_builder.add_node("research", research_agent)
main_builder.add_node("summary", summary_agent)
main_builder.add_edge(START, "research")
main_builder.add_edge("research", "summary")
main_builder.add_edge("summary", END)
```

## 常见问题

### Q1: 代理间如何共享上下文？

**A**: 通过共享的 `messages` 状态字段。每个代理的消息都会添加到列表中，后续代理可以读取所有历史消息。

### Q2: 如何实现代理间的异步通信？

**A**: 使用并行执行模式，多个代理节点并行执行，通过归约器合并结果。

### Q3: 如何防止消息循环？

**A**: 通过合理设计图结构和路由条件，确保消息最终能到达 END 节点。也可以设置最大迭代次数。

### Q4: 代理间通信的性能开销如何？

**A**: 消息传递主要是在内存中操作，开销很小。但对于大量消息或大型状态，建议只传递必要的信息。

## 本节总结

多代理通信协议是 LangGraph 多代理协作系统的核心：

1. 消息是代理间通信的主要载体
2. 支持直接通信、发布-订阅和请求-响应模式
3. Send API 支持广播和扇出通信
4. 子图提供代理隔离和模块化设计

掌握通信协议设计，是构建高效多代理系统的关键。
