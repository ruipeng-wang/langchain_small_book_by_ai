# 13.1.1 主从架构与对等架构

多代理协作系统的架构设计决定了代理之间的协作效率、系统可扩展性和故障恢复能力。LangGraph v1.2.22通过灵活的图计算引擎支持多种多代理架构模式，其中主从架构与对等架构是最基础的两种设计模式，适用于不同的应用场景和业务需求。

## 概念讲解

### 多代理系统的基本概念
在多代理系统中，多个独立的智能代理通过协作完成复杂任务：

1. **代理（Agent）**：具有自主决策能力的智能单元，能够感知环境、做出决策并执行动作
2. **协作（Collaboration）**：代理之间通过通信、协调和资源共享共同完成任务
3. **系统目标**：通过分布式智能提高问题解决能力和系统鲁棒性

### 主从架构（Master-Slave Architecture）
主从架构是一种中心化的协作模式：

1. **主代理（Master）**：负责任务分解、调度和结果聚合，具有全局视角
2. **从代理（Slave/Worker）**：执行具体子任务，向主代理报告结果
3. **通信模式**：星型拓扑，所有从代理只与主代理通信
4. **决策中心**：所有重要决策由主代理统一做出

主从架构适用于任务结构清晰、需要集中控制的场景，如数据处理流水线、分布式计算等。

### 对等架构（Peer-to-Peer Architecture）
对等架构是一种去中心化的协作模式：

1. **对等代理（Peer）**：所有代理具有相同地位和功能，既是任务执行者也是协调者
2. **通信模式**：网状拓扑，代理之间可以直接通信
3. **决策分布**：决策权分散在各个代理，通过协商达成共识
4. **自组织性**：系统能够自适应调整，无单点故障

对等架构适用于动态环境、需要高度自治和容错能力的场景，如多智能体系统、区块链等。

### 混合架构
实际应用中常常采用混合架构：

1. **分层主从**：多级主从结构，上级主代理管理下级主代理
2. **主从+对等**：核心层采用主从架构，执行层采用对等架构
3. **动态架构**：根据任务需求动态切换架构模式

## 核心要点

### LangGraph v1.2.22的多代理支持
LangGraph v1.2.22通过以下机制支持多代理协作：

1. **SubAgentMiddleware**：专门的中间件用于管理子代理任务委派
2. **CompiledSubAgent**：将自定义LangGraph工作流包装为可调用的子代理
3. **StateGraph集成**：支持在状态图中嵌入代理节点和子图
4. **并行执行**：支持多个代理节点并行执行，提高协作效率

### 主从架构的设计原则
在主从架构设计中需要考虑：

1. **任务分解粒度**：合理划分子任务，平衡并行度和协调开销
2. **负载均衡**：根据从代理能力动态分配任务
3. **容错机制**：主代理监控从代理状态，失败任务重新分配
4. **结果聚合**：有效整合从代理的执行结果

### 对等架构的设计原则
在对等架构设计中需要考虑：

1. **通信协议**：设计高效的代理间通信协议
2. **共识机制**：在无中心情况下达成一致决策
3. **冲突解决**：处理代理间的资源竞争和任务冲突
4. **自组织算法**：实现系统的自组织和自适应调整

## 简单示例

### 示例1：主从架构实现
以下示例展示了如何使用LangGraph实现主从架构的多代理系统：

```python
import asyncio
from typing import TypedDict, Annotated, List, Dict, Any
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain.agents import create_agent
from deepagents.middleware.subagents import SubAgentMiddleware
from deepagents import CompiledSubAgent
import operator

class MasterSlaveState(TypedDict):
    """主从架构状态定义"""
    master_instructions: str
    subtasks: List[Dict[str, Any]]
    assigned_tasks: Annotated[List[Dict[str, Any]], operator.add]
    completed_tasks: Annotated[List[Dict[str, Any]], operator.add]
    slave_results: Annotated[List[Dict[str, Any]], operator.add]
    final_result: str

def create_data_processing_subagent() -> CompiledSubAgent:
    """创建数据处理子代理"""
    from langgraph.graph import StateGraph
    
    class DataProcessingState(TypedDict):
        input_data: str
        processed_data: str
        processing_steps: List[str]
    
    def data_cleaning(state: DataProcessingState) -> dict:
        """数据清洗"""
        import re
        # 移除多余空格和特殊字符
        cleaned = re.sub(r'\s+', ' ', state["input_data"]).strip()
        return {
            "processed_data": cleaned,
            "processing_steps": ["数据清洗完成"]
        }
    
    def data_transformation(state: DataProcessingState) -> dict:
        """数据转换"""
        # 转换为大写
        transformed = state["processed_data"].upper()
        return {
            "processed_data": transformed,
            "processing_steps": ["数据转换完成"]
        }
    
    def data_validation(state: DataProcessingState) -> dict:
        """数据验证"""
        # 检查数据是否有效
        is_valid = len(state["processed_data"]) > 0
        return {
            "processed_data": state["processed_data"],
            "processing_steps": ["数据验证完成" if is_valid else "数据验证失败"]
        }
    
    # 构建数据处理工作流
    builder = StateGraph(DataProcessingState)
    builder.add_node("clean", data_cleaning)
    builder.add_node("transform", data_transformation)
    builder.add_node("validate", data_validation)
    
    builder.add_edge(START, "clean")
    builder.add_edge("clean", "transform")
    builder.add_edge("transform", "validate")
    builder.add_edge("validate", END)
    
    data_processing_graph = builder.compile()
    
    # 包装为CompiledSubAgent
    return CompiledSubAgent(
        name="data_processor",
        description="数据处理子代理，负责数据清洗、转换和验证",
        runnable=data_processing_graph
    )

def create_analysis_subagent() -> CompiledSubAgent:
    """创建分析子代理"""
    from langgraph.graph import StateGraph
    
    class AnalysisState(TypedDict):
        input_text: str
        analysis_result: Dict[str, Any]
        insights: List[str]
    
    def sentiment_analysis(state: AnalysisState) -> dict:
        """情感分析（简化版）"""
        text = state["input_text"]
        # 简化版情感分析
        positive_words = ["好", "优秀", "满意", "成功", "高兴"]
        negative_words = ["坏", "糟糕", "失望", "失败", "难过"]
        
        positive_count = sum(1 for word in positive_words if word in text)
        negative_count = sum(1 for word in negative_words if word in text)
        
        sentiment = "积极" if positive_count > negative_count else "消极" if negative_count > positive_count else "中性"
        
        return {
            "analysis_result": {
                "sentiment": sentiment,
                "positive_score": positive_count,
                "negative_score": negative_count
            },
            "insights": [f"情感分析: {sentiment}"]
        }
    
    def topic_extraction(state: AnalysisState) -> dict:
        """主题提取（简化版）"""
        # 简化版主题提取
        common_topics = ["技术", "产品", "市场", "服务", "质量"]
        text = state["input_text"]
        
        detected_topics = [topic for topic in common_topics if topic in text]
        
        current_result = state["analysis_result"]
        current_result["topics"] = detected_topics
        
        return {
            "analysis_result": current_result,
            "insights": state["insights"] + [f"提取主题: {detected_topics}"]
        }
    
    # 构建分析工作流
    builder = StateGraph(AnalysisState)
    builder.add_node("sentiment", sentiment_analysis)
    builder.add_node("topics", topic_extraction)
    
    builder.add_edge(START, "sentiment")
    builder.add_edge("sentiment", "topics")
    builder.add_edge("topics", END)
    
    analysis_graph = builder.compile()
    
    return CompiledSubAgent(
        name="text_analyzer",
        description="文本分析子代理，负责情感分析和主题提取",
        runnable=analysis_graph
    )

def master_task_decomposition(state: MasterSlaveState) -> dict:
    """主代理：任务分解"""
    print("[主代理] 开始任务分解...")
    
    # 根据指令分解任务
    instruction = state["master_instructions"]
    
    # 简化版任务分解逻辑
    subtasks = []
    
    if "处理" in instruction or "清洗" in instruction:
        subtasks.append({
            "id": "task_data_process",
            "type": "data_processing",
            "description": "数据处理任务",
            "input": instruction,
            "assigned_to": None
        })
    
    if "分析" in instruction or "情感" in instruction:
        subtasks.append({
            "id": "task_text_analyze",
            "type": "text_analysis",
            "description": "文本分析任务",
            "input": instruction,
            "assigned_to": None
        })
    
    print(f"[主代理] 分解出 {len(subtasks)} 个子任务")
    return {"subtasks": subtasks}

def master_task_assignment(state: MasterSlaveState) -> dict:
    """主代理：任务分配"""
    print("[主代理] 开始任务分配...")
    
    assigned_tasks = []
    for task in state["subtasks"]:
        # 根据任务类型分配
        if task["type"] == "data_processing":
            task["assigned_to"] = "data_processor"
        elif task["type"] == "text_analysis":
            task["assigned_to"] = "text_analyzer"
        
        assigned_tasks.append(task)
        print(f"[主代理] 分配任务 {task['id']} 给 {task['assigned_to']}")
    
    return {"assigned_tasks": assigned_tasks}

def master_result_aggregation(state: MasterSlaveState) -> dict:
    """主代理：结果聚合"""
    print("[主代理] 聚合子代理结果...")
    
    # 整合所有子代理的结果
    all_results = []
    for result in state["slave_results"]:
        all_results.append({
            "task_id": result.get("task_id"),
            "agent": result.get("agent"),
            "result": result.get("result"),
            "timestamp": result.get("timestamp")
        })
    
    # 生成最终报告
    final_report = f"主从架构任务执行报告\n"
    final_report += f"总任务数: {len(state['assigned_tasks'])}\n"
    final_report += f"完成数: {len(state['completed_tasks'])}\n"
    final_report += f"结果数: {len(state['slave_results'])}\n\n"
    
    for i, result in enumerate(all_results, 1):
        final_report += f"任务{i}: {result['task_id']} - {result['agent']}\n"
        final_report += f"  结果: {str(result['result'])[:100]}...\n"
    
    print(f"[主代理] 生成最终报告，长度: {len(final_report)} 字符")
    return {"final_result": final_report}

def build_master_slave_system():
    """构建主从架构系统"""
    # 创建子代理
    data_processor = create_data_processing_subagent()
    text_analyzer = create_analysis_subagent()
    
    # 创建主代理工作流
    builder = StateGraph(MasterSlaveState)
    
    builder.add_node("decompose", master_task_decomposition)
    builder.add_node("assign", master_task_assignment)
    builder.add_node("aggregate", master_result_aggregation)
    
    # 构建主工作流
    builder.add_edge(START, "decompose")
    builder.add_edge("decompose", "assign")
    builder.add_edge("assign", "aggregate")
    builder.add_edge("aggregate", END)
    
    # 创建包含子代理的主代理
    master_agent = create_agent(
        model="claude-sonnet-4-5-20250929",  # 示例模型
        middleware=[
            SubAgentMiddleware(
                default_model="claude-sonnet-4-5-20250929",
                default_tools=[],
                subagents=[data_processor, text_analyzer],
            )
        ],
    )
    
    return builder.compile(), master_agent

def run_master_slave_example():
    """运行主从架构示例"""
    print("=== 主从架构多代理系统示例 ===\n")
    
    # 构建系统
    master_workflow, master_agent = build_master_slave_system()
    
    # 初始状态
    initial_state = {
        "master_instructions": "请处理用户反馈数据并分析情感：用户表示产品质量优秀但售后服务需要改进，市场推广效果很好。",
        "subtasks": [],
        "assigned_tasks": [],
        "completed_tasks": [],
        "slave_results": [],
        "final_result": ""
    }
    
    # 执行主工作流
    print("1. 主代理工作流执行:")
    final_state = master_workflow.invoke(initial_state)
    
    print(f"\n2. 最终结果:")
    print("-" * 50)
    print(final_state["final_result"])
    print("-" * 50)
    
    print("\n3. 系统状态统计:")
    print(f"   子任务数: {len(final_state['subtasks'])}")
    print(f"   已分配任务: {len(final_state['assigned_tasks'])}")
    
    return final_state

# 运行示例
if __name__ == "__main__":
    run_master_slave_example()
```

### 示例2：对等架构实现
以下示例展示了如何使用LangGraph实现对等架构的多代理系统：

```python
def peer_to_peer_architecture():
    """对等架构实现"""
    import random
    import asyncio
    from typing import Dict, List, Any, Optional
    from dataclasses import dataclass
    from enum import Enum
    
    class TaskStatus(Enum):
        PENDING = "pending"
        IN_PROGRESS = "in_progress"
        COMPLETED = "completed"
        FAILED = "failed"
    
    @dataclass
    class Task:
        """任务定义"""
        id: str
        description: str
        requirements: List[str]
        status: TaskStatus
        assigned_to: Optional[str] = None
        result: Any = None
    
    @dataclass
    class PeerAgent:
        """对等代理定义"""
        id: str
        capabilities: List[str]
        current_tasks: List[Task]
        performance_score: float
        neighbors: List[str]  # 直接通信的邻居代理
        
        def can_handle(self, task: Task) -> bool:
            """检查代理是否能处理任务"""
            # 检查是否具备所需能力
            for req in task.requirements:
                if req not in self.capabilities:
                    return False
            return True
        
        def workload(self) -> float:
            """计算当前工作负载"""
            return len(self.current_tasks) / 10.0  # 简化计算
    
    class PeerToPeerSystem:
        """对等多代理系统"""
        
        def __init__(self):
            self.agents: Dict[str, PeerAgent] = {}
            self.tasks: Dict[str, Task] = {}
            self.communication_log: List[Dict[str, Any]] = []
            
        def add_agent(self, agent: PeerAgent):
            """添加代理到系统"""
            self.agents[agent.id] = agent
        
        def create_task(self, description: str, requirements: List[str]) -> str:
            """创建新任务"""
            task_id = f"task_{len(self.tasks) + 1}"
            task = Task(
                id=task_id,
                description=description,
                requirements=requirements,
                status=TaskStatus.PENDING
            )
            self.tasks[task_id] = task
            return task_id
        
        def broadcast_task(self, task_id: str, sender_id: str):
            """广播任务到网络"""
            task = self.tasks[task_id]
            sender = self.agents[sender_id]
            
            print(f"[{sender_id}] 广播任务 {task_id}: {task.description}")
            
            # 记录通信
            self.communication_log.append({
                "type": "broadcast",
                "sender": sender_id,
                "task": task_id,
                "timestamp": asyncio.get_event_loop().time()
            })
            
            # 广播给所有邻居
            for neighbor_id in sender.neighbors:
                if neighbor_id in self.agents:
                    self.receive_task_proposal(task_id, neighbor_id, sender_id)
        
        def receive_task_proposal(self, task_id: str, receiver_id: str, sender_id: str):
            """接收任务提案"""
            task = self.tasks[task_id]
            receiver = self.agents[receiver_id]
            
            # 检查是否能处理
            if receiver.can_handle(task):
                # 计算投标分数（基于能力和负载）
                capability_match = len([c for c in task.requirements if c in receiver.capabilities])
                load_factor = 1.0 / (1.0 + receiver.workload())
                bid_score = receiver.performance_score * capability_match * load_factor
                
                print(f"[{receiver_id}] 收到任务 {task_id} 提案，投标分数: {bid_score:.2f}")
                
                # 发送投标
                self.send_bid(task_id, receiver_id, sender_id, bid_score)
        
        def send_bid(self, task_id: str, bidder_id: str, receiver_id: str, bid_score: float):
            """发送投标"""
            print(f"[{bidder_id}] 向 {receiver_id} 发送任务 {task_id} 投标: {bid_score:.2f}")
            
            self.communication_log.append({
                "type": "bid",
                "sender": bidder_id,
                "receiver": receiver_id,
                "task": task_id,
                "bid_score": bid_score,
                "timestamp": asyncio.get_event_loop().time()
            })
            
            # 接收方处理投标
            self.process_bids(task_id, receiver_id)
        
        def process_bids(self, task_id: str, task_owner_id: str):
            """处理投标并选择最佳代理"""
            task = self.tasks[task_id]
            
            # 收集所有投标
            bids = []
            for log in self.communication_log:
                if (log["type"] == "bid" and log["task"] == task_id and 
                    log["receiver"] == task_owner_id):
                    bids.append({
                        "agent_id": log["sender"],
                        "score": log["bid_score"]
                    })
            
            if not bids:
                print(f"[{task_owner_id}] 任务 {task_id} 无投标，重新广播")
                self.broadcast_task(task_id, task_owner_id)
                return
            
            # 选择最高分
            best_bid = max(bids, key=lambda x: x["score"])
            winner_id = best_bid["agent_id"]
            
            print(f"[{task_owner_id}] 选择 {winner_id} 执行任务 {task_id}，得分: {best_bid['score']:.2f}")
            
            # 分配任务
            self.assign_task(task_id, winner_id, task_owner_id)
        
        def assign_task(self, task_id: str, agent_id: str, assigner_id: str):
            """分配任务给代理"""
            task = self.tasks[task_id]
            agent = self.agents[agent_id]
            
            task.assigned_to = agent_id
            task.status = TaskStatus.IN_PROGRESS
            agent.current_tasks.append(task)
            
            print(f"[{assigner_id}] 分配任务 {task_id} 给 {agent_id}")
            
            # 记录分配
            self.communication_log.append({
                "type": "assignment",
                "assigner": assigner_id,
                "assignee": agent_id,
                "task": task_id,
                "timestamp": asyncio.get_event_loop().time()
            })
            
            # 模拟任务执行
            self.execute_task(task_id, agent_id)
        
        def execute_task(self, task_id: str, agent_id: str):
            """执行任务"""
            task = self.tasks[task_id]
            agent = self.agents[agent_id]
            
            # 模拟执行过程
            print(f"[{agent_id}] 开始执行任务 {task_id}: {task.description}")
            
            # 模拟执行结果
            success = random.random() > 0.1  # 90%成功率
            
            if success:
                task.status = TaskStatus.COMPLETED
                task.result = {
                    "output": f"任务 {task_id} 完成结果",
                    "quality": random.uniform(0.7, 1.0),
                    "execution_time": random.uniform(0.5, 2.0)
                }
                print(f"[{agent_id}] 任务 {task_id} 执行成功")
            else:
                task.status = TaskStatus.FAILED
                print(f"[{agent_id}] 任务 {task_id} 执行失败")
                
                # 失败后重新广播
                self.broadcast_task(task_id, agent_id)
    
    def run_peer_to_peer_example():
        """运行对等架构示例"""
        print("=== 对等多代理系统示例 ===\n")
        
        # 创建系统
        system = PeerToPeerSystem()
        
        # 创建代理
        agents_data = [
            ("agent_1", ["数据分析", "机器学习"], ["agent_2", "agent_3"], 0.9),
            ("agent_2", ["自然语言处理", "文本分析"], ["agent_1", "agent_4"], 0.8),
            ("agent_3", ["数据清洗", "可视化"], ["agent_1", "agent_4"], 0.85),
            ("agent_4", ["机器学习", "深度学习"], ["agent_2", "agent_3"], 0.95),
        ]
        
        for agent_id, capabilities, neighbors, score in agents_data:
            agent = PeerAgent(
                id=agent_id,
                capabilities=capabilities,
                current_tasks=[],
                performance_score=score,
                neighbors=neighbors
            )
            system.add_agent(agent)
        
        # 创建任务
        tasks_data = [
            "分析用户行为数据并生成报告",
            "处理客户反馈文本进行情感分析",
            "清洗销售数据集并可视化",
            "训练用户偏好预测模型"
        ]
        
        task_ids = []
        for i, description in enumerate(tasks_data, 1):
            # 根据任务描述确定需求
            requirements = []
            if "数据" in description and "分析" in description:
                requirements.append("数据分析")
            if "文本" in description or "情感" in description:
                requirements.append("文本分析")
            if "清洗" in description:
                requirements.append("数据清洗")
            if "可视化" in description:
                requirements.append("可视化")
            if "训练" in description or "模型" in description:
                requirements.append("机器学习")
            
            task_id = system.create_task(description, requirements)
            task_ids.append(task_id)
            print(f"创建任务 {task_id}: {description}")
            print(f"  需求: {requirements}")
        
        print("\n=== 开始任务分配与执行 ===\n")
```

### 混合架构模式
在实际应用中，常常需要结合主从架构和对等架构的优点：

```python
def hybrid_architecture_pattern():
    """混合架构模式"""
    
    class HybridAgentSystem:
        """混合架构代理系统"""
        
        def __init__(self):
            self.master_agents = {}  # 主代理层
            self.worker_agents = {}  # 工作代理层
            self.peer_groups = {}    # 对等代理组
            self.task_queue = []
            self.execution_history = []
            
        def add_master_agent(self, agent_id: str, domain: str):
            """添加主代理"""
            self.master_agents[agent_id] = {
                "id": agent_id,
                "domain": domain,
                "managed_workers": [],
                "assigned_tasks": []
            }
        
        def add_worker_agent(self, agent_id: str, capabilities: list, master_id: str):
            """添加工作代理"""
            self.worker_agents[agent_id] = {
                "id": agent_id,
                "capabilities": capabilities,
                "master": master_id,
                "current_task": None,
                "performance": 1.0
            }
            
            # 注册到主代理
            if master_id in self.master_agents:
                self.master_agents[master_id]["managed_workers"].append(agent_id)
        
        def create_peer_group(self, group_id: str, agent_ids: list, coordination_strategy: str):
            """创建对等代理组"""
            self.peer_groups[group_id] = {
                "id": group_id,
                "members": agent_ids,
                "strategy": coordination_strategy,
                "shared_knowledge": {},
                "consensus_mechanism": "voting"  # 投票、协商、拍卖等
            }
            
            # 更新代理的对等关系
            for agent_id in agent_ids:
                if agent_id in self.worker_agents:
                    if "peer_groups" not in self.worker_agents[agent_id]:
                        self.worker_agents[agent_id]["peer_groups"] = []
                    self.worker_agents[agent_id]["peer_groups"].append(group_id)
        
        def submit_task(self, task_description: str, requirements: dict, priority: int = 1):
            """提交任务"""
            task_id = f"task_{len(self.task_queue) + 1}"
            task = {
                "id": task_id,
                "description": task_description,
                "requirements": requirements,
                "priority": priority,
                "status": "pending",
                "assigned_to": None,
                "result": None,
                "submission_time": asyncio.get_event_loop().time()
            }
            
            self.task_queue.append(task)
            self.task_queue.sort(key=lambda x: (-x["priority"], x["submission_time"]))
            
            print(f"提交任务 {task_id}: {task_description}")
            return task_id
        
        def dynamic_architecture_adjustment(self):
            """动态架构调整"""
            # 根据系统负载和任务特征动态调整架构
            
            # 1. 分析当前负载
            total_tasks = len(self.task_queue)
            busy_workers = sum(1 for w in self.worker_agents.values() if w["current_task"])
            load_ratio = busy_workers / max(1, len(self.worker_agents))
            
            # 2. 分析任务特征
            complex_tasks = 0
            for task in self.task_queue:
                if len(task.get("requirements", {})) > 3:
                    complex_tasks += 1
            
            complexity_ratio = complex_tasks / max(1, total_tasks)
            
            # 3. 决定架构模式
            if load_ratio > 0.8 and complexity_ratio > 0.5:
                # 高负载高复杂度：启用对等协作
                print("切换到对等协作模式")
                return "peer_collaboration"
            elif load_ratio < 0.3:
                # 低负载：集中控制
                print("切换到主从集中模式")
                return "master_slave"
            else:
                # 中等负载：混合模式
                print("使用混合模式")
                return "hybrid"
        
        def execute_with_hybrid_architecture(self):
            """使用混合架构执行任务"""
            architecture_mode = self.dynamic_architecture_adjustment()
            
            if architecture_mode == "master_slave":
                self.execute_master_slave()
            elif architecture_mode == "peer_collaboration":
                self.execute_peer_collaboration()
            else:
                self.execute_hybrid()
            
            # 记录执行历史
            self.execution_history.append({
                "timestamp": asyncio.get_event_loop().time(),
                "mode": architecture_mode,
                "pending_tasks": len(self.task_queue),
                "active_workers": sum(1 for w in self.worker_agents.values() if w["current_task"])
            })
```

## 常见问题

### Q1: 如何选择主从架构还是对等架构？
**A**: 架构选择指南：
1. **任务特征**：结构化任务选主从，复杂动态任务选对等
2. **系统规模**：小规模系统主从简单，大规模系统对等扩展性好
3. **可靠性要求**：主从有单点故障，对等容错性更好
4. **通信开销**：主从通信集中，对等通信分散但可能更复杂
5. **开发复杂度**：主从实现简单，对等实现复杂但灵活

### Q2: 主从架构中主代理单点故障如何解决？
**A**: 主代理单点故障解决方案：
1. **主从热备**：设置备用主代理，主故障时自动切换
2. **主代理集群**：多个主代理形成集群，选举领导
3. **状态同步**：主代理状态定期备份，故障时恢复
4. **去中心化备份**：部分决策权下放到从代理
5. **心跳检测**：监控主代理状态，及时故障转移

### Q3: 对等架构中如何避免通信风暴？
**A**: 避免通信风暴的策略：
1. **通信限制**：限制每个代理的通信频率和数量
2. **消息聚合**：聚合相关消息批量发送
3. **层级通信**：引入局部领导代理减少全网通信
4. **通信调度**：按优先级调度通信消息
5. **流量控制**：实施基于拥塞控制的流量管理

### Q4: 混合架构如何保证一致性？
**A**: 混合架构一致性保证：
1. **全局时钟**：使用逻辑时钟或物理时钟同步
2. **状态同步协议**：定期同步不同架构层的状态
3. **事务协调**：跨架构层的事务协调机制
4. **版本控制**：状态版本管理和冲突解决
5. **最终一致性**：接受短暂不一致，保证最终一致
