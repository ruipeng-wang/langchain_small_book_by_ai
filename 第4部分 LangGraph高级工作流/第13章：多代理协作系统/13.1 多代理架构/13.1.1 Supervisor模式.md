# 13.1.1 Supervisor模式

## 概念讲解

### 什么是Supervisor模式

Supervisor（监督者）模式是多代理协作中最常用的架构。一个专门的Supervisor Agent负责协调和分配任务给多个Worker Agents，最后汇总结果。

**类比理解**：就像项目经理管理一个团队。项目经理不亲自干活，而是：
1. 理解任务需求
2. 分配给合适的团队成员
3. 审核工作成果
4. 汇总交付

### Supervisor架构

```
用户请求
    ↓
Supervisor Agent（决策者）
    ↓
┌───┼───┐
↓   ↓   ↓
W1  W2  W3  ← Worker Agents（执行者）
↑   ↑   ↑
└───┼───┘
    ↓
Supervisor 汇总结果
    ↓
返回用户
```

## 核心要点

1. **Supervisor**：使用LLM决策，决定哪个Worker执行
2. **Worker**：专注于特定任务的专业代理
3. **共享状态**：通过StateGraph传递消息和结果
4. **Command**：Worker使用Command控制流转

## 简单示例

### Supervisor模式实现

```python
from langgraph.graph import StateGraph, START, END
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI

class TeamState(TypedDict):
    messages: Annotated[list, operator.add]
    next: str  # 下一步指示

# 创建Worker Agents
model = ChatOpenAI(model="gpt-4o-mini")

researcher = create_agent(
    model=model,
    tools=[search_tool],
    system_prompt="你是研究员，负责搜索和分析信息。"
)

writer = create_agent(
    model=model,
    tools=[],
    system_prompt="你是写手，负责根据信息撰写内容。"
)

# Supervisor路由函数
def supervisor_route(state: TeamState) -> str:
    last_message = state["messages"][-1]
    if "research" in last_message.content.lower():
        return "researcher"
    elif "write" in last_message.content.lower():
        return "writer"
    return END

# 构建图
graph = StateGraph(TeamState)
graph.add_node("researcher", researcher)
graph.add_node("writer", writer)

graph.add_edge(START, "supervisor")
graph.add_conditional_edges("supervisor", supervisor_route)
graph.add_edge("researcher", "supervisor")
graph.add_edge("writer", "supervisor")

app = graph.compile()
```

### 使用Command控制流转

```python
from langgraph.types import Command

def worker_node(state: TeamState) -> Command:
    # Worker完成任务后，告诉Supervisor下一步是谁
    return Command(
        update={"messages": [...], "result": "done"},
        goto="supervisor"  # 回到Supervisor
    )
```

## 常见问题

**Q: Supervisor模式和Router有什么区别？**

A: Router是简单的条件判断，Supervisor使用LLM进行更智能的决策，可以理解复杂上下文。

**Q: Worker之间能直接通信吗？**

A: 推荐通过Supervisor中转，保持架构清晰。直接通信会增加复杂度。

## 本节总结

Supervisor模式通过一个决策者协调多个Worker Agent，实现了类似团队分工的协作模式。它是最直观、最容易理解的多代理架构。
