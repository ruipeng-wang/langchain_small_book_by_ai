# 13.2.3 共识机制与决策协调

## 概念讲解

在多代理协作系统中，共识机制与决策协调是确保多个代理能够就关键问题达成一致、协调行动的核心技术。共识机制解决"如何让多个独立实体达成一致"的问题，而决策协调关注"如何在达成共识后有效执行决策"。

**共识的核心挑战**：
1. **信息不对称**：不同代理拥有不同信息
2. **通信延迟**：网络延迟导致信息不同步
3. **代理故障**：部分代理可能崩溃或行为异常
4. **恶意行为**：拜占庭故障代理故意提供错误信息

**决策协调的关键问题**：
1. **行动同步**：多个代理如何协调执行时间
2. **资源分配**：决策执行所需的资源如何分配
3. **冲突解决**：不同决策之间的冲突如何处理
4. **状态一致性**：如何确保所有代理状态一致

LangGraph通过其图计算能力和状态管理机制，为构建复杂的共识和决策协调系统提供了强大基础。其条件路由、状态共享和异步执行特性特别适合实现分布式共识算法。

## 核心要点

### 1. 共识算法的类型

**经典共识算法**
- **Paxos**：理论上的完美共识算法，但实现复杂
- **Raft**：易于理解和实现的共识算法
- **Viewstamped Replication**：早期共识算法，为后续算法奠定基础

**拜占庭容错算法**
- **PBFT**：实用拜占庭容错，支持1/3故障节点
- **SBFT**：简化拜占庭容错，性能更好
- **HotStuff**：区块链常用共识算法

**弱一致性共识**
- **最终一致性**：不保证立即一致，但最终会一致
- **因果一致性**：保持因果关系的顺序
- **读写一致性**：读操作能看到之前写操作的结果

### 2. 决策协调策略

**集中式协调**
- 单一协调者负责决策和调度
- 简单高效，但存在单点故障
- 适合小型系统或层级结构

**分布式协调**
- 多个代理共同参与决策过程
- 容错性好，但协调复杂
- 适合大型对等网络

**混合式协调**
- 结合集中式和分布式优势
- 关键决策集中，日常操作分布式
- 灵活平衡性能和可靠性

### 3. LangGraph中的共识实现

LangGraph支持以下共识实现模式：

- **状态投票**：代理通过修改共享状态进行投票
- **条件路由**：基于共识结果选择不同执行路径
- **异步协调**：非阻塞的共识达成过程
- **检查点恢复**：共识过程中的故障恢复

## 简单示例

### 示例1：基于简单多数投票的共识系统

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict, List, Dict, Literal
from datetime import datetime
import asyncio
from collections import defaultdict

# 定义共识状态
class ConsensusState(TypedDict):
    proposal: Dict  # 待决策的提案
    agents: List[Dict]  # 参与决策的代理
    votes: Dict[str, str]  # 代理ID -> 投票结果
    decision: str  # 最终决策结果
    consensus_round: int  # 共识轮次
    history: List[Dict]  # 共识历史记录

# 提案生成节点
def generate_proposal(state: ConsensusState):
    """生成待决策的提案"""
    # 模拟从某个代理生成提案
    proposal = {
        "id": f"proposal_{state.get('consensus_round', 0)}",
        "content": "是否执行系统升级？",
        "options": ["同意", "反对", "弃权"],
        "creator": state["agents"][0]["id"] if state["agents"] else "system",
        "timestamp": datetime.now().isoformat(),
        "deadline": (datetime.now().timestamp() + 30)  # 30秒截止
    }
    
    return {
        "proposal": proposal,
        "consensus_round": state.get("consensus_round", 0) + 1
    }

# 投票收集节点
def collect_votes(state: ConsensusState):
    """收集代理投票"""
    votes = {}
    
    for agent in state["agents"]:
        if agent["status"] == "active":
            # 模拟代理投票逻辑
            # 实际应用中，代理会根据自身知识和策略投票
            if agent["role"] == "maintainer":
                vote = "同意"  # 维护者通常支持升级
            elif agent["role"] == "user":
                vote = "同意" if agent.get("risk_tolerance", "low") == "high" else "反对"
            else:
                vote = "弃权"
            
            votes[agent["id"]] = {
                "vote": vote,
                "timestamp": datetime.now().isoformat(),
                "confidence": 0.8  # 投票置信度
            }
    
    return {"votes": votes}

# 共识达成节点
def reach_consensus(state: ConsensusState):
    """基于投票结果达成共识"""
    if not state["votes"]:
        return {"decision": "pending", "consensus_reached": False}
    
    # 统计投票结果
    vote_counts = defaultdict(int)
    for agent_id, vote_data in state["votes"].items():
        vote = vote_data["vote"]
        vote_counts[vote] += 1
    
    total_votes = sum(vote_counts.values())
    
    # 简单多数规则：超过50%同意则通过
    if vote_counts["同意"] > total_votes / 2:
        decision = "通过"
        consensus_reached = True
    elif vote_counts["反对"] > total_votes / 2:
        decision = "否决"
        consensus_reached = True
    else:
        decision = "pending"
        consensus_reached = False
    
    # 记录共识历史
    history_entry = {
        "round": state["consensus_round"],
        "proposal": state["proposal"]["id"],
        "vote_counts": dict(vote_counts),
        "decision": decision,
        "consensus_reached": consensus_reached,
        "timestamp": datetime.now().isoformat()
    }
    
    history = state.get("history", []) + [history_entry]
    
    return {
        "decision": decision,
        "consensus_reached": consensus_reached,
        "history": history
    }

# 决策执行节点
def execute_decision(state: ConsensusState):
    """执行共识达成的决策"""
    if not state.get("consensus_reached"):
        return {"execution_status": "skipped", "reason": "no_consensus"}
    
    decision = state["decision"]
    proposal = state["proposal"]
    
    execution_result = {
        "proposal_id": proposal["id"],
        "decision": decision,
        "execution_time": datetime.now().isoformat(),
        "status": "completed"
    }
    
    if decision == "通过":
        # 执行提案内容
        execution_result["actions"] = [
            "开始系统升级准备",
            "备份当前系统状态",
            "执行升级程序",
            "验证升级结果"
        ]
        execution_result["outcome"] = "升级流程已启动"
    else:
        execution_result["actions"] = ["维持当前系统状态"]
        execution_result["outcome"] = "提案被否决，无变更"
    
    return {
        "execution_result": execution_result,
        "execution_status": "completed"
    }

# 构建共识图
def build_consensus_graph():
    workflow = StateGraph(ConsensusState)
    
    workflow.add_node("propose", generate_proposal)
    workflow.add_node("vote", collect_votes)
    workflow.add_node("consensus", reach_consensus)
    workflow.add_node("execute", execute_decision)
    
    workflow.add_edge(START, "propose")
    workflow.add_edge("propose", "vote")
    workflow.add_edge("vote", "consensus")
    
    # 条件边：达成共识则执行，否则重新提议
    def should_execute(state):
        return "execute" if state.get("consensus_reached") else "propose"
    
    workflow.add_conditional_edges("consensus", should_execute)
    workflow.add_edge("execute", END)
    
    return workflow.compile()

# 使用示例
if __name__ == "__main__":
    # 初始化系统状态
    initial_state = {
        "agents": [
            {"id": "agent1", "status": "active", "role": "maintainer"},
            {"id": "agent2", "status": "active", "role": "user", "risk_tolerance": "high"},
            {"id": "agent3", "status": "active", "role": "user", "risk_tolerance": "low"},
            {"id": "agent4", "status": "active", "role": "observer"},
            {"id": "agent5", "status": "inactive", "role": "user"}
        ],
        "proposal": {},
        "votes": {},
        "decision": "",
        "consensus_round": 0,
        "history": []
    }
    
    # 编译并执行图
    consensus_graph = build_consensus_graph()
    result = consensus_graph.invoke(initial_state)
    
    print(f"共识轮次: {result['consensus_round']}")
    print(f"最终决策: {result['decision']}")
    print(f"共识达成: {result['consensus_reached']}")
    
    if result.get("execution_result"):
        print(f"执行结果: {result['execution_result']['outcome']}")
```

### 示例2：基于权重投票的决策协调系统

```python
from typing import Tuple

class WeightedVotingConsensus:
    """基于权重投票的共识系统"""
    
    def __init__(self):
        self.agents = {}  # 代理ID -> 权重
        self.proposals = {}  # 提案ID -> 提案内容
        self.votes = defaultdict(dict)  # 提案ID -> 代理ID -> 投票
    
    def register_agent(self, agent_id: str, weight: float, role: str):
        """注册代理并分配权重"""
        self.agents[agent_id] = {
            "weight": weight,
            "role": role,
            "status": "active"
        }
    
    def submit_proposal(self, proposal_id: str, content: str, 
                       options: List[str], creator: str) -> bool:
        """提交提案"""
        if proposal_id in self.proposals:
            return False
        
        self.proposals[proposal_id] = {
            "content": content,
            "options": options,
            "creator": creator,
            "status": "voting",
            "submission_time": datetime.now().isoformat(),
            "deadline": (datetime.now().timestamp() + 60)  # 60秒截止
        }
        return True
    
    def vote(self, agent_id: str, proposal_id: str, option: str) -> bool:
        """代理投票"""
        if (agent_id not in self.agents or 
            self.agents[agent_id]["status"] != "active" or
            proposal_id not in self.proposals):
            return False
        
        # 检查投票选项是否有效
        if option not in self.proposals[proposal_id]["options"]:
            return False
        
        self.votes[proposal_id][agent_id] = {
            "option": option,
            "timestamp": datetime.now().isoformat(),
            "weight": self.agents[agent_id]["weight"]
        }
        return True
    
    def calculate_result(self, proposal_id: str) -> Tuple[str, float]:
        """计算投票结果"""
        if proposal_id not in self.votes:
            return "no_votes", 0.0
        
        option_weights = defaultdict(float)
        total_weight = 0.0
        
        for agent_id, vote_data in self.votes[proposal_id].items():
            option = vote_data["option"]
            weight = vote_data["weight"]
            option_weights[option] += weight
            total_weight += weight
        
        if total_weight == 0:
            return "no_weight", 0.0
        
        # 找到权重最高的选项
        max_option = max(option_weights.items(), key=lambda x: x[1])
        max_weight = max_option[1]
        
        # 计算支持率
        support_rate = max_weight / total_weight
        
        # 阈值判断：需要超过66%权重支持
        if support_rate > 0.66:
            return max_option[0], support_rate
        else:
            return "no_consensus", support_rate
    
    def execute_decision(self, proposal_id: str, decision: str) -> Dict:
        """执行决策"""
        if proposal_id not in self.proposals:
            return {"status": "error", "reason": "proposal_not_found"}
        
        self.proposals[proposal_id]["status"] = "executing"
        self.proposals[proposal_id]["decision"] = decision
        self.proposals[proposal_id]["execution_time"] = datetime.now().isoformat()
        
        # 模拟执行决策
        execution_plan = {
            "proposal_id": proposal_id,
            "decision": decision,
            "steps": [
                "验证决策合法性",
                "分配执行资源",
                "协调相关代理",
                "执行具体操作",
                "验证执行结果"
            ],
            "status": "in_progress"
        }
        
        return execution_plan

# 在LangGraph中集成权重投票
def weighted_voting_node(state: ConsensusState):
    """权重投票节点"""
    voting_system = WeightedVotingConsensus()
    
    # 注册代理
    for agent in state["agents"]:
        weight = agent.get("weight", 1.0)
        role = agent.get("role", "participant")
        voting_system.register_agent(agent["id"], weight, role)
    
    # 提交提案
    proposal = state["proposal"]
    proposal_id = proposal.get("id", "default_proposal")
    voting_system.submit_proposal(
        proposal_id,
        proposal.get("content", ""),
        proposal.get("options", ["同意", "反对"]),
        proposal.get("creator", "system")
    )
    
    # 收集投票
    for agent in state["agents"]:
        if agent["status"] == "active":
            # 模拟代理投票决策
            vote_option = agent.get("preferred_option")
            if not vote_option:
                # 基于代理属性决定投票
                if agent["role"] == "expert":
                    vote_option = "同意" if agent.get("expertise_level", 0) > 0.7 else "反对"
                else:
                    vote_option = "同意"  # 默认同意
            
            voting_system.vote(agent["id"], proposal_id, vote_option)
    
    # 计算结果
    decision, support_rate = voting_system.calculate_result(proposal_id)
    
    return {
        "decision": decision,
        "support_rate": support_rate,
        "consensus_reached": decision not in ["no_votes", "no_weight", "no_consensus"]
    }
```

## 进阶应用

### 1. 基于PBFT的拜占庭容错共识

实用拜占庭容错（PBFT）算法可以容忍最多f个恶意节点，其中总节点数n ≥ 3f + 1。

```python
from enum import Enum
from dataclasses import dataclass
from typing import List, Dict, Optional
import hashlib
import json

class PBFTPhase(Enum):
    """PBFT算法阶段"""
    PRE_PREPARE = "pre-prepare"
    PREPARE = "prepare"
    COMMIT = "commit"
    REPLY = "reply"

@dataclass
class PBFTMessage:
    """PBFT消息"""
    phase: PBFTPhase
    view_number: int
    sequence_number: int
    digest: str
    sender: str
    signature: Optional[str] = None
    content: Optional[Dict] = None

class PBFTConsensus:
    """PBFT共识实现"""
    
    def __init__(self, node_id: str, total_nodes: int):
        self.node_id = node_id
        self.total_nodes = total_nodes
        self.faulty_limit = (total_nodes - 1) // 3  # 最大容错节点数
        
        self.view_number = 0
        self.sequence_number = 0
        self.log = []  # 共识日志
        self.prepare_messages = defaultdict(list)
        self.commit_messages = defaultdict(list)
        
        # 主节点选择（基于view_number）
        self.primary_node = self._select_primary()
    
    def _select_primary(self) -> str:
        """选择主节点"""
        # 简单轮换：view_number % total_nodes
        primary_index = self.view_number % self.total_nodes
        return f"node_{primary_index}"
    
    def _create_digest(self, content: Dict) -> str:
        """创建消息摘要"""
        content_str = json.dumps(content, sort_keys=True)
        return hashlib.sha256(content_str.encode()).hexdigest()
    
    def pre_prepare(self, request: Dict) -> PBFTMessage:
        """主节点：预准备阶段"""
        if self.node_id != self.primary_node:
            raise ValueError("只有主节点可以发送pre-prepare消息")
        
        self.sequence_number += 1
        digest = self._create_digest(request)
        
        message = PBFTMessage(
            phase=PBFTPhase.PRE_PREPARE,
            view_number=self.view_number,
            sequence_number=self.sequence_number,
            digest=digest,
            sender=self.node_id,
            content=request
        )
        
        # 记录到日志
        self.log.append({
            "view": self.view_number,
            "seq": self.sequence_number,
            "digest": digest,
            "phase": "pre-prepare",
            "content": request
        })
        
        return message
    
    def prepare(self, pre_prepare_msg: PBFTMessage) -> PBFTMessage:
        """副本节点：准备阶段"""
        # 验证pre-prepare消息
        if pre_prepare_msg.phase != PBFTPhase.PRE_PREPARE:
            raise ValueError("无效的pre-prepare消息")
        
        # 验证消息摘要
        expected_digest = self._create_digest(pre_prepare_msg.content)
        if expected_digest != pre_prepare_msg.digest:
            raise ValueError("消息摘要不匹配")
        
        prepare_msg = PBFTMessage(
            phase=PBFTPhase.PREPARE,
            view_number=pre_prepare_msg.view_number,
            sequence_number=pre_prepare_msg.sequence_number,
            digest=pre_prepare_msg.digest,
            sender=self.node_id
        )
        
        # 记录准备消息
        key = (pre_prepare_msg.view_number, pre_prepare_msg.sequence_number)
        self.prepare_messages[key].append(prepare_msg)
        
        return prepare_msg
    
    def commit(self, prepare_messages: List[PBFTMessage]) -> Optional[PBFTMessage]:
        """提交阶段"""
        if len(prepare_messages) < 2 * self.faulty_limit:
            # 未收到足够的prepare消息
            return None
        
        # 验证所有prepare消息针对同一个请求
        first_msg = prepare_messages[0]
        for msg in prepare_messages[1:]:
            if (msg.view_number != first_msg.view_number or 
                msg.sequence_number != first_msg.sequence_number or
                msg.digest != first_msg.digest):
                return None
        
        commit_msg = PBFTMessage(
            phase=PBFTPhase.COMMIT,
            view_number=first_msg.view_number,
            sequence_number=first_msg.sequence_number,
            digest=first_msg.digest,
            sender=self.node_id
        )
        
        # 记录提交消息
        key = (first_msg.view_number, first_msg.sequence_number)
        self.commit_messages[key].append(commit_msg)
        
        return commit_msg
    
    def check_commit_quorum(self, view: int, seq: int) -> bool:
        """检查是否达到提交法定人数"""
        key = (view, seq)
        commit_count = len(self.commit_messages.get(key, []))
        
        # 需要至少2f+1个commit消息（包括自己的）
        return commit_count >= 2 * self.faulty_limit + 1
    
    def execute_request(self, view: int, seq: int, content: Dict) -> PBFTMessage:
        """执行请求并返回结果"""
        if not self.check_commit_quorum(view, seq):
            raise ValueError("未达到提交法定人数")
        
        # 执行请求（这里只是模拟）
        result = {
            "request": content,
            "executed_by": self.node_id,
            "execution_time": datetime.now().isoformat(),
            "result": "success"
        }
        
        reply_msg = PBFTMessage(
            phase=PBFTPhase.REPLY,
            view_number=view,
            sequence_number=seq,
            digest=self._create_digest(result),
            sender=self.node_id,
            content=result
        )
        
        return reply_msg
```

### 2. 基于Raft的分布式一致性

Raft算法通过领导者选举、日志复制和安全性保证分布式一致性。

```python
class RaftState(Enum):
    """Raft节点状态"""
    FOLLOWER = "follower"
    CANDIDATE = "candidate"
    LEADER = "leader"

class RaftNode:
    """Raft节点实现"""
    
    def __init__(self, node_id: str, peers: List[str]):
        self.node_id = node_id
        self.peers = peers
        
        # Raft状态
        self.state = RaftState.FOLLOWER
        self.current_term = 0
        self.voted_for = None
        self.log = []  # 日志条目列表
        
        # 领导者状态（仅当leader时有效）
        self.next_index = {}
        self.match_index = {}
        
        # 选举超时（150-300ms）
        self.election_timeout = 0.2
        self.last_heartbeat = time.time()
        
        # 提交索引和最后应用索引
        self.commit_index = 0
        self.last_applied = 0
    
    def become_follower(self, term: int):
        """转换为follower状态"""
        self.state = RaftState.FOLLOWER
        self.current_term = term
        self.voted_for = None
    
    def become_candidate(self):
        """转换为candidate状态"""
        self.state = RaftState.CANDIDATE
        self.current_term += 1
        self.voted_for = self.node_id
        
        # 重置选举超时
        self.last_heartbeat = time.time()
    
    def become_leader(self):
        """转换为leader状态"""
        self.state = RaftState.LEADER
        
        # 初始化next_index和match_index
        for peer in self.peers:
            self.next_index[peer] = len(self.log) + 1
            self.match_index[peer] = 0
        
        # 立即发送心跳
        self.send_heartbeat()
    
    def request_vote(self) -> Dict:
        """请求投票"""
        vote_request = {
            "type": "RequestVote",
            "term": self.current_term,
            "candidate_id": self.node_id,
            "last_log_index": len(self.log),
            "last_log_term": self.log[-1]["term"] if self.log else 0
        }
        return vote_request
    
    def handle_vote_request(self, request: Dict) -> Dict:
        """处理投票请求"""
        response = {
            "type": "RequestVoteResponse",
            "term": self.current_term,
            "vote_granted": False
        }
        
        # 检查任期
        if request["term"] < self.current_term:
            return response
        
        # 更新任期（如果请求任期更大）
        if request["term"] > self.current_term:
            self.become_follower(request["term"])
        
        # 检查是否已投票
        if (self.voted_for is None or self.voted_for == request["candidate_id"]):
            # 检查日志新旧程度
            last_log_term = self.log[-1]["term"] if self.log else 0
            last_log_index = len(self.log)
            
            if (request["last_log_term"] > last_log_term or
                (request["last_log_term"] == last_log_term and 
                 request["last_log_index"] >= last_log_index)):
                response["vote_granted"] = True
                self.voted_for = request["candidate_id"]
                # 重置选举超时
                self.last_heartbeat = time.time()
        
        return response
    
    def append_entries(self, entries: List[Dict]) -> Dict:
        """追加日志条目（仅leader调用）"""
        if self.state != RaftState.LEADER:
            raise ValueError("只有leader可以追加日志")
        
        for entry in entries:
            log_entry = {
                "term": self.current_term,
                "command": entry,
                "index": len(self.log) + 1
            }
            self.log.append(log_entry)
        
        # 尝试提交
        self.advance_commit_index()
        
        return {"status": "appended", "log_length": len(self.log)}
    
    def send_heartbeat(self):
        """发送心跳（仅leader调用）"""
        if self.state != RaftState.LEADER:
            return
        
        heartbeat = {
            "type": "AppendEntries",
            "term": self.current_term,
            "leader_id": self.node_id,
            "prev_log_index": len(self.log),
            "prev_log_term": self.log[-1]["term"] if self.log else 0,
            "entries": [],  # 空表示心跳
            "leader_commit": self.commit_index
        }
        
        # 实际应用中会发送给所有peers
        return heartbeat
    
    def advance_commit_index(self):
        """推进提交索引"""
        if self.state != RaftState.LEADER:
            return
        
        # 找到多数节点已复制的日志索引
        sorted_match_indices = sorted(self.match_index.values())
        n = len(self.peers) + 1  # 包括自己
        
        # 需要多数（n//2 + 1）
        majority_index = sorted_match_indices[n // 2]
        
        # 检查是否可以提交
        if (majority_index > self.commit_index and 
            self.log[majority_index - 1]["term"] == self.current_term):
            self.commit_index = majority_index
    
    def apply_committed_entries(self):
        """应用已提交的日志条目"""
        while self.last_applied < self.commit_index:
            self.last_applied += 1
            entry = self.log[self.last_applied - 1]
            
            # 执行命令（实际应用中这里会执行具体业务逻辑）
            print(f"应用日志条目 {self.last_applied}: {entry['command']}")
```

## 常见问题

### Q1: 如何选择合适的共识算法？

**回答**：选择共识算法需要考虑以下因素：

1. **网络规模**：
   - 小型网络（<10节点）：Paxos、Raft
   - 中型网络（10-100节点）：PBFT、Raft
   - 大型网络（>100节点）：最终一致性、Gossip协议

2. **容错要求**：
   - 崩溃故障：Raft、Paxos
   - 拜占庭故障：PBFT、SBFT
   - 混合故障：混合共识算法

3. **性能需求**：
   - 高吞吐量：最终一致性、分片共识
   - 低延迟：Raft、PBFT（小规模）
   - 高可用性：多主复制、最终一致性

4. **一致性级别**：
   - 强一致性：Raft、Paxos
   - 最终一致性：CRDT、Gossip
   - 因果一致性：向量时钟

**建议**：
- 从简单算法开始，根据需求逐步升级
- 进行性能测试和故障注入测试
- 考虑算法的实现复杂性和维护成本

### Q2: 共识算法如何处理网络分区？

**回答**：网络分区处理策略：

1. **多数派原则**：
   - 只有拥有多数节点的分区可以继续操作
   - 少数派分区拒绝服务，等待分区恢复
   - 保证强一致性，但可能降低可用性

2. **最后写入获胜**：
   - 允许所有分区继续操作
   - 分区恢复后解决冲突
   - 提高可用性，但可能丢失数据

3. **多版本并发控制**：
   - 维护多个版本的数据
   - 分区恢复后合并版本
   - 平衡一致性和可用性

4. **人工干预**：
   - 检测到分区后暂停服务
   - 人工决策如何处理
   - 适合关键系统

**CAP定理权衡**：
- 选择CP（一致性+分区容错）：Raft、Paxos
- 选择AP（可用性+分区容错）：最终一致性
- 实际系统通常需要权衡折中

### Q3: 如何在LangGraph中实现高效的决策协调？

**回答**：LangGraph中高效决策协调的实现技巧：

1. **分层决策**：
   - 局部决策在图节点内部完成
   - 全局决策通过图结构协调
   - 减少协调开销

2. **异步协调**：
   - 使用异步节点处理长时间决策
   - 避免阻塞整个工作流
   - 提高系统吞吐量

3. **检查点机制**：
   - 关键决策点设置检查点
   - 支持回滚和重试
   - 提高系统可靠性

4. **智能路由**：
   - 基于决策结果动态选择执行路径
   - 实现条件分支和循环
   - 提高决策灵活性

**性能优化**：
- 批量处理决策请求
- 缓存频繁使用的决策结果
- 并行处理独立决策
- 监控决策延迟和成功率
