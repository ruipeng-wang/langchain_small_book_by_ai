# 12.2.3 超时处理与异常降级

在复杂工作流系统中，超时处理与异常降级是确保系统可用性和稳定性的关键机制。LangGraph通过内置的超时控制、降级策略和优雅失败机制，提供了完整的异常处理和系统降级支持。

## 概念讲解

### 超时处理的必要性
超时处理机制防止工作流因外部依赖故障而无限等待：
1. **资源保护**：避免线程/连接被长时间占用
2. **用户体验**：提供及时反馈，避免用户长时间等待
3. **系统稳定性**：防止级联故障影响整个系统
4. **故障隔离**：快速识别和隔离故障组件

### 异常降级的设计哲学
当主功能不可用时，异常降级提供备选方案：
1. **功能降级**：使用简化功能替代完整功能
2. **数据降级**：使用缓存数据或默认值
3. **服务降级**：切换到备用服务或本地处理
4. **用户体验降级**：提供简化但可用的用户界面

### LangGraph的超时与降级架构
LangGraph通过以下机制支持超时和降级：
- **节点级超时**：为单个节点设置执行时间限制
- **工作流级超时**：整个工作流的执行时间限制
- **条件降级**：基于错误类型或系统状态触发降级
- **优雅失败**：失败时执行清理操作并保存状态

## 核心要点

### 1. 超时策略的多层次设计
支持从节点级到工作流级的完整超时控制体系。

### 2. 降级条件的智能判断
基于错误类型、系统负载、历史成功率等因素智能触发降级。

### 3. 状态保存与恢复
超时或降级发生时保存当前状态，支持后续恢复或重试。

### 4. 监控与告警集成
超时和降级事件触发监控告警，便于及时干预。

### 示例1：基础超时控制
以下示例展示了如何为节点和工作流设置超时控制：

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.types import RetryPolicy
import time
import concurrent.futures

class TimeoutState(TypedDict):
    external_service_result: str
    execution_time: float
    timeout_occurred: bool
    fallback_used: bool
    retry_count: int

def external_service_with_timeout(state: TimeoutState) -> dict:
    """调用外部服务，带有超时控制"""
    import time
    
    start_time = time.time()
    retry_count = state.get("retry_count", 0)
    
    def call_external_service():
        """模拟不可靠的外部服务"""
        # 模拟服务响应时间，有时会很慢
        time.sleep(2.5)  # 设置2.5秒延迟，超过2秒超时
        return "外部服务结果"
    
    try:
        # 使用线程池执行带超时的调用
        with concurrent.futures.ThreadPoolExecutor(max_workers=1) as executor:
            future = executor.submit(call_external_service)
            
            # 设置2秒超时
            result = future.result(timeout=2.0)
            
            execution_time = time.time() - start_time
            
            return {
                "external_service_result": result,
                "execution_time": execution_time,
                "timeout_occurred": False,
                "retry_count": retry_count + 1
            }
    
    except concurrent.futures.TimeoutError:
        # 超时处理
        execution_time = time.time() - start_time
        
        return {
            "external_service_result": "",
            "execution_time": execution_time,
            "timeout_occurred": True,
            "retry_count": retry_count + 1
        }
    
    except Exception as e:
        # 其他异常处理
        execution_time = time.time() - start_time
        
        return {
            "external_service_result": f"错误: {str(e)}",
            "execution_time": execution_time,
            "timeout_occurred": False,
            "retry_count": retry_count + 1
        }

def fallback_service(state: TimeoutState) -> dict:
    """降级服务：当主服务超时或失败时使用"""
    # 模拟降级服务的快速响应
    time.sleep(0.1)
    
    # 基于缓存或简化逻辑提供结果
    fallback_result = "降级服务结果（基于缓存数据）"
    
    return {
        "external_service_result": fallback_result,
        "fallback_used": True,
        "execution_time": state.get("execution_time", 0) + 0.1
    }

def build_timeout_workflow():
    """构建带超时控制的工作流"""
    builder = StateGraph(TimeoutState)
    
    # 添加主服务节点
    builder.add_node("external_service", external_service_with_timeout)
    
    # 添加降级服务节点
    builder.add_node("fallback_service", fallback_service)
    
    # 定义执行流程
    builder.add_edge(START, "external_service")
    
    # 条件边：根据服务调用结果决定下一步
    def after_external_service(state: TimeoutState):
        if state.get("timeout_occurred", False):
            # 超时，使用降级服务
            return "fallback_service"
        elif not state.get("external_service_result"):
            # 服务调用失败（非超时），使用降级服务
            return "fallback_service"
        else:
            # 服务调用成功
            return END
    
    builder.add_conditional_edges("external_service", after_external_service)
    builder.add_edge("fallback_service", END)
    
    return builder.compile(checkpointer=InMemorySaver())

def test_timeout_workflow():

    config = {"configurable": {"thread_id": "timeout-test-001"}}
    
    print("=== 超时处理工作流测试 ===")

        return {
            "external_service_result": "快速服务结果",
            "execution_time": 0.5,

            "retry_count": state.get("retry_count", 0) + 1
        }
    
    # 临时替换服务函数
    graph_with_fast_service = build_timeout_workflow()
    # 这里需要重新构建图，实际应用中可以通过配置实现
    
    initial_state = {
```

```python
    """系统监控器，模拟系统状态检测"""
    
    @staticmethod
    def get_current_load() -> float:

        # 模拟负载检测
        return random.uniform(0.1, 0.9)
    
    @staticmethod
    def check_service_health(service_name: str) -> tuple[bool, str]:
        """检查服务健康状态"""
        # 模拟健康检查
        is_healthy = random.random() > 0.3  # 70%健康概率
        status = "healthy" if is_healthy else "unhealthy"
        
        if not is_healthy:
            # 模拟不同的故障类型

            error_type = random.choice(error_types)
            return False, error_type
        
        return True, "healthy"
    
    @staticmethod
    def should_degrade(current_load: float, request_priority: str) -> tuple[bool, str, Literal["none", "partial", "full"]]:
        """判断是否需要降级"""
        # 基于负载和优先级决定降级策略
        
        if current_load > 0.8:
            # 高负载，所有请求都降级
            if request_priority == "high_priority":
                return True, "system_overloaded", "partial"
            else:
                return True, "system_overloaded", "full"
        
        elif current_load > 0.6:
            # 中等负载，低优先级请求降级
            if request_priority == "low_priority":
                return True, "moderate_load", "full"
        
        # 检查外部服务健康状态
        is_healthy, error_type = SystemMonitor.check_service_health("external_api")
        if not is_healthy:
            if error_type == "resource_exhausted":
                return True, f"service_{error_type}", "full"
            else:
                return True, f"service_{error_type}", "partial"
        
        return False, "", "none"

def intelligent_processor(state: DegradationState) -> dict:
    """智能处理器，根据系统状态决定是否降级"""
    start_time = time.time()
    
    # 获取当前系统状态
    current_load = SystemMonitor.get_current_load()
    request_priority = state.get("request_type", "normal")
    
    # 判断是否需要降级
    should_degrade, reason, level = SystemMonitor.should_degrade(current_load, request_priority)
    
    if should_degrade:
        # 执行降级逻辑
        degraded_result = execute_degraded_function(state, level)
        
        response_time = time.time() - start_time
        
        return {
            "system_load": current_load,
            "degradation_level": level,
            "degraded_function_result": degraded_result,
            "degradation_reason": reason,
            "response_time": response_time
        }

        response_time = time.time() - start_time
        
        return {
            "system_load": current_load,
            "degradation_level": "none",
            "original_function_result": full_result,
            "degradation_reason": "no_degradation_needed",
            "response_time": response_time
        }

def execute_full_function(state: DegradationState) -> str:

        # 部分降级：使用简化算法
        time.sleep(0.3)
```

### 自适应超时调整
基于历史性能数据动态调整超时时间：

```python
def adaptive_timeout_management():
    """自适应超时管理"""
    from collections import deque
    import statistics
    
    class AdaptiveTimeoutManager:
        """自适应超时管理器"""
        
        def __init__(self, initial_timeout: float = 5.0, min_timeout: float = 1.0, max_timeout: float = 30.0):
            self.initial_timeout = initial_timeout
            self.min_timeout = min_timeout
            self.max_timeout = max_timeout
            self.response_times = deque(maxlen=100)  # 保存最近100次响应时间
            self.success_rate = 1.0
            self.timeout_factor = 1.0
        
        def record_response(self, response_time: float, success: bool):
            """记录响应时间和成功率"""
            self.response_times.append((response_time, success))
            
            # 计算成功率和平均响应时间
            if len(self.response_times) > 0:
                success_count = sum(1 for _, s in self.response_times if s)
                self.success_rate = success_count / len(self.response_times)
                
                successful_times = [rt for rt, s in self.response_times if s]
                if successful_times:
                    avg_response = statistics.mean(successful_times)
                    
                    # 基于成功率和响应时间调整超时因子
                    if self.success_rate < 0.8:
                        # 成功率低，增加超时时间
                        self.timeout_factor = min(2.0, self.timeout_factor * 1.1)
                    elif self.success_rate > 0.95 and avg_response < self.get_current_timeout() * 0.5:
                        # 成功率高且响应时间远小于超时时间，减少超时时间
                        self.timeout_factor = max(0.5, self.timeout_factor * 0.9)
        
        def get_current_timeout(self) -> float:
            """获取当前建议的超时时间"""
            if not self.response_times:
                return self.initial_timeout
            
            # 基于历史响应时间计算超时时间
            successful_times = [rt for rt, s in self.response_times if s]
            if successful_times:
                # 使用第95百分位数加上安全边际
                p95 = statistics.quantiles(successful_times, n=20)[-1]  # 第95百分位
                adaptive_timeout = p95 * 2.0 * self.timeout_factor  # 2倍第95百分位
                
                # 确保在最小和最大范围内
                return max(self.min_timeout, min(self.max_timeout, adaptive_timeout))
            
            return self.initial_timeout
        
        def get_recommendation(self) -> dict:
            """获取调整建议"""
            return {
                "current_timeout": self.get_current_timeout(),
                "success_rate": self.success_rate,
                "sample_size": len(self.response_times),
                "timeout_factor": self.timeout_factor
            }
    
    # 在节点函数中使用自适应超时
    def adaptive_service_call(state: dict, timeout_manager: AdaptiveTimeoutManager) -> dict:
        """使用自适应超时的服务调用"""
        import time
        import concurrent.futures
        
        start_time = time.time()
        current_timeout = timeout_manager.get_current_timeout()
        
        def call_service():
            # 模拟服务调用
            processing_time = random.uniform(0.1, current_timeout * 0.8)
            time.sleep(processing_time)
            
            # 模拟随机失败
            if random.random() < 0.1:  # 10%失败率
                raise Exception("服务调用失败")
            
            return f"服务结果，处理时间: {processing_time:.2f}秒"
        
        try:
            with concurrent.futures.ThreadPoolExecutor(max_workers=1) as executor:
                future = executor.submit(call_service)
                result = future.result(timeout=current_timeout)
                
                response_time = time.time() - start_time
                timeout_manager.record_response(response_time, True)
                
                return {
                    "result": result,
                    "response_time": response_time,
                    "timeout_used": current_timeout,
                    "success": True
                }
        
        except concurrent.futures.TimeoutError:
            response_time = time.time() - start_time
            timeout_manager.record_response(response_time, False)
            
            return {
                "result": "超时",
                "response_time": response_time,
                "timeout_used": current_timeout,
                "success": False
            }
        
        except Exception as e:
            response_time = time.time() - start_time
            timeout_manager.record_response(response_time, False)
            
            return {
                "result": f"错误: {str(e)}",
                "response_time": response_time,
                "timeout_used": current_timeout,
                "success": False
            }
```

```python
def multi_level_degradation():
    """多级降级策略"""
    from enum import Enum
    from typing import Dict, Any
    
    class DegradationLevel(Enum):
        """降级级别"""
        NONE = 0      # 无降级，完整功能
        LEVEL_1 = 1   # 轻度降级，简化非核心功能
        LEVEL_2 = 2   # 中度降级，使用缓存数据
        LEVEL_3 = 3   # 重度降级，仅核心功能
        LEVEL_4 = 4   # 极限降级，只返回默认值
    
    class MultiLevelDegradationManager:
        """多级降级管理器"""
        
        def __init__(self):
            self.current_level = DegradationLevel.NONE
            self.level_thresholds = {
                "load": [0.7, 0.8, 0.9, 0.95],      # 负载阈值
                "error_rate": [0.1, 0.2, 0.3, 0.5],  # 错误率阈值
                "response_time": [1.0, 2.0, 3.0, 5.0] # 响应时间阈值（秒）
            }
            self.metrics_history = []
        
        def assess_situation(self, metrics: Dict[str, float]) -> DegradationLevel:
            """评估当前情况，确定降级级别"""
            # 计算各项指标的超标程度
            load_score = self._calculate_score(metrics.get("load", 0), self.level_thresholds["load"])
            error_score = self._calculate_score(metrics.get("error_rate", 0), self.level_thresholds["error_rate"])
            response_score = self._calculate_score(metrics.get("avg_response_time", 0), self.level_thresholds["response_time"])
            
            # 取最高分作为降级级别
            max_score = max(load_score, error_score, response_score)
            
            # 转换为降级级别
            if max_score == 0:
                return DegradationLevel.NONE
            elif max_score == 1:
                return DegradationLevel.LEVEL_1
            elif max_score == 2:
                return DegradationLevel.LEVEL_2
            elif max_score == 3:
                return DegradationLevel.LEVEL_3
            else:
                return DegradationLevel.LEVEL_4
        
        def _calculate_score(self, value: float, thresholds: list) -> int:
            """计算指标得分"""
            for i, threshold in enumerate(thresholds):
                if value <= threshold:
                    return i
            return len(thresholds)  # 超过所有阈值
        
        def apply_degradation(self, level: DegradationLevel, original_function, context: Dict[str, Any]):
            """应用指定级别的降级"""
            if level == DegradationLevel.NONE:
                # 无降级，使用完整功能
                return original_function(context)
            
            elif level == DegradationLevel.LEVEL_1:
                # 轻度降级：简化非核心功能
                simplified_context = {
                    **context,
                    "simplify_non_core": True,
                    "use_optimized_algorithm": True
                }
                return self._level1_function(simplified_context)
            
            elif level == DegradationLevel.LEVEL_2:
                # 中度降级：使用缓存数据
                cached_data = self._get_cached_data(context)
                if cached_data:
                    return cached_data
                else:
                    # 缓存未命中，进一步降级
                    return self.apply_degradation(DegradationLevel.LEVEL_3, original_function, context)
            
            elif level == DegradationLevel.LEVEL_3:
                # 重度降级：仅核心功能
                core_context = {
                    "only_core_features": True,
                    "skip_validation": True
                }
                return self._level3_function(core_context)
            
            else:  # LEVEL_4
                # 极限降级：只返回默认值
                return self._get_default_response(context)
```

## 常见问题

### Q1: 超时时间应该如何设置？
**A**: 超时时间设置策略：
1. **基准测试**：基于历史响应时间的第95百分位设置
2. **动态调整**：根据成功率动态调整超时时间
3. **分级设置**：不同服务级别设置不同的超时时间
4. **安全边际**：在基准值上增加合理的安全边际

### Q2: 降级可能导致数据不一致吗？
**A**: 通过以下策略确保数据一致性：
1. **只读降级**：降级操作只影响读取，不修改数据
2. **补偿事务**：降级后记录待补偿的操作
3. **数据版本**：使用数据版本标识降级期间的数据
4. **同步恢复**：系统恢复后同步降级期间的操作

### Q3: 如何测试超时和降级功能？
**A**: 测试策略：
1. **故障注入**：模拟各种超时和故障场景
2. **混沌测试**：在生产环境注入随机故障
3. **负载测试**：测试不同负载下的降级行为
4. **恢复测试**：测试系统从降级状态恢复的能力

### Q4: 降级策略会影响业务逻辑吗？
**A**: 合理设计的降级策略影响有限：
1. **业务感知**：降级策略考虑业务重要性和优先级
2. **渐进式**：从非核心功能开始降级
3. **可逆性**：降级操作可逆，系统恢复后能正常处理
4. **透明度**：向用户适当说明降级情况
