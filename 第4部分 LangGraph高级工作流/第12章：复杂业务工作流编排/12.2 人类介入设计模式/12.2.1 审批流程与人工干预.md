# 12.2.1 审批流程与人工干预

在复杂的业务工作流中，某些关键决策需要人工参与和审批。LangGraph提供了完整的"人在回路"（Human-in-the-Loop）机制，通过`interrupt()`函数实现工作流的暂停和人工干预，确保关键决策由人类审批者做出。

## 概念讲解

### 人在回路模式的核心价值
人在回路模式将人工智能的自动化处理与人类的判断能力相结合，适用于以下场景：
1. **高风险决策**：涉及资金、法律或重大业务影响的决策
2. **模糊边界**：AI无法明确判断的情况需要人类介入
3. **合规要求**：法规要求必须有人工审核的流程
4. **质量控制**：确保自动化处理的质量符合标准

### LangGraph的interrupt机制
LangGraph通过`interrupt()`函数提供标准化的人工介入机制：
- **工作流暂停**：在特定节点暂停执行，等待人工输入
- **上下文保持**：保存完整的执行状态和上下文
- **恢复控制**：支持多种方式恢复工作流执行
- **异步处理**：人工审批可以异步进行，不阻塞系统

### 审批流程的类型
根据业务需求，审批流程可以分为：
- **单级审批**：只需一个审批者批准
- **多级审批**：需要多个层级依次审批
- **并行审批**：多个审批者同时审批
- **条件审批**：根据条件触发不同的审批路径

## 核心要点

### 1. interrupt()函数的工作原理
`interrupt()`会暂停工作流执行，并向外部系统发送中断信息，等待人工处理结果后恢复执行。

### 2. Command对象的恢复机制
使用`Command(resume=...)`可以从暂停点恢复执行，携带人工决策结果。

### 3. 检查点与状态持久化
结合检查点机制，即使系统重启，暂停的工作流状态也能完整恢复。

### 4. 超时与默认处理
为审批流程设置合理的超时时间，超时后自动执行默认操作。

## 简单示例

### 示例1：基础审批流程
以下示例展示了如何使用interrupt实现简单的人工审批流程：

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.types import interrupt, Command

# 定义审批状态
class ApprovalState(TypedDict):
    proposal_content: str
    approval_status: str  # "pending", "approved", "rejected"
    approver_comment: str
    approval_history: list[str]

# 生成提案节点
def generate_proposal(state: ApprovalState) -> dict:
    """生成业务提案"""
    proposal = "提案：将营销预算增加20%，用于Q3季度的社交媒体推广活动"
    return {
        "proposal_content": proposal,
        "approval_history": ["提案已生成"]
    }

# 人工审批节点
def human_approval(state: ApprovalState) -> dict:
    """请求人工审批"""
    # 暂停工作流，等待人工审批
    decision = interrupt({
        "title": "营销预算增加审批",
        "proposal": state["proposal_content"],
        "approver_required": "财务经理",
        "deadline": "2024-09-30 18:00:00",
        "options": {
            "approve": "批准提案",
            "reject": "拒绝提案",
            "modify": "需要修改"
        },
        "context": {
            "budget_amount": "¥200,000",
            "current_budget": "¥1,000,000",
            "increase_percentage": "20%"
        }
    })
    
    # 中断后，工作流会暂停在这里
    # 当通过Command(resume=...)恢复时，decision参数会包含人工选择
    
    return {
        "approval_status": decision,
        "approval_history": [f"人工审批完成，结果：{decision}"]
    }

# 提案执行节点
def execute_proposal(state: ApprovalState) -> dict:
    """执行批准的提案"""
    if state["approval_status"] == "approve":
        action = "开始执行营销预算增加计划"
    elif state["approval_status"] == "reject":
        action = "提案被拒绝，维持原预算"
    else:
        action = "提案需要修改，通知申请人"
    
    return {
        "approval_history": [f"执行阶段：{action}"],
        "approver_comment": f"最终处理：{action}"
    }

# 构建审批工作流
def build_approval_workflow():
    builder = StateGraph(ApprovalState)
    
    # 添加节点
    builder.add_node("generate", generate_proposal)
    builder.add_node("approval", human_approval)
    builder.add_node("execute", execute_proposal)
    
    # 定义执行路径
    builder.add_edge(START, "generate")
    builder.add_edge("generate", "approval")
    builder.add_edge("approval", "execute")
    builder.add_edge("execute", END)
    
    # 编译带检查点的图，支持持久化
    return builder.compile(checkpointer=InMemorySaver())

# 测试审批流程
def test_approval_workflow():
    """测试审批工作流"""
    graph = build_approval_workflow()
    config = {"configurable": {"thread_id": "approval-001"}}
    
    print("=== 开始审批工作流测试 ===")
    
    # 初始状态
    initial_state = {
        "proposal_content": "",
        "approval_status": "pending",
        "approver_comment": "",
        "approval_history": []
    }
    
    # 步骤1：执行到人工审批节点
    print("\n1. 生成提案...")
    for chunk in graph.stream(initial_state, config, stream_mode="values"):
        node_name = list(chunk.keys())[0] if chunk else "start"
        print(f"  执行节点: {node_name}")
        
        # 检查是否到达中断点
        if "__interrupt__" in chunk:
            interrupt_data = chunk["__interrupt__"][1]  # 获取中断数据
            print(f"\n2. 工作流暂停，等待人工审批")
            print(f"   审批标题: {interrupt_data.get('title')}")
            print(f"   审批内容: {interrupt_data.get('proposal')[:50]}...")
            print(f"   审批选项: {list(interrupt_data.get('options', {}).keys())}")
            break
    
    # 模拟人工审批过程（在实际应用中，这里会等待人工操作）
    print("\n3. 模拟人工审批过程...")
    print("   [在实际系统中，这里会显示审批界面，等待审批者操作]")
    print("   [审批者查看提案详情，做出决策]")
    
    # 模拟审批者选择"批准"
    human_decision = "approve"
    print(f"   审批者决定: {human_decision}")
    
    # 步骤2：恢复工作流
    print("\n4. 恢复工作流执行...")
    resume_command = Command(resume=human_decision)
    
    for chunk in graph.stream(resume_command, config, stream_mode="values"):
        node_name = list(chunk.keys())[0] if chunk else "resume"
        print(f"  执行节点: {node_name}")
        
        # 显示执行结果
        if "execute" in chunk:
            result = chunk["execute"]
            print(f"\n5. 最终结果:")
            print(f"   审批状态: {result.get('approval_status', 'unknown')}")
            print(f"   执行动作: {result.get('approver_comment', '')}")
            print(f"   历史记录: {result.get('approval_history', [])}")

# 运行测试
test_approval_workflow()
```

### 示例2：多级审批流程
对于需要多级审批的复杂场景，可以设计级联审批工作流：

```python
from typing import Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.types import interrupt, Command

class MultiLevelApprovalState(TypedDict):
    proposal: str
    department_approval: Literal["pending", "approved", "rejected"]
    finance_approval: Literal["pending", "approved", "rejected"]
    executive_approval: Literal["pending", "approved", "rejected"]
    current_approver: str
    approval_comments: dict[str, str]

def department_approval(state: MultiLevelApprovalState) -> dict:
    """部门经理审批"""
    decision = interrupt({
        "level": "department",
        "approver_role": "部门经理",
        "proposal": state["proposal"],
        "budget_impact": "部门预算内",
        "options": ["approve", "reject", "request_changes"]
    })
    
    comment = f"部门经理审批：{decision}"
    return {
        "department_approval": decision,
        "current_approver": "finance",
        "approval_comments": {"department": comment}
    }

def finance_approval(state: MultiLevelApprovalState) -> dict:
    """财务审批"""
    # 只有部门批准后才能进行财务审批
    if state["department_approval"] != "approved":
        return {"current_approver": "end", "approval_comments": {"finance": "前置审批未通过"}}
    
    decision = interrupt({
        "level": "finance",
        "approver_role": "财务总监",
        "proposal": state["proposal"],
        "budget_impact": "跨部门预算",
        "financial_analysis": "ROI预计为150%",
        "options": ["approve", "reject", "adjust_budget"]
    })
    
    comment = f"财务审批：{decision}"
    return {
        "finance_approval": decision,
        "current_approver": "executive" if decision == "approved" else "end",
        "approval_comments": {"finance": comment}
    }

def executive_approval(state: MultiLevelApprovalState) -> dict:
    """高管审批"""
    # 只有财务批准后才能进行高管审批
    if state["finance_approval"] != "approved":
        return {"current_approver": "end", "approval_comments": {"executive": "财务审批未通过"}}
    
    decision = interrupt({
        "level": "executive",
        "approver_role": "执行副总裁",
        "proposal": state["proposal"],
        "strategic_alignment": "符合公司年度战略",
        "risk_assessment": "低风险",
        "options": ["approve", "reject", "defer"]
    })
    
    comment = f"高管审批：{decision}"
    return {
        "executive_approval": decision,
        "current_approver": "end",
        "approval_comments": {"executive": comment}
    }

def build_multi_level_approval():
    """构建多级审批工作流"""
    builder = StateGraph(MultiLevelApprovalState)
    
    builder.add_node("department", department_approval)
    builder.add_node("finance", finance_approval)
    builder.add_node("executive", executive_approval)
    
    # 定义审批流程
    builder.add_edge(START, "department")
    
    # 条件流转：根据部门审批结果决定下一步
    def after_department(state: MultiLevelApprovalState) -> Literal["finance", "end"]:
        if state["department_approval"] == "approved":
            return "finance"
        return "end"
    
    builder.add_conditional_edges("department", after_department)
    
    # 财务审批后的流转
    def after_finance(state: MultiLevelApprovalState) -> Literal["executive", "end"]:
        if state["finance_approval"] == "approved":
            return "executive"
        return "end"
    
    builder.add_conditional_edges("finance", after_finance)
    builder.add_edge("executive", END)
    
    return builder.compile(checkpointer=InMemorySaver())
```

## 进阶应用

### 动态审批路由
根据提案内容和上下文动态决定审批路径：

```python
def dynamic_approval_routing():
    """动态审批路由"""
    from typing import List
    
    class DynamicApprovalState(TypedDict):
        proposal: dict
        approval_path: List[str]
        current_step: int
        approvals: dict
    
    def route_approval(state: DynamicApprovalState) -> dict:
        """根据提案内容动态决定审批路径"""
        proposal = state["proposal"]
        amount = proposal.get("amount", 0)
        department = proposal.get("department")
        risk_level = proposal.get("risk_level", "low")
        
        # 根据金额决定审批路径
        if amount <= 10000:
            path = ["direct_manager"]
        elif amount <= 50000:
            path = ["direct_manager", "department_head"]
        elif amount <= 200000:
            path = ["direct_manager", "department_head", "finance"]
        else:
            path = ["direct_manager", "department_head", "finance", "executive"]
        
        # 高风险项目需要额外审批
        if risk_level == "high":
            path.append("risk_committee")
        
        return {
            "approval_path": path,
            "current_step": 0
        }
    
    def execute_approval_step(state: DynamicApprovalState) -> dict:
        """执行当前审批步骤"""
        current_step = state["current_step"]
        approval_path = state["approval_path"]
        
        if current_step >= len(approval_path):
            return {"current_step": current_step}  # 审批完成
        
        approver = approval_path[current_step]
        
        # 请求当前审批者审批
        decision = interrupt({
            "step": current_step + 1,
            "total_steps": len(approval_path),
            "approver": approver,
            "proposal_summary": state["proposal"],
            "previous_approvals": state.get("approvals", {})
        })
        
        # 记录审批结果
        approvals = state.get("approvals", {})
        approvals[approver] = decision
        
        return {
            "approvals": approvals,
            "current_step": current_step + 1
        }
```

def approval_delegation():
    """审批委托机制"""
    import datetime
    
    class DelegationState(TypedDict):
        original_approver: str
        delegated_to: str
        delegation_reason: str
        delegation_expiry: datetime.datetime
        approval_result: str
    
    def check_delegation(state: DelegationState) -> dict:
        """检查是否需要委托审批"""
        original_approver = state["original_approver"]
        
        # 检查原始审批者是否可用
        if not is_approver_available(original_approver):
            # 查找可用的委托审批者
            delegate = find_available_delegate(original_approver)
            if delegate:
                return {
                    "delegated_to": delegate,
                    "delegation_reason": f"{original_approver}暂时不可用",
                    "delegation_expiry": datetime.datetime.now() + datetime.timedelta(days=3)
                }
        
        return {"delegated_to": original_approver}
    
    def request_approval_with_delegation(state: DelegationState) -> dict:
        """请求审批（支持委托）"""
        approver = state["delegated_to"]
        is_delegated = approver != state["original_approver"]
        
        decision = interrupt({
            "original_approver": state["original_approver"],
            "current_approver": approver,
            "is_delegated": is_delegated,
            "delegation_reason": state.get("delegation_reason", ""),
            "delegation_expiry": state.get("delegation_expiry"),
            "options": ["approve", "reject", "defer_to_original"]
        })
        
        return {"approval_result": decision}

## 常见问题

### Q1: 审批超时如何处理？
**A**: LangGraph支持审批超时处理策略：
1. **自动提醒**：超时前发送提醒通知
2. **默认处理**：超时后执行预设的默认操作
3. **审批升级**：超时后自动转交给上级审批
4. **流程跳过**：在特定条件下跳过超时审批步骤

### Q2: 如何确保审批的安全性？
**A**: 通过以下措施确保审批安全：
1. **身份验证**：确保审批者身份真实有效
2. **权限控制**：基于角色的审批权限管理
3. **操作审计**：完整记录审批操作日志
4. **数据加密**：敏感审批数据传输加密

### Q3: 审批流程可以回退吗？
**A**: 支持多种回退策略：
1. **审批撤回**：在一定时间内允许审批者撤回决定
2. **流程回退**：支持将流程回退到特定节点重新审批
3. **异常回退**：审批出现异常时自动回退到安全状态

### Q4: 如何处理并行审批？
**A**: 并行审批处理策略：
1. **全部通过**：所有审批者都必须批准
2. **多数通过**：超过一定比例的审批者批准即可
3. **任意通过**：任意一个审批者批准即可
4. **分级并行**：不同层级的审批可以并行进行
