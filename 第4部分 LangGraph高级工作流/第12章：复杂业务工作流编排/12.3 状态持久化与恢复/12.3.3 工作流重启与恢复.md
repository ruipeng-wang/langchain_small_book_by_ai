# 12.3.3 工作流重启与恢复

工作流重启与恢复是确保系统可靠性的关键技术，允许工作流在故障、维护或人为中断后从最近的检查点恢复执行。LangGraph提供了完整的重启与恢复机制，支持精确的状态恢复和继续执行。

## 概念讲解

### 重启与恢复的价值
工作流重启与恢复机制为生产系统提供关键保障：

1. **故障容错**：系统崩溃后自动恢复，避免数据丢失和重新计算
2. **维护友好**：支持计划内维护后的无缝恢复
3. **人为干预**：允许人工暂停和恢复复杂工作流
4. **负载均衡**：支持在不同实例间迁移和恢复工作流
5. **版本升级**：系统升级后恢复旧版本的工作流

### 恢复的三种模式
根据恢复场景和需求的不同，可以分为：

1. **精确恢复**：从特定检查点恢复，完全重现之前的状态
2. **补偿恢复**：恢复后执行补偿操作，修复可能的中间状态不一致
3. **乐观恢复**：假设中间状态正确，直接继续执行后续步骤
4. **渐进恢复**：分阶段恢复，先恢复核心功能再恢复辅助功能

### LangGraph的恢复架构
LangGraph通过以下组件支持完整的工作流恢复：

- **检查点管理器**：管理检查点的创建、存储和检索
- **状态恢复器**：从检查点重建工作流状态
- **恢复验证器**：验证恢复状态的完整性和一致性
- **继续执行器**：从恢复状态继续执行工作流
- **补偿管理器**：处理恢复后的状态补偿和修复

## 核心要点

### 1. Thread ID的持久化标识
通过唯一的`thread_id`标识工作流实例，确保恢复时能定位正确的状态。

### 2. 检查点链的完整性
维护完整的检查点链，支持按时间顺序恢复任意检查点。

### 3. 状态一致性保证
确保恢复后的状态与原始执行状态完全一致，包括所有中间变量。

### 4. 恢复后执行上下文
恢复完整的执行上下文，包括环境变量、配置参数和外部依赖状态。

## 简单示例

### 示例1：基础工作流重启
以下示例展示了如何实现工作流的中断和恢复：

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
import time
import random

class ProcessingState(TypedDict):
    items: list[str]
    processed_items: list[str]
    current_index: int
    status: str  # "running", "paused", "completed", "failed"
    last_checkpoint_time: float
    recovery_count: int

def data_processor(state: ProcessingState) -> dict:
    """数据处理节点"""
    import time
    
    # 检查是否需要恢复
    if state.get("status") == "paused":
        print(f"从暂停状态恢复，继续处理第 {state.get('current_index', 0)} 项")
    
    items = state.get("items", [])
    current_index = state.get("current_index", 0)
    processed_items = state.get("processed_items", [])
    
    if current_index >= len(items):
        return {
            "status": "completed",
            "processed_items": processed_items
        }
    
    # 处理当前项
    current_item = items[current_index]
    
    # 模拟处理逻辑（可能失败）
    processing_time = random.uniform(0.1, 0.5)
    time.sleep(processing_time)
    
    # 随机模拟处理失败
    if random.random() < 0.1:  # 10%失败率
        raise ValueError(f"处理项目失败: {current_item}")
    
    # 成功处理
    processed_items.append(f"{current_item} (处理时间: {processing_time:.2f}s)")
    
    # 定期创建检查点（每3个项目）
    checkpoint_time = time.time()
    should_checkpoint = (current_index + 1) % 3 == 0
    
    return {
        "processed_items": processed_items,
        "current_index": current_index + 1,
        "status": "running",
        "last_checkpoint_time": checkpoint_time if should_checkpoint else state.get("last_checkpoint_time", 0),
        "recovery_count": state.get("recovery_count", 0)
    }

def build_restartable_workflow():
    """构建可重启的工作流"""
    builder = StateGraph(ProcessingState)
    
    builder.add_node("process", data_processor)
    
    # 条件边：根据处理进度决定下一步
    def check_completion(state: ProcessingState):
        current_index = state.get("current_index", 0)
        items = state.get("items", [])
        
        if current_index >= len(items):
            return END  # 完成
        else:
            return "process"  # 继续处理
    
    builder.add_edge(START, "process")
    builder.add_conditional_edges("process", check_completion)
    
    return builder.compile(checkpointer=InMemorySaver())

def simulate_workflow_with_failure_and_recovery():
    """模拟工作流故障和恢复场景"""
    print("=== 工作流重启与恢复测试 ===")
    
    # 创建可重启的工作流
    graph = build_restartable_workflow()
    
    # 准备测试数据
    test_items = [f"Item-{i}" for i in range(10)]
    
    # 场景1：正常执行
    print("\n1. 场景1: 正常执行（无故障）")
    thread_id = "normal-execution"
    config = {"configurable": {"thread_id": thread_id}}
    
    initial_state = {
        "items": test_items,
        "processed_items": [],
        "current_index": 0,
        "status": "running",
        "last_checkpoint_time": 0,
        "recovery_count": 0
    }
    
    try:
        result = graph.invoke(initial_state, config)
        print(f"   成功处理项目数: {len(result['processed_items'])}")
        print(f"   最终状态: {result['status']}")
    except Exception as e:
        print(f"   执行失败: {e}")
    
    # 场景2：模拟故障并恢复
    print("\n2. 场景2: 模拟故障和恢复")
    
    # 创建新工作流实例（模拟系统重启）
    recovered_graph = build_restartable_workflow()
    
    # 使用相同的thread_id恢复状态
    recovered_config = {"configurable": {"thread_id": thread_id}}
    
    try:
        # 尝试从检查点恢复并继续执行
        recovered_result = recovered_graph.invoke(None, recovered_config)
        print(f"   恢复后处理项目数: {len(recovered_result.get('processed_items', []))}")
        print(f"   恢复后状态: {recovered_result.get('status', 'unknown')}")
        
        # 检查恢复的完整性
        original_state = graph.get_state(config)
        recovered_state = recovered_graph.get_state(recovered_config)
        
        print(f"   原始检查点ID: {original_state.config['configurable'].get('checkpoint_id', 'N/A')[:16]}...")
        print(f"   恢复检查点ID: {recovered_state.config['configurable'].get('checkpoint_id', 'N/A')[:16]}...")
        
    except Exception as e:
        print(f"   恢复失败: {e}")
    
    # 场景3：手动暂停和恢复
    print("\n3. 场景3: 手动暂停和恢复")
    
    manual_thread_id = "manual-pause-resume"
    manual_config = {"configurable": {"thread_id": manual_thread_id}}
    
    # 第一阶段：执行部分工作
    print("   第一阶段: 执行前5个项目")
    
    stage1_state = {
        "items": test_items[:5],  # 只处理前5个
        "processed_items": [],
        "current_index": 0,
        "status": "running",
        "last_checkpoint_time": 0,
        "recovery_count": 0
    }
    
    stage1_result = graph.invoke(stage1_state, manual_config)
    print(f"   第一阶段完成，处理了 {len(stage1_result['processed_items'])} 个项目")
    
    # 模拟手动暂停
    print("   模拟: 手动暂停工作流")
    
    # 第二阶段：从暂停状态恢复，处理剩余项目
    print("   第二阶段: 从暂停状态恢复，处理所有10个项目")
    
    stage2_state = {
        "items": test_items,  # 所有10个项目
        "processed_items": [],
        "current_index": 0,
        "status": "paused",  # 标记为暂停状态
        "last_checkpoint_time": 0,
        "recovery_count": 1  # 增加恢复计数
    }
    
    # 从检查点恢复（不传递初始状态）
    try:
        final_result = graph.invoke(None, manual_config)
        print(f"   最终处理项目数: {len(final_result.get('processed_items', []))}")
        print(f"   恢复次数: {final_result.get('recovery_count', 0)}")
    except Exception as e:
        print(f"   恢复执行失败: {e}")

# 运行测试
simulate_workflow_with_failure_and_recovery()
```

### 示例2：多阶段恢复策略
以下示例展示了针对不同故障类型采用不同的恢复策略：

```python
from typing import Literal, Optional
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
import time

class MultiStageRecoveryState(TypedDict):
    processing_stage: Literal["stage1", "stage2", "stage3", "completed", "failed"]
    stage_data: dict
    recovery_strategy: Literal["full", "partial", "incremental", "compensating"]
    recovery_history: list[dict]
    consistency_checks: dict

def detect_failure_type(state: MultiStageRecoveryState) -> dict:
    """检测故障类型并选择恢复策略"""
    # 模拟故障检测逻辑
    import random
    
    failure_types = [
        "network_timeout",
        "resource_exhausted", 
        "data_corruption",
        "external_service_down",
        "validation_failed"
    ]
    
    # 随机选择故障类型（测试用）
    failure_type = random.choice(failure_types) if random.random() < 0.3 else None
    
    if failure_type:
        # 根据故障类型选择恢复策略
        if failure_type in ["network_timeout", "external_service_down"]:
            strategy = "incremental"  # 增量恢复，重试失败部分
        elif failure_type == "resource_exhausted":
            strategy = "partial"  # 部分恢复，跳过非关键任务
        elif failure_type == "data_corruption":
            strategy = "compensating"  # 补偿恢复，修复损坏数据
        else:
            strategy = "full"  # 完全恢复，从头开始
        
        return {
            "failure_detected": True,
            "failure_type": failure_type,
            "recovery_strategy": strategy,
            "recovery_history": [{
                "timestamp": time.time(),
                "failure_type": failure_type,
                "strategy": strategy,
                "stage": state.get("processing_stage", "unknown")
            }]
        }
    
    return {"failure_detected": False}

def stage1_processing(state: MultiStageRecoveryState) -> dict:
    """第一阶段处理"""
    print("执行: 第一阶段处理")
    
    # 模拟处理逻辑
    time.sleep(0.3)
    
    # 检测故障
    failure_info = detect_failure_type(state)
    
    if failure_info.get("failure_detected", False):
        print(f"  检测到故障: {failure_info['failure_type']}")
        print(f"  恢复策略: {failure_info['recovery_strategy']}")
        
        # 执行故障处理
        return execute_recovery(state, failure_info)
    
    # 正常处理
    stage_data = state.get("stage_data", {})
    stage_data["stage1"] = {
        "completed": True,
        "timestamp": time.time(),
        "data": "stage1_processed_data"
    }
    
    return {
        "processing_stage": "stage2",
        "stage_data": stage_data,
        "consistency_checks": {"stage1": "passed"}
    }

def stage2_processing(state: MultiStageRecoveryState) -> dict:
    """第二阶段处理"""
    print("执行: 第二阶段处理")
    
    # 检查前置阶段是否完成
    stage_data = state.get("stage_data", {})
    if "stage1" not in stage_data:
        return {"processing_stage": "failed", "recovery_strategy": "full"}
    
    # 模拟处理逻辑
    time.sleep(0.4)
    
    # 检测故障
    failure_info = detect_failure_type(state)
    
    if failure_info.get("failure_detected", False):
        print(f"  检测到故障: {failure_info['failure_type']}")
        
        # 根据故障类型决定恢复策略
        if failure_info["failure_type"] == "data_corruption":
            # 数据损坏，需要补偿恢复
            return {
                "processing_stage": "stage1",  # 回退到第一阶段
                "recovery_strategy": "compensating",
                "recovery_history": state.get("recovery_history", []) + [failure_info.get("recovery_history", [{}])[0]]
            }
        else:
            return execute_recovery(state, failure_info)
    
    # 正常处理
    stage_data["stage2"] = {
        "completed": True,
        "timestamp": time.time(),
        "data": "stage2_processed_data",
        "depends_on": "stage1"
    }
    
    return {
        "processing_stage": "stage3",
        "stage_data": stage_data,
        "consistency_checks": {**state.get("consistency_checks", {}), "stage2": "passed"}
    }

def stage3_processing(state: MultiStageRecoveryState) -> dict:
    """第三阶段处理"""
    print("执行: 第三阶段处理")
    
    # 检查前置阶段是否完成
    stage_data = state.get("stage_data", {})
    if "stage2" not in stage_data:
        return {"processing_stage": "failed", "recovery_strategy": "full"}
    
    # 模拟处理逻辑
    time.sleep(0.5)
    
    # 最终一致性检查
    consistency_checks = state.get("consistency_checks", {})
    all_passed = all(v == "passed" for v in consistency_checks.values())
    
    if not all_passed:
        print("  一致性检查失败，需要补偿恢复")
        return {
            "processing_stage": "stage1",  # 回退到第一阶段
            "recovery_strategy": "compensating",
            "recovery_history": state.get("recovery_history", []) + [{
                "timestamp": time.time(),
                "failure_type": "consistency_failure",
                "strategy": "compensating",
                "stage": "stage3"
            }]
        }
    
    # 完成处理
    stage_data["stage3"] = {
        "completed": True,
        "timestamp": time.time(),
        "data": "stage3_processed_data",
        "depends_on": ["stage1", "stage2"]
    }
    
    return {
        "processing_stage": "completed",
        "stage_data": stage_data,
        "consistency_checks": {**consistency_checks, "stage3": "passed"}
    }

def execute_recovery(state: MultiStageRecoveryState, failure_info: dict) -> dict:
    """执行恢复操作"""
    strategy = failure_info.get("recovery_strategy", "full")
    
    print(f"  执行恢复策略: {strategy}")
    
    if strategy == "full":
        # 完全恢复：从头开始
        return {
            "processing_stage": "stage1",
            "recovery_strategy": strategy,
            "recovery_history": state.get("recovery_history", []) + failure_info.get("recovery_history", [])
        }
    
    elif strategy == "partial":
        # 部分恢复：跳过非关键任务，继续当前阶段
        time.sleep(0.2)  # 模拟恢复操作
        return {
            "processing_stage": state.get("processing_stage", "stage1"),
            "recovery_strategy": strategy,
            "stage_data": state.get("stage_data", {}),
            "recovery_history": state.get("recovery_history", []) + failure_info.get("recovery_history", [])
        }
    
    elif strategy == "incremental":
        # 增量恢复：重试当前操作
        print("  执行增量恢复：重试当前操作")
        time.sleep(0.1)
        # 返回相同阶段，让工作流重试
        return {
            "processing_stage": state.get("processing_stage", "stage1"),
            "recovery_strategy": strategy,
            "recovery_history": state.get("recovery_history", []) + failure_info.get("recovery_history", [])
        }
    
    else:  # compensating
        # 补偿恢复：执行补偿操作后继续
        print("  执行补偿恢复：修复数据后继续")
        time.sleep(0.3)
        
        # 模拟数据修复
        stage_data = state.get("stage_data", {}).copy()
        if "stage2" in stage_data:
            stage_data["stage2"]["repaired"] = True
            stage_data["stage2"]["repair_timestamp"] = time.time()
        
        return {
            "processing_stage": state.get("processing_stage", "stage1"),
            "recovery_strategy": strategy,
            "stage_data": stage_data,
            "recovery_history": state.get("recovery_history", []) + failure_info.get("recovery_history", [])
        }

def build_multi_stage_recovery_workflow():
    """构建多阶段恢复工作流"""
    builder = StateGraph(MultiStageRecoveryState)
    
    builder.add_node("stage1", stage1_processing)
    builder.add_node("stage2", stage2_processing)
    builder.add_node("stage3", stage3_processing)
    
    # 定义执行流程
    builder.add_edge(START, "stage1")
    
    # 条件边：根据处理阶段决定下一步
    def route_by_stage(state: MultiStageRecoveryState):
        stage = state.get("processing_stage", "stage1")
        
        if stage == "completed":
            return END
        elif stage == "failed":
            # 故障处理：根据恢复策略决定
            strategy = state.get("recovery_strategy", "full")
            if strategy == "compensating":
                return "stage1"  # 补偿恢复需要回退
            else:
                return "stage1"  # 其他恢复从头开始
        else:
            return stage  # stage1, stage2, stage3
    
    builder.add_conditional_edges("stage1", route_by_stage)
    builder.add_conditional_edges("stage2", route_by_stage)
    builder.add_conditional_edges("stage3", route_by_stage)
    
    return builder.compile(checkpointer=InMemorySaver())

def test_multi_stage_recovery():
    """测试多阶段恢复"""
    print("=== 多阶段恢复策略测试 ===")
    
    graph = build_multi_stage_recovery_workflow()
    
    # 设置随机种子以获得可重复的测试结果
    import random
    random.seed(42)  # 固定随机种子
    
    test_cases = [
        {"name": "正常流程", "seed": 42},  # 这个种子会产生正常流程
        {"name": "网络超时恢复", "seed": 123},
        {"name": "数据损坏恢复", "seed": 456},
        {"name": "资源耗尽恢复", "seed": 789}
    ]
    
    for i, test_case in enumerate(test_cases, 1):
        print(f"\n{i}. 测试用例: {test_case['name']}")
        
        random.seed(test_case["seed"])
        
        config = {"configurable": {"thread_id": f"recovery-test-{i}"}}
        
        initial_state = {
            "processing_stage": "stage1",
            "stage_data": {},
            "recovery_strategy": "full",
            "recovery_history": [],
            "consistency_checks": {}
        }
        
        try:
            result = graph.invoke(initial_state, config)
            
            print(f"   最终阶段: {result.get('processing_stage', 'unknown')}")
            print(f"   恢复策略使用次数: {len(result.get('recovery_history', []))}")
            print(f"   完成的阶段: {list(result.get('stage_data', {}).keys())}")
            
            if result.get("processing_stage") == "completed":
                print(f"   状态: 成功完成")
            else:
                print(f"   状态: 未完成，最后阶段: {result.get('processing_stage')}")
        
        except Exception as e:
            print(f"   执行异常: {e}")
    
    random.seed()  # 恢复随机种子

# 运行测试
test_multi_stage_recovery()
```

### 分布式工作流恢复
在分布式环境中恢复工作流：

```python
def distributed_workflow_recovery():
    """分布式工作流恢复"""
    from typing import Dict, Any, Optional
    import uuid
    import threading
    
    class DistributedRecoveryManager:
        """分布式恢复管理器"""
        
        def __init__(self, coordination_service):
            self.coordination = coordination_service
            self.local_checkpoints = {}
            self.recovery_lock = threading.Lock()
            self.active_recoveries = {}
        
        def register_workflow_instance(self, workflow_id: str, instance_id: str, 
                                     node_mapping: Dict[str, str]):
            """注册工作流实例"""
            # 记录工作流实例到节点的映射
            instance_info = {
                "workflow_id": workflow_id,
                "instance_id": instance_id,
                "nodes": node_mapping,
                "status": "running",
                "registered_at": time.time()
            }
            
            self.coordination.set(f"workflow:{workflow_id}:instance:{instance_id}", 
                                instance_info)
            
            # 发布注册事件
            self.coordination.publish(f"workflow:{workflow_id}:registered", 
                                    {"instance_id": instance_id})
        
        def create_distributed_checkpoint(self, workflow_id: str, instance_id: str,
                                        checkpoint_data: Dict[str, Any]) -> str:
            """创建分布式检查点"""
            checkpoint_id = str(uuid.uuid4())
            
            # 将检查点数据分布存储到不同节点
            for node_id, node_state in checkpoint_data.get("node_states", {}).items():
                checkpoint_key = f"checkpoint:{checkpoint_id}:node:{node_id}"
                self.coordination.set(checkpoint_key, {
                    "workflow_id": workflow_id,
                    "instance_id": instance_id,
                    "node_id": node_id,
                    "state": node_state,
                    "timestamp": time.time()
                })
            
            # 记录检查点元数据
            checkpoint_meta = {
                "checkpoint_id": checkpoint_id,
                "workflow_id": workflow_id,
                "instance_id": instance_id,
                "created_at": time.time(),
                "node_count": len(checkpoint_data.get("node_states", {})),
                "coordinator_node": self._get_current_node_id()
            }
            
            self.coordination.set(f"checkpoint:{checkpoint_id}:meta", checkpoint_meta)
            
            # 更新实例的最新检查点
            self.coordination.set(f"workflow:{workflow_id}:instance:{instance_id}:latest_checkpoint",
                                checkpoint_id)
            
            return checkpoint_id
        
        def recover_workflow_instance(self, workflow_id: str, instance_id: str,
                                    checkpoint_id: Optional[str] = None) -> Dict[str, Any]:
            """恢复工作流实例"""
            with self.recovery_lock:
                recovery_id = str(uuid.uuid4())
                self.active_recoveries[recovery_id] = {
                    "workflow_id": workflow_id,
                    "instance_id": instance_id,
                    "status": "starting",
                    "started_at": time.time()
                }
                
                try:
                    # 获取检查点ID（如果不提供则使用最新的）
                    if checkpoint_id is None:
                        checkpoint_id = self.coordination.get(
                            f"workflow:{workflow_id}:instance:{instance_id}:latest_checkpoint")
                    
                    if not checkpoint_id:
                        raise ValueError(f"No checkpoint found for instance {instance_id}")
                    
                    # 获取检查点元数据
                    checkpoint_meta = self.coordination.get(f"checkpoint:{checkpoint_id}:meta")
                    if not checkpoint_meta:
                        raise ValueError(f"Checkpoint {checkpoint_id} not found")
                    
                    # 恢复各节点状态
                    recovered_states = {}
                    for node_id in checkpoint_meta.get("node_ids", []):
                        checkpoint_key = f"checkpoint:{checkpoint_id}:node:{node_id}"
                        node_state = self.coordination.get(checkpoint_key)
                        
                        if node_state:
                            recovered_states[node_id] = node_state["state"]
                    
                    # 获取实例信息
                    instance_info = self.coordination.get(
                        f"workflow:{workflow_id}:instance:{instance_id}")
                    
                    recovery_result = {
                        "recovery_id": recovery_id,
                        "checkpoint_id": checkpoint_id,
                        "workflow_id": workflow_id,
                        "instance_id": instance_id,
                        "recovered_states": recovered_states,
                        "instance_info": instance_info,
                        "recovery_timestamp": time.time(),
                        "node_count": len(recovered_states)
                    }
                    
                    # 更新恢复状态
                    self.active_recoveries[recovery_id]["status"] = "completed"
                    self.active_recoveries[recovery_id]["completed_at"] = time.time()
                    self.active_recoveries[recovery_id]["result"] = recovery_result
                    
                    # 发布恢复完成事件
                    self.coordination.publish(f"workflow:{workflow_id}:recovered", {
                        "instance_id": instance_id,
                        "checkpoint_id": checkpoint_id,
                        "recovery_id": recovery_id
                    })
                    
                    return recovery_result
                
                except Exception as e:
                    # 记录恢复失败
                    self.active_recoveries[recovery_id]["status"] = "failed"
                    self.active_recoveries[recovery_id]["error"] = str(e)
                    self.active_recoveries[recovery_id]["failed_at"] = time.time()
                    
                    raise
```

## 常见问题

### Q1: 恢复后状态不一致怎么办？
**A**: 状态不一致处理策略：
1. **状态验证**：恢复后立即验证状态一致性
2. **补偿事务**：执行补偿操作修复不一致状态
3. **回滚机制**：状态不一致时回滚到上一个一致检查点
4. **人工干预**：复杂不一致时请求人工处理
5. **渐进修复**：分步骤修复不一致状态

### Q2: 如何确保恢复的原子性？
**A**: 恢复原子性保障：
1. **事务性存储**：使用支持事务的存储后端
2. **两阶段提交**：采用两阶段提交协议
3. **检查点锁**：恢复期间锁定检查点防止并发修改
4. **幂等恢复**：确保恢复操作可以安全重试
5. **状态快照**：恢复前创建状态快照用于回滚

### Q3: 恢复性能如何优化？
**A**: 恢复性能优化策略：
1. **增量恢复**：只恢复变更部分而非完整状态
2. **并行恢复**：并行恢复多个组件状态
3. **缓存预热**：恢复后预热关键缓存
4. **懒加载**：延迟加载非关键状态
5. **预测恢复**：基于历史数据预测和预加载

### Q4: 分布式恢复如何协调？
**A**: 分布式恢复协调策略：
1. **协调服务**：使用ZooKeeper等协调服务
2. **共识算法**：采用Raft等共识算法确保一致性
3. **领导者选举**：选举恢复协调者
4. **状态同步**：确保各节点状态同步
5. **故障检测**：实时检测节点故障
