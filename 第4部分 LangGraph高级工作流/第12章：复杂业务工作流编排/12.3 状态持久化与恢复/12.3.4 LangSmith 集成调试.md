# 12.3.4 LangSmith 集成调试

## 概念讲解

随着 LangGraph 工作流变得越来越复杂，传统的日志和打印调试方式已经无法满足需求。**LangSmith** 是 LangChain 官方提供的可观测性平台，为 LangGraph 工作流提供深度的可视化调试能力，包括执行路径追踪、状态转换记录、延迟分析和输出质量评估。

### LangSmith 的核心价值

1. **可视化执行路径**：图形化展示工作流的实际执行路径，包括分支选择、循环次数等
2. **节点级别追踪**：记录每个节点的输入输出、执行时间和错误信息
3. **状态历史回溯**：查看工作流在每个检查点的完整状态
4. **性能分析**：识别工作流中的性能瓶颈

### 启用 LangSmith 追踪

启用 LangSmith 非常简单，只需设置环境变量即可。启用后，所有 LangGraph 的操作都会被自动记录并上传到 LangSmith 平台。

## 核心要点

- **环境变量一键启用**：设置 `LANGSMITH_TRACING=true` 即可启用
- **自动追踪图执行**：每个节点的输入输出自动记录
- **调试模式实时查看**：使用 `stream_mode="debug"` 实时查看执行详情
- **选择性追踪**：使用 `tracing_context` 精确控制追踪范围
- **Xray 视图查看子图**：`get_graph(xray=True)` 展开子图查看内部结构

## 简单示例

### 启用 LangSmith 追踪

```python
# 设置环境变量（在执行任何 LangGraph 代码前设置）
import os
os.environ["LANGSMITH_TRACING"] = "true"
os.environ["LANGSMITH_API_KEY"] = "your-api-key-here"
os.environ["LANGSMITH_PROJECT"] = "my-langgraph-project"  # 可选，自定义项目名

# 之后所有的 LangGraph 操作都会被自动追踪
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    message: str
    result: str

def step_one(state: State):
    return {"result": f"步骤一处理: {state['message']}"}

def step_two(state: State):
    return {"result": state["result"] + "，步骤二完成"}

builder = StateGraph(State)
builder.add_node("one", step_one)
builder.add_node("two", step_two)
builder.add_edge(START, "one")
builder.add_edge("one", "two")
builder.add_edge("two", END)

graph = builder.compile()
result = graph.invoke({"message": "测试消息", "result": ""})
# 执行完成后，可以在 LangSmith 平台查看完整的执行轨迹
```

### 本地调试模式

不想使用云端平台时，可以使用本地调试模式：

```python
# 使用 debug 模式实时查看执行信息
for chunk in graph.stream({"message": "测试", "result": ""}, stream_mode="debug"):
    if chunk["type"] == "debug":
        data = chunk["data"]
        print(f"节点: {data.get('name')}")
        print(f"状态: {data.get('payload')}")
        print("---")
```

### 可视化图结构

```python
from IPython.display import Image, display

# 生成图的可视化（包含子图内部结构）
display(Image(graph.get_graph(xray=True).draw_mermaid_png()))
```

## 进阶应用

### 选择性追踪

只想追踪特定部分的代码，使用 `tracing_context`：

```python
import langsmith as ls

# 这段代码会被追踪
with ls.tracing_context(enabled=True):
    result = graph.invoke({"message": "重要请求", "result": ""}, config)

# 这段代码不会被追踪（如果 LANGSMITH_TRACING 未设置）
result = graph.invoke({"message": "普通请求", "result": ""})
```

### 自定义项目名和标签

```python
from langsmith import traceable

@traceable(project_name="my-custom-project", tags=["production", "v2"])
def run_workflow(message: str):
    return graph.invoke({"message": message, "result": ""})
```

### 查看状态历史

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)
config = {"configurable": {"thread_id": "demo-001"}}

# 执行工作流
graph.invoke({"message": "测试", "result": ""}, config)

# 查看状态历史
for state in graph.get_state_history(config):
    print(f"步骤: {state.metadata.get('step')}")
    print(f"状态: {state.values}")
    print(f"下一个节点: {state.next}")
    print("---")
```

## 常见问题

### Q1: LangSmith 是免费的吗？

**A**: LangSmith 提供免费额度，超出后需要付费。对于个人开发者和小型团队，免费额度通常足够日常开发和调试使用。

### Q2: 不想使用云端追踪怎么办？

**A**: 可以不设置 `LANGSMITH_TRACING` 环境变量，或使用本地 `stream_mode="debug"` 进行调试。

### Q3: 如何追踪子图的执行？

**A**: 使用 `get_graph(xray=True)` 可以在可视化时展开子图内部结构。LangSmith 平台会自动记录子图的执行详情。

### Q4: 如何过滤噪音日志？

**A**: 使用 `tracing_context(enabled=True)` 只追踪关键代码路径，或使用 `@traceable` 装饰器精确控制追踪范围。

## 本节总结

LangSmith 集成调试为 LangGraph 工作流提供了强大的可观测性能力：

1. 环境变量一键启用自动追踪
2. 可视化执行路径和节点级别详情
3. 调试模式支持本地实时查看
4. 选择性追踪精确控制追踪范围
5. 状态历史回溯支持时间旅行调试

掌握 LangSmith 集成调试，是高效开发和运维 LangGraph 工作流的必备技能。
