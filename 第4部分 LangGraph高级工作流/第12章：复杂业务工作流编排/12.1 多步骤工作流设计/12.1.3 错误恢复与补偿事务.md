# 12.1.3 错误恢复与补偿事务

在复杂业务工作流中，错误恢复与补偿事务是确保系统健壮性和数据一致性的关键机制。LangGraph提供了完整的错误处理框架，支持从简单的重试策略到复杂的补偿事务模式。

## 概念讲解

### 错误处理的三层架构
LangGraph的错误处理机制分为三个层次：

1. **节点级错误处理**：单个节点的异常捕获和恢复
2. **图级错误处理**：整个工作流的错误传播和恢复策略
3. **事务级补偿**：业务操作失败时的逆向补偿操作

### 错误类型分类
根据错误的可恢复性，可以分为：

- **瞬时错误**：网络波动、临时资源不可用等，适合重试
- **持久错误**：数据格式错误、权限不足等，需要人工干预
- **业务逻辑错误**：违反业务规则，需要特定补偿逻辑

### 补偿事务模式
补偿事务（Compensating Transaction）是分布式系统中确保最终一致性的关键模式：
- **正向操作**：执行业务逻辑
- **补偿操作**：当正向操作失败时，执行逆向操作回滚已完成的步骤
- **幂等性**：补偿操作必须支持多次执行，确保一致性

## 核心要点

### 1. 重试策略的灵活配置
LangGraph的`RetryPolicy`支持多种重试策略，包括指数退避、抖动、自定义重试条件等。

### 2. 错误传播与隔离控制
可以配置错误是否在节点间传播，或者被隔离在特定执行路径中。

### 3. 补偿事务链自动执行
当工作流部分失败时，自动触发已成功节点的补偿操作，确保系统状态一致性。

### 4. 检查点与状态恢复
结合检查点机制，可以在错误恢复时从最近的成功状态继续执行。

## 简单示例

### 示例1：基础重试策略
以下示例展示了如何使用LangGraph的RetryPolicy处理瞬时错误：

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy
import random
import time

class OrderState(TypedDict):
    order_id: str
    attempts: int
    payment_status: str
    errors: list[str]

# 自定义重试条件：只重试特定类型的错误
def should_retry(error: Exception) -> bool:
    """判断是否应该重试"""
    # 重试网络错误和超时错误
    retryable_errors = (ConnectionError, TimeoutError, ConnectionRefusedError)
    return isinstance(error, retryable_errors)

# 创建重试策略
payment_retry_policy = RetryPolicy(
    initial_interval=1.0,     # 初始间隔1秒
    backoff_factor=2.0,       # 指数退避因子
    max_interval=30.0,        # 最大间隔30秒
    max_attempts=3,           # 最多重试3次
    jitter=True,              # 添加随机抖动避免惊群效应
    retry_on=should_retry     # 自定义重试条件
)

# 模拟不可靠的支付服务
def process_payment(state: OrderState) -> dict:
    """处理支付（模拟不可靠服务）"""
    attempts = state.get("attempts", 0) + 1
    
    # 模拟70%的失败率
    if random.random() < 0.7 and attempts < 4:
        raise ConnectionError(f"支付服务暂时不可用（第{attempts}次尝试）")
    
    # 模拟支付成功
    time.sleep(0.5)  # 模拟处理时间
    return {
        "payment_status": "paid",
        "attempts": attempts
    }

# 补偿操作：支付失败时的处理
def compensate_payment(state: OrderState) -> dict:
    """支付补偿操作"""
    return {
        "payment_status": "cancelled",
        "errors": ["支付失败，订单已取消"]
    }

# 构建带错误处理的工作流
def build_payment_workflow():
    builder = StateGraph(OrderState)
    
    # 添加支付节点，附带重试策略
    builder.add_node(
        "process_payment", 
        process_payment,
        retry_policy=payment_retry_policy  # 附加重试策略
    )
    
    # 添加补偿节点
    builder.add_node("compensate_payment", compensate_payment)
    
    # 正常流程
    builder.add_edge(START, "process_payment")
    
    # 条件边：根据支付结果决定下一步
    def check_payment_result(state: OrderState):
        if state.get("payment_status") == "paid":
            return END  # 支付成功，结束
        else:
            return "compensate_payment"  # 支付失败，执行补偿
    
    builder.add_conditional_edges("process_payment", check_payment_result)
    builder.add_edge("compensate_payment", END)
    
    return builder.compile()

# 测试重试机制
graph = build_payment_workflow()

# 测试用例1：支付成功的情况
print("=== 测试用例1: 支付成功 ===")
try:
    result = graph.invoke({"order_id": "ORD001", "attempts": 0, "payment_status": "", "errors": []})
    print(f"支付结果: {result['payment_status']}")
    print(f"尝试次数: {result['attempts']}")
except Exception as e:
    print(f"最终错误: {e}")

# 测试用例2：支付失败的情况
print("\n=== 测试用例2: 支付失败 ===")
# 设置随机种子以模拟失败场景
random.seed(42)  # 这会确保支付失败
try:
    result = graph.invoke({"order_id": "ORD002", "attempts": 0, "payment_status": "", "errors": []})
    print(f"最终状态: {result}")
except Exception as e:
    print(f"工作流执行失败: {e}")
```

### 示例2：补偿事务链
以下示例展示了如何实现完整的补偿事务链：

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
import operator

class TransactionState(TypedDict):
    # 正向操作状态
    inventory_reserved: bool
    payment_processed: bool
    shipping_scheduled: bool
    
    # 补偿操作状态
    inventory_compensated: bool
    payment_refunded: bool
    shipping_cancelled: bool
    
    # 执行日志
    actions_log: Annotated[list[str], operator.add]
    errors: Annotated[list[str], operator.add]

# 正向操作1：库存预留
def reserve_inventory(state: TransactionState) -> dict:
    """预留库存"""
    # 模拟库存预留逻辑
    print("执行: 库存预留")
    return {
        "inventory_reserved": True,
        "actions_log": ["库存预留成功"]
    }

# 正向操作2：支付处理
def process_payment_transaction(state: TransactionState) -> dict:
    """处理支付"""
    # 检查前置条件
    if not state.get("inventory_reserved", False):
        raise ValueError("库存未预留，无法处理支付")
    
    # 模拟支付处理（可能失败）
    import random
    if random.random() < 0.3:  # 30%失败率
        raise ConnectionError("支付网关连接失败")
    
    print("执行: 支付处理")
    return {
        "payment_processed": True,
        "actions_log": ["支付处理成功"]
    }

# 正向操作3：发货安排
def schedule_shipping(state: TransactionState) -> dict:
    """安排发货"""
    # 检查前置条件
    if not state.get("payment_processed", False):
        raise ValueError("支付未处理，无法安排发货")
    
    print("执行: 发货安排")
    return {
        "shipping_scheduled": True,
        "actions_log": ["发货安排成功"]
    }

# 补偿操作1：库存释放
def compensate_inventory(state: TransactionState) -> dict:
    """释放预留的库存"""
    if state.get("inventory_reserved", False):
        print("补偿: 释放库存")
        return {
            "inventory_compensated": True,
            "actions_log": ["库存释放成功"]
        }
    return {}

# 补偿操作2：支付退款
def compensate_payment(state: TransactionState) -> dict:
    """处理支付退款"""
    if state.get("payment_processed", False):
        print("补偿: 支付退款")
        return {
            "payment_refunded": True,
            "actions_log": ["支付退款成功"]
        }
    return {}

# 补偿操作3：取消发货
def compensate_shipping(state: TransactionState) -> dict:
    """取消发货安排"""
    if state.get("shipping_scheduled", False):
        print("补偿: 取消发货")
        return {
            "shipping_cancelled": True,
            "actions_log": ["发货取消成功"]
        }
    return {}

# 构建补偿事务工作流
def build_compensation_workflow():
    builder = StateGraph(TransactionState)
    
    # 添加正向操作节点
    builder.add_node("reserve_inventory", reserve_inventory)
    builder.add_node("process_payment", process_payment_transaction)
    builder.add_node("schedule_shipping", schedule_shipping)
    
    # 添加补偿操作节点
    builder.add_node("compensate_inventory", compensate_inventory)
    builder.add_node("compensate_payment", compensate_payment)
    builder.add_node("compensate_shipping", compensate_shipping)
    
    # 正向执行路径
    builder.add_edge(START, "reserve_inventory")
    builder.add_edge("reserve_inventory", "process_payment")
    builder.add_edge("process_payment", "schedule_shipping")
    builder.add_edge("schedule_shipping", END)
    
    # 错误处理：捕获异常并执行补偿链
    def handle_error(state: TransactionState, error: Exception) -> dict:
        """错误处理函数"""
        print(f"捕获到错误: {error}")
        
        # 决定执行哪些补偿操作（基于已完成的步骤）
        compensation_steps = []
        
        if state.get("shipping_scheduled", False):
            compensation_steps.append("compensate_shipping")
        if state.get("payment_processed", False):
            compensation_steps.append("compensate_payment")
        if state.get("inventory_reserved", False):
            compensation_steps.append("compensate_inventory")
        
        # 返回错误信息和补偿步骤
        return {
            "errors": [f"错误: {str(error)}"],
            "__next__": compensation_steps[0] if compensation_steps else END
        }
    
    # 为可能失败的节点配置错误处理
    builder.add_error_handler("process_payment", handle_error)
    builder.add_error_handler("schedule_shipping", handle_error)
    
    # 补偿链执行顺序
    builder.add_edge("compensate_shipping", "compensate_payment")
    builder.add_edge("compensate_payment", "compensate_inventory")
    builder.add_edge("compensate_inventory", END)
    
    # 编译带检查点的图
    return builder.compile(checkpointer=InMemorySaver())

# 测试补偿事务
def test_compensation_workflow():
    graph = build_compensation_workflow()
    config = {"configurable": {"thread_id": "tx-001"}}
    
    print("=== 测试补偿事务工作流 ===")
    
    # 初始状态
    initial_state = {
        "inventory_reserved": False,
        "payment_processed": False,
        "shipping_scheduled": False,
        "inventory_compensated": False,
        "payment_refunded": False,
        "shipping_cancelled": False,
        "actions_log": [],
        "errors": []
    }
    
    try:
        # 执行工作流
        result = graph.invoke(initial_state, config)
        print(f"\n工作流执行成功!")
        print(f"最终状态: {result}")
    except Exception as e:
        print(f"\n工作流执行失败: {e}")
        
        # 检查补偿执行情况
        print("\n检查补偿执行状态...")
        # 可以通过检查点获取中间状态
        # 在实际应用中，可以查询检查点存储来验证补偿是否执行

# 运行测试
test_compensation_workflow()
```

```python
def implement_saga_pattern():
    """实现Saga事务模式"""
    from typing import Literal
    from langgraph.types import Command
    
    class SagaState(TypedDict):
        steps_completed: list[str]
        steps_failed: list[str]
        compensation_executed: list[str]
        current_step: str
        
    def saga_coordinator(state: SagaState) -> Command[Literal["step1", "step2", "step3", "compensate", "end"]]:
        """Saga协调器"""
        # 根据当前状态决定下一步
        if state.get("current_step") == "compensate":
            # 执行补偿
            return Command(goto="execute_compensation")
        elif len(state.get("steps_failed", [])) > 0:
            # 有步骤失败，开始补偿
            return Command(goto="compensate")
        elif len(state.get("steps_completed", [])) < 3:
            # 继续执行下一步
            next_step = f"step{len(state.get('steps_completed', [])) + 1}"
            return Command(goto=next_step)
        else:
            # 所有步骤完成
            return Command(goto="end")
    
    # 构建Saga工作流
    builder = StateGraph(SagaState)
    # ... 实现细节
```

### 断路器模式集成
结合断路器模式防止级联故障：

```python
def circuit_breaker_integration():
    """断路器模式集成"""
    import time
    from collections import deque
    
    class CircuitBreaker:
        """简单的断路器实现"""
        def __init__(self, failure_threshold=5, reset_timeout=60):
            self.failure_threshold = failure_threshold
            self.reset_timeout = reset_timeout
            self.failure_count = 0
            self.last_failure_time = 0
            self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
        
        def can_execute(self) -> bool:
            """检查是否允许执行"""
            current_time = time.time()
            
            if self.state == "OPEN":
                # 检查是否需要进入半开状态
                if current_time - self.last_failure_time > self.reset_timeout:
                    self.state = "HALF_OPEN"
                    return True
                return False
            return True
        
        def record_success(self):
            """记录成功"""
            if self.state == "HALF_OPEN":
                self.state = "CLOSED"
            self.failure_count = 0
        
        def record_failure(self):
            """记录失败"""
            self.failure_count += 1
            self.last_failure_time = time.time()
            
            if self.failure_count >= self.failure_threshold:
                self.state = "OPEN"
    
    # 在节点函数中使用断路器
    def protected_service(state: dict, circuit_breaker: CircuitBreaker) -> dict:
        """受断路器保护的服务"""
        if not circuit_breaker.can_execute():
            raise CircuitBreakerOpen("服务暂时不可用")
        
        try:
            # 调用实际服务
            result = call_external_service()
            circuit_breaker.record_success()
            return result
        except Exception as e:
            circuit_breaker.record_failure()
            raise
```

## 常见问题

### Q1: 重试策略应该如何选择？
**A**: 根据错误类型选择重试策略：
- **瞬时错误**：使用指数退避+抖动，避免惊群效应
- **网络错误**：快速重试（间隔短，次数少）
- **业务错误**：不自动重试，需要人工干预
- **资源竞争**：随机延迟重试，避免冲突

### Q2: 补偿事务失败怎么办？
**A**: 补偿事务失败需要特殊处理：
1. **记录失败**：详细记录补偿失败的原因和上下文
2. **告警通知**：立即通知运维人员
3. **手动干预**：提供手动补偿工具
4. **定期重试**：实现补偿任务的定期重试机制

### Q3: 如何测试错误恢复逻辑？
**A**: 使用以下测试策略：
1. **故障注入**：模拟各种错误场景
2. **混沌测试**：随机注入故障，测试系统恢复能力
3. **集成测试**：测试完整的事务回滚流程
4. **性能测试**：测试重试机制对性能的影响

### Q4: 错误处理对性能有什么影响？
**A**: 合理设计的错误处理对性能影响有限：
1. **重试开销**：重试间隔和次数决定额外耗时
2. **补偿开销**：补偿操作应尽可能轻量
3. **状态保存**：检查点机制有存储开销
4. **监控开销**：错误追踪和日志记录有额外成本
