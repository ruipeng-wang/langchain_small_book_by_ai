# 4.2.4 实时数据流处理

## 概念讲解

在现代AI应用中，实时数据处理能力至关重要。传统的批量加载模式已无法满足实时分析、动态响应和流式决策的需求。LangChain提供了完整的实时数据流处理解决方案，支持从各种流式数据源中实时加载和处理文档数据。

### 实时数据流的独特价值

实时数据流处理与传统的批量加载有本质区别：

1. **时间敏感性**：数据产生后需要立即处理，延迟敏感
2. **无限数据流**：数据源是持续不断的，没有明确的结束点
3. **状态管理**：需要维护处理状态，支持窗口操作和聚合计算
4. **容错性**：需要处理网络抖动、服务中断等异常情况
5. **资源优化**：按需处理，避免一次性加载大量数据

### LangChain流式处理架构

LangChain采用分层架构支持实时数据流处理：

- **数据源层**：支持Kafka、Redis Streams、WebSocket、API流等实时数据源
- **连接器层**：提供标准化的流式连接器接口
- **处理层**：内置流式文本分割、实时转换和过滤
- **输出层**：支持流式输出到向量存储、消息队列或下游应用

## 核心要点

### 流式数据源支持
- **消息队列**：Kafka、RabbitMQ、Redis Streams、AWS Kinesis
- **API流**：RESTful流式API、WebSocket连接、Server-Sent Events
- **文件流**：实时日志文件、监控数据流、传感器数据
- **数据库变更**：CDC（Change Data Capture）流、数据库触发器

### 关键技术特性
1. **增量处理**：支持按数据块（chunk）逐步处理，避免内存溢出
2. **背压控制**：自动调节处理速度，防止数据积压
3. **检查点机制**：支持断点续传，确保数据不丢失
4. **并行处理**：支持多线程/多进程并发处理流数据
5. **时间窗口**：支持滑动窗口、滚动窗口等时间窗口操作

### 流式文档加载模式
- **拉取模式**：主动从数据源拉取数据
- **推送模式**：被动接收数据源推送的数据
- **混合模式**：结合拉取和推送，适应不同场景需求

### Kafka流式数据集成

```python
from langchain_community.document_loaders import KafkaLoader
from confluent_kafka import Consumer, KafkaError
import json

class RealTimeKafkaProcessor:
    def __init__(self, bootstrap_servers, topics):
        """初始化Kafka流处理器"""
        self.consumer = Consumer({
            'bootstrap.servers': bootstrap_servers,
            'group.id': 'langchain-processor',
            'auto.offset.reset': 'earliest',
            'enable.auto.commit': False
        })
        self.consumer.subscribe(topics)
        
    def process_stream(self):
        """处理Kafka数据流"""
        try:
            while True:
                msg = self.consumer.poll(1.0)
                
                if msg is None:
                    continue
                if msg.error():
                    if msg.error().code() == KafkaError._PARTITION_EOF:
                        continue
                    else:
                        print(f"Kafka错误: {msg.error()}")
                        break
                
                # 解析消息内容
                data = json.loads(msg.value().decode('utf-8'))
                
                # 创建文档对象
                document = {
                    'page_content': data.get('content', ''),
                    'metadata': {
                        'topic': msg.topic(),
                        'partition': msg.partition(),
                        'offset': msg.offset(),
                        'timestamp': msg.timestamp()[1] if msg.timestamp() else None,
                        'source': 'kafka',
                        **data.get('metadata', {})
                    }
                }
                
                yield document
                
                # 手动提交偏移量
                self.consumer.commit(msg)
                
        except KeyboardInterrupt:
            print("用户中断处理")
        finally:
            self.consumer.close()

# 使用示例
processor = RealTimeKafkaProcessor(
    bootstrap_servers='localhost:9092',
    topics=['documents', 'logs']
)

for doc in processor.process_stream():
    print(f"实时处理文档: {doc['metadata']['source']}")
    # 这里可以添加向量化、索引等实时处理逻辑
```

### WebSocket实时数据流

```python
import websockets
import asyncio
from langchain_core.documents import Document

class WebSocketStreamLoader:
    def __init__(self, ws_url, message_handler=None):
        self.ws_url = ws_url
        self.message_handler = message_handler or self.default_handler
        
    async def default_handler(self, message):
        """默认消息处理器"""
        return Document(
            page_content=message.get('text', ''),
            metadata={
                'source': 'websocket',
                'timestamp': message.get('timestamp'),
                'type': message.get('type', 'text')
            }
        )
    
    async def connect_and_stream(self):
        """连接WebSocket并流式接收数据"""
        async with websockets.connect(self.ws_url) as websocket:
            print(f"已连接到WebSocket: {self.ws_url}")
            
            while True:
                try:
                    # 接收消息
                    message = await websocket.recv()
                    
                    # 解析消息
                    data = json.loads(message)
                    
                    # 处理消息并生成文档
                    document = await self.message_handler(data)
                    
                    yield document
                    
                except websockets.exceptions.ConnectionClosed:
                    print("WebSocket连接已关闭")
                    break
                except Exception as e:
                    print(f"处理消息时出错: {e}")
                    continue

async def main():
    loader = WebSocketStreamLoader("ws://localhost:8765")
    
    async for document in loader.connect_and_stream():
        print(f"实时接收文档: {document.metadata['timestamp']}")
        # 实时处理逻辑
        process_realtime_document(document)

def process_realtime_document(document):
    """实时文档处理函数"""
    # 这里可以实现实时分析、分类、向量化等操作
    content_length = len(document.page_content)
    print(f"处理文档，长度: {content_length} 字符")
    
    if content_length > 1000:
        # 大文档需要分块处理
        chunks = split_document_in_realtime(document)
        for chunk in chunks:
            # 实时处理每个分块
            process_chunk_in_realtime(chunk)
```

### 实时文档处理管道

构建端到端的实时文档处理管道，支持复杂的流式转换：

```python
from typing import AsyncGenerator
from dataclasses import dataclass
from langchain.text_splitter import RecursiveCharacterTextSplitter

@dataclass
class ProcessingConfig:
    chunk_size: int = 1000
    chunk_overlap: int = 200
    max_concurrency: int = 5
    buffer_size: int = 100

class RealTimeDocumentPipeline:
    def __init__(self, config: ProcessingConfig):
        self.config = config
        self.text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=config.chunk_size,
            chunk_overlap=config.chunk_overlap
        )
        self.processing_queue = asyncio.Queue(maxsize=config.buffer_size)
        
    async def ingest_stream(self, document_stream: AsyncGenerator):
        """流式注入文档数据"""
        async for document in document_stream:
            try:
                # 将文档放入处理队列
                await self.processing_queue.put(document)
                print(f"已接收文档: {document.metadata.get('source', 'unknown')}")
            except asyncio.QueueFull:
                print("处理队列已满，等待处理...")
                await asyncio.sleep(0.1)
                
    async def process_stream(self):
        """并行处理流数据"""
        tasks = []
        for i in range(self.config.max_concurrency):
            task = asyncio.create_task(self._process_worker(f"worker-{i}"))
            tasks.append(task)
            
        await asyncio.gather(*tasks)
        
    async def _process_worker(self, worker_name: str):
        """处理工作线程"""
        while True:
            try:
                # 从队列获取文档
                document = await asyncio.wait_for(
                    self.processing_queue.get(),
                    timeout=5.0
                )
                
                print(f"{worker_name} 开始处理文档")
                
                # 实时文本分割
                chunks = self.text_splitter.split_documents([document])
                
                # 实时向量化（模拟）
                for chunk in chunks:
                    vector = await self.embed_chunk_in_realtime(chunk)
                    chunk.metadata['embedding'] = vector
                    
                    # 发布到下游系统
                    await self.publish_to_downstream(chunk)
                
                # 标记任务完成
                self.processing_queue.task_done()
                
            except asyncio.TimeoutError:
                print(f"{worker_name} 等待超时，继续等待...")
                continue
            except Exception as e:
                print(f"{worker_name} 处理出错: {e}")
                continue
    
    async def embed_chunk_in_realtime(self, chunk):
        """实时向量化（这里简化为模拟）"""
        # 实际应用中应调用embedding模型
        import numpy as np
        return np.random.randn(768).tolist()
    
    async def publish_to_downstream(self, chunk):
        """发布到下游系统"""
        # 这里可以发布到向量数据库、消息队列等
        print(f"发布处理后的分块: {chunk.metadata.get('source')}")
```

### 实时监控与告警系统

```python
from datetime import datetime
from collections import defaultdict
import time

class StreamProcessingMonitor:
    def __init__(self, alert_thresholds=None):
        self.metrics = defaultdict(int)
        self.start_time = time.time()
        self.alert_thresholds = alert_thresholds or {
            'error_rate': 0.05,  # 5%错误率
            'processing_lag': 10.0,  # 10秒处理延迟
            'queue_size': 1000  # 队列积压
        }
        
    def record_metric(self, metric_name, value=1):
        """记录指标"""
        self.metrics[metric_name] += value
        
    def get_processing_rate(self):
        """计算处理速率"""
        elapsed = time.time() - self.start_time
        if elapsed == 0:
            return 0
        return self.metrics.get('documents_processed', 0) / elapsed
    
    def check_alerts(self, current_metrics):
        """检查告警条件"""
        alerts = []
        
        # 检查错误率
        total_docs = current_metrics.get('documents_processed', 0)
        errors = current_metrics.get('processing_errors', 0)
        if total_docs > 0:
            error_rate = errors / total_docs
            if error_rate > self.alert_thresholds['error_rate']:
                alerts.append(f"错误率过高: {error_rate:.2%}")
        
        # 检查处理延迟
        current_lag = current_metrics.get('processing_lag', 0)
        if current_lag > self.alert_thresholds['processing_lag']:
            alerts.append(f"处理延迟过高: {current_lag:.1f}秒")
            
        # 检查队列积压
        queue_size = current_metrics.get('queue_size', 0)
        if queue_size > self.alert_thresholds['queue_size']:
            alerts.append(f"队列积压严重: {queue_size}个文档")
            
        return alerts

# 集成监控的流处理器
class MonitoredStreamProcessor:
    def __init__(self):
        self.monitor = StreamProcessingMonitor()
        
    async def process_with_monitoring(self, document_stream):
        """带监控的流处理"""
        async for document in document_stream:
            start_time = time.time()
            
            try:
                # 处理文档
                await self.process_document(document)
                
                # 记录成功指标
                self.monitor.record_metric('documents_processed')
                processing_time = time.time() - start_time
                self.monitor.record_metric('total_processing_time', processing_time)
                
            except Exception as e:
                # 记录错误指标
                self.monitor.record_metric('processing_errors')
                print(f"处理文档时出错: {e}")
                
            # 定期检查告警
            if self.monitor.metrics['documents_processed'] % 100 == 0:
                alerts = self.monitor.check_alerts(self.monitor.metrics)
                if alerts:
                    print(f"告警: {alerts}")
                    # 这里可以触发通知机制
```

## 常见问题

### Q1: 如何保证实时数据处理的Exactly-Once语义？
**A:** LangChain提供多种机制保证处理语义：
1. **检查点机制**：定期保存处理状态，支持断点续传
2. **幂等处理**：设计幂等的处理逻辑，避免重复处理
3. **事务支持**：与支持事务的下游系统集成
4. **去重标识**：为每个数据项生成唯一ID，用于去重

```python
class ExactlyOnceProcessor:
    def __init__(self, checkpoint_store):
        self.checkpoint_store = checkpoint_store
        self.processed_ids = set()
        
    async def process_with_exactly_once(self, data_stream):
        """保证Exactly-Once语义的处理"""
        # 加载检查点
        checkpoint = await self.checkpoint_store.load()
        if checkpoint:
            self.processed_ids = set(checkpoint.get('processed_ids', []))
            
        async for data in data_stream:
            data_id = data.get('id')
            
            # 检查是否已处理
            if data_id in self.processed_ids:
                continue
                
            # 处理数据
            await self.process_data(data)
            
            # 更新已处理集合
            self.processed_ids.add(data_id)
            
            # 定期保存检查点
            if len(self.processed_ids) % 100 == 0:
                await self.checkpoint_store.save({
                    'processed_ids': list(self.processed_ids)
                })
```

### Q2: 实时处理如何应对数据峰值？
**A:** 采用弹性处理策略：
1. **动态扩缩容**：根据队列长度动态调整处理线程数
2. **速率限制**：实现背压控制，避免系统过载
3. **数据降级**：峰值时降低处理质量以保证可用性
4. **缓冲队列**：使用有界队列平滑处理压力

### Q3: 流式处理如何与批量处理结合？
**A:** 采用Lambda架构或Kappa架构：
- **Lambda架构**：实时流处理提供低延迟视图，批量处理提供准确视图
- **Kappa架构**：所有数据处理都通过流式处理完成，通过重播实现批量计算

```python
class HybridProcessingSystem:
    def __init__(self):
        self.realtime_processor = RealTimeProcessor()
        self.batch_processor = BatchProcessor()
        
    async def process_hybrid(self, data_stream):
        """混合处理：实时+批量"""
        # 实时处理快速路径
        async for data in data_stream:
            # 实时处理
            realtime_result = await self.realtime_processor.process(data)
            yield realtime_result
            
            # 同时累积数据用于批量处理
            self.batch_processor.accumulate(data)
            
        # 定期执行批量处理
        if self.batch_processor.should_process_batch():
            batch_results = await self.batch_processor.process_batch()
            yield from batch_results
```

## 本节总结

实时数据流处理是LangChain文档加载系统的先进特性，它使AI应用能够处理动态变化的数据环境。通过本章学习，我们了解到：

1. **流式架构设计**：LangChain提供了标准化的流式处理接口，支持多种实时数据源。

2. **性能优化机制**：增量处理、背压控制、检查点等机制确保了流式处理的效率和可靠性。

3. **企业级特性**：Exactly-Once语义、弹性伸缩、全面监控等特性满足生产环境需求。

4. **生态集成**：与Kafka、WebSocket、CDC等现代数据系统深度集成。

5. **混合处理模式**：支持实时处理与批量处理的有机结合，适应不同业务场景。

在实际应用中，建议根据数据特性和业务需求设计合适的流式处理架构。对于延迟敏感的场景，优先考虑实时处理；对于数据一致性要求高的场景，采用检查点和幂等设计。随着LangChain生态的不断发展，更多流式处理组件和优化策略将持续涌现，进一步提升实时AI应用的能力。

实时数据流处理不仅是技术选择，更是架构哲学。它代表着从"数据仓库"到"数据流"的思维转变，推动AI应用向更实时、更智能的方向发展。
