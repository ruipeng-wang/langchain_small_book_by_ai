# 4.4 文档预处理管道

## 概念讲解

加载后的文档通常需要进行清洗、转换和格式化，才能用于后续的向量化。文档预处理是提升RAG系统质量的重要环节。

### 为什么需要预处理？

```
原始文档内容:
<html><body><h1>标题</h1><p>正文内容...</p></body></html>
                    ↓ 预处理
清洗后内容:
标题
正文内容...
```

预处理的目标：
- **去除噪声**：HTML标签、特殊字符、多余空白
- **标准化格式**：统一编码、换行符、日期格式
- **增强信息**：添加摘要、关键词、结构化元数据

### LangChain的转换器架构

LangChain使用`DocumentTransformer`进行文档转换：

```
List[Document] → DocumentTransformer.transform() → List[Document]
```

## 核心要点

| 转换器 | 功能 | 适用场景 |

### 🔑 自定义文档转换

可以创建自定义的转换函数：

```python
from langchain_core.documents import Document
import re

def clean_document(doc: Document) -> Document:
    """清理文档内容"""
    text = doc.page_content
    
    # 去除多余空白
    text = re.sub(r'\s+', ' ', text)
    
    # 去除特殊字符
    text = text.replace('\xa0', ' ')
    
    # 标准化换行符
    text = text.replace('\r\n', '\n').replace('\r', '\n')
    
    return Document(
        page_content=text.strip(),
        metadata=doc.metadata
    )

# 批量应用
cleaned_docs = [clean_document(doc) for doc in raw_docs]
```

### 🔑 文本规范化

```python
def normalize_text(text: str) -> str:
    """文本规范化处理"""
    # 全角转半角
    text = text.replace('０', '0').replace('１', '1').replace('２', '2')
    
    # 去除行内多余符号
    text = re.sub(r'[。]{2,}', '。', text)
    text = re.sub(r'[，]{2,}', '，', text)
    
    return text.strip()

# 应用到文档列表
for doc in documents:
    doc.page_content = normalize_text(doc.page_content)
```

## 进阶应用

### 构建完整的预处理管道

```python
from typing import List, Callable
from langchain_core.documents import Document

class DocumentPipeline:
    """文档预处理管道"""
    
    def __init__(self):
        self.steps: List[Callable] = []
    
    def add_step(self, fn: Callable[[Document], Document]):
        """添加处理步骤"""
        self.steps.append(fn)
    
    def process(self, documents: List[Document]) -> List[Document]:
        """执行完整管道"""
        results = []
        for doc in documents:
            processed = doc
            for step in self.steps:
                processed = step(processed)
            results.append(processed)
        return results

# 创建管道
pipeline = DocumentPipeline()

# 添加处理步骤
pipeline.add_step(
    lambda doc: Document(
        page_content=re.sub(r'<[^>]+>', '', doc.page_content),
        metadata=doc.metadata
    )
)

pipeline.add_step(
    lambda doc: Document(
        page_content=re.sub(r'\s+', ' ', doc.page_content).strip(),
        metadata=doc.metadata
    )
)

pipeline.add_step(
    lambda doc: Document(
        page_content=doc.page_content,
        metadata={**doc.metadata, "processed": True}
    )
)

# 执行处理
processed_docs = pipeline.process(raw_documents)
```

### 文档质量检查与过滤

```python
def quality_check(doc: Document) -> tuple[bool, list[str]]:
    """文档质量检查"""
    issues = []
    
    # 检查内容长度
    if len(doc.page_content) < 10:
        issues.append("内容过短")
    
    # 检查特殊字符比例
    special_chars = sum(1 for c in doc.page_content if ord(c) > 127)
    if special_chars / max(len(doc.page_content), 1) > 0.5:
        issues.append("乱码比例过高")
    
    # 检查重复内容
    words = doc.page_content.split()
    if len(set(words)) / max(len(words), 1) < 0.3:
        issues.append("重复率过高")
    
    return (len(issues) == 0, issues)

# 过滤不合格文档
valid_docs = []
for doc in documents:
    is_valid, issues = quality_check(doc)
    if is_valid:
        valid_docs.append(doc)
    else:
        print(f"文档 {doc.metadata.get('source')} 存在问题: {issues}")
```

## 常见问题

### Q1: 如何处理文档中的图片和表格？

**A:** 图片和表格内容需要特殊处理：
1. 表格：使用`Unstructured`库提取为结构化数据
2. 图片：使用OCR工具（如Tesseract）提取文字
3. 将提取的内容合并到文档的`page_content`中

### Q2: 预处理是否会影响元数据？

**A:** 转换器通常保留元数据不变，只修改`page_content`。如果需要修改元数据，应在自定义转换函数中显式处理。

### Q3: 如何处理文档中的多语言内容？

**A:** 可以使用`langdetect`库检测语言，然后按语言分别处理：

```python
from langdetect import detect

def detect_language(doc: Document) -> Document:
    try:
        lang = detect(doc.page_content)
        doc.metadata["language"] = lang
    except:
        doc.metadata["language"] = "unknown"
    return doc
```

文档预处理是提升RAG系统质量的关键步骤：

3. 使用管道模式组织多个处理步骤
4. 质量检查过滤低质量文档
