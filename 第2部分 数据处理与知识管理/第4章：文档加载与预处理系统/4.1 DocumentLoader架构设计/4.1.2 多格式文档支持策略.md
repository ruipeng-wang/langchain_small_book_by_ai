# 4.1.2 多格式文档支持策略

## 概念讲解（文字+图示）

在现代AI应用开发中，文档数据通常以多种格式存在：文本文件、PDF文档、Word文档、Excel表格、HTML网页、JSON数据、CSV表格等。这些格式各有特点，解析方式也各不相同，给开发者带来了巨大的集成复杂性。

LangChain的多格式文档支持策略采用了**分层设计理念**：

```
┌─────────────────────────────────────────────┐
│           多格式文档支持策略                 │
├─────────────────────────────────────────────┤
│   策略层：MultiFileLoader、UnstructuredAPI  │
├─────────────────────────────────────────────┤
│   解析层：PDF、DOCX、HTML、CSV、JSON解析器   │
├─────────────────────────────────────────────┤
│   抽象层：统一的Document对象                 │
└─────────────────────────────────────────────┘
```

### 设计原则

1. **格式无关性**：为开发者提供统一的编程接口，无论底层文档是什么格式，都使用相同的Document对象表示
2. **插件化架构**：每种文件格式对应一个专门的加载器，可以独立注册、卸载和更新
3. **智能探测**：系统能自动识别文件格式并选择合适的加载器
4. **批量处理**：支持批量加载多个文件，自动处理格式混合的场景
5. **元数据保留**：从原始文档中提取并保留尽可能多的元数据信息

这种设计帮助开发者屏蔽了复杂的文件解析细节，使他们能够专注于业务逻辑的实现。

## 核心要点（重点标记）

### 🔑 **统一加载接口**

- **MultiFileLoader**：核心的多格式文件加载器，支持一次性加载多个不同格式的文件
- **统一返回格式**：所有加载器都返回标准`Document`对象列表，确保接口一致性
- **文件扩展名映射**：通过文件扩展名自动选择对应的加载器

### 🔑 **支持的文档格式**

根据LangChain v1.2.22官方文档，支持的主要格式包括：

1. **文本类**：`.txt`、`.md`（Markdown）、`.rtf`
2. **办公文档**：`.pdf`、`.docx`、`.pptx`、`.xlsx`
3. **结构化数据**：`.csv`、`.json`、`.jsonl`（JSON Lines）
4. **网页格式**：`.html`、`.htm`
5. **图像文件**：`.png`、`.jpg`、`.jpeg`、`.bmp`、`.tiff`
6. **特殊格式**：`.epub`（电子书）、`.ipynb`（Jupyter Notebook）

### 🔑 **智能格式探测机制**

LangChain采用多层格式探测策略：

1. **扩展名识别**：首先基于文件扩展名选择加载器
2. **内容嗅探**：对于未知扩展名的文件，通过文件头（Magic Number）识别格式
3. **降级策略**：当专用解析器失败时，自动降级为文本解析器
4. **错误恢复**：单个文件解析失败不影响其他文件的处理

### 🔑 **外部服务集成**

对于复杂的文档格式，LangChain集成了多个专业解析服务：

- **Unstructured API**：处理复杂布局的PDF、Word文档
- **Azure AI Document Intelligence**：企业级文档智能解析
- **Google Drive API**：云端文档的直接访问

## 简单示例（代码演示）

### 示例1：使用MultiFileLoader加载多种格式文件

```python
# LangChain v1.2.22 多格式文档加载示例
import os
from langchain_community.document_loaders import (
    MultiFileLoader, TextLoader, PyPDFLoader,
    CSVLoader, DocxLoader, JSONLoader
)

# 1. 定义文件扩展名到加载器的映射
loader_mapping = {
    '.txt': TextLoader,
    '.pdf': PyPDFLoader,
    '.csv': CSVLoader,
    '.docx': DocxLoader,
    '.json': JSONLoader
}

# 2. 准备要加载的文件列表
file_paths = [
    "documents/report.txt",
    "documents/paper.pdf", 
    "documents/data.csv",
    "documents/contract.docx",
    "documents/config.json"
]

# 3. 创建MultiFileLoader实例
loader = MultiFileLoader(
    file_paths=file_paths,
    loader_mapping=loader_mapping,
    silent_errors=True  # 单个文件错误不中断整个加载过程
)

# 4. 加载所有文档
documents = loader.load()
print(f"成功加载了 {len(documents)} 个文档片段")

# 5. 查看第一个文档的内容和元数据
if documents:
    first_doc = documents[0]
    print(f"文档内容前100字符: {first_doc.page_content[:100]}...")
    print(f"文档元数据: {first_doc.metadata}")
```

### 示例2：基于文件扩展名自动选择加载器

```python
from langchain_community.document_loaders import (
    TextLoader, PyPDFLoader, CSVLoader,
    DocxLoader, JSONLoader, BSHTMLLoader
)
from pathlib import Path

class SmartDocumentLoader:
    """智能文档加载器，根据扩展名自动选择加载器"""
    
    # 扩展名到加载器的映射表
    LOADER_MAPPING = {
        '.txt': TextLoader,
        '.md': TextLoader,
        '.pdf': PyPDFLoader,
        '.csv': CSVLoader,
        '.docx': DocxLoader,
        '.json': JSONLoader,
        '.html': BSHTMLLoader,
        '.htm': BSHTMLLoader,
    }
    
    @classmethod
    def load_file(cls, file_path: str):
        """加载单个文件"""
        path = Path(file_path)
        ext = path.suffix.lower()
        
        if ext not in cls.LOADER_MAPPING:
            # 未知格式，尝试作为文本文件处理
            loader_class = TextLoader
        else:
            loader_class = cls.LOADER_MAPPING[ext]
        
        loader = loader_class(file_path)
        return loader.load()
    
    @classmethod
    def load_directory(cls, directory_path: str):
        """加载目录中的所有支持格式文件"""
        directory = Path(directory_path)
        all_documents = []
        
        for ext in cls.LOADER_MAPPING.keys():
            # 查找该扩展名的所有文件
            files = list(directory.glob(f"**/*{ext}"))
            
            for file_path in files:
                try:
                    documents = cls.load_file(str(file_path))
                    all_documents.extend(documents)
                    print(f"✓ 成功加载: {file_path.name}")
                except Exception as e:
                    print(f"✗ 加载失败: {file_path.name} - {e}")
        
        return all_documents

# 使用智能加载器
loader = SmartDocumentLoader()
docs = loader.load_directory("data/")
print(f"总共加载了 {len(docs)} 个文档片段")
```

## 进阶应用（可选内容）

### 1. 自定义格式支持

对于LangChain尚未支持的特定格式，可以轻松扩展：

```python
from langchain_core.documents import Document
from langchain_community.document_loaders.base import BaseLoader

class CustomXMLFormatLoader(BaseLoader):
    """自定义XML格式加载器"""
    
    def __init__(self, file_path: str):
        self.file_path = file_path
    
    def load(self):
        """加载XML文件并转换为Document对象"""
        import xml.etree.ElementTree as ET
        
        tree = ET.parse(self.file_path)
        root = tree.getroot()
        
        # 提取文本内容（这里简化处理，实际需要更复杂的解析）
        text_content = " ".join([elem.text for elem in root.iter() if elem.text])
        
        # 提取元数据
        metadata = {
            "source": self.file_path,
            "format": "xml",
            "root_tag": root.tag,
            "element_count": len(list(root.iter()))
        }
        
        return [Document(page_content=text_content, metadata=metadata)]

# 注册自定义加载器到MultiFileLoader
loader_mapping = {
    '.xml': CustomXMLFormatLoader,
    # ... 其他加载器
}
```

### 2. 性能优化策略

处理大量文档时，需要考虑性能优化：

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor
from langchain_community.document_loaders import TextLoader

class ParallelDocumentLoader:
    """并行文档加载器，提高大文件集合加载速度"""
    
    def __init__(self, max_workers: int = 4):
        self.max_workers = max_workers
    
    def load_files_parallel(self, file_paths, loader_mapping):
        """并行加载多个文件"""
        from pathlib import Path
        
        with ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            futures = []
            
            for file_path in file_paths:
                path = Path(file_path)
                ext = path.suffix.lower()
                
                if ext in loader_mapping:
                    loader_class = loader_mapping[ext]
                    future = executor.submit(
                        self._load_single_file,
                        loader_class,
                        file_path
                    )
                    futures.append((file_path, future))
            
            # 收集结果
            all_documents = []
            for file_path, future in futures:
                try:
                    documents = future.result(timeout=30)
                    all_documents.extend(documents)
                    print(f"✓ 完成: {Path(file_path).name}")
                except Exception as e:
                    print(f"✗ 失败: {Path(file_path).name} - {e}")
            
            return all_documents
    
    def _load_single_file(self, loader_class, file_path):
        """加载单个文件（在独立线程中执行）"""
        loader = loader_class(file_path)
        return loader.load()

# 使用并行加载器
parallel_loader = ParallelDocumentLoader(max_workers=8)
file_paths = ["data/*.txt", "data/*.pdf", "data/*.docx"]  # 实际需要展开文件列表
documents = parallel_loader.load_files_parallel(file_paths, loader_mapping)
print(f"并行加载完成，共 {len(documents)} 个文档片段")
```

### 3. 格式转换中间件

在某些场景下，可能需要将不支持的格式转换为支持的格式：

```python
class FormatConverterMiddleware:
    """格式转换中间件，将不支持的格式转换为支持的格式"""
    
    @staticmethod
    def convert_to_txt(input_path, output_path):
        """将多种格式转换为文本文件"""
        from pathlib import Path
        import subprocess
        
        path = Path(input_path)
        ext = path.suffix.lower()
        
        if ext == '.pdf':
            # 使用pdftotext工具转换PDF
            cmd = ["pdftotext", input_path, output_path]
            subprocess.run(cmd, check=True)
        elif ext in ['.docx', '.doc']:
            # 使用python-docx库转换Word文档
            from docx import Document as DocxDocument
            doc = DocxDocument(input_path)
            with open(output_path, 'w', encoding='utf-8') as f:
                for para in doc.paragraphs:
                    f.write(para.text + '\n')
        elif ext == '.pptx':
            # 转换PowerPoint演示文稿
            from pptx import Presentation
            prs = Presentation(input_path)
            text_content = []
            for slide in prs.slides:
                for shape in slide.shapes:
                    if hasattr(shape, "text"):
                        text_content.append(shape.text)
            
            with open(output_path, 'w', encoding='utf-8') as f:
                f.write('\n'.join(text_content))
        
        return output_path

# 使用转换中间件
converter = FormatConverterMiddleware()
txt_file = converter.convert_to_txt("presentation.pptx", "presentation.txt")

# 然后使用TextLoader加载转换后的文件
from langchain_community.document_loaders import TextLoader
loader = TextLoader(txt_file)
documents = loader.load()
```

## 常见问题

### Q1: LangChain支持哪些具体的文件格式？

**A:** LangChain v1.2.22支持超过20种常见文档格式，主要包括：
- 文本文件：.txt、.md、.rtf
- 办公文档：.pdf、.docx、.pptx、.xlsx
- 结构化数据：.csv、.json、.jsonl、.xml
- 网页文件：.html、.htm
- 图像文件：.png、.jpg、.jpeg、.bmp、.tiff
- 电子书：.epub
- 代码文件：.py、.js、.java、.ipynb（作为文本处理）

对于不支持的格式，可以通过自定义加载器或格式转换中间件来处理。

### Q2: 如何处理超大文件或文件集合？

**A:** 处理大文件集合时，建议采用以下策略：
1. **分批处理**：使用`lazy_load()`方法而不是`load()`，避免一次性加载所有内容到内存
2. **并行加载**：使用`ParallelDocumentLoader`或类似机制加速加载
3. **内存优化**：对于单个大文件，考虑分块加载（chunking）
4. **进度反馈**：实现进度回调机制，跟踪加载状态

### Q3: MultiFileLoader在文件加载失败时会怎样？

**A:** MultiFileLoader提供了多种错误处理策略：
- **silent_errors=True**：单个文件加载失败不会中断整个进程，会记录错误并继续
- **error_callback**：可以传入自定义错误处理函数
- **skip_failed=True**：自动跳过无法加载的文件

建议在生产环境中启用错误日志记录，并实现适当的监控。

### Q4: 如何添加对新文件格式的支持？

**A:** 添加新格式支持非常简单：
1. 继承`BaseLoader`基类实现自定义加载器
2. 实现`load()`方法，返回`Document`对象列表
3. 将自定义加载器注册到`loader_mapping`中
4. 使用`MultiFileLoader`或直接调用自定义加载器

### Q5: 格式探测失败时会发生什么？

**A:** 当LangChain无法识别文件格式时，会采用以下降级策略：
1. 首先尝试作为二进制文件处理
2. 如果失败，尝试作为纯文本文件处理（使用`TextLoader`）
3. 最后记录错误并跳过该文件（如果配置了错误容忍）

可以通过扩展`LOADER_MAPPING`或实现自定义格式探测逻辑来改进识别精度。

1. **统一接口设计**：LangChain通过MultiFileLoader提供了统一的多格式文档加载接口，屏蔽了底层格式差异的复杂性
