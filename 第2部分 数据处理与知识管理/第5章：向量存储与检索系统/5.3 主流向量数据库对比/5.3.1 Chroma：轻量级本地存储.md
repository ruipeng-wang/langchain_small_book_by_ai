# 5.3.1 Chroma：轻量级本地存储

## 概念讲解

ChromaDB 是一款专门为 AI 应用设计的开源向量数据库，以其轻量级、开发友好和 Python 原生的特性，成为 LangChain 生态系统中最受欢迎的向量存储解决方案之一。Chroma 的设计哲学是"简单至上"，它屏蔽了向量数据库的复杂性，让开发者能够快速构建和迭代 AI 应用。

### Chroma 的技术定位

Chroma 在向量数据库生态中的定位十分独特：

1. **开发者优先**：API 设计直观易懂，学习曲线平缓
2. **Python 原生**：深度集成 Python 生态，无需复杂的序列化转换
3. **轻量级架构**：单文件部署，无外部依赖，资源消耗低
4. **快速迭代**：支持内存模式和持久化模式，便于开发测试

### Chroma 的核心架构

```
Chroma 客户端 (Python/JS) 
    ↓
Chroma 服务层 (REST API)
    ↓
存储引擎层 (SQLite/ClickHouse)
    ↓
向量索引层 (HNSW/IVF)
```

**架构特点**：
- **多语言支持**：Python 和 JavaScript 官方客户端
- **多存储后端**：支持 SQLite（本地）和 ClickHouse（分布式）
- **可插拔索引**：支持 HNSW、IVF 等多种向量索引算法
- **云原生就绪**：支持 Docker 部署和 Chroma Cloud 服务

### Chroma 的发展历程

Chroma 起源于 2022 年，最初是为简化向量检索而设计的开源项目。经过几年的发展，Chroma 已成为 LangChain 生态中集成度最高的向量数据库之一：

```
2022.06: Chroma 首次发布，支持基础向量存储
2023.01: 集成 HNSW 索引，性能大幅提升
2023.06: 发布 Chroma Cloud 服务
2024.01: LangChain v1.0 深度集成
2024.12: 支持多模态向量和元数据过滤
```

## 核心要点

### ⭐ 四大部署模式

Chroma 提供四种灵活的部署模式，满足不同场景需求：

**1. 内存模式**：数据仅存在于内存中，适合快速原型验证
```python
# 无需持久化目录
vector_store = Chroma(collection_name="test", embedding_function=embeddings)
```

**2. 本地持久化模式**：数据保存到本地文件系统，适合单机应用
```python
# 指定持久化目录
vector_store = Chroma(
    collection_name="docs", 
    embedding_function=embeddings,
    persist_directory="./chroma_db"
)
```

**3. 服务端模式**：连接独立的 Chroma 服务器，适合团队协作
```python
# 连接远程 Chroma 服务器
vector_store = Chroma(
    collection_name="prod_docs",
    embedding_function=embeddings,
    host="localhost",
    port=8000
)
```

**4. 云端模式**：使用 Chroma Cloud 托管服务，免运维
```python
# 使用 Chroma Cloud
vector_store = Chroma(
    collection_name="cloud_docs",
    embedding_function=embeddings,
    chroma_cloud_api_key=os.getenv("CHROMA_API_KEY"),
    tenant="my-tenant",
    database="my-database"
)
```

### ⭐ 性能特征

Chroma 的性能表现具有明显的场景特征：

1. **小数据集优势**：10万向量以内，性能接近内存数据库
2. **中等数据集**：100万向量，需要合理配置索引参数
3. **大数据集挑战**：超过1000万向量，建议使用分布式方案

**性能基准**（基于 AWS t3.medium）：
- 查询延迟：< 50ms（10万向量，k=10）
- 写入吞吐：1000 向量/秒（批量模式）
- 内存占用：约 1GB/100万向量（768维）

### 多模态向量存储

```python
from typing import Union, List
import base64
from PIL import Image
import io

class MultimodalChromaStore:
    """多模态 Chroma 向量存储"""
    
    def __init__(self, persist_directory: str = "./multimodal_chroma"):
        self.text_embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
        self.image_embeddings = CLIPEmbeddings()  # 假设的CLIP嵌入模型
        
        # 创建文本集合
        self.text_store = Chroma(
            collection_name="text_documents",
            embedding_function=self.text_embeddings,
            persist_directory=persist_directory
        )
        
        # 创建图像集合
        self.image_store = Chroma(
            collection_name="image_documents",
            embedding_function=self.image_embeddings,
            persist_directory=persist_directory
        )
    
    def add_multimodal_document(
        self,
        text_content: str = None,
        image_path: str = None,
        metadata: dict = None
    ):
        """添加多模态文档"""
        
        documents = []
        
        # 处理文本内容
        if text_content:
            text_doc = Document(
                page_content=text_content,
                metadata={
                    **metadata,
                    "modality": "text",
                    "timestamp": datetime.now().isoformat()
                }
            )
            documents.append(text_doc)
        
        # 处理图像内容
        if image_path:
            # 读取并编码图像
            with open(image_path, "rb") as f:
                image_bytes = f.read()
                image_b64 = base64.b64encode(image_bytes).decode("utf-8")
            
            image_doc = Document(
                page_content=f"图像: {image_path}",  # 占位文本
                metadata={
                    **metadata,
                    "modality": "image",
                    "image_b64": image_b64,
                    "image_path": image_path,
                    "timestamp": datetime.now().isoformat()
                }
            )
            documents.append(image_doc)
        
        # 存储到对应集合
        for doc in documents:
            if doc.metadata["modality"] == "text":
                self.text_store.add_documents([doc])
            else:
                self.image_store.add_documents([doc])
        
        return len(documents)
    
    def multimodal_search(
        self,
        query: Union[str, str],  # 文本查询或图像路径
        k: int = 5,
        modality: str = "both"  # text/image/both
    ) -> List[Document]:
        """多模态检索"""
        
        results = []
        
        # 文本检索
        if modality in ["text", "both"] and isinstance(query, str):
            text_results = self.text_store.similarity_search(query, k=k)
            results.extend([(r, "text") for r in text_results])
        
        # 图像检索（如果是图像路径）
        if modality in ["image", "both"] and query.endswith((".jpg", ".png", ".jpeg")):
            # 使用CLIP生成图像嵌入
            image_embedding = self.image_embeddings.embed_image(query)
            
            # Chroma 支持直接使用向量搜索
            image_results = self.image_store.similarity_search_by_vector(
                embedding=image_embedding,
                k=k
            )
            results.extend([(r, "image") for r in image_results])
        
        # 按相似度排序
        results.sort(key=lambda x: x[0].metadata.get("similarity", 0), reverse=True)
        
        return results[:k]
```

### 故障恢复与数据迁移

```python
import sqlite3
import json
from pathlib import Path
from datetime import datetime

class ChromaDataManager:
    """Chroma 数据管理器"""
    
    def __init__(self, chroma_dir: str):
        self.chroma_dir = Path(chroma_dir)
        self.backup_dir = self.chroma_dir / "backups"
        self.backup_dir.mkdir(parents=True, exist_ok=True)
    
    def create_backup(self, backup_name: str = None) -> str:
        """创建数据备份"""
        if not backup_name:
            backup_name = f"backup_{datetime.now().strftime('%Y%m%d_%H%M%S')}"
        
        backup_path = self.backup_dir / backup_name
        backup_path.mkdir(parents=True, exist_ok=True)
        
        # 备份 SQLite 数据库
        db_files = list(self.chroma_dir.glob("*.sqlite3"))
        for db_file in db_files:
            backup_db = backup_path / db_file.name
            
            # 使用 SQLite 备份 API
            source = sqlite3.connect(db_file)
            destination = sqlite3.connect(backup_db)
            
            with destination:
                source.backup(destination)
            
            source.close()
            destination.close()
        
        # 备份配置文件
        config_files = list(self.chroma_dir.glob("*.json"))
        for config_file in config_files:
            backup_config = backup_path / config_file.name
            backup_config.write_text(config_file.read_text())
        
        # 创建备份元数据
        metadata = {
            "backup_time": datetime.now().isoformat(),
            "original_path": str(self.chroma_dir),
            "file_count": len(db_files) + len(config_files),
            "total_size_mb": sum(f.stat().st_size for f in 
                               list(db_files) + list(config_files)) / 1024 / 1024
        }
        
        metadata_file = backup_path / "backup_metadata.json"
        metadata_file.write_text(json.dumps(metadata, indent=2))
        
        print(f"备份创建完成: {backup_path}")
        return str(backup_path)
    
    def restore_backup(self, backup_name: str) -> bool:
        """从备份恢复数据"""
        backup_path = self.backup_dir / backup_name
        
        if not backup_path.exists():
            print(f"备份不存在: {backup_name}")
            return False
        
        # 清空当前目录
        for file in self.chroma_dir.glob("*"):
            if file.is_file():
                file.unlink()
        
        # 恢复数据库文件
        for db_file in backup_path.glob("*.sqlite3"):
            target_db = self.chroma_dir / db_file.name
            
            source = sqlite3.connect(db_file)
            destination = sqlite3.connect(target_db)
            
            with destination:
                source.backup(destination)
            
            source.close()
            destination.close()
        
        # 恢复配置文件
        for config_file in backup_path.glob("*.json"):
            if config_file.name != "backup_metadata.json":
                target_config = self.chroma_dir / config_file.name
                target_config.write_text(config_file.read_text())
        
        print(f"数据恢复完成: {backup_name}")
        return True
    
    def migrate_to_new_version(self, new_chroma_dir: str):
        """迁移数据到新版本"""
        new_dir = Path(new_chroma_dir)
        new_dir.mkdir(parents=True, exist_ok=True)
        
        # 读取所有集合数据
        collections = []
        for db_file in self.chroma_dir.glob("*.sqlite3"):
            conn = sqlite3.connect(db_file)
            cursor = conn.cursor()
            
            # 获取集合信息
            cursor.execute("SELECT name FROM collections")
            collections.extend([row[0] for row in cursor.fetchall()])
            
            conn.close()
        
        # 在新目录中重新创建集合
        print(f"开始迁移 {len(collections)} 个集合到新版本...")
        
        # 这里需要根据具体迁移逻辑实现
        # 通常涉及数据格式转换和重新索引
        
        return len(collections)
```

## 常见问题

### ❓ Chroma 适合处理多大数量的向量？

**回答**：Chroma 的适用规模取决于部署模式：
- **内存模式**：适合 1-10 万向量，主要用于开发和测试
- **本地持久化模式**：适合 10-100 万向量，中小型生产应用
- **服务端模式**：适合 100-1000 万向量，中大型应用
- **分布式模式**（需要自定义）：适合 1000 万以上向量

**性能建议**：
1. 超过 100 万向量建议使用 HNSW 索引并优化参数
2. 超过 1000 万向量建议考虑分布式方案或迁移到 PGVector/Milvus
3. 定期监控查询延迟和内存使用情况

### ❓ Chroma 与其他向量数据库相比有什么优势？

**回答**：Chroma 的核心优势体现在：

**开发体验**：
- API 设计直观，学习成本低
- 与 Python 生态深度集成
- 丰富的文档和示例代码

**部署灵活性**：
- 支持四种部署模式，适应不同场景
- 单文件部署，无复杂依赖
- 快速启动，分钟级即可运行

**生态集成**：
- LangChain 官方推荐，集成度最高
- 支持多种嵌入模型和索引算法

# 调整 HNSW 参数

**硬件优化**：
1. 使用 SSD 而不是 HDD

3. 多核 CPU 可以提升批量处理性能

### ❓ Chroma 的数据如何备份和迁移？

**回答**：Chroma 的数据管理需要特别注意：

**备份策略**：
1. **文件系统备份**：直接复制整个 `persist_directory`
2. **SQLite 备份**：使用 `.backup()` API 保证一致性
3. **增量备份**：只备份变更的集合

**迁移方案**：
1. **同版本迁移**：直接复制数据库文件
2. **跨版本迁移**：需要导出数据并重新导入
3. **数据库间迁移**：使用 LangChain 的 `from_documents` 重新向量化

**最佳实践**：
- 定期自动备份（每天/每周）
- 测试恢复流程确保可用性
- 保留多个历史备份版本
- 监控存储空间使用情况
