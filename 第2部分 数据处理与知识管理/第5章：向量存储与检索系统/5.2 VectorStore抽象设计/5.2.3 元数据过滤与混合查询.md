# 5.2.3 元数据过滤与混合查询

## 概念讲解

在现代AI应用中，单纯的向量相似性搜索往往无法满足复杂的业务需求。LangChain v1.2.22 提供了强大的元数据过滤和混合查询能力，将语义理解与结构化数据查询完美结合，实现了真正智能化的信息检索。

### 元数据过滤的核心价值

元数据（Metadata）是描述数据的数据，在文档检索场景中，常见的元数据包括：
- **文档属性**：作者、创建时间、修改时间、文件类型、文件大小
- **业务属性**：产品类别、价格区间、地理位置、用户评分
- **内容属性**：语言、主题标签、情感倾向、关键词频率
- **系统属性**：数据来源、版本号、访问权限、过期时间

元数据过滤的价值体现在：
1. **精度提升**：在语义相似的基础上增加结构化约束
2. **性能优化**：通过预过滤减少向量计算量
3. **业务适配**：将业务规则直接嵌入检索过程
4. **多维度分析**：支持基于时间、分类等多维度查询

### 混合查询的技术演进

混合查询技术经历了从简单到复杂的发展：

```
单一搜索 → 并行搜索 → 级联搜索 → 融合搜索 → 自适应搜索
```

1. **单一搜索**：仅使用向量相似性或关键词匹配
2. **并行搜索**：同时执行多种搜索，结果简单合并
3. **级联搜索**：先用快速方法初筛，再用精确方法精炼
4. **融合搜索**：使用加权融合算法（如RRF）智能合并
5. **自适应搜索**：根据查询特点动态选择最优策略

### 融合算法对比

| 融合算法 | 原理 | 优点 | 缺点 |
|---------|------|------|------|
| **RRF** | 基于排名的倒数融合 | 简单高效，无需分数标准化 | 可能忽略绝对相关性 |
| **加权平均** | 加权平均归一化分数 | 考虑绝对相关性 | 需要分数标准化 |
| **CombSUM** | 直接求和各系统分数 | 实现简单 | 对高分系统有偏 |
| **CombMNZ** | 求和 × 系统数量 | 鼓励多系统一致 | 可能过度奖励一致 |

## 核心要点

### ⭐ 元数据过滤操作符体系

LangChain v1.2.22 提供了完整的过滤操作符体系：

```python
# 基础比较操作符
$eq:  等于           # filter={"price": {"$eq": 100}}
$ne:  不等于         # filter={"status": {"$ne": "deleted"}}
$lt:  小于           # filter={"age": {"$lt": 18}}
$lte: 小于等于       # filter={"score": {"$lte": 90}}
$gt:  大于           # filter={"price": {"$gt": 50}}
$gte: 大于等于       # filter={"quantity": {"$gte": 10}}

# 集合操作符
$in:   在列表中       # filter={"category": {"$in": ["tech", "sports"]}}
$nin:  不在列表中     # filter={"tag": {"$nin": ["deprecated", "test"]}}

# 字符串操作符
$like:    文本包含     # filter={"title": {"$like": "Python%"}}
$ilike:   忽略大小写包含 # filter={"content": {"$ilike": "%database%"}}

# 范围操作符
$between: 介于之间     # filter={"date": {"$between": ["2023-01-01", "2023-12-31"]}}

# 逻辑操作符
$and: 逻辑与         # filter={"$and": [{"price": {"$lt": 100}}, {"rating": {"$gte": 4}}]}
$or:  逻辑或         # filter={"$or": [{"type": "article"}, {"type": "tutorial"}]}
$not: 逻辑非         # filter={"$not": {"status": "archived"}}
```

### ⭐ 混合查询的三种模式

**1. 并行混合查询**
```python
# 同时执行向量搜索和关键词搜索，然后融合结果
vector_results = vector_search(query, k=20)
keyword_results = keyword_search(query, k=20)
final_results = fuse_results(vector_results, keyword_results)
```

**2. 级联混合查询**
```python
# 先用关键词快速筛选，再用向量精确排序
candidate_docs = keyword_search(query, k=100)  # 快速但不够精确
refined_results = vector_search(query, k=10, filter_docs=candidate_docs)
```

**3. 自适应混合查询**
```python
# 根据查询特点选择最优策略
query_type = analyze_query(query)
if query_type == "factual":
    results = keyword_search(query, k=10)  # 事实性查询用关键词
elif query_type == "semantic":
    results = vector_search(query, k=10)   # 语义查询用向量
else:
    results = hybrid_search(query, k=10)   # 混合查询
```

### ⭐ 互惠排名融合（RRF）算法

RRF（Reciprocal Rank Fusion）是混合查询的核心算法：

**公式**：
```
RRF_score(d) = Σ(1 / (k + rank_i(d)))
```
其中：
- `d`：文档
- `rank_i(d)`：文档在第i个检索系统中的排名
- `k`：平滑常数（通常为60）

**特点**：
1. **无需分数标准化**：直接使用排名，避免不同系统分数尺度差异
2. **偏向高排名**：排名越靠前，贡献越大（倒数关系）
3. **鼓励多样性**：多个系统都认为相关的文档得分更高

### 自定义混合查询策略

```python
from typing import List, Dict, Any
from langchain.schema import Document
import numpy as np

class CustomHybridSearch:
    """自定义混合查询策略"""
    
    def __init__(self, vector_store, keyword_store):
        self.vector_store = vector_store
        self.keyword_store = keyword_store
        self.rrf_k = 60  # RRF平滑常数
    
    def hybrid_search(
        self, 
        query: str, 
        k: int = 10,
        vector_weight: float = 0.7,
        keyword_weight: float = 0.3,
        filter_dict: Dict[str, Any] = None
    ) -> List[Document]:
        """执行混合查询"""
        
        # 并行执行两种搜索
        vector_results = self.vector_store.similarity_search(
            query, k=k*2, filter=filter_dict
        )
        keyword_results = self.keyword_store.search(
            query, k=k*2, filter=filter_dict
        )
        
        # 应用RRF融合
        fused_results = self._rrf_fusion(
            vector_results, keyword_results, 
            vector_weight, keyword_weight, k
        )
        
        return fused_results
    
    def _rrf_fusion(
        self,
        vector_results: List[Document],
        keyword_results: List[Document],
        vector_weight: float,
        keyword_weight: float,
        k: int
    ) -> List[Document]:
        """RRF结果融合"""
        
        # 创建文档到排名的映射
        vector_ranks = {
            self._get_doc_id(doc): i 
            for i, doc in enumerate(vector_results)
        }
        keyword_ranks = {
            self._get_doc_id(doc): i 
            for i, doc in enumerate(keyword_results)
        }
        
        # 收集所有唯一文档
        all_doc_ids = set(vector_ranks.keys()) | set(keyword_ranks.keys())
        doc_map = {
            self._get_doc_id(doc): doc 
            for doc in vector_results + keyword_results
        }
        
        # 计算RRF分数
        doc_scores = {}
        for doc_id in all_doc_ids:
            vector_rank = vector_ranks.get(doc_id, len(vector_results))
            keyword_rank = keyword_ranks.get(doc_id, len(keyword_results))
            
            # 加权RRF
            vector_score = vector_weight / (self.rrf_k + vector_rank + 1)
            keyword_score = keyword_weight / (self.rrf_k + keyword_rank + 1)
            
            doc_scores[doc_id] = vector_score + keyword_score
        
        # 按分数排序
        sorted_doc_ids = sorted(
            doc_scores.keys(), 
            key=lambda x: doc_scores[x], 
            reverse=True
        )[:k]
        
        # 返回排序后的文档
        results = []
        for doc_id in sorted_doc_ids:
            doc = doc_map[doc_id]
            # 添加融合分数到元数据
            doc.metadata["fusion_score"] = doc_scores[doc_id]
            doc.metadata["vector_rank"] = vector_ranks.get(doc_id, -1)
            doc.metadata["keyword_rank"] = keyword_ranks.get(doc_id, -1)
            results.append(doc)
        
        return results
    
    def _get_doc_id(self, doc: Document) -> str:
        """生成文档唯一标识"""
        # 使用内容和元数据的组合作为ID
        return f"{hash(doc.page_content)}:{hash(str(sorted(doc.metadata.items())))}"
```

### 动态过滤条件生成器

```python
from typing import Optional, Dict, Any
from datetime import datetime, timedelta
from enum import Enum

class TimeRange(Enum):
    TODAY = "today"
    LAST_7_DAYS = "last_7_days"
    LAST_30_DAYS = "last_30_days"
    THIS_YEAR = "this_year"
    CUSTOM = "custom"

class DynamicFilterGenerator:
    """动态过滤条件生成器"""
    
    def __init__(self):
        self.user_preferences = {}
        self.business_rules = {}
    
    def generate_filters(
        self,
        query: str,
        user_context: Optional[Dict[str, Any]] = None,
        time_range: Optional[TimeRange] = None,
        custom_filters: Optional[Dict[str, Any]] = None
    ) -> Dict[str, Any]:
        """生成动态过滤条件"""
        
        filters = {}
        
        # 1. 基于时间范围的过滤
        if time_range:
            filters.update(self._get_time_filters(time_range))
        
        # 2. 基于用户上下文的过滤
        if user_context:
            filters.update(self._get_user_filters(user_context))
        
        # 3. 基于查询内容的过滤
        query_filters = self._analyze_query_for_filters(query)
        if query_filters:
            filters.update(query_filters)
        
        # 4. 自定义过滤条件
        if custom_filters:
            filters.update(custom_filters)
        
        # 5. 应用业务规则
        filters = self._apply_business_rules(filters)
        
        return filters
    
    def _get_time_filters(self, time_range: TimeRange) -> Dict[str, Any]:
        """生成时间过滤条件"""
        
        now = datetime.now()
        
        if time_range == TimeRange.TODAY:
            start_date = now.replace(hour=0, minute=0, second=0, microsecond=0)
            return {"created_at": {"$gte": start_date.isoformat()}}
        
        elif time_range == TimeRange.LAST_7_DAYS:
            start_date = now - timedelta(days=7)
            return {"created_at": {"$gte": start_date.isoformat()}}
        
        elif time_range == TimeRange.LAST_30_DAYS:
            start_date = now - timedelta(days=30)
            return {"created_at": {"$gte": start_date.isoformat()}}
        
        elif time_range == TimeRange.THIS_YEAR:
            start_date = now.replace(month=1, day=1, hour=0, minute=0, second=0, microsecond=0)
            return {"created_at": {"$gte": start_date.isoformat()}}
        
        else:
            return {}
    
    def _get_user_filters(self, user_context: Dict[str, Any]) -> Dict[str, Any]:
        """基于用户上下文生成过滤条件"""
        
        filters = {}
        
        # 用户权限过滤
        if "permissions" in user_context:
            permissions = user_context["permissions"]
            if "view_confidential" not in permissions:
                filters["confidential"] = {"$ne": True}
        
        # 用户偏好过滤
        if "preferences" in user_context:
            preferences = user_context["preferences"]
            if "preferred_languages" in preferences:
                filters["language"] = {"$in": preferences["preferred_languages"]}
        
        # 地理位置过滤
        if "location" in user_context:
            location = user_context["location"]
            # 可以添加基于位置的过滤逻辑
        
        return filters
    
    def _analyze_query_for_filters(self, query: str) -> Dict[str, Any]:
        """从查询内容中提取过滤条件"""
        
        # 简单的关键词提取（实际应用中可以使用NLP模型）
        filters = {}
        
        # 检查价格相关关键词
        price_keywords = ["便宜", "实惠", "昂贵", "价格", "￥", "$"]
        for keyword in price_keywords:
            if keyword in query:
                if keyword in ["便宜", "实惠"]:
                    filters["price_tier"] = "low"
                elif keyword in ["昂贵"]:
                    filters["price_tier"] = "high"
                break
        
        # 检查时间相关关键词
        time_keywords = ["最新", "最近", "今天", "今年", "2023", "2024"]
        for keyword in time_keywords:
            if keyword in query:
                if keyword in ["最新", "最近", "今天"]:
                    filters["recency"] = "recent"
                break
        
        return filters
    
    def _apply_business_rules(self, filters: Dict[str, Any]) -> Dict[str, Any]:
        """应用业务规则"""
        
        # 示例：确保某些文档始终被排除
        if "exclude_categories" in self.business_rules:
            exclude_cats = self.business_rules["exclude_categories"]
            if "category" in filters:
                # 确保不排除已经指定的类别
                current_cats = filters["category"].get("$in", [])
                if current_cats:
                    filtered_cats = [cat for cat in current_cats if cat not in exclude_cats]
                    if filtered_cats:
                        filters["category"]["$in"] = filtered_cats
                    else:
                        # 如果所有类别都被排除，移除该条件
                        del filters["category"]
            else:
                # 添加排除条件
                filters["category"] = {"$nin": exclude_cats}
        
        return filters
```

### 查询性能优化策略

```python
import time
from functools import lru_cache
from typing import Tuple, List

class QueryOptimizer:
    """查询性能优化器"""
    
    def __init__(self, vector_store):
        self.vector_store = vector_store
        self.query_cache = {}
        self.stats = {
            "total_queries": 0,
            "cache_hits": 0,
            "avg_response_time": 0,
            "filter_efficiency": 0
        }
    
    @lru_cache(maxsize=1000)
    def cached_similarity_search(
        self, 
        query: str, 
        k: int = 10,
        filter_hash: str = ""
    ) -> List[Document]:
        """带缓存的相似性搜索"""
        
        start_time = time.time()
        
        # 解析过滤条件
        filter_dict = self._parse_filter_hash(filter_hash) if filter_hash else None
        
        # 执行查询
        results = self.vector_store.similarity_search(
            query, k=k, filter=filter_dict
        )
        
        # 更新统计信息
        query_time = time.time() - start_time
        self._update_stats(query_time, filter_dict)
        
        return results
    
    def optimize_filter_order(
        self, 
        filters: Dict[str, Any]
    ) -> Tuple[Dict[str, Any], float]:
        """优化过滤条件顺序"""
        
        if not filters:
            return filters, 1.0
        
        # 估计每个过滤条件的 selectivity（选择性）
        # selectivity = 满足条件的文档比例，越低表示过滤效果越好
        estimated_selectivity = {}
        
        for field, condition in filters.items():
            # 这里使用启发式规则估计选择性
            if isinstance(condition, dict):
                operator = next(iter(condition.keys()))
                value = condition[operator]
                
                if operator == "$eq":
                    # 等值查询通常具有较低的选择性
                    selectivity = 0.1  # 估计10%的文档满足条件
                elif operator == "$in":
                    # 集合查询的选择性取决于集合大小
                    selectivity = min(0.3, len(value) * 0.05)
                elif operator in ["$lt", "$lte", "$gt", "$gte"]:
                    # 范围查询通常具有中等选择性
                    selectivity = 0.4
                elif operator == "$like":
                    # 模糊匹配通常具有较高选择性
                    selectivity = 0.6
                else:
                    selectivity = 0.5
            else:
                # 简单等值查询
                selectivity = 0.1
            
            estimated_selectivity[field] = selectivity
        
        # 按选择性排序（选择性低的先执行）
        sorted_fields = sorted(
            estimated_selectivity.keys(),
            key=lambda x: estimated_selectivity[x]
        )
        
        # 构建优化后的过滤条件
        optimized_filters = {}
        for field in sorted_fields:
            optimized_filters[field] = filters[field]
        
        # 计算总体选择性（多个条件独立时相乘）
        overall_selectivity = 1.0
        for selectivity in estimated_selectivity.values():
            overall_selectivity *= selectivity
        
        return optimized_filters, overall_selectivity
    
    def _parse_filter_hash(self, filter_hash: str) -> Dict[str, Any]:
        """解析过滤条件哈希（简化实现）"""
        # 实际应用中可能需要更复杂的解析逻辑
        try:
            import json
            return json.loads(filter_hash)
        except:
            return {}
    
    def _update_stats(self, query_time: float, filter_dict: Dict[str, Any]):
        """更新性能统计"""
        
        self.stats["total_queries"] += 1
        self.stats["avg_response_time"] = (
            (self.stats["avg_response_time"] * (self.stats["total_queries"] - 1) + query_time)
            / self.stats["total_queries"]
        )
        
        # 估计过滤效率
        if filter_dict:
            # 简单的启发式：过滤条件越多，效率越高
            filter_efficiency = min(0.9, len(filter_dict) * 0.2)
            self.stats["filter_efficiency"] = (
                (self.stats["filter_efficiency"] * (self.stats["total_queries"] - 1) + filter_efficiency)
                / self.stats["total_queries"]
            )
```

## 常见问题

### ❓ 元数据过滤会影响查询性能吗？

**回答**：正确使用元数据过滤可以**提升**查询性能：
1. **预过滤优势**：在向量相似性计算前排除不相关文档，减少计算量
2. **索引利用**：大多数向量数据库对元数据字段建立倒排索引，过滤速度极快
3. **查询优化**：选择性高的过滤条件先执行，大幅减少搜索空间

**性能建议**：
- 为高频过滤字段建立索引
- 将选择性高的条件放在前面
- 避免过于复杂的逻辑组合（如深层嵌套的$and/$or）

### ❓ 混合查询中如何设置权重？

**回答**：权重设置需要根据场景调整：
1. **语义主导场景**（如概念搜索）：vector_weight=0.8, text_weight=0.2
2. **关键词主导场景**（如精确匹配）：vector_weight=0.3, text_weight=0.7
3. **平衡场景**（通用搜索）：vector_weight=0.5, text_weight=0.5

**调优方法**：
- A/B测试不同权重配置
- 使用用户反馈数据优化
- 根据查询类型动态调整

### ❓ 如何处理缺失或不完整的元数据？

**回答**：缺失元数据是常见问题：
1. **默认值策略**：为缺失字段设置合理的默认值
2. **条件回退**：当过滤条件涉及缺失字段时，使用备选策略
3. **数据质量监控**：定期检查和修复元数据问题
4. **渐进式增强**：优先处理完整元数据，逐步扩展

### ❓ RRF算法中k值（平滑常数）如何选择？

**回答**：k值影响排名融合的敏感度：
1. **小k值**（如10-30）：对高排名敏感，偏向排名靠前的文档
2. **中等k值**（如60-100）：平衡排名和多样性，通用推荐
3. **大k值**（如100+）：降低排名影响，增加结果多样性

**经验值**：
- 文献推荐：k=60
- 实际应用：根据数据规模和业务需求在40-80间调整
- 调优方法：使用NDCG等指标评估不同k值的效果
