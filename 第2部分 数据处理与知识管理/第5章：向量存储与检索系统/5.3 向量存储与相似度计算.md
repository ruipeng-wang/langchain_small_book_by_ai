# 5.3 向量存储与相似度计算

## 概念讲解

向量存储（VectorStore）是LangChain中管理向量数据的核心抽象。它负责存储文档的向量表示，并高效的相似度搜索。

### VectorStore的统一抽象

LangChain的`VectorStore`类提供了统一的接口，底层可以对接FAISS、Chroma、Pinecone等多种向量数据库：

```
应用层 → VectorStore接口 → 具体实现(FAISS/Chroma/Pinecone/...)
```

**核心方法**：
- `add_texts()` / `add_documents()`：添加向量
- `similarity_search()`：相似度搜索
- `delete()`：删除向量
- `save_local()` / `load_local()`：持久化

## 核心要点

### 🔑 FAISS - 内存型向量存储

FAISS（Facebook AI Similarity Search）是Facebook开源的高效相似度搜索库，适合中小规模数据：

```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import FAISS

# 初始化
embeddings = OpenAIEmbeddings()

# 从文本创建
texts = ["文档1内容", "文档2内容", "文档3内容"]
vector_store = FAISS.from_texts(texts, embeddings)

# 从Document创建
from langchain_core.documents import Document
docs = [
    Document(page_content="文档1", metadata={"source": "file1"}),
    Document(page_content="文档2", metadata={"source": "file2"}),
]
vector_store = FAISS.from_documents(docs, embeddings)

# 检索
results = vector_store.similarity_search("查询文本", k=2)
```

### 🔑 带分数的相似度搜索

返回相似度分数，便于结果排序和过滤：

```python
# 返回带分数的结果
results_with_scores = vector_store.similarity_search_with_score(
    "查询文本", 
    k=3
)

for doc, score in results_with_scores:
    print(f"分数: {score:.4f}")  # 分数越小越相似（FAISS使用L2距离）
    print(f"内容: {doc.page_content}")
```

### 🔑 Chroma - 持久化向量存储

Chroma是开源的向量数据库，支持数据持久化：

```python
from langchain_chroma import Chroma

# 创建或加载
vector_store = Chroma(
    persist_directory="./chroma_db",  # 持久化路径
    embedding_function=embeddings,
    collection_name="my_documents"
)

# 添加文档
vector_store.add_documents(docs)

# 检索
results = vector_store.similarity_search("查询文本", k=3)

# 持久化到磁盘
vector_store.persist()
```

### 🔑 元数据过滤检索

通过元数据条件过滤，提高检索精确度：

```python
from langchain_core.documents import Document

# 添加带元数据的文档
docs = [
    Document(page_content="Python教程", metadata={"type": "tutorial", "lang": "python"}),
    Document(page_content="Java教程", metadata={"type": "tutorial", "lang": "java"}),
    Document(page_content="Python进阶", metadata={"type": "advanced", "lang": "python"}),
]

vector_store.add_documents(docs)

# 按元数据过滤检索
results = vector_store.similarity_search(
    "教程",
    k=2,
    filter={"lang": "python"}  # 只搜索Python相关
)
```

## 进阶应用

### 增量添加文档

向量存储支持逐步添加，不需要重建：

```python
# 初始添加
initial_docs = [Document(page_content="初始文档1")]
vector_store = FAISS.from_documents(initial_docs, embeddings)

# 后续增量添加
new_docs = [Document(page_content="新增文档1")]
vector_store.add_documents(new_docs)

# 验证
print(f"当前文档数: {vector_store.index.ntotal}")
```

### 向量存储持久化

FAISS支持保存和加载：

```python
# 保存
vector_store.save_local("faiss_index")

# 加载
loaded_store = FAISS.load_local(
    "faiss_index",
    embeddings,
    allow_dangerous_deserialization=True
)

# 验证
results = loaded_store.similarity_search("查询文本", k=2)
```

### 相似度搜索方法对比

VectorStore提供多种搜索方式：

| 方法 | 返回值 | 适用场景 |
|------|--------|---------|
| `similarity_search()` | List[Document] | 只需要文档内容 |
| `similarity_search_with_score()` | List[(Document, float)] | 需要分数排序 |
| `similarity_search_by_vector()` | List[Document] | 已有查询向量 |
| `max_marginal_relevance_search()` | List[Document] | 需要结果多样性 |

### 多样性搜索（MMR）

MMR算法在相关性和多样性之间取得平衡：

```python
# 最大边际相关性搜索
results = vector_store.max_marginal_relevance_search(
    "查询文本",
    k=5,           # 返回数量
    fetch_k=20,    # 候选池大小
    lambda_mult=0.5  # 多样性权重(0-1)
)
```

## 常见问题

### Q1: FAISS和Chroma如何选择？

**A:**
- **FAISS**：适合中小规模数据（百万级），内存计算，速度快
- **Chroma**：支持持久化，适合需要数据持久化的场景
- **大规模**：建议使用Pinecone、Milvus等分布式向量数据库

### Q2: FAISS的分数含义是什么？

**A:** FAISS默认使用L2距离（欧氏距离），分数越小表示越相似。如果使用余弦相似度，分数越大表示越相似。

### Q3: 如何实现向量更新？

**A:** FAISS不支持直接更新，需要删除后重新添加。Chroma支持通过ID更新：

```python
# Chroma支持upsert（更新或插入）
vector_store.upsert(
    ids=["doc1"],
    documents=[Document(page_content="更新后的内容")]
)
```

## 本节总结

向量存储是管理向量和执行检索的核心组件：
1. VectorStore提供统一的检索接口
2. FAISS适合内存计算，Chroma支持持久化
3. 元数据过滤可以提高检索精确度
4. MMR算法平衡相关性和多样性
5. 合理选择存储方案取决于数据规模和性能需求
