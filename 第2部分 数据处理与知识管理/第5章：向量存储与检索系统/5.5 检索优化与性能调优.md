# 5.5 检索优化与性能调优

## 概念讲解

检索质量直接影响RAG系统的整体表现。即使使用了最好的Embedding模型，不合理的检索策略也会导致结果不佳。

### 检索优化的核心维度

```
检索优化
├── 查询优化    - 查询改写、扩展
├── 索引优化    - 分块策略、Embedding选择
├── 检索优化    - 混合检索、重排序
└── 性能优化    - 缓存、批处理、索引结构
```

## 核心要点

### 🔑 查询改写

用户的原始查询往往不够精确，可以通过改写提高检索效果：

```python
# 多查询策略：生成多个相关查询
from langchain.retrievers.multi_query import MultiQueryRetriever
from langchain_openai import ChatOpenAI

# 使用LLM生成多个查询变体
retriever = MultiQueryRetriever.from_llm(
    retriever=vector_store.as_retriever(),
    llm=ChatOpenAI(temperature=0),
    prompt=custom_prompt  # 可选的自定义提示
)

# 自动扩展查询并合并结果
results = retriever.invoke("用户原始查询")
```

### 🔑 混合检索

结合关键词检索和向量检索的优势：

```python
from langchain.retrievers import BM25Retriever, EnsembleRetriever
from langchain_community.vectorstores import FAISS

# BM25关键词检索
bm25 = BM25Retriever.from_documents(documents)
bm25.k = 2

# 向量检索
faiss = FAISS.from_documents(documents, embeddings)
faiss_retriever = faiss.as_retriever(search_kwargs={"k": 2})

# 混合检索
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25, faiss_retriever],
    weights=[0.5, 0.5]  # 权重分配
)

results = ensemble_retriever.invoke("查询文本")
```

### 🔑 上下文增强检索

检索相关文档后，添加上下文信息：

```python
from langchain.retrievers import ParentDocumentRetriever
from langchain.storage import InMemoryStore
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 父文档存储完整内容
store = InMemoryStore()

# 子文档用于检索
child_splitter = RecursiveCharacterTextSplitter(chunk_size=200)
# 父文档保留完整上下文
parent_splitter = RecursiveCharacterTextSplitter(chunk_size=1000)

retriever = ParentDocumentRetriever(
    vectorstore=faiss,
    docstore=store,
    child_splitter=child_splitter,
    parent_splitter=parent_splitter,
)

# 添加文档
retriever.add_documents(documents)

# 检索返回父文档（完整上下文）
results = retriever.invoke("查询")
```

## 进阶应用

### 重排序（Re-ranking）

检索结果使用更精确的模型重新排序：

```python
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import EmbeddingsFilter

# 使用Embeddings过滤低相关性文档
compressor = EmbeddingsFilter(
    embeddings=embeddings,
    similarity_threshold=0.7  # 相似度阈值
)

compression_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=vector_store.as_retriever()
)

results = compression_retriever.invoke("查询")
```

### 缓存优化

对于重复查询，使用缓存减少重复计算：

```python
from langchain.globals import set_llm_cache
from langchain_community.cache import InMemoryCache

# 启用LLM缓存
set_llm_cache(InMemoryCache())

# 相同查询将使用缓存结果
result1 = llm.invoke("相同查询")  # 计算
result2 = llm.invoke("相同查询")  # 使用缓存
```

### 批量处理优化

对于大量文档的向量化，使用批处理提高效率：

```python
# 批量添加文档
batch_size = 100
for i in range(0, len(documents), batch_size):
    batch = documents[i:i+batch_size]
    vector_store.add_documents(batch)
    print(f"已处理 {min(i+batch_size, len(documents))}/{len(documents)} 个文档")
```

## 性能调优指南

### 检索性能瓶颈分析

| 瓶颈 | 表现 | 解决方案 |
|------|------|---------|
| **Embedding计算** | 添加文档慢 | 批量处理、缓存 |
| **向量检索** | 查询响应慢 | 索引优化、降维 |
| **网络延迟** | API调用慢 | 本地模型、异步 |
| **内存占用** | OOM错误 | 分块处理、流式 |

### 索引优化建议

1. **HNSW索引**：适合大规模数据，查询速度快
2. **IVF索引**：平衡查询速度和精度
3. **降维**：使用PCA等算法减少向量维度

## 常见问题

### Q1: 如何评估检索效果？

**A:** 可以使用以下指标：
- **命中率（Hit Rate）**：相关文档是否被检索到
- **MRR（Mean Reciprocal Rank）**：第一个相关文档的排名
- **NDCG**：考虑排名的相关性评分

### Q2: 检索结果不相关怎么办？

**A:** 几种优化策略：
1. 检查Embedding模型是否适合你的领域
2. 优化分块策略（大小、重叠）
3. 使用混合检索
4. 添加查询改写
5. 使用重排序

### Q3: 如何处理大规模数据的检索？

**A:** 
1. 使用分布式向量数据库（Milvus、Pinecone）
2. 按类别/时间分区索引
3. 使用元数据预过滤减少搜索空间
4. 考虑使用近似最近邻（ANN）索引

## 本节总结

检索优化是持续提升RAG系统效果的关键：
1. 查询改写可以提高召回率
2. 混合检索结合不同检索策略的优势
3. 上下文增强检索提供更完整的信息
4. 重排序提升最终结果的相关性
5. 缓存和批处理优化系统性能
