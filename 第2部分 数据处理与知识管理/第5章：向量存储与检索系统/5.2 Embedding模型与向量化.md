# 5.2 Embedding模型与向量化

## 概念讲解

Embedding模型是RAG系统的**语义引擎**，决定了文本如何被转换为向量表示。选择合适的Embedding模型直接影响检索效果。

### Embedding模型的工作原理

Embedding模型通过深度学习将文本映射到高维向量空间。训练过程中，模型学习到语义相似的句子在向量空间中距离也相似。

```
输入文本 → Tokenizer → Embedding Model → 输出向量
  "猫"     → [23,45]  →  神经网络处理   → [0.1, -0.3, 0.5, ...]
  "狗"     → [67,89]  →  神经网络处理   → [0.15, -0.28, 0.52, ...]
```

**关键指标**：
- **维度**：向量的长度，通常768-3072维
- **上下文窗口**：支持的最大token数，通常512-8192
- **语言支持**：单语/多语

## 核心要点

### 🔑 Embedding模型选择指南

| 模型 | 提供商 | 维度 | 最大Token | 特点 |
|------|--------|------|----------|------|
| `text-embedding-3-small` | OpenAI | 1536 | 8191 | 性价比高 |
| `text-embedding-3-large` | OpenAI | 3072 | 8191 | 高质量 |
| `text-embedding-004` | Google | 768 | 2048 | 多语言 |
| `bge-large-zh` | 北京智源 | 1024 | 512 | 中文优化 |
| `bge-m3` | 北京智源 | 1024 | 8192 | 多语言多粒度 |
| `nomic-embed-text` | Ollama | 768 | 8192 | 本地部署 |

### 🔑 LangChain的Embedding抽象

LangChain通过统一的`Embeddings`接口支持多种Embedding模型：

```python
from langchain_core.embeddings import Embeddings

class Embeddings:
    def embed_query(self, text: str) -> list[float]:
        """将单个文本转换为向量"""
        ...
    
    def embed_documents(self, texts: list[str]) -> list[list[float]]:
        """将多个文本批量转换为向量"""
        ...
```

这种设计让开发者可以轻松切换不同的Embedding提供商，而无需修改业务代码。

### 🔑 OpenAI Embedding

```python
from langchain_openai import OpenAIEmbeddings

# 基础使用
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 指定维度（可降维减少存储）
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512  # 原始1536维，降至512维
)

# 单个文本
vector = embeddings.embed_query("什么是机器学习？")
print(f"向量维度: {len(vector)}")

# 批量文本
texts = ["机器学习", "深度学习", "自然语言处理"]
vectors = embeddings.embed_documents(texts)
print(f"生成 {len(vectors)} 个向量")
```

### Embedding缓存优化

Embedding计算是耗时的操作，对于重复文本可以使用缓存：

```python
import hashlib
from functools import lru_cache

class CachedEmbeddings:
    """带缓存的Embedding封装"""
    
    def __init__(self, base_embeddings: Embeddings, cache_size: int = 10000):
        self.base_embeddings = base_embeddings
        self._cache = {}
    
    def embed_query(self, text: str) -> list[float]:
        text_hash = hashlib.md5(text.encode()).hexdigest()
        if text_hash not in self._cache:
            self._cache[text_hash] = self.base_embeddings.embed_query(text)
        return self._cache[text_hash]
    
    def embed_documents(self, texts: list[str]) -> list[list[float]]:
        results = []
        for text in texts:
            text_hash = hashlib.md5(text.encode()).hexdigest()
            if text_hash in self._cache:
                results.append(self._cache[text_hash])
            else:
                vector = self.base_embeddings.embed_query(text)
                self._cache[text_hash] = vector
                results.append(vector)
        return results
```

## 常见问题

### Q1: OpenAI的text-embedding-3-small和3-large如何选择？

**A:** 
- **small**：性价比高，适合大多数场景
- **large**：质量更高，适合对检索精度要求高的场景
- **建议**：先用small测试效果，不满足再升级到large

### Q2: 中文应该选择什么Embedding模型？

**A:** 
- **在线**：OpenAI的`text-embedding-3-small`支持多语言
- **本地**：`bge-large-zh`或`bge-m3`（北京智源）对中文优化
- **注意**：一些英文优化模型处理中文效果可能不佳

### Q3: 如何评估Embedding模型的效果？

**A:** 可以使用以下方法评估：
1. **人工评估**：检查检索结果的相关性
2. **MTEB基准**：Massive Text Embedding Benchmark
3. **A/B测试**：对比不同模型在实际业务中的表现
