# 5.1 向量检索基础

## 概念讲解

向量检索是RAG系统的核心能力之一。在理解向量检索之前，我们需要先了解**嵌入（Embedding）**的概念。

### 从文本到向量

语言模型无法直接理解文字，但能处理数字。嵌入模型将文本转换为高维空间中的向量：

```
文本: "人工智能是未来的方向"
         ↓ Embedding模型
向量: [0.12, -0.34, 0.56, ..., 0.78]  (768维或更多)
```

**关键理解**：语义相近的文本在向量空间中距离也相近。这就是向量检索的基础。

### 向量检索的工作原理

```
查询: "如何学习机器学习？"
         ↓ Embedding
查询向量: [0.23, -0.45, 0.67, ...]
         ↓ 相似度计算
与数据库中所有向量比较
         ↓ 排序
返回最相似的Top-K文档
```

### 为什么需要向量数据库？

传统的关键词匹配无法理解语义。例如搜索"汽车"，关键词匹配无法找到"汽车"的同义词"车辆"、"automobile"。而向量检索通过语义相似度，可以自然地找到相关内容。

| 检索方式 | 原理 | 优点 | 缺点 |
|---------|------|------|------|
| **关键词检索** | 精确匹配词汇 | 速度快、可解释 | 无法理解语义 |
| **向量检索** | 比较向量距离 | 理解语义、跨语言 | 需要Embedding计算 |
| **混合检索** | 两者结合 | 兼顾精确和语义 | 实现复杂 |

## 核心要点

### 🔑 相似度度量方法

向量数据库中常用的距离计算方式：

| 度量方法 | 说明 | 适用场景 |
|---------|------|---------|
| **余弦相似度** | 计算向量夹角 | 文本相似度（最常用） |
| **欧氏距离** | 计算直线距离 | 通用距离度量 |
| **点积** | 向量内积 | 与余弦类似，但对向量大小敏感 |

**余弦相似度公式**：

```
cos(A, B) = (A·B) / (|A| × |B|)
```

取值范围[-1, 1]，值越大越相似。

### 🔑 Top-K检索

向量检索返回最相似的K个文档：

```python
# 伪代码示意
query_vector = embedding_model.embed("查询文本")
results = vector_store.similarity_search(
    query_vector,
    k=3  # 返回最相似的3个结果
)
```

K值的选择影响检索结果：
- **K太小**：可能遗漏相关文档
- **K太大**：引入不相关文档，增加LLM处理负担
- **建议值**：3-5（问答场景）、5-10（知识库场景）

### 🔑 相似度阈值

除了Top-K，还可以设置相似度阈值过滤结果：

```python
# 只返回相似度大于0.7的结果
results = vector_store.similarity_search_with_score(
    query="查询文本",
    k=5,
    score_threshold=0.7
)
```

## 简单示例

### 基础的向量检索流程

```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import FAISS

# 1. 初始化Embedding模型
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 2. 准备文档
texts = [
    "机器学习是人工智能的一个分支",
    "深度学习使用神经网络进行学习",
    "自然语言处理关注文本和语言理解",
]

# 3. 创建向量存储
vector_store = FAISS.from_texts(texts, embedding=embeddings)

# 4. 执行检索
query = "什么是AI？"
results = vector_store.similarity_search(query, k=2)

for doc in results:
    print(f"内容: {doc.page_content}")
```

## 常见问题

### Q1: 中文和英文需要使用不同的Embedding模型吗？

**A:** 建议使用支持多语言的模型（如OpenAI的`text-embedding-3-small`），或者使用专门的中文Embedding模型（如`bge-large-zh`）。

### Q2: 向量维度应该选择多大？

**A:** 这取决于使用的Embedding模型。OpenAI提供1536维， Cohere提供1024维等。更高维度通常能捕获更多语义信息，但计算量也更大。

### Q3: 如何提高检索的相关性？

**A:** 
1. 选择更好的Embedding模型
2. 优化文档分块策略
3. 使用混合检索（关键词+向量）
4. 添加元数据过滤
5. 对查询进行改写增强

## 本节总结

向量检索是RAG系统实现语义搜索的基础：
1. Embedding模型将文本转换为高维向量
2. 向量距离反映文本语义相似度
3. Top-K检索返回最相关文档
4. 合理选择K值和阈值平衡精度和召回
